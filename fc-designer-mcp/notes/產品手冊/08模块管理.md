# 模組管理 ​

模組管理功能，主要用於設計和管理表單中的子彈窗。它適合把“複雜但不適合直接攤在主畫布上”的錄入內容拆出去，透過彈窗、抽屜等形式分步完成。

## 設計子彈窗 ​

第一步：點選新增模組，選擇彈出框

![设计子弹窗 1](https://pro.form-create.com/doc/img/help/module-01.png)

第二步：點選新增的彈出框

![设计子弹窗 2](https://pro.form-create.com/doc/img/help/module-02.png)

第三步：可切換到元件列表新增需要的元件到彈窗中，即可完成彈窗表單資料的新增

![设计子弹窗 3](https://pro.form-create.com/doc/img/help/module-03.png)

## 管理子彈窗 ​

支援刪除已經設計的彈窗，或複製現有彈窗以便快速建立類似樣式的彈窗

![管理子弹窗](https://pro.form-create.com/doc/img/help/module-04.png)

## API 支援 ​

可以透過API在表單中開啟和關閉彈窗，實現靈活的彈窗控制和操作。實際接入時，這類能力通常會和[表單 API](https://pro.form-create.com/doc/api)以及[$inject](https://pro.form-create.com/doc/inject)一起使用：先在事件裡拿到當前上下文，再決定開啟哪個模組、傳什麼引數、何時回寫。

## 模組使用 ​

模組設計完成後，通常會在元件事件、全域事件或動作配置中被引用。常見用法是在點選按鈕、選擇欄位或觸發某個動作後開啟模組，待使用者在模組內完成選擇或填寫後，再回寫到當前表單。如果你正準備把這類流程串起來，可以順著閱讀[事件配置](https://pro.form-create.com/doc/help/component/event)、[全域事件](https://pro.form-create.com/doc/help/global/event)以及[$inject](https://pro.form-create.com/doc/inject)中的引數結構說明。

![模块使用](https://pro.form-create.com/doc/img/help/module-06.png)

## 典型業務場景 ​

- 商品選擇彈窗：在訂單表單中點選“選擇商品”後開啟模組，選擇完成後自動回填商品資訊。
- 詳情檢視彈窗：在閱讀模式或列表操作中開啟詳情模組，僅檢視不編輯。
- 輔助錄入模組：在主表單之外承載複雜欄位錄入，減少主畫布的視覺負擔。
