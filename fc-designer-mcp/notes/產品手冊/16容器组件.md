# 容器元件 ​

用於包裹或動態控制其他元件的顯示與佈局。相比基礎元件更關注“怎麼組織內容”，它們通常決定的是結構、分組方式和互動承載區域。

## 插槽區域 ​

提供可自定義內容的區域，允許父元件動態插入子元件或內容。

### 核心功能 ​

容器元件的自定義內容擴展區域，無固定表單邏輯，專門用於插入非標準表單項的自定義內容（純展示 / 互動類），支援 HTML、文字、按鈕組等任意內容，不繫結表單資料，僅做介面補充。

### 功能特性 ​

- 定義插槽：使用者可以在元件中定義一個或多個插槽，作為佔位符，用於填充不同內容。
- 內容填充：透過插槽，使用者可以在特定的元件區域中插入動態內容，從而實現更靈活的佈局和內容展示。
- 任意型別：支援任意型別的內容插入，包括文字、圖片、其他元件等，使得表單設計更加靈活和多樣。

### 適用場景 ​

- 表單分組頭部新增模組說明文字（如 “收貨地址資訊僅用於本次配送”）；
- 表單模組底部插入操作按鈕（如 “清空當前模組內容”）；
- 插入業務相關的圖片、二維碼或連結（如報銷表單插入發票示例圖）。

## 動態區域 ​

根據條件或資料動態渲染不同內容，適用於需要靈活切換的佈局場景。

### 核心功能 ​

支援動態增刪行的表單區域，可配置 “新增 / 刪除” 按鈕，區域內的表單項可批次複製、移除，資料以陣列形式儲存，適配 “一組資料需重複錄入” 的場景。

### 功能特性 ​

- 載入 JSON 規則：使用者可以將預定義的 JSON 規則匯入到動態區域元件中，自動生成對應的表單佈局和元件。
- 動態填充：根據 JSON 規則，動態區域元件能夠自動填充相應的元件，減少手動配置的工作量，提高效率。
- 靈活性：透過 JSON 規則，可以靈活定義各類元件及其屬性，適應不同業務需求。
- 實時更新：支援根據實時資料動態更新元件內容，使得表單能夠及時反映最新的資料變化。
- 模組化：使用者可以預定義一組 JSON 規則，用於重複使用和快速配置，實現表單設計的模組化管理。

### 適用場景 ​

- 訂單表單中新增多商品明細（可新增 / 刪除商品行，每行含商品名稱、數量、單價）；
- 入職表單中錄入多個緊急聯絡人（可新增聯絡人，每行含姓名、電話、關係）；
- 報銷表單中新增多筆費用明細（可刪除錯誤的費用行）。

## Vue元件 ​

用於嵌入自研元件，擴展表單業務能力

### 核心功能 ​

容器內嵌入自定義 Vue 元件的專屬入口，可繫結 FormCreate 的表單資料、觸發表單事件，打通視覺化表單與自研業務元件的聯動，支援傳參、事件回撥。

### 適用場景 ​

- 嵌入自研的驗證碼輸入元件（繫結表單驗證邏輯）；
- 插入業務專屬的簽名元件（收集手寫簽名並同步至表單資料）；
- 整合自定義的門店選擇元件（適配企業專屬的門店列表）。

![Vue组件](https://pro.form-create.com/doc/img/help/container-components-03.png)

如果你需要把企業內部的專屬能力正式接進設計器，這類場景通常會繼續檢視[擴展元件](https://pro.form-create.com/doc/component)與[原型方法](https://pro.form-create.com/doc/proto)。

## 表單項 ​

作為表單欄位的容器，支援統一管理欄位樣式、驗證規則或佈局。

### 核心功能 ​

容器內最基礎的表單元素承載區域，用於放置輸入框、選擇器、開關等標準表單項，支援資料繫結、校驗規則、顯隱控制等核心表單邏輯，是容器內資料錄入的核心載體。

### 適用場景 ​

- 個人資訊容器內放置姓名輸入框、手機號輸入框、性別單選框；
- 收貨地址容器內放置省市區選擇器、詳細地址輸入框、郵編輸入框；
- 支付資訊容器內放置支付方式單選框、銀行卡號輸入框。

![表单项](https://pro.form-create.com/doc/img/help/container-components-04.png)

從結構上看，容器元件往往對應規則中的巢狀層級與拖拽約束；如果你需要進一步理解哪些元件可拖入、子節點如何組織，可以參考[TS型別定義](https://pro.form-create.com/doc/ts)。
