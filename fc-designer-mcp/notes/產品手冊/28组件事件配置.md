# 元件事件配置 ​

元件事件配置用於監聽和處理元件互動過程中的各類事件，以實現更靈活的聯動邏輯和業務行為。

## 典型場景 ​

### 動態表單驗證 ​

- 場景：使用者輸入手機號時，實時校驗格式並提示錯誤。
- 實現：為輸入框繫結 `onChange` 事件，觸發正則校驗邏輯，動態更新錯誤提示。若需要在事件裡繼續呼叫校驗、取值或設定欄位狀態，可以結合[表單 API](https://pro.form-create.com/doc/api)中的 `validateField`、`getValue`、`setValue` 等方法使用。

### 級聯選擇器 ​

- 場景：選擇省份後，自動載入對應的城市列表。
- 實現：為省份下拉框繫結 `onChange` 事件，觸發介面請求獲取城市資料，並更新城市下拉框的選項。若需要讀取表單 API、當前元件規則或原始事件引數，結合[$inject](https://pro.form-create.com/doc/inject)會更方便。

### 實時資料同步 ​

- 場景：使用者修改表單資料時，實時儲存到本地或同步到伺服器。
- 實現：為表單繫結 `onBlur` 或 `onChange` 事件，觸發資料儲存邏輯。若需要把元件內部事件繼續拋給業務頁面統一處理，可以參考[渲染時監聽事件](https://pro.form-create.com/doc/event-emit)中的事件丟擲與監聽方式。

### 動作配置 ​

![image](https://pro.form-create.com/doc/img/help/event-16.png)

![image](https://pro.form-create.com/doc/img/help/event-17.png)

動作管理功能提供了便捷的操作方式，使使用者可以透過指定事件觸發一系列動作，具體包括以下操作：

**視覺化操作設定** 透過直觀的視覺化介面，使用者可以為各個元件設定在特定事件下觸發的一系列動作。

**內建常用動作** 系統內建了多種常用動作，已經能夠覆蓋頁面跳轉、彈窗控制、表單操作、請求處理和業務事件丟擲等大部分場景。實際配置時，通常先確定“事件發生後要做什麼”，再從對應分類中選擇動作；如果你需要逐項檢視更完整的動作配置結構，也可以繼續閱讀[內建動作列表](https://pro.form-create.com/doc/behavior-list)。

##### 頁面操作動作 ​

| 動作名稱 | 用途說明 |
| --- | --- |
| redirectPage | 頁面跳轉 |
| reloadPage | 頁面重新整理 |

| 動作名稱 | 用途說明 |
| --- | --- |

| message | 顯示訊息提示 |
| confirm | 顯示確認對話方塊 |

##### 表單操作動作 ​

| validate | 驗證整個表單 |
| validateFields | 驗證指定欄位 |
| submit | 提交表單 |

##### 其他動作 ​

| 動作名稱 | 用途說明 |
| --- | --- |
| fetch | 傳送請求 |

##### 動作使用提示 ​

- 動作執行條件：支援為每個動作設定執行條件，只有在條件滿足時動作才會執行。
- 阻斷條件：支援為動作設定阻斷條件，如果阻斷條件成立，則中斷執行，這使得動作的執行流程更加可控。
- 流程控制：透過條件設定和阻斷機制，使用者可以高度定製動作的執行流程，實現複雜的業務邏輯控制。
- 支援擴展：使用者可以根據需求擴展自定義動作，滿足更復雜和特定的業務需求。若需要繼續檢視內建動作能力、註冊方式和擴展示例，可直接閱讀[內建動作列表](https://pro.form-create.com/doc/behavior-list)與[擴展動作操作](https://pro.form-create.com/doc/behavior)。

## inject 引數說明 ​

在自定義事件或全域事件的函式編輯區中，通常可以透過 `inject` 獲取設計器提供的上下文能力，例如表單 API、原始引數、當前事件資訊等。它適合用於讀取表單資料、操作欄位狀態、呼叫模組或執行自定義處理邏輯；如果你需要了解完整欄位結構、`api`、`self`、`args` 等屬性含義，可以繼續檢視[$inject](https://pro.form-create.com/doc/inject)。

## 開發示例 ​

下面這段示例可以直接放到 JS 面板中，用於在元件值變化後同步更新其他欄位；如果你還需要把這個變化繼續拋到業務頁面中處理，可以自然配合[渲染時監聽事件](https://pro.form-create.com/doc/event-emit)一起使用。

js

```
const api = $inject.api;
const value = $inject.args[0];

if (!value) {
    api.setValue('remark', '');
} else {
    api.setValue('remark', `已选择：${value}`);
}
```

這個示例適合用在選擇器、單選框或輸入框的 `change` 類事件中。實際接入時，只需要把 `remark` 替換成目標欄位的標識即可；若要進一步執行開啟模組、提交表單或條件流程控制，可繼續結合本頁上方的動作配置使用。
