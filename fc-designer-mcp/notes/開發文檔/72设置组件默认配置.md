# 设置组件默认配置

> Source: https://pro.form-create.com/doc/default-rule

---

# 設定元件預設配置 ​

FormCreate 提供了兩種方式來設定元件的預設配置：

1. **設計器配置**：透過 `updateDefaultRule` 配置項，在設計器中設定元件拖入時的預設規則
2. **渲染時配置**：透過 `option.global` 配置項，在表單渲染時設定元件的公共配置

## 設計器配置 ​

在設計器中，您可以透過 `updateDefaultRule` 配置項設定元件的初始化規則。這允許您在拖拽元件到設計器時，自動應用某些預設配置，進而提高使用者體驗和效率。

### 資料結構 ​

ts

```
type UpdateDefaultRule = {
    //组件拖拽组件规则的id, 设置组件的初始化规则
    [id: string]: Partial<Omit<Rule, "field" | "children" | "component">> | ((Rule) => void);
}
```

注意

定義元件預設值後，不會影響歷史已有資料，僅對重新拖入的元件生效。

### 修改輸入框元件的預設規則 ​

在這個示例中，讓輸入框拖入時自動禁用

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
    const config = {
        updateDefaultRule: {
            input: {
                props: {
                    disabled: true
                }
            }
        }
    }
</script>
```

### 設定選擇框的預設選項 ​

您可以讓選擇框元件必填並且預設一個預設值，例如“請選擇”：

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
    const config = {
        updateDefaultRule: {
            select: {
                $required: true,
                props: {
                    options: [
                        { label: "请选择", value: "" },
                        { label: "选项1", value: "option1" },
                        { label: "选项2", value: "option2" }
                    ]
                }
            }
        }
    }
</script>
```

### 設定文字域的行數 ​

您可以在文字域元件中設定預設的行數限制：

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
    const config = {
        updateDefaultRule: {
            textarea: {
                props: {
                    rows: 5 // 默认行数
                }
            }
        }
    }
</script>
```

## 全域替換配置 ​

透過 `formCreate.parser` 註冊解析器，可以直接修改規則本身。Parser 是全域處理，會在 `options.global` 之前觸發，適合需要強制統一規則的場景。

重要提示

`formCreate.parser` 需要在應用啟動時註冊，通常寫在 `main.js` 中，而不是在元件中。

### 基本用法 ​

在 `main.js` 中註冊 Parser：

js

```
// main.js
import formCreate from '@form-create/element-ui';

formCreate.parser({
  name: 'upload',
  merge: true,
  init({rule}) {
    // 直接修改规则本身
    rule.props.action = '/api/upload';
    rule.props.onError = function (r) {
      alert('统一的上传失败处理');
    };
    rule.props.onSuccess = function (file, res) {
      file.url = res.data.url;
    };
  }
});
```

### 使用示例 ​

vue

```
<template>
  <form-create :rule="rule" v-model:api="fApi" :option="options"/>
</template>

<script>
export default {
  data() {
    return {
      fApi: {},
      options: {
        // options.global 会在 Parser 之后执行
        global: {
          upload: {
            props: {
              // 如果规则中没有定义 listType，会应用此配置
              listType: 'picture-card'
            }
          }
        }
      },
      rule: [
        {
          type: 'upload',
          field: 'upload1',
          title: '上传图片1',
          props: {
            // action 会被 Parser 修改为 '/api/upload'
            // onError 会被 Parser 修改为统一处理函数
          }
        },
        {
          type: 'upload',
          field: 'upload2',
          title: '上传图片2',
          props: {
            action: '/custom-upload',  // 这个会被 Parser 覆盖为 '/api/upload'
            onError: function (r) {
              console.log('自定义错误处理');  // 这个会被 Parser 覆盖
            }
          }
        }
      ]
    }
  }
}
</script>
```

### 資料結構 ​

ts

```
type Parser = (name: string, parser: ParserConfig) => void;

interface ParserConfig {
    // 是否合并内置解析规则, 内置组件请保持 true
    merge?: true;
    // 初始化组件规则
    init?: (ctx: ParserContext) => void;
}

interface ParserContext {
    // 当前组件的规则配置
    rule: Rule;
    // 表单 API 实例
    $api: Api;
}
```

merge 引數說明

- 內建元件（如 `input`、`select` 等）在新增解析器時，請配置 `merge: true`，以免覆蓋系統內建的解析邏輯。
- 自定義擴展元件若無預設解析邏輯，可以省略該引數。

Parser vs options.global

- **Parser**：直接修改規則本身，執行時機最早，適合需要強制統一規則的場景
- **options.global**：透過合併方式應用配置，執行時機在 Parser 之後，適合需要靈活配置的場景

如果需要在 Parser 之後再次覆蓋配置，可以使用 `options.global.deep`。

## 渲染時替換配置 ​

在表單渲染時，透過全域配置中的 `global` 配置項，可以實現對元件的公共配置。此功能允許您為所有元件設定統一的預設值和行為，從而簡化配置和維護工作。

### 基本用法 ​

vue

```
<template>
  <form-create :rule="rule" v-model:api="fApi" :option="options"/>
</template>

<script>
export default {
  data() {
    return {
      fApi: {},
      options: {
        global: {
          // 全局配置：所有组件都应用
          '*': {
            props: {
              disabled: true
            }
          },
          // 特定组件配置：只对 upload 组件生效
          upload: {
            props: {
              onError: function (r) {
                alert('上传失败');
              }
            }
          }
        }
      },
      rule: [
        // 表单规则
      ]
    }
  }
}
</script>
```

### 配置規則 ​

`global` 配置支援以下規則：

- **`'*'`**：萬用字元，表示所有元件都會應用此配置
- **元件型別名**：如 `input`、`select`、`upload` 等，只對指定型別的元件生效
- **配置優先順序**：元件型別配置會覆蓋萬用字元配置，元件規則中的配置會覆蓋全域配置

### 配置合併方式 ​

`global` 配置支援兩種合併方式：

**1. 普通配置（預設合併）**：

當規則中已存在相同屬性時，全域配置不會覆蓋規則中的配置，只在規則中沒有該屬性時才會生效。

js

```
global: {
  upload: {
    props: {
      action: '/api/upload',  // 默认上传地址
      onError: function (r) {
        alert('上传失败');
      }
    }
  }
}
```

**2. 深度合併配置（強制覆蓋）**：

使用 `deep` 屬性，透過點號路徑指定要強制覆蓋的屬性，會強制覆蓋規則中的同名屬性。

js

```
global: {
  upload: {
    deep: {
      'props.action': '/api/upload',  // 强制覆盖规则中的 action
      'props.onError': function (r) {
        alert('上传失败');
      },
      'props.onSuccess': function (file, res) {
        file.url = file.response.url;
      }
    }
  }
}
```

重要提示

**執行順序**：Parser（全域替換配置） → options.global（普通配置） → options.global.deep（深度合併）

- **普通配置**：只在規則中沒有對應屬性時生效，不會覆蓋規則中已存在的屬性
- **深度合併配置**：使用 `deep` 物件，透過點號路徑（如 `'props.onError'`）指定要覆蓋的屬性，會強制覆蓋規則中的同名屬性，即使規則中已經定義了該屬性

### 使用示例 ​

**1. 設定所有元件的預設禁用狀態和 col 佈局**：

vue

```
<template>
  <form-create :rule="rule" v-model:api="fApi" :option="options"/>
</template>

<script>
export default {
  data() {
    return {
      fApi: {},
      options: {
        global: {
          '*': {
            props: {
              disabled: true  // 所有组件默认禁用
            },
            col: {
              span: 12  // 所有组件默认占 12 列（一半宽度）
            }
          }
        }
      },
      rule: [
        {
          type: 'input',
          field: 'input1',
          title: 'input1'
          // 会自动应用 disabled: true 和 col: { span: 12 }
        },
        {
          type: 'input',
          field: 'input2',
          title: 'input2'
          // 会自动应用 disabled: true 和 col: { span: 12 }
        },
        {
          type: 'input',
          field: 'input3',
          title: 'input3',
          col: {
            span: 24  // 覆盖全局配置，占满整行
          }
        }
      ]
    }
  }
}
</script>
```

**2. 設定 upload 元件的上傳回撥事件（普通配置）**：

配置 upload 元件的全域回撥事件，使用普通配置方式。這種方式只在規則中沒有定義對應回撥時才會生效。

vue

```
<template>
  <form-create :rule="rule" v-model:api="fApi" :option="options"/>
</template>

<script>
export default {
  data() {
    return {
      fApi: {},
      options: {
        onSubmit: (formData) => {
          alert(JSON.stringify(formData));
        },
        global: {
          '*': {
            props: {
              disabled: true
            },
            col: {
              span: 12
            }
          },
          // 普通配置：只在规则中没有对应属性时才会生效
          upload: {
            props: {
              action: '/api/upload',  // 默认上传地址
              onError: function (r) {
                alert('上传失败');
              },
              onSuccess: function (file, res) {
                file.url = file.response.url;
              }
            }
          }
        }
      },
      rule: [
        {
          type: 'input',
          field: 'input1',
          title: 'input1'
        },
        {
          type: 'input',
          field: 'input2',
          title: 'input2'
        },
        {
          type: 'input',
          field: 'input3',
          title: 'input3',
          col: {
            span: 24
          }
        },
        {
          type: 'input',
          field: 'input4',
          title: 'input4',
          col: {
            span: 24
          }
        },
        {
          type: 'upload',
          field: 'upload',
          title: '上传图片',
          value: '',
          col: {
            span: 24
          },
          props: {
            listType: 'picture-card'
            // 如果规则中没有定义 action，会自动应用全局配置的 action: '/api/upload'
            // 如果规则中没有定义 onError，会自动应用全局配置的 onError
            // 如果规则中定义了 onError，则使用规则中的 onError
          }
        },
        {
          type: 'upload',
          field: 'upload2',
          title: '上传文件',
          props: {
            action: '/',
            onError: function (r) {
              console.log('自定义错误处理');  // 这个会覆盖全局配置
            }
            // 因为规则中定义了 onError，全局配置的 onError 不会生效
          }
        }
      ]
    }
  }
}
</script>
```

**3. 強制覆蓋規則中的配置（深度合併）**：

使用 `deep` 屬性，透過點號路徑指定要強制覆蓋的屬性，可以強制覆蓋規則中的同名屬性，即使規則中已經定義了該屬性。

vue

```
<template>
  <form-create :rule="rule" v-model:api="fApi" :option="options"/>
</template>

<script>
export default {
  data() {
    return {
      fApi: {},
      options: {
        global: {
          upload: {
            // 深度合并配置：使用点号路径指定要覆盖的属性
            deep: {
              'props.action': '/api/upload',  // 强制覆盖规则中的 action
              'props.onError': function (r) {
                alert('统一的上传失败处理');
              },
              'props.onSuccess': function (file, res) {
                file.url = file.response.url;
              }
            }
          }
        }
      },
      rule: [
        {
          type: 'upload',
          field: 'upload1',
          title: '上传图片1',
          props: {
            action: '/old-upload',  // 这个会被全局配置覆盖为 '/api/upload'
            onError: function (r) {
              console.log('规则中的错误处理');  // 这个会被全局配置覆盖
            }
            // 即使规则中定义了 action 和 onError，也会被全局配置覆盖
          }
        },
        {
          type: 'upload',
          field: 'upload2',
          title: '上传图片2',
          props: {
            // 没有定义 action 和 onError，会使用全局配置的 action 和 onError
          }
        }
      ]
    }
  }
}
</script>
```

## 三種配置方式的區別 ​

| 配置方式 | 應用場景 | 生效時機 | 影響範圍 | 執行順序 |
| --- | --- | --- | --- | --- |
| formCreate.parser | 全域處理 | 規則解析時 | 直接修改規則本身 | 最先執行 |
| option.global | 渲染環境 | 表單渲染時 | 透過合併方式應用配置 | Parser 之後 |
| updateDefaultRule | 設計器環境 | 元件拖入設計器時 | 僅影響新拖入的元件 | 設計器專用 |

使用建議

- **Parser 配置**：適用於需要強制統一規則、直接修改規則本身的場景，執行時機最早
- **渲染時配置（option.global）**：適用於在執行時統一元件的顯示和行為，實現業務邏輯的統一管理
- **設計器配置（updateDefaultRule）**：適用於在設計階段統一元件的初始狀態，提高設計效率

**執行順序**：Parser → option.global（普通配置） → option.global.deep（深度合併）
