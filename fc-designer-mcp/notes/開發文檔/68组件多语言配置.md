# 元件多語言配置 ​

本文將詳細介紹如何透過設計器配置元件的多語言功能，從而使您的元件更易於本地化和國際化使用。

**閱讀本文前，請先查閱[擴展元件](https://pro.form-create.com/doc/component)文件以瞭解基礎概念。**

## 定義拖拽規則 ​

在設計表單元件時，可以使用 `languageKey` 來宣告元件的多語言 ID。這使得在元件插入表單時，能夠自動管理和匹配對應的多語言配置。

ts

```
const dragRule = {
    type: 'upload2',
    languageKey: ['clickUpload']
    //...
}
```

## 元件中使用 ​

在元件實現中，透過 `formCreateInject.t` 方法可以從多語言配置中讀取和顯示對應文字。這種方式能夠確保文字自動根據語言環境顯示相應內容。

vue

```
<template>
    <button>{{formCreateInject.t('clickUpload') || '点击上传'}}</button>
</template>

<script>
    export default {
        props: {
            formCreateInject: Object
        }
    }
</script>
```

在上述程式碼中，`formCreateInject.t('clickUpload')` 將獲取並顯示“clickUpload” ID 對應的多語言字串。

## 資料結構 ​

ts

```
typeof t = (id: string, params?: Object) => string | undefined;
```

- 可以使用`{param}`這種大括號的方式在多語言字元中引入變數。
- `params`物件中的值將自動替換多語言字串中的變數佔位符。

### 示例：傳遞引數到多語言字串 ​

假設您有以下的多語言配置和實現：

多語言配置：

json

```
{
  "welcomeMessage": "Welcome, {name}!"
}
```

使用程式碼：

vue

```
<template>
    <div>{{formCreateInject.t('welcomeMessage', { name: 'John' })}}</div>
</template>
```

將生成：

```
Welcome, John!
```

透過對多語言功能的有效利用，您可以顯著提升元件的國際化能力，而不妨礙開發的靈活性。希望這份指南能夠幫助您更好地配置和實現多語言支援，使應用更具國際吸引力。
