# 儲存欄位為模板 ​

提供了儲存欄位的功能，使開發者能夠將當前選中的欄位儲存為可複用的模板，以便在其他表單中快速使用。這對於標準化欄位設計和提高開發效率具有重要意義。

![save.png](https://pro.form-create.com/doc/img/saveField.png)

注意

**儲存欄位功能僅在選中具有 `field` 屬性的欄位時可用，確保欄位具有完整的配置資訊。**

## 顯示儲存欄位按鈕 ​

設計器中內建了一個儲存欄位按鈕，透過 `config.showSaveFieldBtn` 配置項來控制其顯示。此功能允許使用者將當前選中的欄位儲存為模板。

以下程式碼展示瞭如何在配置中啟用儲存欄位按鈕：

js

```
{
    config: {
        showSaveFieldBtn: true //默认开启, 为 false 时关闭
    }
}
```

## 儲存欄位資料 ​

當使用者點選儲存欄位按鈕時，會觸發 `saveField` 事件。開發者可以自定義處理該事件以實現欄位模板的儲存功能。

### 儲存欄位示例 ​

以下是儲存欄位的一個簡單實現示例：

vue

```
<template>
    <fc-designer ref="designer" @saveField="handleSaveField" :config="config"/>
</template>
<script setup>
    const config = {
        showSaveFieldBtn: true
    }
    function handleSaveField(activeRule) {
        // 示例：调用后端接口保存字段模板
        axios.post('/api/saveFieldTemplate', {
            fieldConfig: activeRule,
            fieldName: activeRule.field,
            fieldType: activeRule.type,
            fieldTitle: activeRule.title,
        }).then(response => {
            // 处理保存成功的逻辑
            console.log('字段模板保存成功', response.data);
            // 可以显示成功提示
            ElMessage.success('字段模板保存成功');
        }).catch(error => {
            // 错误处理
            console.error('字段模板保存失败', error);
            ElMessage.error('字段模板保存失败');
        });
    }
</script>
```

透過上述程式碼，當使用者點選儲存欄位按鈕時，當前選中欄位的配置資訊會被髮送到指定的 API 介面進行儲存。

## 讀取欄位列表 ​

儲存的欄位模板可以透過介面讀取，並新增到設計器的欄位列表中，方便使用者快速複用已儲存的欄位配置。

### 讀取欄位列表示例 ​

以下示例展示瞭如何從後端介面讀取欄位模板，並將其新增到設計器的欄位列表中：

vue

```
<template>
    <fc-designer 
        ref="designer" 
        :field="fieldList"
        :config="config"
    />
</template>
<script setup>
import { ref, onMounted } from 'vue'
const fieldList = ref([])
const config = {
    showSaveFieldBtn: true
}
// 读取字段模板
async function loadFieldTemplates() {
    try {
        const response = await axios.get('/api/getFieldTemplates');
        const templates = response.data;
        // 将模板转换为字段列表格式
        const templateFields = templates.map(template => ({
            label: template.fieldTitle,
            field: template.fieldName,
            rule: template.fieldConfig,
        }));
        // 添加到现有字段列表
        fieldList.value = [
            ...fieldList.value,
            {
                label: '自定义模板',
                children: templateFields
            }
        ];
    } catch (error) {
        console.error('读取字段模板失败', error);
        ElMessage.error('读取字段模板失败');
    }
}
// 组件挂载时加载字段模板
onMounted(() => {
    loadFieldTemplates();
});
</script>
```

## 新增到欄位ID選擇列表 ​

除了將儲存的欄位模板新增到左側欄位列表（用於拖拽）外，您還可以將這些模板新增到 `config.fieldList` 中，使其成為欄位ID的候選選項。這樣使用者在選擇欄位ID時，可以直接從已儲存的模板中選擇。

### 欄位ID選擇列表配置 ​

透過設定 `config.fieldList` 屬性，可以將欄位模板配置為欄位ID的選擇項。這種配置通常用於：

1. 限制使用者只能選擇預定義的欄位
2. 提供標準的欄位命名規範
3. 在子表單元件中建立欄位聯動關係

### 配置示例 ​

以下示例展示如何將儲存的欄位模板新增到欄位ID選擇列表中：

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
import { ref, onMounted } from 'vue'
const config = ref({
    showSaveFieldBtn: true,
    // 字段ID只能从选择项中选择，不能手动输入
    fieldReadonly: true,
    // 字段ID选择项列表
    fieldList: []
})
// 读取字段模板并配置到fieldList
async function loadFieldList() {
    try {
        const response = await axios.get('/api/getFieldTemplates');
        const templates = response.data;
        // 将模板转换为fieldList格式
        const fieldItems = templates.map(template => ({
            value: template.fieldName,  // 字段值
            label: template.fieldTitle, // 显示标签
        }));
        // 配置fieldList
        config.value.fieldList = [
            {
                value: 'custom_template',
                label: '自定义模板',
                children: fieldItems
            }
        ];
    } catch (error) {
        console.error('读取字段列表失败', error);
        ElMessage.error('读取字段列表失败');
    }
}
// 组件挂载时加载字段列表
onMounted(() => {
    loadFieldList();
});
</script>
```

透過以上配置，您可以將儲存的欄位模板充分利用，既能在左側欄位列表中進行拖拽使用，又能在欄位ID選擇器中快速選擇，大大提高了表單設計的效率和規範性。
