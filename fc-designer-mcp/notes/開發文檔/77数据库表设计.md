# 資料庫表設計 ​

在 FormCreate 環境中管理表單時，一個合理的資料庫設計至關重要，以便儲存表單設計的規則以及使用者提交的資料。下述內容詳細闡述了幾種核心資料庫表設計，幫助您更有效地組織和管理資料。

### 1. 表單表 (`forms`) ​

此表用於儲存每個表單的整體資訊，包括名稱、描述以及設計的規則等：

| 欄位名 | 資料型別 | 描述 |
| --- | --- | --- |
| id | INT | 主鍵ID，自增 |
| title | VARCHAR | 表單名稱，用於標識表單的唯一標識 |
| description | TEXT | 表單的描述資訊 |
| rules | JSON | 表單的規則和欄位的整體配置資料，通常包含多個欄位的配置 |
| options | JSON | 表單的配置資料（例如：佈局、尺寸、全域資料等） |
| created_at | DATETIME | 表單建立時間 |
| updated_at | DATETIME | 表單更新時間 |

sql

```
CREATE TABLE forms (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID，自增',
    title VARCHAR(64) NOT NULL UNIQUE COMMENT '表单名称，用于标识表单的唯一标识',
    description TEXT COMMENT '表单的描述信息',
    rules JSON COMMENT '表单的规则和字段的整体配置数据，通常包含多个字段的配置',
    options JSON COMMENT '表单的配置数据（例如：布局、尺寸、全局数据等）',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '表单创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '表单更新时间'
) COMMENT='表单信息表';
```

**示例程式碼:**

表單元件的插入資料物件，包含標題、描述、表單配置等關鍵資訊，用於將表單設計資料序列化並儲存到後端資料庫。

ts

```
//rules 是表单的组件设计规则
//options 是表单配置
const insert = {
    title: options.formName || '',
    description: options.description || '',
    rules: JSON.stringify(rules),
    options: JSON.stringify(options),
}
```

### 2. 欄位資料表 (`form_data`) ​

此表用於儲存使用者提交的每個欄位的具體值。每次表單提交都會產生一條新的紀錄，用於追蹤表單的歷史記錄。

| 欄位名 | 資料型別 | 描述 |
| --- | --- | --- |
| id | INT | 主鍵ID，自增 |
| form_id | INT | 表單ID，外來鍵關聯至 forms 表 |
| field_name | VARCHAR | 欄位名稱 |
| field_value | JSON | 使用者提交的欄位值 |
| created_at | DATETIME | 資料建立時間 |
| updated_at | DATETIME | 表單更新時間 |

sql

```
CREATE TABLE form_data (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID，自增',
    form_id INT NOT NULL COMMENT '表单ID',
    field VARCHAR(32) NOT NULL COMMENT '字段名称，指向 fields 表中的 field',
    value JSON COMMENT '用户提交的字段值',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '数据创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '表单更新时间'
) COMMENT='用户提交的表单字段数据表';
```

**示例程式碼:**

將表單提交的資料轉換為可插入資料庫的結構，透過遍歷表單資料的每個欄位，生成包含欄位ID、欄位名和序列化值的物件陣列，便於批次儲存表單提交記錄。

ts

```
//formData 是表单提交的数据
const inserts = Object.keys(formData).map(field => {
    return {
        field: field,
        value: JSON.stringify(formData[field]),
    };
})
```

透過上述資料庫表結構示例，您可以為基於 FormCreate 的應用搭建一個良好的資料儲存基礎，方便儲存和管理表單設計與使用者資料。
