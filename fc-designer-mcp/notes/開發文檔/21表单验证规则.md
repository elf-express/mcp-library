# 表單驗證規則 ​

表單驗證是確保使用者輸入資料有效性的重要功能。FormCreate 設計器提供了豐富的驗證規則型別，幫助您快速配置各種驗證場景。

注意

- **驗證觸發時機**：支援 `blur`（失焦）、`change`（值改變）、`submit`（提交）三種觸發方式
- 自定義驗證中可透過 this.rule 和 this.api 獲取元件和表單的相關資訊，便於在驗證時訪問其他元件的值。
- pattern 型別為 string 時，前後不能包含 / 符號，否則會導致正規表示式無效。
- 自定義驗證中無論成功或失敗，都必須執行 callback 回撥函式。

## 驗證規則型別 ​

FormCreate 設計器支援以下驗證規則型別：

### 基礎驗證規則 ​

| Key | 規則名稱 | 功能描述 |
| --- | --- | --- |
| len | 長度 | 驗證字串長度等於指定值 |
| minLen | 最小長度 | 驗證字串最小長度 |
| maxLen | 最大長度 | 驗證字串最大長度 |
| min | 最小值 | 驗證數值最小值 |
| max | 最大值 | 驗證數值最大值 |

### 格式驗證規則 ​

| Key | 規則名稱 | 功能描述 |
| --- | --- | --- |
| pattern | 正規表示式 | 自定義正規表示式驗證 |
| email | 郵箱格式 | 驗證郵箱格式 |
| url | URL格式 | 驗證URL格式 |
| ip | IP地址 | 驗證IP地址格式 |
| phone | 手機號 | 驗證手機號格式 |

### 字元驗證規則 ​

| Key | 規則名稱 | 功能描述 |
| --- | --- | --- |
| uppercase | 全大寫 | 驗證必須為大寫字母 |
| lowercase | 全小寫 | 驗證必須為小寫字母 |
| positive | 正數 | 驗證必須為正數 |
| negative | 負數 | 驗證必須為負數 |
| integer | 整數 | 驗證必須為整數 |
| number | 數字 | 驗證必須為數字 |

### 高階驗證規則 ​

| Key | 規則名稱 | 功能描述 |
| --- | --- | --- |
| validator | 自定義驗證 | 自定義驗證函式 |
| computed | 條件驗證 | 基於其他欄位的驗證 |

透過這些驗證配置，您可以構建出功能完善、使用者友好的表單驗證系統。
