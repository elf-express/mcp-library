# 視覺化設計元件插槽 ​

`FcDesigner` 支援對元件插槽內容區域進行視覺化拖拽。這樣，開發者可以更加靈活地在插槽中放置和配置元件，從而增強表單設計的自定義能力。

**閱讀本文前，請先查閱[擴展元件](https://pro.form-create.com/doc/component)文件以瞭解基礎概念。**

## 示例 ​

一個有多個插槽的自定義元件

vue

```
<template>
  <div>
      <p>默认插槽</p>
      <div>
          <slot></slot>
      </div>
      <p>content插槽</p>
      <div>
          <slot name="content"></slot>
      </div>
      <p>footer插槽</p>
      <div>
          <slot name="footer"></slot>
      </div>
  </div>
</template>
<script>
export default {
    name: 'ShopForm',
    props:{
      disabled: Boolean
    }
}
</script>
```

在上述元件中，定義了三個插槽：

- `default` 插槽
- `content` 插槽
- `footer` 插槽

## 定義拖拽規則 ​

透過 slot 欄位配置需要支援拖拽的插槽區域：

js

```
const ShopFormRule = {
    //插入菜单位置
    menu: 'subform',
    //图标
    icon: 'icon-subform',
    //名称
    label: '商品表单',
    //id,唯一!
    name: 'ShopForm',
    //是否可以操作, 需要支持拖拽所以为true
    mask: false,
    drag: true,
    //配置插槽
    slot: [
        //支持默认插槽内拖入组件
        'default',
        //支持content插槽内拖入组件, 拖拽区域固定为子表单容器
        {
            name: 'content',
            //子表单组件的name
            type: 'fcRow'
        },
        //支持footer插槽内拖入组件
        'footer',
    ]
    //定义组件的渲染规则
    rule({t}) {
      	//自定义组件的生成规则
        return {
            type: 'ShopForm',
            props: {},
            children: []
        };
    },
  	//自定义组件的属性配置
    props() {
        return [
            {
                type: 'switch',
                field: 'disabled',
                title: '是否禁用'
            }
        ];
    }
};
```

在 `slot` 配置中，`type` 欄位用於定義插槽區域的容器元件，預設為拖拽容器。您可以自定義 type 為 row 或其他型別，以匹配您的實際需求。

## 掛載元件和拖拽規則 ​

js

```
//挂载组件
FcDesigner.addCompoent(ShopForm);
//挂载拖拽规则
this.$refs.designer.addComponent(ShopFormRule);
```

透過上述步驟，您可以將自定義元件和拖拽規則新增到設計器中，實現對插槽區域內容的視覺化拖拽。
