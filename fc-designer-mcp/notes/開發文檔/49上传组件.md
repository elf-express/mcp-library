# 上傳元件 ​

上傳元件提供了檔案上傳功能，支援圖片、文件、影片、音訊等多種檔案型別的上傳。元件提供了豐富的配置選項，支援多檔案上傳、檔案預覽、上傳進度監控等功能，有助於實現靈活的檔案管理場景。

## 基礎使用 ​

### 設定上傳成功回撥 ​

在上傳元件的 `onSuccess` 回撥中，必須將上傳成功後的 URL 賦值給 `file.url`，否則表單將無法獲取檔案資料：

js

```
function onSuccess($inject) {
   const res = $inject.args[0];
   const file = $inject.args[1];
   // 将上传后的URL赋值给file.url
    file.url = res.data.url;
}
```

這樣可以確保表單提交時能夠正確獲取上傳的檔案資訊。

### 設定請求頭 ​

檔案上傳時將觸發表單的 `beforeFetch` 事件，可在該事件回撥中修改請求頭(headers)配置，例如新增認證token：

js

```
function beforeFetch(config, data) {
    // 从外部数据获取全局 token
    const token = data.api.getData('globalToken');
    if (!config.headers) {
        config.headers = {};
    }
    config.headers.token = token;
}
```

### 儲存更多資料 ​

如果期望上傳元件的 value 型別為 `object[]`，需要將上傳的結果資料賦值給 `file.value`：

js

```
function onSuccess($inject) {
   const res = $inject.args[0];
   const file = $inject.args[1];
    file.url = res.data.url;
    file.name = res.data.name;
    file.value = res.data;  // 保存完整的返回数据
}
```

## 全域配置 ​

在實際專案中，通常需要為所有上傳元件設定統一的上傳地址、成功回撥和請求頭。FormCreate 提供了三種方式來設定全域配置，按執行順序依次為：

1. **formCreate.parser**（全域替換配置）- 直接修改規則本身，執行時機最早
2. **option.global**（普通配置）- 只在規則中沒有對應屬性時生效
3. **option.global.deep**（深度合併配置）- 強制覆蓋規則中的同名屬性

執行順序

**執行順序**：Parser → option.global（普通配置） → option.global.deep（深度合併）

詳細說明請檢視：[設定元件預設配置](https://pro.form-create.com/doc/default-rule)

### 方式一：使用 Parser 全域替換（推薦用於強制統一） ​

透過 `formCreate.parser` 註冊解析器，直接修改規則本身。適合需要強制統一所有上傳元件配置的場景。

**在 `main.js` 中註冊 Parser**：

js

```
// main.js
import formCreate from '@form-create/element-ui';

formCreate.parser({
  name: 'upload',
  merge: true,
  init({rule}) {
    // 直接修改规则本身
    rule.props.action = '/api/upload';
    rule.props.onSuccess = function (res, file) {
      // 必须设置 file.url，否则表单无法获取文件数据
      file.url = res.data.url;
      file.name = res.data.name;
    };
    rule.props.onError = function (err) {
      console.error('上传失败：', err);
    };
  }
});
```

**在元件中使用**：

vue

```
<template>
  <form-create :rule="rule" v-model:api="fApi" :option="options"/>
</template>

<script>
export default {
  data() {
    return {
      fApi: {},
      options: {
        // 表单级别的 beforeFetch 事件，用于设置请求头
        beforeFetch: function (config, data) {
          // 从外部数据获取全局 token
          const token = data.api.getData('globalToken');
          if (!config.headers) {
            config.headers = {};
          }
          config.headers.Authorization = 'Bearer ' + token;
          return config;
        }
      },
      rule: [
        {
          type: 'upload',
          field: 'upload1',
          title: '上传图片',
          props: {
            listType: 'picture-card'
            // action 和 onSuccess 会被 Parser 自动设置
          }
        },
        {
          type: 'upload',
          field: 'upload2',
          title: '上传文件',
          props: {
            action: '/custom-upload',  // 这个会被 Parser 覆盖为 '/api/upload'
            onSuccess: function (res, file) {
              console.log('自定义处理');  // 这个会被 Parser 覆盖
            }
          }
        }
      ]
    }
  }
}
</script>
```

### 方式二：使用 option.global 普通配置 ​

使用 `option.global` 設定全域配置，只在規則中沒有定義對應屬性時才會生效。

vue

```
<template>
  <form-create :rule="rule" v-model:api="fApi" :option="options"/>
</template>

<script>
export default {
  data() {
    return {
      fApi: {},
      options: {
        // 表单级别的 beforeFetch 事件
        beforeFetch: function (config, data) {
          const token = data.api.getData('globalToken');
          if (!config.headers) {
            config.headers = {};
          }
          config.headers.Authorization = 'Bearer ' + token;
          return config;
        },
        global: {
          upload: {
            props: {
              // 只在规则中没有定义 action 时才会生效
              action: '/api/upload',
              // 只在规则中没有定义 onSuccess 时才会生效
              onSuccess: function (res, file) {
                file.url = res.data.url;
                file.name = res.data.name;
              },
              onError: function (err) {
                console.error('上传失败：', err);
              }
            }
          }
        }
      },
      rule: [
        {
          type: 'upload',
          field: 'upload1',
          title: '上传图片',
          props: {
            listType: 'picture-card'
            // 没有定义 action 和 onSuccess，会使用全局配置
          }
        },
        {
          type: 'upload',
          field: 'upload2',
          title: '上传文件',
          props: {
            action: '/custom-upload',  // 规则中定义了 action，全局配置不会生效
            onSuccess: function (res, file) {
              console.log('自定义处理');  // 规则中定义了 onSuccess，全局配置不会生效
            }
          }
        }
      ]
    }
  }
}
</script>
```

### 方式三：使用 option.global.deep 強制覆蓋 ​

使用 `deep` 屬性，透過點號路徑指定要強制覆蓋的屬性，會強制覆蓋規則中的同名屬性。

vue

```
<template>
  <form-create :rule="rule" v-model:api="fApi" :option="options"/>
</template>

<script>
export default {
  data() {
    return {
      fApi: {},
      options: {
        // 表单级别的 beforeFetch 事件
        beforeFetch: function (config, data) {
          const token = data.api.getData('globalToken');
          if (!config.headers) {
            config.headers = {};
          }
          config.headers.Authorization = 'Bearer ' + token;
          return config;
        },
        global: {
          upload: {
            // 深度合并配置：使用点号路径指定要覆盖的属性
            deep: {
              'props.action': '/api/upload',  // 强制覆盖规则中的 action
              'props.onSuccess': function (res, file) {
                // 强制覆盖规则中的 onSuccess
                file.url = res.data.url;
                file.name = res.data.name;
              },
              'props.onError': function (err) {
                console.error('统一的上传失败处理：', err);
              }
            }
          }
        }
      },
      rule: [
        {
          type: 'upload',
          field: 'upload1',
          title: '上传图片',
          props: {
            action: '/old-upload',  // 这个会被全局配置覆盖为 '/api/upload'
            onSuccess: function (res, file) {
              console.log('自定义处理');  // 这个会被全局配置覆盖
            }
          }
        },
        {
          type: 'upload',
          field: 'upload2',
          title: '上传文件',
          props: {
            // 没有定义 action 和 onSuccess，会使用全局配置
          }
        }
      ]
    }
  }
}
</script>
```

### 設定全域 beforeFetch 事件 ​

`beforeFetch` 是表單級別的事件，需要在 `option` 中配置，而不是在元件的 `props` 中。該事件會在所有上傳請求之前觸發，適合統一設定請求頭、修改請求引數等。

vue

```
<template>
  <form-create :rule="rule" v-model:api="fApi" :option="options"/>
</template>

<script>
export default {
  data() {
    return {
      fApi: {},
      options: {
        // 全局 beforeFetch 事件，所有上传请求都会触发
        beforeFetch: function (config, data) {
          // 从外部数据获取全局 token
          const token = data.api.getData('globalToken') || 
                       data.api.getData('$cookie.token') || 
                       '';

                       // 设置请求头
          if (!config.headers) {
            config.headers = {};
          }
          config.headers.Authorization = 'Bearer ' + token;
          config.headers['X-Requested-With'] = 'XMLHttpRequest';

          // 可以修改上传地址
          // config.url = '/api/custom-upload';

          // 可以添加额外的请求参数
          // config.data = {
          //   ...config.data,
          //   category: 'image'
          // };

          return config;
        }
      },
      rule: [
        {
          type: 'upload',
          field: 'upload1',
          title: '上传图片',
          props: {
            action: '/api/upload'
            // beforeFetch 会在上传前自动触发
          }
        }
      ]
    }
  },
  mounted() {
    // 设置全局 token，供 beforeFetch 使用
    this.$formCreate.setData({
      globalToken: 'your-token-here'
    });
  }
}
</script>
```

使用建議

- **Parser 配置**：適用於需要強制統一所有上傳元件配置的場景，執行時機最早，會覆蓋規則中的配置
- **普通配置（option.global）**：適用於需要為未配置的上傳元件提供預設值的場景，不會覆蓋規則中已存在的配置
- **深度合併配置（option.global.deep）**：適用於需要強制覆蓋規則中配置的場景，即使規則中已經定義了對應屬性
- **beforeFetch 事件**：適用於需要統一設定請求頭、修改請求引數等場景，在表單級別配置即可

## 配置項 ​

上傳元件提供了豐富的配置選項，您可以透過在設計器中配置屬性來自定義上傳元件的行為。

| 屬性名 | 型別 | 預設值 | 必需 | 說明 |
| --- | --- | --- | --- | --- |
| disabled | Boolean | - | 否 | 是否禁用 |
| listType | String | picture-card | 否 | 檔案列表型別（text/picture/picture-card） |
| multiple | Boolean | - | 否 | 是否支援多檔案上傳 |
| name | String | - | 否 | 上傳的檔案欄位名 |
| accept | String | - | 否 | 接受上傳的檔案型別 |
| action | String | / | 否 | 上傳的地址 |
| beforeUpload | Function | - | 否 | 上傳檔案之前的鉤子 |
| beforeRemove | Function | - | 否 | 刪除檔案之前的鉤子 |
| onSuccess | Function | - | 否 | 檔案上傳成功時的鉤子 |
| headers | Object | - | 否 | 設定上傳請求頭 |
| data | Object | - | 否 | 上傳時附帶的額外引數 |
| withCredentials | Boolean | - | 否 | 支援傳送 cookie 憑證資訊 |
| autoUpload | Boolean | true | 否 | 是否自動上傳 |
| limit | Number | - | 否 | 最大允許上傳個數 |

**重要說明：**

- **listType**：設定上傳檔案列表的展示方式，`picture-card` 適用於圖片上傳
- **onSuccess**：必須在此回撥中設定 `file.url`，否則表單無法獲取檔案資料
- **accept**：可以設定為檔案型別（如 `image/*`、`.pdf` 等）來限制上傳檔案型別
- **action**：上傳介面地址，可以是完整的 URL 或相對路徑

## 事件 ​

上傳元件提供了豐富的事件，方便您監聽上傳狀態變化並執行相應的處理。

| 事件名 | 引數 | 說明 |
| --- | --- | --- |
| change | file, fileList | 檔案狀態改變時的鉤子 |
| remove | file, fileList | 檔案列表移除檔案時的鉤子 |
| preview | file | 點選檔案列表中已上傳的檔案時的鉤子 |
| error | error, file, fileList | 上傳失敗時的鉤子 |
| progress | event, file, fileList | 上傳進度時的鉤子 |
| exceed | files, fileList | 檔案超出個數限制時的鉤子 |

## 方法 ​

上傳元件內部提供了多種方法，用於處理檔案上傳的各種操作。

| 方法名 | 引數 | 說明 | 返回值 |
| --- | --- | --- | --- |
| clearFiles | - | 清空已上傳的檔案列表 | - |
| abort | file | 取消上傳 | - |
| submit | - | 手動上傳檔案列表 | - |

### 方法使用示例 ​

**清空檔案列表**：

js

```
function clearUpload($inject) {
    const uploadInstance = $inject.api.el('ref_F2vulxvqc841dac');
    uploadInstance.clearFiles();
}
```

## 注意事項 ​

1. **必須設定 file.url**：在 `onSuccess` 回撥中必須設定 `file.url = res.data.url`，否則表單提交時無法獲取檔案資訊。
2. **beforeFetch 事件**：如果需要新增上傳請求頭（如認證 token），應該在表單的 `beforeFetch` 事件中設定，而不是在元件的 `headers` 屬性中設定。
3. **不同 UI 框架的配置差異**：element-plus 和 ant-design-vue 在上傳元件的配置上存在差異，請根據使用的 UI 框架檢視對應文件：

- **Element Plus**：[Upload 上傳元件](https://element-plus.org/zh-CN/component/upload.html)
- **Ant Design Vue**：[Upload 上傳元件](https://www.antdv.com/components/upload-cn)
