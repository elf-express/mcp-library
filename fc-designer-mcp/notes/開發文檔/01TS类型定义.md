# TS型別定義 ​

本文件詳細介紹了 `fcDesigner` 中各個核心資料結構的 `TypeScript` 定義。這些資料結構定義瞭如何生成元件、拖拽元件、配置設計器等。透過理解這些資料結構，你可以自定義和擴展設計器的功能，以滿足不同的業務需求。

## 元件生成規則 ​

**元件的 JSON 資料結構**

`Rule` 是描述元件生成規則的核心資料結構。每個元件都有一個唯一的 _fc_id 識別符號和其他屬性，用於定義元件的型別、樣式、事件、驗證規則等，檢視[完整配置說明](https://form-create.com/v3/guide/rule)。

ts

```
type Rule = {
    //规则唯一值
    _fc_id: string;
    //拖拽规则ID
    _fc_drag_tag: string;
    //拖拽模板规则ID
    _fc_template?: string;
    //生成组件名称
    type: string;
    //字段 ID
    field?: string;
    //编号
    name?: string;
    //字段名称
    title?: string;
    //提示信息
    info?: string;
    //组件的值
    value?: any;
    //组件的配置
    props: Object;
    //组件样式
    style: string | Object;
    //组件class
    class: string | Array;
    //组件事件
    on?: { [key: string]: Function | Function[] };
    //插槽名
    slot?: string;
    //组件的key
    key?: string;
    //是否必填
    $required: boolean | string;
    //组件的选择项
    options?: Array;
    //选择项对应的组件prop名称
    optionsTo?: string;
    //不使用formItem和wrap包裹
    native?: boolean;
    //是否隐藏组件(无dom)
    hidden?: boolean;
    //是否显示组件(有dom)
    display?: boolean;
    //是否进入阅读模式
    preview?: boolean;
    //自定义阅读模式回显
    readMode?: 'custom';
    //组件验证规则
    validate?: Object[];
    //子组件
    children?: Rule[];
    //联动数据
    control?: Array;
    //FormItem配置
    wrap?: Object;
    //逻辑条件/计算公式
    computed?: {
        [key: string]: string | Object;
    };
    //自定义属性
    effect?: {
        //远程数据
        fetch?: Object,
    };
    //其他扩展属性
    [key: string]: any;
}
```

**說明**

- **_fc_id:** 元件規則的唯一識別符號，用於在設計器中唯一定位每個元件。
- **props:** 元件的屬性配置，用於定義元件的行為和外觀。
- **on:** 元件的事件處理器，可以配置多個事件監聽。
- **validate:** 用於表單驗證的規則列表。
- **children:** 可以包含子元件，用於生成複雜的元件結構。

## 容器元件可拖入規則 ​

`AllowDrag` 和 `DenyDrag` 用於定義容器元件的拖拽規則，分別控制哪些元件可以被拖拽到容器中，以及哪些元件不能被拖拽。

ts

```
//可拖入的组件列表
type AllowDrag = string[] | {
    //可拖入的菜单列表
    menu: string[];
    //可拖入的组件列表
    item: string[];
}
//不可拖入的组件列表
type DenyDrag = string[] | {
    //不可拖入的菜单列表
    menu: string[];
    //不可拖入的组件列表
    item: string[];
}
```

**說明**

- **AllowDrag:** 定義容器中允許拖拽的元件或選單。可以是一個元件 `name` 的陣列，或者一個物件，其中 `menu` 和 `item` 分別指定允許的選單和元件。
- **DenyDrag:** 定義容器中不允許拖拽的元件或選單。可以是一個元件 `name` 的陣列，或者一個物件，其中 `menu` 和 `item` 分別指定禁止的選單和元件。
- **優先順序:** `AllowDrag` 配置優先順序高於 `DenyDrag`。

**使用方法**

在容器元件或者`config`中配置這些規則，以控制拖拽行為：

js

```
// 设置可拖拽规则
const allowDrag = {
    menu: ['main'],
    item: ['button', 'checkbox']
};
// 设置不可拖拽规则
const denyDrag = {
    menu: ['subform'],
    item: ['date-picker']
};
```

## 拖拽元件 ​

**描述規則結構**

`DragRule` 用於定義設計器中的拖拽元件規則。每個拖拽元件都有一個唯一的 `name` 識別符號，並透過 `rule` 方法生成元件規則。

ts

```
//多语言读取函数
type t = (name, ...args) => string;
//拖拽组件描述规则结构
interface DragRule {
    //组件id，不能重复
    name: string;
    //组件的名称
    label: string;
    //组件的图标
    icon: string;
    //插入的分类
    menu?: MenuName;
    //是否支持样式配置
    style?: boolean;
    //是否是行内组件
    inline?: boolean;
    //是否是表单组件
    input?: boolean;
    //控制最多拖入几个子组件
    maxChildren?: number;
    //判断组件是否可以拖入
    checkDrag?: (drag: {rule: Rule | undefined, menu: DragRule, toRule: Rule, toMenu: DragRule})=> boolean;
    //如果是子表单组件,需要定义`value`的类型
    subForm?: 'object' | 'array';
    //可拖入的组件列表(容器组件可用)
    allowDrag?: AllowDrag;
    //不可拖入的组件列表(容器组件可用)
    denyDrag?: DenyDrag;
    //允许拖入到那些组件中
    allowDragTo?: string | string[];
    //组件，不建议使用
    component?: Component;
    //渲染子组件的列表
    subRender: ({h, resolveComponent, subRule, rule}:{h:  typeof H<any>, resolveComponent: typeof ResolveComponent, subRule: Rule, rule:Rule})=>Array<{
        label: string;
        vnode: VNode
    }>;
    //配置组件条件选择方式
    condition: conditionComponentType | ((rule: Rule) => ({
        //组件类型
        type: conditionComponentType;
        //选择项
        options?: any[];
        //组件类型
        props?: Object;
    }));
    //模块组件配置参数
    container?: {
        //formData保存字段位置
        formDataField: string;
        //模块名称字段位置
        labelField: string;
    };
    //多语言配置项
    languageKey: string[];
    //表单全局配置
    formOptions?: {
        globalClass?: GlobalClass;
        globalVariable?: GlobalVariable;
        globalData?: GlobalData;
        globalEvent?: GlobalEvent;
        language?: Object;
        style?: string;
    } | string;
    //组件的生成规则
    rule(arg: { t: t }): Rule;
    //组件属性配置的规则
    props(rule: Rule, arg: { t: t, api: Api }): Rule[];
    //导出规则时通过这个方法转成最终规则
    parseRule?: (rule: Rule) => void;
    //导入规则时通过这个方法转成设计器中的渲染规则
    loadRule?: (rule: Rule) => void;
    //当props中的字段变化时触发
    watch?: {
        [key: string]: (arg: { value: any, rule: Rule, api: Api, field: string }) => void;
    };
    //是否有配套的子组件，例如Row和Col
    children?: string;
    //初始化时渲染几个子组件
    childrenLen?: number;
    //当前组件的操作容器是否显示在组件内部，为false时操作容器包裹当前组件
    inside?: true | boolean;
    //是否可以拖入其他组件到当前组件内部
    drag?: true | string | boolean;
    //是否显示拖拽按钮
    dragBtn?: false | boolean;
    //控制操作操作按钮是否显示，显示哪些
    handleBtn?: true | boolean | Array<'create' | 'copy' | 'addChild' | 'delete'>;
    //隐藏基础配置中的字段
    hiddenBaseField?: string[];
    //是否显示遮罩, 避免对组件操作. 建议有子组件时为true,其他为false
    mask?: false | boolean;
    //是否只能拖入一个
    only?: boolean;
    //是否生成name
    aide?: boolean;
    //当前组件的插槽
    easySlots?: Array<string| {
        //插槽名称
        value: string;
        //插槽简介
        label?: string;
        //类型
        type?: 'icon' | 'text'
    }>;
    //当前组件支持的事件
    event?: Array<string | { name: string; info: string }>;
    //当前组件`value`的数据类型
    validate?: string[] | 'required' | boolean;
}
```

**說明**

- **rule方法:** 生成元件的規則，返回一個 `Rule` 物件。
- **props方法:** 用於定義元件的屬性配置表單。
- **watch:** 監聽元件屬性變化，並作出相應處理。
- **children:** 用於支援具有子元件的元件，例如 `Row` 和 `Col`。

**使用方法**

定義好 DragRule 後，透過 designer.addComponent 方法匯入元件規則。例如：

js

```
designer.addComponent(dragRule);
```

你也可以批次匯入多個規則：

js

```
designer.addComponent([dragRule1, dragRule2, dragRule3]);
```

## 拖拽模板 ​

**描述規則結構**`DragTemplateRule` 用於定義拖拽模板的規則結構。這些模板會顯示在設計器左側的模板區域，並可以拖拽到表單中使用。

ts

```
//拖拽模板描述规则结构
interface DragTemplateRule {
    //固定
    menu: 'template';
    //模板的id,不能重复
    name: string;
    //模板的名称
    label: string;
    //是否只能拖入一个
    only?: boolean;
    //是否自动替换规则中的字段名, 表单中可能存在多个时建议为true
    autoField?: false | boolean;
    //表单全局配置
    formOptions?: {
        globalClass?: GlobalClass;
        globalVariable?: GlobalVariable;
        globalData?: GlobalData;
        globalEvent?: GlobalEvent;
        language?: Object;
        style?: string;
    } | string;
    //模板的渲染规则,或者json规则,或者通过函数返回json规则
    template: Rule[] | string | ((arg: { t: t }) => string);
}
```

**說明**

- **template:** 模板的渲染規則，支援多種格式，包括 JSON、字串或函式返回的規則。
- **autoField:** 自動生成欄位 ID，當表單中可能存在多個相同模板時建議設定為 true。
- **only:** 是否只能拖入一個。

**使用方法**

與拖拽元件類似，透過 designer.addComponent 方法匯入模板規則：

js

```
designer.addComponent(dragTemplateRule);
```

## 設計器左側選單列表 ​

ts

```
//拖拽组件
interface MenuItem {
    //拖拽组件名
    label: string;
    //拖拽组件id
    name: string;
    //拖拽组件图标
    icon: string;
}
//菜单
interface Menu {
    //菜单名
    title: string;
    //菜单id
    name: string;
    //拖拽组件列表
    list: MenuItem[];
    //放在列表最顶部
    before?: boolean;
}
type MenuList = Menu[];
type MenuName = 'main' | 'aide' | 'layout' | 'subform' | 'template' | 'container' | 'chart' | string;
```

**說明** 內建的選單 id 分別為：`template`、`main`、`subform`、`container`、`chart`、`aide`、`layout`

- template 模板元件
- main 基礎元件
- subform 子表單元件
- container 容器元件
- chart 圖表元件
- aide 輔助元件
- layout 佈局元件

## 設計器配置 ​

**設計器的`props.config`的配置**

`props.config` 用於配置設計器的各類行為和介面元素。透過這些配置項，你可以對設計器進行定製化調整，以便更好地滿足專案需求。以下是該配置的詳細資料結構及其使用說明。

    //控制组件验证是否只显示必填验证

    //自定义变量列表
    varList?: VarItem[];
    //隐藏拖拽操作按钮
    hiddenDragMenu?: boolean;
    //隐藏拖拽按钮
    hiddenDragBtn?: boolean;
    //控制组件的配置权限
    componentPermission?: {
        //组件的 field
        field?: string | string[];
        //组件的 name

        //组件的 _fc_drag_tag

        //组件的 _fc_id

            //是否可以配置事件

    };

```

**使用方法**

透過 props.config 設定設計器的行為。例如：

vue

```
<fc-designer :config="designerConfig" />
```

其中 designerConfig 是基於上文所述結構的配置物件。

js

```
export default {
    data() {
        return {
            designerConfig: {
                formOptions: {},
                fieldReadonly: false,

## 全域資料來源 ​

全域資料來源用於提供表單中可能需要的各種資料。資料來源可以是靜態資料（如預定義的資料列表）或動態資料（透過遠端 API 獲取的資料）。

ts

```
interface StaticDataItem {
    //数据名称
    label: string;
    //是否可以删除
    deletable: boolean;
    //数据类型
    type: 'static';
    //数据
    result: any;
}

interface FetchDataItem {
    //数据名称
    label: string;
    //是否可以删除
    deletable: boolean;
    //数据类型
    type: 'fetch';
    //请求链接
    action: string;
    //请求方式
    method: 'GET' | 'POST';
    //请求头部
    headers?: Object;
    //附带数据
    data?: Object;
    //远程数据解析
    parse?: string | ((res: any) => any);
    //远程异常处理
    onError?: string | ((e) => void);
}

export interface GlobalData {
    [id:string] : StaticDataItem | FetchDataItem;
}
```

**使用說明**

- 使用 setGlobalData 方法匯入預設的資料。
- 也可以在表單配置中透過 options.globalData 來指定全域資料來源。
