# 管理欄位ID ​

在表單設計器中，欄位ID用於唯一標識每個元件的欄位。預設情況下，欄位ID是隨機生成的，且使用者無法輸入。您可以透過以下配置自定義欄位ID的輸入和選擇方式。

注意

1. 欄位 ID 必須以字母開頭，且只能包含字母、數字和下劃線。有效示例：'goods_brandName_01'。
2. 同一層級表單中的欄位 ID 必須保持唯一，重複會導致功能異常。

## 資料結構 ​

`FieldItem` 定義了欄位ID選擇項的結構：

ts

```
type FieldItem = {
    //字段名
    value: string;
    //别名
    label: string;
    //修改当前规则的必填,禁用和说明
    update?: {
        required?: Boolean;
        disabled?: Boolean;
        info?: string;
    };
    //子级字段列表
    children?: FieldItem[];
}
```

## 允許欄位ID可以輸入 ​

透過設定 config.fieldReadonly 屬性，您可以控制欄位ID輸入框是否可編輯：

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
    const config = {
        //控制字段ID输入框能否输入
        fieldReadonly: false
    }
</script>
```

- **fieldReadonly: true：** 欄位ID只能從選擇項中選擇，不能手動輸入。
- **fieldReadonly: false：** 欄位ID可以手動輸入或從選擇項中選擇。

## 自定義欄位ID的選擇項 ​

![custom-field.png](https://pro.form-create.com/doc/img/custom-field.png)

透過設定 config.fieldList 屬性，您可以定義欄位ID的選擇項列表。選擇項支援巢狀結構，便於組織和管理複雜的欄位列表。

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
    const config = {
        //控制字段ID只能选择,不能输入
        fieldReadonly: true,
        fieldList: [
            {
                value: 'goods',
                label: '商品表',
                children: [
                    {
                        value: 'goods_name',
                        label: '商品名称',
                    },
                    {
                        value: 'goods_info',
                        label: '商品简介',
                    },
                    {
                        value: 'goods_cate',
                        label: '商品分类',
                    },
                ],
            },
            {
                value: 'order',
                label: '订单表',
                children: [
                    {
                        value: 'order_id',
                        label: '订单id',
                    },
                    {
                        value: 'order_sn',
                        label: '订单号',
                    },
                    {
                        value: 'order_time',
                        label: '订单时间',
                    },
                ],
            },
        ]
    }
</script>
```

## 選擇欄位時修改元件規則 ​

透過 update 欄位，您可以在選擇欄位時動態修改元件的必填、禁用和說明等規則：

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
    const config = {
        //控制字段ID只能选择,不能输入
        fieldReadonly: false,
        fieldList: [
            {
                value: 'goods',
                label: '商品表',
                children: [
                    {
                        value: 'goods_name',
                        label: '商品名称',
                        update: {
                            required: true,
                            info: '请输入商品名称',
                            dislabed: false,
                        }
                    },
                    {
                        value: 'goods_info',
                        label: '商品简介',
                    },
                    {
                        value: 'goods_cate',
                        label: '商品分类',
                        update: {
                            required: false,
                            info: '请选择商品分类',
                            dislabed: true,
                        }
                    },
                ],
            },
        ]
    }
</script>
```

在上述配置中，選擇“商品名稱”欄位後，該元件會自動變為必填狀態，並顯示提示資訊。

## 子表單元件與內部表單元件聯動 ​

預設情況下，當子表單元件選擇的欄位存在 `children` 時，內部的其他元件只能從 `children` 中選擇。可以透過設定 `config.relationField` 來控制此功能：

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
    const config = {
        // 控制是否启用字段联动
        relationField: true // 设置为 false 以禁用字段联动
    }
</script>
```

- **relationField: true：** 啟用欄位聯動，子表單選擇的欄位會影響其他元件的選擇範圍。
- **relationField: false：** 禁用欄位聯動，所有元件可以自由選擇欄位。

## 動態修改欄位ID選項 ​

除了靜態設定欄位ID選擇項，您還可以根據業務需求動態修改欄位ID選項。這在需要根據使用者的選擇或其他條件動態調整欄位選項時特別有用。

**1. 定義動態欄位ID選項更新方法** 在元件內部定義一個方法，用於更新欄位ID選項列表。

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
    <button @click="updateFieldList">更新字段列表</button>
</template>
<script setup>
    import { ref } from 'vue';
    const config = ref({
        fieldReadonly: false,
        fieldList: [
            // 初始字段选项
        ]
    });

    // 更新字段列表的方法
    const updateFieldList = () => {
        config.value.fieldList = [
            {
                value: 'user',
                label: '用户表',
                children: [
                    { value: 'user_name', label: '用户名' },
                    { value: 'user_email', label: '用户邮箱' }
                ]
            },
            {
                value: 'product',
                label: '产品表',
                children: [
                    { value: 'product_name', label: '产品名称' },
                    { value: 'product_price', label: '产品价格' }
                ]
            }
        ];
    }
</script>
```

**2.在使用者操作時更新欄位ID選項**

透過使用者操作或其他事件觸發欄位ID選項的更新。例如，可以在表單載入後從伺服器獲取欄位ID選項，並根據業務邏輯進行更新。

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
    import { ref, onMounted } from 'vue';
    const config = ref({
        fieldReadonly: false,
        fieldList: []
    });

    // 获取动态字段列表的函数
    const fetchFieldList = async () => {
        // 模拟从服务器获取字段列表
        const response = await fetch('/api/fields');
        const data = await response.json();
        config.value.fieldList = data;
    }

    onMounted(() => {
        fetchFieldList();
    });
</script>
```

這樣，欄位ID列表會在元件載入時從伺服器獲取，並且可以根據實際業務需求進行調整。
