# 內建動作列表 ​

本文為你列舉了設計器中目前支援的所有內建動作，按分類如下：

![内置动作列表](https://pro.form-create.com/doc/img/book/image-20241016224333555.png)

## 頁面操作動作 (page) ​

| 動作名稱 | 功能描述 | 配置說明 |
| --- | --- | --- |
| redirectPage | 頁面跳轉 | 支援 location（當前視窗/新視窗）和 router（路由跳轉）兩種型別，使用 router 型別時需要注入 $router 例項 |
| reloadPage | 頁面重新整理 | 無配置引數，直接重新整理當前頁面 |

## 模型操作動作 (model) ​

| 動作名稱 | 功能描述 | 配置說明 |
| --- | --- | --- |
| openModel | 開啟彈窗 | 需要選擇已定義的彈窗元件（透過 model 欄位選擇） |
| closeModel | 關閉彈窗 | 需要選擇要關閉的彈窗元件（透過 model 欄位選擇） |
| message | 顯示訊息提示 | 支援 info、warning、success、error 四種型別，可配置訊息內容、顯示時長、是否顯示關閉按鈕 |
| confirm | 顯示確認對話方塊 | 需要配置標題和訊息內容，返回 Promise，使用者確認後繼續執行後續動作 |

## 表單操作動作 (form) ​

| 動作名稱 | 功能描述 | 配置說明 |
| --- | --- | --- |
| hidden | 隱藏/顯示欄位 | 支援靜態配置或公式計算，可同時操作多個欄位，透過 id 欄位選擇目標欄位 |
| disabled | 禁用/啟用欄位 | 支援靜態配置或公式計算，可同時操作多個欄位，透過 id 欄位選擇目標欄位 |
| resetFields | 重置表單 | 無配置引數，重置所有欄位到初始值 |
| clearFields | 清空表單 | 無配置引數，清空所有欄位的值 |
| validate | 驗證表單 | 無配置引數，驗證整個表單，返回 Promise |
| validateFields | 驗證指定欄位 | 透過 id 欄位選擇要驗證的欄位，可同時驗證多個欄位 |
| submit | 提交表單 | 無配置引數，觸發表單提交，返回 Promise |
| setValue | 設定欄位值 | 支援靜態值或公式計算，可同時設定多個欄位的值，透過 formData 陣列配置欄位和值 |

## 其他動作 (other) ​

| 動作名稱 | 功能描述 | 配置說明 |
| --- | --- | --- |
| fetch | 傳送請求 | 支援完整的請求配置（URL、方法、引數等），可配置是否追加表單資料，支援設定響應資料儲存位置 |
| print | 列印表單 | 無配置引數，呼叫瀏覽器列印當前表單內容 |
| exportPdf | 匯出 PDF | 無配置引數，將表單內容匯出為 PDF 檔案 |
| emitEvent | 丟擲業務事件 | 需要配置 eventName（事件名會加上 biz- 字首）和可選引數 args，用於設計器與業務邏輯解耦 |
| copy | 複製文字 | 支援靜態文字或公式計算，將內容複製到剪貼簿 |
| callback | 自定義回撥 | 需要編寫回撥函式，接收 $inject 引數，丟擲異常時會阻止後續動作執行 |

## 使用注意事項 ​

內建動作使用注意事項

- **路由依賴**：`redirectPage` 動作在使用 `router` 型別時需要透過 `formCreate.setData('$router', router)` 注入 `$router` 例項
- **欄位引用**：表單操作動作（`hidden`、`disabled`、`validateFields`、`setValue`）需要確保欄位ID正確且欄位存在
- **公式計算**：支援公式計算的動作（`hidden`、`disabled`、`setValue`、`copy`）可以透過 `compute` 選項啟用公式模式
- **Promise 處理**：返回 Promise 的動作（`validate`、`validateFields`、`submit`、`fetch`、`confirm`、`print`、`exportPdf`）會等待執行完成後再繼續後續動作
- **動作鏈執行**：多個動作按順序執行，如果某個動作返回 Promise，會等待其完成後再執行下一個動作
- **業務事件**：`emitEvent` 動作會觸發以 `biz-` 為字首的事件，可透過 `api.top.on('biz-事件名', handler)` 監聽
