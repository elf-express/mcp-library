# 表單渲染 ​

在使用設計好的規則進行表單設計和回顯時，正確地處理 JSON 資料格式是至關重要的。以下文件詳細介紹瞭如何渲染表單，幷包含了動態載入和儲存表單配置的高階用法示例。

注意!!!!!

**必須使用 `formCreate.parseJson` 代替 `JSON.parse` 方法，使用 `formCreate.toJson` 代替 `JSON.stringify` 方法來轉換 JSON 資料，以確保資料格式正確。**

## 表單渲染 ​

要渲染表單，您需要掛載 FormCreate 到 Vue 應用中，並載入表單規則和配置。

### 掛載 formCreate 示例 ​

js

```
//从设计器中导入 formCreate
import {formCreate} from 'path/to/fcDesignerPro';
//挂载 formCreate
app.use(formCreate);
```

**表單渲染示例**

以下示例演示如何獲取後端儲存的表單規則，並透過 formCreate 渲染表單。

vue

```
<template>
    <div id="app">
        <form-create v-if="rule.length" v-model="formData" v-model:api="api" @submit="handleSubmit" :rule="rule"
                     :option="option"></form-create>
    </div>
</template>
<script setup>
    import {formCreate} from 'path/to/fcDesignerPro';
    const rule = ref([]);
    const option = ref({});
    const api = ref(null);
    //表单数据
    const formData = ref({});
    function handleSubmit(formData) {
        // 模拟提交表单数据到后端
        axios.post('/api/submitFormData', formData)
            .then(response => {
                console.log('提交成功:', response.data);
            })
            .catch(error => {
                console.error('提交失败:', error);
            });
    }
    onMounted(() => {
        // 模拟从后端加载表单JSON规则
        axios.get('/api/getFormRules')
            .then(response => {
                const {ruleJson, optionsJson, formData: initFormData} = response.data;
                rule.value = formCreate.parseJson(ruleJson);
                option.value = formCreate.parseJson(optionsJson);
                formData.value = initFormData || {};
            })
            .catch(error => {
                console.error('加载表单规则失败:', error);
            });
    });
</script>
```

## 載入表單資料 ​

表單渲染解決的是 `rule/option`，而“載入表單資料”解決的是給表單欄位賦初始值（例如編輯已有記錄、草稿恢復、從 URL 帶預設值等）。

### 方式一：透過 `v-model` 直接繫結物件（推薦預設用法） ​

當你已經有一份資料物件（例如介面返回的詳情）時，直接賦值給 `formData` 即可觸發表單回顯。

vue

```
<template>
  <form-create
    v-if="rule.length"
    v-model="formData"
    v-model:api="api"
    :rule="rule"
    :option="option"
  />
</template>

<script setup>
import {formCreate} from 'path/to/fcDesignerPro';

const rule = ref([]);
const option = ref({});
const api = ref(null);
const formData = ref({});

async function loadSchema() {
  const res = await axios.get('/api/getFormRules');
  const {ruleJson, optionsJson} = res.data;
  rule.value = formCreate.parseJson(ruleJson);
  option.value = formCreate.parseJson(optionsJson);
}

async function loadRecord(recordId) {
  const res = await axios.get('/api/getRecordDetail', {params: {id: recordId}});
  // 这里的 key 应与 rule 中的字段 field 对应
  formData.value = res.data || {};
}

onMounted(async () => {
  await loadSchema();
  await loadRecord('10001');
});
</script>
```

Vue2 提示（使用 `value.sync`）

如果你在 **Vue2** 專案中使用表單渲染，常見做法是使用 `:value.sync="formData"` 來實現雙向繫結。

vue

```
<template>
  <form-create
    v-if="rule.length"
    :value.sync="formData"
    v-model="api"
    :rule="rule"
    :option="option"
  />
</template>
```

### 方式二：使用 `api` 主動設定值（適合“分批載入/區域更新”） ​

有些場景資料是分段到達的（例如先渲染再非同步補充某些欄位），或只想更新少數字段，這時可以使用 `api` 設定值。

js

```
// 等表单实例可用后（rule 渲染完成），再 setValue
await nextTick();
api.value?.setValue({
  name: '张三',
  age: 18,
});

// 或者只更新单个字段
api.value?.setValue('name', '李四');
```

## 注意事項 ​

- 確保從後端返回的 JSON 資料格式正確，以保證 `formCreate.parseJson` 的正常工作。
- 在實際實現時，根據你的 API 結構調整請求URL和資料欄位名稱。
- 表單回顯的關鍵是：**資料物件的 key 需要與規則中欄位的 `field` 一致**，否則不會對映到對應元件。

透過這些示例和說明，開發者可以在 Vue 應用中實現動態表單渲染與資料持久化提交，高效管理表單的設計和使用。
