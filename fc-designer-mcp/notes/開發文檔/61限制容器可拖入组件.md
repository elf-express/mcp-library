# 限制容器可拖入元件 ​

透過配置 `allowDrag` 和 `denyDrag`，您可以精細控制容器元件允許或禁止拖入的元件。這種控制可以幫助您根據具體的業務需求，定製拖拽功能的行為，確保表單設計的一致性和規範性。

## 資料結構 ​

ts

```
//可拖入的组件列表
type AllowDrag = string[] | {
    //可拖入的菜单列表
    menu: string[];
    //可拖入的组件列表
    item: string[];
}
//不可拖入的组件列表
type DenyDrag = string[] | {
    //不可拖入的菜单列表
    menu: string[];
    //不可拖入的组件列表
    item: string[];
}
```

- **AllowDrag:** 定義容器中允許拖拽的元件或選單。可以是一個元件 `name` 的陣列，或者一個物件，其中 `menu` 和 `item` 分別指定允許的選單和元件。
- **DenyDrag:** 定義容器中不允許拖拽的元件或選單。可以是一個元件 `name` 的陣列，或者一個物件，其中 `menu` 和 `item` 分別指定禁止的選單和元件。
- **優先順序:** `AllowDrag` 配置優先順序高於 `DenyDrag`。

## 基本示例 ​

在一個“佈局”容器中，只允許拖入佈局相關的元件，如行和列元件。

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
    const config = {
        allowDrag: {
            group: {
                item: ['fcRow']
                menu: ['main']
            }
        }
    }
</script>
```

## 限制不能拖入的元件 ​

在一個分組元件中，禁止拖入子表單元件，但允許其他元件。

vue

```
<template>
    <fc-designer ref="designer" :config="config"/>
</template>
<script setup>
    const config = {
        denyDrag: {
            subForm: {
                menu: ['subform']
            }
        }
    }
</script>
```

## 透過元件的拖拽規則限制 ​

在一個格子元件中，僅允許拖入基礎元件。

js

```
const colRule = {
    menu: 'layout',
    icon: 'icon-col',
    label: '格子',
    name: 'col',
    //可以向内部拖入组件
    drag: true,
    //控制可拖入的组件
    allowDrag: {
        menu: ['main']
    },
    inside: true,
    mask: false,
    rule() {
        return {
            type: 'Col',
            props: {span: 12},
            children: []
        };
    },
    props(_, {t}) {
        return [{
            type: 'slider',
            title: '宽度',
            field: 'span',
            value: 12,
            props: {min: 0, max: 24}
        }];
    }
};
```
