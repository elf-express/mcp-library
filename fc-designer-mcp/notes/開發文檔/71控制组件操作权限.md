# 控制元件操作許可權 ​

`config.componentPermission` 配置項允許開發者對錶單設計器中的元件進行細粒度的許可權控制，包括操作許可權和配置項許可權。

## 資料結構 ​

ts

```
//控制组件的配置权限
type ComponentPermission = {
    // 匹配组件标识字段（可单选或多选）
    //组件的 field
    field?: string | string[];
    //组件的 name
    name?: string | string[];
    //组件的 _fc_drag_tag
    tag?: string | string[];
    //组件的 _fc_id
    id?: string | string[];
    //权限
    permission: {
        // 基础操作权限
        delete?: boolean    // 是否允许删除
        copy?: boolean     // 是否允许复制
        move?: boolean     // 是否允许拖动
        // 配置项权限
        validate?: boolean  // 验证规则配置
        event?: boolean     // 事件配置
        advanced?: boolean  // 高级配置
        base?: boolean      // 基础配置
        props?: boolean     // 组件属性配置
        style?: boolean     // 样式配置
        slots?: boolean     // 插槽配置
        switchType?: boolean // 组件类型切换
        name?: boolean      // 是否显示组件名称
        aiEdit?: boolean    // 是否显示 AI 编辑按钮
        // 精细化控制
        hiddenConfig?: string[]   // 需要隐藏的配置项名称
        disabledConfig?: string[] // 需要禁用的配置项名称
    },
}[],
```

**在 JSON 面板中可以檢視對應元件的相關 id 和配置名稱**

## input 元件禁止刪除和複製 ​

限制 input 元件的操作許可權，禁止使用者刪除和複製該型別的表單元件

vue

```
<template>
    <fc-designer ref="designer" :config="config"></fc-designer>
</template>
<script>
    export default {
        name: 'Component',
        data() {
            return {
                config: {
                    componentPermission: [
                        {
                            tag: 'input',
                            permission: {
                                delete: false,
                                copy: false,
                            }
                        }
                    ]
                },
            };
        }
    };
</script>
```

## 透過元件的 field 隱藏對應元件的 validate 配置項 ​

透過指定欄位名（field1、field2），來隱藏對應元件的驗證（validate）配置項

vue

```
<template>
    <fc-designer ref="designer" :config="config"></fc-designer>
</template>
<script>
    export default {
        name: 'Component',
        data() {
            return {
                config: {
                    componentPermission: [
                        {
                            field: ['field1', 'field2'],
                            permission: {
                                validate: false,
                            }
                        }
                    ]
                },
            };
        }
    };
</script>
```

## 隱藏 input 元件的disabled 配置項 ​

透過隱藏 input 元件的 disabled 配置項，從而限制使用者在表單設計器中修改該元件的禁用狀態屬性。

vue

```
<template>
    <fc-designer ref="designer" :config="config"></fc-designer>
</template>
<script>
    export default {
        name: 'Component',
        data() {
            return {
                config: {
                    componentPermission: [
                        {
                            tag: 'input',
                            permission: {
                                hiddenConfig: ['disabled'],
                            }
                        }
                    ]
                },
            };
        }
    };
</script>
```

## 禁用 select 元件的樣式配置 ​

透過配置 ComponentPermission 來禁用 select 元件的樣式配置功能，從而限制使用者在表單設計器中修改該元件的樣式屬性。

vue

```
<template>
    <fc-designer ref="designer" :config="config"></fc-designer>
</template>
<script>
    export default {
        name: 'Component',
        data() {
            return {
                config: {
                    componentPermission: [
                        {
                            tag: 'select',
                            permission: {
                                style: false,
                            }
                        }
                    ]
                },
            };
        }
    };
</script>
```
