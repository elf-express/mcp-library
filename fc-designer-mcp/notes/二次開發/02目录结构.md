# 目錄結構 ​

本文介紹設計器專案的目錄結構與各模組職責，幫助你在二次開發時快速定位“入口在哪裡、規則在哪裡、元件在哪裡、構建產物會輸出到哪”。

專案概況：

- **技術棧**：Vue 3 + Element Plus/Ant Design Vue（PC 設計器/渲染器），同時包含 Vant（移動端渲染器）
- **構建工具**：Vite（多份 config 按目標產物拆分）、Gulp（多語言產物打包）
- **核心目標**：視覺化拖拽生成規則（DragRule），並在設計態/執行態以資料驅動渲染表單

## 快速定位：從哪裡開始看？ ​

如果你第一次看這個專案，建議按下面順序“自頂向下”閱讀：

- **入口匯出與全域註冊**：`src/index.js`
- **設計器畫布主體**：`src/designer/`（核心入口檔案在該目錄下）
- **左側元件面板 / 拖拽規則集合**：`src/config/index.js`、`src/config/rule/*`
- **選單與分組**：`src/config/menu.js`
- **渲染器（runtime）註冊**： 

- Element Plus：`src/form/elm.js`
- Ant Design Vue：`src/form/antdv.js`
- Vant（mobile）：`src/form/mobile.js`
- **擴展點（元件/能力注入）**：`src/extension/setting.js`
- **本地開發示例專案**：`examples/*`（`pnpm run dev` 直接跑這裡）

## 根目錄：專案級配置與多目標構建 ​

根目錄主要是“專案配置 + 構建配置 + 演示入口 + 產物輸出”。

### `package.json` ​

- **`vite.dev.config.js`**：開發環境配置，`root` 指向 `examples/`
- **`vite.config.build.js`**：完整包構建，lib entry 為 `src/index.js`，輸出到 `dist/`（`index.es.js` / `index.umd.js`）
- **`vite.config.pc.js`**：PC 設計器包構建，輸出到 `dist/pc/`
- **`vite.config.elm.js`**：Element Plus runtime 渲染器構建，entry 為 `src/form/elm.js`，輸出到 `dist/render/element-plus/`
- **`vite.config.antdv.js`**：Ant Design Vue runtime 渲染器構建，entry 為 `src/form/antdv.js`，輸出到 `dist/render/ant-design-vue/`
- **`vite.config.mobile.js`**：Vant runtime 渲染器構建，entry 為 `src/form/mobile.js`，輸出到 `dist/render/vant/`
- **`vite.config.preview.js`**：預覽頁構建（預設 `base: '/view/'`，按部署需要調整）
- **`vite.config.locale.js`**：多語言構建（entry 為 `src/locale/<lang>.js`，輸出到根目錄 `locale/`）

### `gulpfile.js`：多語言批次構建 ​

`build:locale` 會：

- 先清理根目錄 `locale/`
- 遍歷 `src/locale/*.js`
- 按語言逐個呼叫 `vite build --config vite.config.locale.js -m <lang>`

### `examples/`：本地開發與演示專案 ​

開發時你執行的不是 `src/` 直接啟動，而是 `examples/`：

- `examples/main.js` / `examples/App.vue` 作為演示入口
- `vite.dev.config.js` 把 Vite 的 root 設定為 `examples/`，因此本地除錯體驗更接近真實使用方

## `src/`：核心原始碼分層 ​

### `src/index.js`：包入口（註冊與匯出） ​

你可以把它理解為“總裝配線”：

- **匯入設計器核心元件**：例如 `src/designer/` 下的主入口元件
- **註冊設計態需要的元件**：透過 `designerForm.component(...)`
- **註冊執行態元件**：透過 `formCreate.component(...)`（同時相容設計態/預覽態）
- **掛載擴展點**：讀取 `extension/setting.js` 的擴展元件與更新鉤子
- **匯出 API**：對外暴露 `addDragRule`、`addMenu`、`useLocale`、`setFormula`、`setBehavior` 等

做二次開發時，絕大多數“全域層面的新增能力”最終都會落在這裡完成接線。

### `src/designer/`：設計器 UI（拖拽畫布 + 配置面板） ​

這裡是設計器的主體實現，典型職責包括：

- **畫布與拖拽互動**：拖入元件、排序、選中、快捷佈局等
- **右側配置面板**：屬性（Props）、樣式（Style）、校驗（Validate）、事件（Event）、計算（Computed）等
- **匯入匯出/預覽/列印**：如 `ImportDialog.vue`、`JsonPreview.vue`、`PrintForm.vue`
- **高階能力**： 

- `computed/`：條件/變數/表示式等配置元件
- `fetch/`：資料來源/介面配置
- `language/`：多語言配置面板
- `style/`：樣式編輯器與各類樣式輸入控制元件
- `ai/`：AI 相關面板與渲染

當你需要“改介面、改互動、改右側面板的配置能力”，大機率就在這裡動手。

### `src/components/`：執行態元件（包含 View/預覽元件） ​

這裡存放的是表單渲染時真正參與互動的元件，以及一些“只讀/預覽展示”的 View 元件：

- **PC 元件**：如 `SignaturePad.vue`、`DataTable.vue`、`TableForm.vue` 等
- **配套 View 元件**：如 `DialogView.vue`、`JsonComponentView.vue`、`ValueView.vue`
- **移動端元件**：集中在 `components/mobile/`（配合 Vant 渲染器）

通常規則（DragRule）決定了你最終會渲染哪個元件型別；元件實現則決定了互動細節。

### `src/config/`：選單、規則、模板與基礎配置 ​

這是“讓設計器知道有哪些元件、如何拖拽生成規則、右側顯示哪些配置項”的核心目錄：

- **`config/menu.js`**：左側面板選單結構與分組
- **`config/index.js`**：拖拽規則的彙總入口（`dragRuleList`）
- **`config/rule/`**：每個元件一個檔案，定義 DragRule（最常改）
- **`config/template/`**：預置模板（如快速生成幾列佈局、表格佈局片段等）
- **`config/base/`**：通用配置片段（欄位/表單/樣式/校驗等“基礎項”）

新增一個可拖拽元件的常見落點：

- 元件實現：`src/components/<YourComp>.vue`（必要時也加移動端版本）
- DragRule：`src/config/rule/<yourRule>.js`
- 規則彙總/選單配置：`src/config/index.js`、`src/config/menu.js`
- 註冊接線：`src/index.js`（設計態/執行態註冊）

### `src/form/`：執行態渲染器入口（PC/移動） ​

這層負責把元件與渲染器環境串起來，並提供各端樣式：

- Element Plus runtime：`form/elm.js` + `form/elm.css`
- Ant Design Vue runtime：`form/antdv.js` + `form/antdv.css`
- Vant runtime：`form/mobile.js` + `form/mobile.css`

當你遇到“某個 rule.type 在執行態渲染不出來 / 樣式不一致”，優先看這裡是否註冊/引入齊全。

### `src/extension/`：二次擴展入口 ​

這裡提供“面向外部擴展”的集中出口，典型包含：

- **擴展元件集合**：例如 `setting.extend.components`
- **設計器更新鉤子**：例如 `setting.updateDesigner(...)`
- **元件檢視對映**：例如 `setting.componentViews`

建議把“專案級自定義擴展”儘量收斂到這裡，避免把業務邏輯散落在 `src/index.js`。

### `src/utils/`：跨模組通用能力 ​

包括但不限於：

- **公式/行為**：`formulas.js`、`behavior.js`
- **預覽/匯出/模板**：`preview.js`、`template.js`、`sfc.js`
- **多語言工具**：`locale.js`

當你想複用能力或抽離通用邏輯時，優先考慮放在這裡。

### `src/locale/` 與根目錄 `locale/` ​

- `src/locale/`：原始碼層語言包（如 `en.js`、`zh-cn.js`）
- 根目錄 `locale/`：構建後的語言產物（`build:locale` 生成，包含 `*.es.js` 等）

## 構建產物：會輸出到哪裡？ ​

結合 `package.json` 的 scripts 與 Vite 配置，主要產物路徑如下：

- **完整包**：`dist/index.es.js`、`dist/index.umd.js`（`vite.config.build.js`）
- **PC 設計器包**：`dist/pc/*`（`vite.config.pc.js`）
- **PC 渲染器（Element Plus runtime）**：`dist/render/element-plus/form-create.*.js`（`vite.config.elm.js`）
- **PC 渲染器（Ant Design Vue runtime）**：`dist/render/ant-design-vue/form-create.*.js`（`vite.config.antdv.js`）
- **移動端渲染器（Vant runtime）**：`dist/render/vant/form-create.*.js`（`vite.config.mobile.js`）
- **多語言產物**：`locale/*`（`gulpfile.js` + `vite.config.locale.js`）
