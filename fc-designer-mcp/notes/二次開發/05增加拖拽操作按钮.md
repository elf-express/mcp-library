# 增加拖拽操作按鈕 ​

在二開場景中，你可能希望給畫布上的元件增加“額外的快捷操作”，例如：一鍵高亮、快速生成聯動、繫結業務資料、開啟自定義彈窗配置等。本文講清楚在設計器中如何在元件右下角的操作區新增一個按鈕，並透過事件把“目標元件規則（rule）”傳遞出去執行自定義邏輯。

> 你可以把它理解為：仿照已有的 `copy` / `delete`，新增一個 `custom`（或任意名稱）的“元件操作”事件。

## 你需要了解 ​

- 畫布上每個元件都會被 `DragTool` 包一層，用於渲染“移動/複製/刪除”等操作按鈕
- `DragTool` 只負責 `$emit(...)` 拋事件，真正的“複製/刪除”等邏輯在 `FcDesigner.vue` 的 `handleToolCopy / handleToolDelete` 中實現

## 你要改哪些地方？ ​

增加一個“元件操作按鈕”通常涉及 2 個檔案：

- **操作按鈕 UI 與事件丟擲**：`src/designer/DragTool.vue`

- 新增一個按鈕
- 增加對應的 `emits` 事件名
- **事件接收與執行自定義邏輯**：`src/designer/FcDesigner.vue`

- 在構造 `DragTool` 節點時補齊 `on: { custom: ... }` 的監聽
- 新增一個 `handleToolCustom`（命名自定）方法，統一拿到 `targetRule` 並執行自定義操作

## 參考樣例：`copy` / `delete` 的完整鏈路 ​

在原始碼中，`copy/delete` 的鏈路非常典型：

### 1. `DragTool.vue` 裡只做 `$emit` ​

`copy/delete` 按鈕點選後，直接觸發：

- `$emit('copy')`
- `$emit('delete')`

你新增按鈕時沿用這個模式即可。

### 2. `FcDesigner.vue` 裡把事件對映到處理函式 ​

`FcDesigner.vue` 在把元件規則包裝成 `DragTool` 時，會在 `on` 中繫結事件處理：

- `copy: ({ self }) => methods.handleToolCopy({ self, inside: true/false })`
- `delete: ({ self }) => methods.handleToolDelete({ self, inside: true/false })`

### 3. `handleToolCopy/handleToolDelete` 統一拿到 `targetRule` ​

關鍵點在於：**同一個按鈕事件，需要相容兩種包裝模式**：

- `inside === true`：`DragTool` 在規則的 `children` 內（“內嵌模式”）
- `inside === false`：`DragTool` 是外層 wrapper（“外包裹模式”）

所以原始碼裡會用同樣的判斷統一得到“真正被操作的規則”：

- `targetRule = inside ? methods.getParent(self).parent : self.children[0]`

新增自定義操作，也建議複用這一套 `targetRule` 計算方式。

## 方案設計與約定 ​

### 事件命名建議 ​

- **按鈕事件名**建議用語義清晰、可擴展的名字，例如： 

- `custom`（通用）
- `highlight`（高亮）
- `bind`（繫結）
- `linkage`（聯動）

### 顯隱與許可權 ​

`DragTool.vue` 對 `copy/delete` 的顯示控制包含兩層：

- **按鈕白名單**：透過 `handleBtn`（陣列）控制顯示哪些按鈕（例如 `['copy','delete']`）
- **許可權物件**：透過 `permission.xxx !== false` 控制某個按鈕是否允許

新增按鈕時建議也按同樣方式接入（這樣業務方可以用“配置/許可權”控制按鈕顯隱，而不是寫死在原始碼裡）。

## 實現步驟 ​

下面以新增一個 `custom` 操作為例（你可以替換成任意事件名）。

### 1. 在 `DragTool.vue` 新增按鈕並丟擲事件 ​

檔案：`src/designer/DragTool.vue`

#### 1.1 增加 emits 宣告 ​

把 `custom` 加到 `emits` 陣列中（示例）：

js

```
emits: ['create', 'copy', 'addChild', 'delete', 'custom', 'active', 'action', 'fc.el'],
```

#### 1.2 在操作區增加按鈕 ​

參照 `copy/delete` 的寫法，新增一段按鈕（示例用 `icon-setting`，你也可以換成你們的業務圖示）：

vue

```
<div
  class="_fd-drag-btn"
  @click.stop
  v-if="permission.custom !== false && (btns === true || btns.indexOf('custom') > -1)"
  @click="$emit('custom')"
>
  <i class="fc-icon icon-setting"></i>
</div>
```

建議

如果你希望按鈕只在“選中啟用”時顯示，不需要額外處理：`DragTool` 已有樣式邏輯，會在 `active/hover` 時顯示按鈕。

### 2. 在 `FcDesigner.vue` 接收事件並執行自定義邏輯 ​

檔案：`src/designer/FcDesigner.vue`

#### 2.1 在 `DragTool` 的事件對映裡補齊 `custom` ​

在構造 `DragTool` 的兩處 `on: { ... }` 裡都加上（`inside: true` 與 `inside: false` 各一處）：

js

```
custom: ({ self }) => methods.handleToolCustom({ self, inside: true }),
```

js

```
custom: ({ self }) => methods.handleToolCustom({ self, inside: false }),
```

#### 2.2 增加處理函式 `handleToolCustom` ​

仿照 `handleToolCopy/handleToolDelete`，統一拿到 `targetRule` 後執行你的業務操作。最推薦的方式是：

- 在 `FcDesigner` 內部完成“預設行為”（例如彈窗、改規則、寫入擴展欄位等）
- 同時對外 `emit('custom', targetRule)`，讓整合專案可監聽（按需）

示例（虛擬碼）：

js

```
handleToolCustom({ self, inside }) {
  const targetRule = inside ? methods.getParent(self).parent : self.children[0];

  // 1) 执行你的自定义逻辑（示例：给规则打标记）
  targetRule.props = targetRule.props || {};
  targetRule.props.__bizTagged = true;

  // 2) 如果影响了右侧面板/画布展示，可按需刷新
  if (data.activeRule === targetRule) {
    methods.updateRuleFormData();
  }

  // 3) 对外抛事件（按需：让集成项目接管）
  vm.emit('custom', targetRule);
},
```

#### 2.3（可選）把 `custom` 加到元件的 `emits` 列表 ​

如果你希望在整合專案裡這樣監聽：

vue

```
<FcDesigner @custom="onCustomTool" />
```

那麼需要把 `custom` 加到 `FcDesigner.vue` 的 `emits: [...]` 中（與 `copy/delete/create` 同級）。

## 常見問題排查 ​

### 1. 按鈕顯示了，但點選沒有反應 ​

- **最可能原因**：`DragTool.vue` 裡只新增了按鈕，但沒有把事件名加入 `emits`，導致 Vue 丟棄事件
- **定位路徑**：`src/designer/DragTool.vue` 的 `emits: [...]`
- **修復方式**：把 `custom` 加入 `emits`

### 2. 點選觸發了，但拿到的 rule 不對（刪錯/改錯元件） ​

- **最可能原因**：沒有按 `inside` 區分取 `targetRule`
- **定位路徑**：`src/designer/FcDesigner.vue` 的 `handleToolCopy/handleToolDelete`（對照它們的取值方式）
- **修復方式**：使用統一寫法  
`targetRule = inside ? methods.getParent(self).parent : self.children[0]`

### 3. 整合專案監聽不到 `@custom` ​

- **最可能原因**：`FcDesigner.vue` 的 `emits` 列表沒加入 `custom`
- **定位路徑**：`src/designer/FcDesigner.vue` 頂部 `emits: [...]`
- **修復方式**：加入 `custom`，並在 `handleToolCustom` 中 `vm.emit('custom', targetRule)`
