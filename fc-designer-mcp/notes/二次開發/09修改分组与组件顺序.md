# 修改分組與元件順序 ​

在設計器中，左側“元件面板”的展示順序通常由兩部分共同決定：

- **分組（選單）順序**：由選單配置檔案控制（分組在陣列中的先後順序，就是最終顯示順序）
- **元件（DragRule）順序**：由 DragRule 彙總檔案中的陣列順序控制（元件在陣列中的先後順序，就是分組內元件的展示順序）

本文講清楚如何在原始碼層二開實現：

- 調整左側面板 **分組的排列順序**
- 調整每個分組下 **元件的排列順序**

## 你要改哪些地方？ ​

本主題只涉及兩處配置檔案，屬於**低風險 UI 排序調整**：

- **分組配置（選單）**：`/src/config/menu.js`
- **元件彙總（DragRule 列表）**：`/src/config/index.js`

> 核心原則：**陣列順序 = 展示順序**。通常不需要改元件實現、渲染器註冊、或資料結構。

## 參考樣例 ​

可以先對照你專案裡“當前顯示順序”和“配置檔案陣列順序”是否一致：

- 左側面板看到的分組順序，是否等同於 `menu.js` 裡配置的順序
- 某個分組下元件的順序，是否等同於 `index.js` 裡 DragRule 陣列的順序（結合每個 DragRule 的 `menu` 欄位歸類）

## 修改分組順序 ​

### 1. 定位分組配置檔案 ​

開啟：

- `/src/config/menu.js`

你通常會看到一個“分組陣列”的匯出結構。

### 2. 調整陣列順序 ​

把你希望“更靠前展示”的分組物件移動到陣列更前面即可。

示例（僅示意：把“佈局元件”移動到“基礎元件”之前）：

js

```
// before
export default [
  { name: 'main', title: '基础组件', /* ... */ },
  { name: 'layout', title: '布局组件', /* ... */ },
  { name: 'advanced', title: '高级组件', /* ... */ },
];

// after
export default [
  { name: 'layout', title: '布局组件', /* ... */ },
  { name: 'main', title: '基础组件', /* ... */ },
  { name: 'advanced', title: '高级组件', /* ... */ },
];
```

### 3. 新增一個分組 ​

如果你想新增一個“業務元件”分組，本質就是在 `menu.js` 裡**追加一個分組物件**（並按你希望的展示順序放到陣列對應位置）。

示例（僅示意：新增 `biz` 分組，並放到 `main` 後面）：

js

```
export default [
  { name: 'main', title: '基础组件', /* ... */ },

  // 新增：业务组件分组
  {
    name: 'biz',             // 分组 key：需要与 DragRule.menu 对应
    title: '业务组件',        // 左侧分组标题
    // icon: 'icon-app',     // 如果你的项目支持分组图标，可按既有字段补充
    // show: true,           // 若项目有显隐字段，可按既有字段补充
  },

  { name: 'layout', title: '布局组件', /* ... */ },
  { name: 'advanced', title: '高级组件', /* ... */ },
];
```

## 修改元件順序 ​

### 1. 定位 DragRule 彙總檔案 ​

開啟：

- `/src/config/index.js`

該檔案通常會：

- `import` 一批 DragRule（每個元件一個規則檔案）
- 把它們按陣列順序放入一個列表匯出

### 2. 明確“元件歸屬哪個分組” ​

元件屬於哪個分組，一般看 DragRule 自身的欄位（常見是 `menu: 'main'` 這類 key）。  
 因此你在調整順序時要注意：只需要調整該分組對應的 DragRule 在陣列中的相對順序

### 3. 調整陣列順序 ​

示例 1：透過“調整陣列中 push/排列順序”改變展示順序（最常見）

js

```
// before（main 分组内：input 在 select 前）
export const ruleList = [
  input,
  select,
  radio,
  // ...
];

// after（main 分组内：select 提前）
export const ruleList = [
  select,
  input,
  radio,
  // ...
];
```

示例 2：如果你的彙總方式是“先定義陣列，再 push”，也同理

js

```
// before
const ruleList = [];
ruleList.push(input);
ruleList.push(select);

// after
const ruleList = [];
ruleList.push(select);
ruleList.push(input);
```

### 4. 在分組中新增元件 ​

“在某個分組中新增元件”通常分兩步：

- **讓元件歸屬到目標分組**：在該元件的 DragRule 中設定 `menu: '<分组 key>'`（例如 `menu: 'biz'`）
- **控制它在分組內的位置**：在 `index.js` 的 DragRule 陣列裡，把它放到你希望的相對順序位置

示例（僅示意：把一個新 DragRule `userSelect` 加到 `biz` 分組，並排在 `customerSelect` 前面）：

js

```
// /src/config/index.js
import userSelect from './rule/userSelect';
import customerSelect from './rule/customerSelect';

export const ruleList = [
  // ...其他分组/其他组件

  // biz 分组内：userSelect 在 customerSelect 前面
  userSelect,
  customerSelect,

  // ...其他组件
];
```

並確保對應 DragRule 裡寫了 `menu: 'biz'`：

js

```
// /src/config/rule/userSelect.js（仅示意）
export default {
  menu: 'biz',
  name: 'userSelect',
  label: '用户选择',
  // ...
};
```

## 常見問題排查 ​

### 1. 改了 `menu.js` 或 `index.js`，介面順序沒變化 ​

- **最可能原因 1：改錯了專案/改錯了檔案**  
 一些團隊同時維護多份設計器原始碼（不同版本/不同端/不同 UI），請確認你執行的專案引用的確實是：  
`designer3.0/src/config/menu.js` 與 `designer3.0/src/config/index.js`
- **最可能原因 2：有二次排序邏輯覆蓋了陣列順序**  
 例如渲染時對分組/元件做了 `sort()`，或按 `sort` 欄位重新排序。需要找到最終渲染處，確認是否存在強制排序。
- **最可能原因 3：快取未重新整理**  
 嘗試重啟 dev、清理構建快取、或強制重新整理頁面。

### 2. 順序變化了，但某些元件“跑到了別的分組” ​

- **原因**：元件歸屬分組通常由 DragRule 的 `menu` 決定。  
 如果你只改陣列順序，不應發生跨分組變化；若發生了，說明專案裡存在“分組白名單/分組對映”邏輯或 DragRule 配置被同時改動。

## 附錄：相關檔案路徑速查 ​

- 分組配置：`/src/config/menu.js`
- DragRule 彙總：`/src/config/index.js`
