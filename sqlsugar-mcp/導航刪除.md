# SqlSugar — 導航刪除（級聯刪除）筆記

[//]: # (sqlsugar-source)
> 📖 官方文件:[果糖網 SqlSugar 5x 文件](https://www.donet5.com/Home/Doc?typeId=2431)

## 核心邏輯

導航刪除的順序：**先刪子表，再刪主表**（避免外鍵約束錯誤）。
SqlSugar 不依賴資料庫外鍵，只需設定實體導航特性即可使用。

---

## 速查表

| 需求 | 語法 |
|---|---|
| 依條件導航刪除 | db.DeleteNav\<T\>(it => it.Id == 1).Include(...).ExecuteCommand() |
| 依集合導航刪除 | db.DeleteNav\<T\>(list).Include(...).ExecuteCommand() |
| 多層級刪除 | .Include(z => z.A).ThenInclude(z => z.B) |
| 自動刪除第一層所有導航 | .IncludesAllFirstLayer() |
| 多對多 — 只刪中間表 | .Include(z => z.Roles)（預設）|
| 多對多 — 刪主表 + 中間表 | new DeleteNavOptions { ManyToManyIsDeleteA = true } |
| 多對多 — 刪子表 + 中間表 | new DeleteNavOptions { ManyToManyIsDeleteB = true } |
| 多對多 — 刪三張表 | ManyToManyIsDeleteA = true, ManyToManyIsDeleteB = true |

---

## 一、多層級導航刪除語法

```csharp
// 依條件刪除
db.DeleteNav<Student>(it => it.Id == 1)
    .Include(z => z.SchoolA).ThenInclude(z => z.RoomList)  // Root → SchoolA → RoomList（兩層）
    .Include(z => z.Books)                                   // Root → Books（一層）
    .ExecuteCommand();

// 依集合刪除（List 不需要有子物件資料，有主鍵即可）
var list = db.Queryable<Student>().Where(it => it.Id == 1).ToList();
db.DeleteNav(list)
    .Include(z => z.SchoolA).ThenInclude(z => z.RoomList)
    .Include(z => z.Books)
    .ExecuteCommand();

// 異步
await db.DeleteNav<Student>(it => it.Id == 1)
    .Include(z => z.Books)
    .ExecuteCommandAsync();
```

層級說明：
- Include 從第一層開始
- ThenInclude 在上一層繼續向下
- 可以有多條 Include 支鏈同時進行

## 二、一對一導航刪除

邏輯：先刪子表（SchoolA），再刪主表（Student）。

```csharp
db.DeleteNav<Student>(it => it.Id == 1)
    .Include(z => z.SchoolA)
    .ExecuteCommand();
// → DELETE SchoolA WHERE Id IN (...) 先執行
// → DELETE Student WHERE Id = 1    後執行
```

## 三、一對多導航刪除

邏輯：先刪子表（Books），再刪主表（Student）。

```csharp
db.DeleteNav<Student>(it => it.Id == 1)
    .Include(z => z.Books)
    .ExecuteCommand();
// → DELETE Books WHERE StudentId IN (1)  先執行
// → DELETE Student WHERE Id = 1         後執行
```

**✅ 實際案例：刪除訂單及其所有明細**

```csharp
[HttpDelete("{id}")]
public async Task<ApiResult> DeleteOrder(int id)
{
    await db.DeleteNav<Order>(o => o.Id == id)
        .Include(o => o.Items)          // 先刪訂單明細
        .ExecuteCommandAsync();

    return ApiResult.Success();
}
```

## 四、多對多導航刪除

多對多有四種刪除組合，用 DeleteNavOptions 控制要刪哪些表。

設定說明：A 表 = 主表（UserInfo），B 表 = 子表（Roles），中間表 = 關聯表（UserRole）

### 4.1 只刪中間表（預設）

```csharp
// 解除使用者和角色的關聯，但不刪 UserInfo 也不刪 Roles
db.DeleteNav<UserInfo>(x => x.Id == 1)
    .Include(x => x.Roles)
    .ExecuteCommand();
// → DELETE UserRole WHERE UserId = 1
```

### 4.2 刪主表 + 中間表（A + 中間表）

```csharp
// 刪除使用者 + 關聯（不刪角色本身）
db.DeleteNav<UserInfo>(x => x.Id == 1)
    .Include(x => x.Roles, new DeleteNavOptions { ManyToManyIsDeleteA = true })
    .ExecuteCommand();
// → DELETE UserRole WHERE UserId = 1
// → DELETE UserInfo WHERE Id = 1
```

### 4.3 刪子表 + 中間表（B + 中間表）

```csharp
// 刪除關聯的角色本身 + 關聯（不刪使用者）
db.DeleteNav<UserInfo>(x => x.Id == 1)
    .Include(x => x.Roles, new DeleteNavOptions { ManyToManyIsDeleteB = true })
    .ExecuteCommand();
// → DELETE Roles WHERE Id IN (關聯的 RoleId)
// → DELETE UserRole WHERE UserId = 1
```

### 4.4 刪三張表（A + B + 中間表）

```csharp
// 刪除使用者 + 關聯角色 + 關聯記錄（完整清除）
db.DeleteNav<UserInfo>(x => x.Id == 1)
    .Include(x => x.Roles, new DeleteNavOptions
    {
        ManyToManyIsDeleteA = true,
        ManyToManyIsDeleteB = true
    })
    .ExecuteCommand();
// → DELETE Roles WHERE Id IN (...)
// → DELETE UserRole WHERE UserId = 1
// → DELETE UserInfo WHERE Id = 1
```

## 五、自動刪除第一層所有導航（5.1.4.108+）

```csharp
// 自動包含所有第一層導航一起刪除
db.DeleteNav(list).IncludesAllFirstLayer().ExecuteCommand();

// 排除不需要的導航
db.DeleteNav(list)
    .IncludesAllFirstLayer(nameof(Student.TempLog))  // 排除 TempLog
    .ExecuteCommand();

// 自動兩層 + 手動指定第三層
db.DeleteNav(list)
    .IncludesAllFirstLayer()
    .IncludeByNameString(nameof(Student.SchoolA))
        .ThenIncludeByNameString(nameof(SchoolA.RoomList))
    .ExecuteCommand();
```

## 六、軟刪除（邏輯刪除）建議

軟刪除時只需刪（更新）主表的 IsDeleted 欄位，子表透過 INNER JOIN 過濾即可：

```csharp
// 軟刪除：只更新主表
await db.Updateable<Order>()
    .SetColumns(o => new Order { IsDeleted = true, DeleteTime = DateTime.Now })
    .Where(o => o.Id == orderId)
    .ExecuteCommandAsync();

// 子表查詢時加上 INNER JOIN 主表，自動過濾已刪除主表的子表
var items = db.Queryable<OrderItem>()
    .InnerJoin<Order>((i, o) => i.OrderId == o.Id)
    .Where((i, o) => o.IsDeleted == false)
    .ToList();
```

多對多軟刪除中間表：可用導航更新設定中間表的邏輯刪除欄位。

---

## 注意事項

- 導航刪除的執行順序：子表 → 孫表 → 主表（由下往上刪，避免 FK 錯誤）
- 多對多預設只刪中間表，不影響 A/B 兩張實體表，需明確設定 DeleteNavOptions 才會刪實體表
- 集合（List）傳入時，List 中的物件只需有主鍵，不需要預先載入導航屬性
- 軟刪除場景一般不需要用導航刪除，只更新主表 IsDeleted，子表透過 JOIN 過濾即可
