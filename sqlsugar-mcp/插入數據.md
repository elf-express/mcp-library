## 速查表

| 需求 | 語法 |
|---|---|
| 單筆插入，返回影響行數 | .ExecuteCommand() |
| 單筆插入，返回自增 ID | .ExecuteReturnIdentity() |
| 單筆插入，返回雪花 ID | .ExecuteReturnSnowflakeId() |
| 單筆插入，返回實體（含自增）| .ExecuteReturnEntity() |
| 批量插入（萬金油）| db.Insertable(list).ExecuteCommand() |
| 批量插入，返回主鍵集合 | .ExecuteReturnPkList\<int\>() |
| 批量插入（參數化分頁）| .UseParameter().ExecuteCommand() |
| 大數據插入（BulkCopy）| db.Fastest\<T\>().BulkCopy(list) |
| 分頁插入 | .PageSize(1000).ExecuteCommand() |
| Select Into（表複製）| .IntoTable\<T\>() |
| 忽略欄位插入 | .IgnoreColumns(it => new { it.Name }) |
| 只插入指定欄位 | .InsertColumns(it => new { it.Name }) |
| NULL 欄位不插入 | .IgnoreColumns(ignoreNullColumn: true) |
| 字典插入 | db.Insertable(dc).AS("表名").ExecuteCommand() |
| 強制插入自增列 | .OffIdentity().ExecuteCommand() |
| 導航插入 | db.InsertNav(list).Include(...).ExecuteCommand() |

## 一、單筆插入

```csharp
var order = new Order { Name = "測試", Amount = 100, CreateTime = DateTime.Now };

// 返回影響行數
int rows = db.Insertable(order).ExecuteCommand();

// 返回自增 ID（int）
int newId = db.Insertable(order).ExecuteReturnIdentity();

// 返回自增 ID（long）
long newId = db.Insertable(order).ExecuteReturnBigIdentity();

// 返回雪花 ID（詳見雪花ID筆記）
long snowId = db.Insertable(order).ExecuteReturnSnowflakeId();

// 返回完整實體（含自增 ID 回填）
Order inserted = db.Insertable(order).ExecuteReturnEntity();

// 自增 ID 回填到傳入實體
db.Insertable(order).ExecuteCommandIdentityIntoEntity();
// 執行後 order.Id 已被賦值

// 異步
int newId = await db.Insertable(order).ExecuteReturnIdentityAsync();

// 強制指定資料表名
db.Insertable(order).AS("OrderBackup").ExecuteCommand();
```

**✅ 實際案例：新增訂單並取得 ID**

```csharp
public async Task<int> CreateOrderAsync(CreateOrderRequest req)
{
    var order = new Order
    {
        OrderNo    = GenerateOrderNo(),
        CustomerId = req.CustomerId,
        Amount     = req.Amount,
        Status     = 1,
        CreateTime = db.GetDate()  // 取資料庫時間
    };

    int newId = await db.Insertable(order).ExecuteReturnIdentityAsync();
    return newId;
}
```

## 二、返回值方法一覽

| 方法 | 說明 |
|---|---|
| ExecuteCommand() | 返回影響行數，批量插入效能最好 |
| ExecuteReturnIdentity() | 返回自增 ID（int）|
| ExecuteReturnBigIdentity() | 返回自增 ID（long）|
| ExecuteReturnEntity() | 返回完整實體（含自增 ID，不支援批量自增）|
| ExecuteCommandIdentityIntoEntity() | 自增 ID 回填到傳入實體（不支援批量）|
| ExecuteReturnSnowflakeId() | 返回雪花 ID（詳見雪花 ID 章節）|
| ExecuteReturnSnowflakeIdList() | 返回雪花 ID 集合 |
| ExecuteReturnPkList\<T\>() | 批量返回主鍵集合（5.1.2.4+）|

## 三、批量插入（四種方式）

### 3.1 普通批量插入（萬金油，推薦）

```csharp
// 適合：列少、1萬條以內、萬金油寫法
// 缺點：超過 5 萬條記憶體占用較大
db.Insertable(list).ExecuteCommand();

// 分頁批量插入（5.1.4.103+，大量資料時減少記憶體）
db.Insertable(list).PageSize(1000).ExecuteCommand();

// 注意：泛型寫法
// ❌ 錯誤：db.Insertable<List<Order>>(list)
// ✅ 正確：db.Insertable<Order>(list) 或 db.Insertable(list)
```

### 3.2 參數化分頁插入

```csharp
// 適合：筆數固定且較少、高頻請求（利用執行計畫快取）
// 缺點：500 筆以上某些資料庫開始變慢，需要加事務才能回滾
db.Insertable(list).UseParameter().ExecuteCommand();
```

### 3.3 大數據 BulkCopy（效能最強）

```csharp
// 適合：1000 筆以上，效能比所有框架快 30%
// 缺點：不支援資料庫預設值，API 簡單
db.Fastest<Order>().BulkCopy(list);

// 分頁大數據插入
db.Fastest<Order>().PageSize(100000).BulkCopy(list);

// MySQL 需在連線字串加：AllowLoadLocalInfile=true
```

### 3.4 Select Into（從查詢結果插入另一張表）

```csharp
// 不同實體：查詢 Order 插入 OrderArchive
db.Queryable<Order>()
    .Where(o => o.CreateTime < DateTime.Now.AddYears(-1))
    // .IgnoreColumns(it => it.Id)  // 自增欄位可忽略
    .Select(it => new OrderArchive { Name = it.Name, Amount = it.Amount })
    .IntoTable<OrderArchive>();

// 相同實體，不同表名
db.Queryable<Order>()
    .Where(o => o.Status == 3)
    .IntoTable<Order>("OrderCompleted");  // 指定目標表名
```

### 3.5 批量返回主鍵集合（5.1.2.4+）

```csharp
// 返回自增 ID 集合
List<int> ids = db.Insertable(list).ExecuteReturnPkList<int>();

// 返回雪花 ID 集合
List<long> snowIds = db.Insertable(list).ExecuteReturnSnowflakeIdList();
```

## 四、欄位控制

```csharp
// 忽略指定欄位（不插入）
db.Insertable(order)
    .IgnoreColumns(it => new { it.Remark, it.UpdateTime })
    .ExecuteCommand();

// 只插入指定欄位
db.Insertable(order)
    .InsertColumns(it => new { it.Name, it.Amount, it.Status })
    .ExecuteCommand();

// NULL 值欄位不插入（使用資料庫預設值）
// 注意：因每筆插入的欄位不同，不支援批量，需自行迴圈
db.Insertable(order)
    .IgnoreColumns(ignoreNullColumn: true)
    .ExecuteCommand();
```

## 五、插入預設值

### 5.1 欄位設定（實體特性）

```csharp
public class Order
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }

    public string Name { get; set; }

    // 方式一：插入時忽略，使用資料庫欄位預設值
    [SugarColumn(IsOnlyIgnoreInsert = true)]
    public DateTime CreateTime { get; set; }

    // 方式二：插入時自動取資料庫當前時間（5.1.3.41+）
    [SugarColumn(InsertServerTime = true, IsOnlyIgnoreUpdate = true)]
    public DateTime CreateTime2 { get; set; }

    // 方式三：插入時使用指定 SQL 表達式
    [SugarColumn(InsertSql = "getdate()")]
    public DateTime CreateTime3 { get; set; }

    [SugarColumn(InsertSql = "''")]
    public string DefaultStr { get; set; }
}
```

## 六、其他插入方式

### 6.1 字典插入

```csharp
var dc = new Dictionary<string, object>
{
    ["Name"]       = "測試訂單",
    ["Amount"]     = 1000m,
    ["CreateTime"] = DateTime.Now
};
db.Insertable(dc).AS("Order").ExecuteCommand();

// 批量字典插入
db.Insertable(new List<Dictionary<string, object>> { dc1, dc2 }).AS("Order").ExecuteCommand();
```

### 6.2 DataTable 插入

```csharp
// 方式一：轉成字典再插入
var dcList = db.Utilities.DataTableToDictionaryList(dataTable);
db.Insertable(dcList).AS("Order").ExecuteReturnIdentity();

// 方式二：直接 BulkCopy
db.Fastest<System.Data.DataTable>().AS("Order").BulkCopy(dataTable);
```

### 6.3 匿名物件 / Object 插入

```csharp
// 匿名物件
db.InsertableByDynamic(new { Name = "A", Amount = 100 })
    .AS("Order")
    .ExecuteCommand();

// Object / 介面 / 抽象類（必須是真實類別物件）
db.InsertableByObject(obj).ExecuteCommand();
```

### 6.4 強制插入自增列（5.1.4.73+）

```csharp
// 插入時帶入自增欄位的值（需資料庫支援）
db.Insertable(order).OffIdentity().ExecuteCommand();
```

### 6.5 Guid 主鍵自動賦值

只要實體主鍵類型是 Guid 且未傳值，SqlSugar 會自動賦值 NewGuid()。
使用 ExecuteCommand 方法，不能使用自增相關方法。

```csharp
var item = new OrderItem { OrderId = 1, Price = 100 };
// item.Id（Guid）未賦值，SqlSugar 自動產生
db.Insertable(item).ExecuteCommand();
```

## 七、導航插入（級聯）

db.InsertNav(orderList)

## 效能選用建議

| 資料量 | 推薦方式 |
|---|---|
| 單筆 | ExecuteReturnIdentity / ExecuteReturnEntity |
| 少量（< 500）且高頻 | UseParameter().ExecuteCommand() |
| 中量（500 ~ 5萬）| Insertable(list).ExecuteCommand() |
| 中量且記憶體有限 | .PageSize(1000).ExecuteCommand() |
| 大量（> 1萬）| db.Fastest\<T\>().BulkCopy(list) |
| 超大量（> 10萬）| db.Fastest\<T\>().PageSize(100000).BulkCopy(list) |

---

## 注意事項

- 批量插入用 ExecuteCommand 效能最好，不要用 ExecuteReturnIdentity（批量效能差）
- ExecuteReturnEntity 不支援批量自增回填，也不支援資料庫預設值
- IgnoreColumns(ignoreNullColumn: true) 因每筆欄位不同，不支援批量，需自行迴圈
- Guid 主鍵自動賦值只能搭配 ExecuteCommand，不能用自增方法
- BulkCopy 不支援資料庫預設值（如 IsOnlyIgnoreInsert 設定的欄位）
- MySQL BulkCopy 需在連線字串加 AllowLoadLocalInfile=true
- 雪花 ID 詳細設定見雪花 ID 章節
