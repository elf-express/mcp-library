## 速查表

| 需求 | 語法 |
|---|---|
| Queryable 當主表再 Join | db.Queryable(queryable).LeftJoin\<T\>(...) |
| 表 Join Queryable（右邊是子查詢）| .LeftJoin(queryableRight, (o,d) => ...) |
| 多個 Queryable 互 Join | q1.LeftJoin(q2, ...).LeftJoin(q3, ...) |
| 表 Join 原生 SQL | .LeftJoin(db.SqlQueryable\<T\>("select..."), ...) |
| 表 Join 記憶體集合 | db.Reportable(list).ToQueryable() 後再 Join |
| 匿名類型嵌套 Join | .Select(匿名).MergeTable().LeftJoin\<T\>(...) |
| 多表合一後動態搜尋 | .Select(...).MergeTable().Where(...).OrderBy(...) |
| 一合一嵌套（CTE 風格）| .Select(...).MergeTable().ToList() |

## 一、嵌套聯表

### 1.1 Queryable 當主表再 Join 實體表

```csharp
// 寫法一：直接包成 Queryable
var queryable = db.Queryable<Order>();
var list = db.Queryable(queryable)
    .LeftJoin<OrderDetail>((o, d) => o.Id == d.OrderId)
    .Select(o => o)
    .ToList();

// 寫法二：透過 MergeTable（MergeTable 前要有 Select，且不能有 OrderBy）
var list = db.Queryable<Order>()
    .Select<Order>()
    .MergeTable()
    .LeftJoin<OrderDetail>((o, d) => o.Id == d.OrderId)
    .Select(o => o)
    .ToList();

// 寫法三：5.1.3.41+ 只有 Where 時不需要 MergeTable
var list = db.Queryable<Order>()
    .Where(o => o.Id == 1)
    .LeftJoin<OrderDetail>((o, d) => o.Id == d.OrderId)
    .Select(o => o)
    .ToList();
```

**✅ 實際案例：先過濾有效訂單，再聯出貨明細**

```csharp
// 先篩選未刪除的訂單，再聯明細資料
var validOrders = db.Queryable<Order>()
    .Where(o => o.IsDeleted == false && o.Status != OrderStatus.Cancelled)
    .Select<Order>();

var list = db.Queryable(validOrders)
    .LeftJoin<OrderDetail>((o, d) => o.Id == d.OrderId)
    .Select((o, d) => new OrderDetailDto
    {
        OrderNo     = o.OrderNo,
        ProductName = d.ProductName,
        Qty         = d.Qty,
        Amount      = d.Amount
    })
    .ToList();
```

### 1.2 表 Join Queryable（右邊是子查詢）

```csharp
var queryableRight = db.Queryable<OrderItem>().Where(x => x.OrderId > 0);

var list = db.Queryable<Order>()
    .LeftJoin(queryableRight, (o, d) => o.Id == d.OrderId)
    .Select(o => o)
    .ToList();
```

**✅ 實際案例：訂單 Join 只取金額大於 0 的明細**

```csharp
// 右側先過濾，再和訂單 Join，避免把無效明細帶進來
var validDetails = db.Queryable<OrderDetail>()
    .Where(d => d.Amount > 0 && d.IsDeleted == false);

var list = db.Queryable<Order>()
    .LeftJoin(validDetails, (o, d) => o.Id == d.OrderId)
    .Select((o, d) => new
    {
        o.OrderNo,
        d.ProductName,
        d.Amount
    })
    .ToList();
```

### 1.3 多個 Queryable 互相 Join

```csharp
var q1 = db.Queryable<Order>().Where(it => it.Id > 1);
var q2 = db.Queryable<Order>().Where(it => it.Id > 2);
var q3 = db.Queryable<Order>().Where(it => it.Id > 3);

var list = q1
    .LeftJoin(q2, (x, y) => x.Id == y.Id)
    .LeftJoin(q3, (x, y, z) => x.Id == z.Id)
    .ToList();
```

**✅ 實際案例：分別取當月、上月、去年同月資料進行比對**

```csharp
var thisMonth  = db.Queryable<SalesReport>().Where(r => r.Month == thisM);
var lastMonth  = db.Queryable<SalesReport>().Where(r => r.Month == lastM);
var lastYear   = db.Queryable<SalesReport>().Where(r => r.Month == lastYearM);

var list = thisMonth
    .LeftJoin(lastMonth,  (a, b) => a.ProductId == b.ProductId)
    .LeftJoin(lastYear,   (a, b, c) => a.ProductId == c.ProductId)
    .Select((a, b, c) => new SalesCompareDto
    {
        ProductId      = a.ProductId,
        ThisMonthSales = a.Sales,
        LastMonthSales = b.Sales,
        LastYearSales  = c.Sales
    })
    .ToList();
```

### 1.4 表 Join 原生 SQL

```csharp
// SqlQueryable 把 SQL 轉成 Queryable
var list = db.Queryable<Order>()
    .LeftJoin(
        db.SqlQueryable<Custom>("SELECT * FROM Custom WHERE IsActive = 1"),
        (o, c) => o.CustomId == c.Id)
    .Select(o => o)
    .ToList();
// → SELECT o.* FROM [Order] o LEFT JOIN (SELECT * FROM Custom WHERE IsActive=1) c ON o.CustomId = c.Id

// 反過來也支援（5.1.3.41+）
db.SqlQueryable<Custom>("SELECT * FROM Custom")
    .LeftJoin<Order>((c, o) => c.Id == o.CustomId)
    .ToList();
```

**✅ 實際案例：Join 複雜統計 SQL 結果**

```csharp
// 右側是一段複雜統計 SQL，不易用 Queryable 表達
string statSql = @"
    SELECT OrderId, SUM(Amount) AS TotalAmount, COUNT(*) AS DetailCount
    FROM OrderDetail
    WHERE IsDeleted = 0
    GROUP BY OrderId";

var list = db.Queryable<Order>()
    .LeftJoin(
        db.SqlQueryable<OrderStat>(statSql),
        (o, s) => o.Id == s.OrderId)
    .Select((o, s) => new OrderDto
    {
        OrderNo     = o.OrderNo,
        TotalAmount = s.TotalAmount,
        DetailCount = s.DetailCount
    })
    .ToList();
```

### 1.5 表 Join 記憶體集合

```csharp
// 把 List<T> 轉成可 Join 的 Queryable
var memoryList   = new List<OrderItem> { ... };

var list = db.Queryable<Order>()
```

## 二、匿名類型嵌套聯表

### 單個匿名物件 Join

```csharp
// MergeTable 前不能有 OrderBy，OrderBy 要寫在 MergeTable 後
db.Queryable<Order>()
    .Select(it => new { id = it.Id })  // 先轉匿名物件
    .MergeTable()                       // 合併成新表
    .LeftJoin<OrderItem>((x, y) => x.id == y.ItemId)  // 最後一個表不能是匿名物件
    .ToList();

// 技巧：若匿名物件在右邊，把 LeftJoin 換成 RightJoin 即可
```

### 多個匿名物件連續 Join

```csharp
db.Queryable<Order>()
    .Select(it => new { id = it.Id })
    .MergeTable()
    .LeftJoin<OrderItem>((x, y) => x.id == y.ItemId)
    .Select((x, y) => new { xid = x.id, yid = y.ItemId })  // 再轉匿名物件
    .MergeTable()
    .LeftJoin<Custom>((a, b) => a.xid == b.Id)
    .ToList();
```

**✅ 實際案例：先取訂單計算欄位，再聯客戶資料**

```csharp
// Step1：先計算每筆訂單總金額（計算欄位）
// Step2：再 Join 客戶資料排序
db.Queryable<Order>()
    .LeftJoin<OrderDetail>((o, d) => o.Id == d.OrderId)
    .GroupBy(o => new { o.Id, o.CustomerId, o.OrderNo })
    .Select(o => new
    {
        id         = o.Id,
        customerId = o.CustomerId,
        orderNo    = o.OrderNo,
        total      = SqlFunc.AggregateSum(o.Amount)
    })
    .MergeTable()
    .LeftJoin<Customer>((a, c) => a.customerId == c.Id)
    .OrderBy((a, c) => a.total, OrderByType.Desc)
    .Select((a, c) => new OrderSummaryDto
    {
        OrderNo      = a.orderNo,
        CustomerName = c.Name,
        Total        = a.total
    })
    .ToList();
```

## 三、多合一（MergeTable 動態搜尋）

多表 Join 後透過 MergeTable 合成單表，後續 Where / OrderBy 不需要加表別名前綴

```csharp
var list = db.Queryable<Student, School>((st, sc) => new JoinQueryInfos(
        JoinType.Left, st.SchoolId == sc.Id))
    .Select((st, sc) => new
    {
        id   = st.Id,
        name = sc.Name
    })
    .MergeTable()                    // 合成單表，之後不需 st. sc. 前綴
    .Where(it => it.id == 1)
    .OrderBy("name asc")
    .ToList();
```

生成 SQL：

```sql
SELECT * FROM
  (SELECT st.Id AS id, sc.Name AS name
   FROM Student st LEFT JOIN School sc ON st.SchoolId = sc.Id) MergeTable
WHERE id = @id0
ORDER BY name asc
```

**✅ 實際案例：訂單聯客戶後，支援前端動態條件與排序**

```csharp
public List<OrderDto> GetOrders(string? customerName, string? sortField, string? sortOrder)
{
    var query = db.Queryable<Order, Customer>((o, c) => new JoinQueryInfos(
            JoinType.Left, o.CustomerId == c.Id))
        .Select((o, c) => new
        {
            orderId      = o.Id,
            orderNo      = o.OrderNo,
            customerName = c.Name,
            createTime   = o.CreateTime
        })
        .MergeTable();

    if (!string.IsNullOrEmpty(customerName))
        query = query.Where(it => it.customerName.Contains(customerName));

    if (!string.IsNullOrEmpty(sortField))
        query = query.OrderByPropertyName(sortField,
            sortOrder == "desc" ? OrderByType.Desc : OrderByType.Asc);

    return query.ToList();
}
```

## 四、一合一嵌套（CTE 風格，三種寫法）

三種寫法產生的 SQL 完全相同

```csharp
// 寫法一：先建 subQuery 再包
var subQuery = db.Queryable<Order>().Select(it => new OrderDto { ... });
var list     = db.Queryable(subQuery).ToList();

// 寫法二：SelectMergeTable（高版本，最簡潔）
var list = db.Queryable<Order>()
    .SelectMergeTable(it => new OrderDto { ... })
    .ToList();

// 寫法三：Select + MergeTable 分開
var list = db.Queryable<Order>()
    .Select(it => new OrderDto { ... })
    .MergeTable()
    .ToList();
```

生成 SQL：

```sql
SELECT t.* FROM (SELECT Id, Name, Price FROM [Order]) t
```

**✅ 實際案例：先計算後過濾（無法在原始查詢直接 WHERE 計算欄位）**

```csharp
// 計算欄位 TotalAmount 無法直接在原始 WHERE 使用，需先 MergeTable
var list = db.Queryable<OrderDetail>()
    .GroupBy(d => d.OrderId)
    .Select(d => new
    {
        orderId     = d.OrderId,
        totalAmount = SqlFunc.AggregateSum(d.Amount),
        detailCount = SqlFunc.AggregateCount(d.Id)
    })
    .MergeTable()
    .Where(it => it.totalAmount > 10000)  // 對計算結果過濾
    .OrderBy(it => it.totalAmount, OrderByType.Desc)
    .ToList();
```

## 五、跨庫特殊情況

```csharp
// QueryableWithAttr：切換庫時主表名不改變
db.QueryableWithAttr<T>()
    .LeftJoin<T2>(...)
    .ToList();

// AsWithAttr：不切換庫，但強制改表名
db.Queryable<T>().AsWithAttr()
    .LeftJoin<T2>(...)
    .ToList();

// UnionAll 跨庫特殊寫法（需用 AsWithAttr）
var q1 = db.Queryable<Order>().AsWithAttr();
var q2 = db.Queryable<Order>().AsWithAttr();
db.UnionAll(q1, q2).ToList();
```

## 注意事項

- MergeTable 前不能有 OrderBy / Skip / Take / 分頁，這些要寫在 MergeTable 後面
- MergeTable 前必須有 Select，否則無法合併
- 5.1.3.41+ 只有 Where 時可以省略 MergeTable 直接接 LeftJoin
- 匿名物件 Join 時，最後一個 Join 的右側表不能是匿名物件
- 多表合一後的 Where / OrderBy 不需要加表別名前綴（st. / sc. 等）
- 跨庫 UnionAll 需用 AsWithAttr，不能用 QueryableWithAttr
