# **修改配置**

## **修改示例**

我們可以在運行當前方法中修改DB對象的配置來滿足一些需求

### **例1**

比如DB已經是自動釋放了，我們可以動態修改成不自動釋放

```
db.CurrentConnectionConfig.IsAutoCloseConnection = false;
```

### **例2**

```
//注意new ConnMoreSettings會清空所有配置
db.CurrentConnectionConfig.MoreSettings=new ConnMoreSettings(){...}

//更安全寫法
if(db.CurrentConnectionConfig.MoreSettings!=null)
{
db.CurrentConnectionConfig.MoreSettings.xxxx=xxx;
}
else
{
db.CurrentConnectionConfig.MoreSettings=new ConnMoreSettings(){ .... }
}
```

## **禁止修改的屬性**

1. DbType 數據庫類型

2. ConnectionString 連接字符串

3. ConfigId 多租戶唯一標識

4. EntityService和EntityNameService 實體AOP

其它都可以使用過程中修改

## **MoreSettings明細**

```
ConnectionString =... ,
DbType = SqlSugar.DbType.MySql,
IsAutoCloseConnection = true,
MoreSettings=new ConnMoreSettings 
{
}
```

- **二級緩存功能：**
  - `IsAutoRemoveDataCache`：（沒用二級緩存忽略不看）控制是否啟用自動刪除數據緩存。
  - `DefaultCacheDurationInSeconds`：（沒用二級緩存忽略不看）設置緩存的默認持續時間（秒）。

- **查詢選項：**
  - `IsWithNoLockQuery`：啟用或禁用NOLOCK查詢提示。
  - `IsWithNoLockSubquery`：啟用或禁用在子查詢中使用NOLOCK。
  - `IsAutoUpdateQueryFilter`：控制是否自動更新查詢過濾器。
  - `IsAutoDeleteQueryFilter`：控制是否自動刪除查詢過濾器。

- **字符串處理：**
  - `DisableNvarchar`：禁用NVARCHAR數據類型的使用。
  - `PgSqlIsAutoToLower`：（默認小寫）設置是否將 PostgreSQL 查詢自動轉換為小寫。
  - `PgSqlIsAutoToLowerCodeFirst`：（默認小寫）設置是否將 PostgreSQL 代碼優先查詢自動轉換為小寫。
  - `IsAutoToUpper`：（oracle 達夢 金倉默認大寫）設置是否自動將字符串轉換為大寫。

- **日期和時間處理：**
  - `DisableMillisecond`：禁用毫秒的使用。（只支持部分庫）
  - `DbMinDate`：數據庫最小日期，默認為"1900-01-01"。

- **創建表庫或CRUD：**
  - `EnableOracleIdentity`：啟用或禁用 Oracle 自增列 12C以Oracle上才支持。
  - `SqlServerCodeFirstNvarchar`：SQL Server Code First 中是否使用 NVARCHAR。
  - `SqliteCodeFirstEnableDefaultValue`：啟用或禁用 SQLite Code First 中的默認值。
  - `SqliteCodeFirstEnableDescription`：啟用或禁用 SQLite Code First 中的描述。
  - `SqliteCodeFirstEnableDropColumn`：啟用或禁用 SQLite Code First 中的刪除列。
  - `EnableCodeFirstUpdatePrecision`：啟用或禁用 Code First 更新精度。
  - `IsCorrectErrorSqlParameterName`：字段有特殊字符導致SQL參數錯誤的兼容配置（性能有影響，可以通過修改配置局部啟用 db.CurrentConfig.xxx.xx=true 這樣）
  - `EnableModelFuncMappingColumn`：低碼碼API支持SQL函數（非低代碼忽略這個選項）
