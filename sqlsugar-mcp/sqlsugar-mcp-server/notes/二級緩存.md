## 速查表

| 需求 | 語法 |
|---|---|
| 啟用快取 | 查詢加 .WithCache() |
| 自動 Key | 以 SQL 語句為 Key:.WithCache() |
| 自訂 Key | .WithCache(cacheKey) / WithCacheFilter |
| 手動清快取 | db.DataCache.RemoveDataCache() |
| 全自動清快取 | 設定後增刪改自動清 |
| 快取服務 | 實作 ICacheService(Redis / MemoryCache) |
| 注入 | ConfigureExternalServices.DataInfoCacheService |
| 適用 | 字典表、選單等低頻更新資料 |

---

## 優點

- **維護方便**：支援單表和多表 CRUD 自動清除快取，減少複雜 Key 維護
- **使用方便**：直接在 `Queryable.WithCache()` 啟用，一行搞定
- **外部整合**：實作 `ICacheService` 介面，可接入 Redis / MemoryCache 等任意快取

---

## 一、原理與適用場景

```
二級快取 = 將查詢結果集快取起來
當 SQL 語句和參數完全相同時 → 直接從快取讀取，跳過資料庫查詢
```

### ✅ 適合快取的查詢

- SQL 和參數**相對固定**的查詢（如字典表、設定表、選單清單）
- 一次性查全部、之後在記憶體操作的場景
- 系統同時存在的快取 Key 在 **1000 個以下**

### ❌ 不適合快取的查詢

- 每次 SQL 都在變的查詢（如 `WHERE Id = @id` 且 id 種類極多）
- 參數值數量巨大的動態查詢
- 即時性要求很高的資料

## 二、建立快取服務

### 2.1 實作 ICacheService（Redis 版，複製即用）

```csharp
// NuGet 安裝：SugarRedis
public class SqlSugarRedisCache : ICacheService
{
    // 單例模式（不要放在建構函式中 new）
    public static SugarRedisClient Redis = new SugarRedisClient();
    // 多個參數：new SugarRedisClient("127.0.0.1:6379,password=,connectTimeout=3000,...,DefaultDatabase=0")

    public void Add<V>(string key, V value)
        => Redis.Set(key, value);

    public void Add<V>(string key, V value, int cacheDurationInSeconds)
        => Redis.SetBySeconds(key, value, cacheDurationInSeconds);

    public bool ContainsKey<V>(string key)
        => Redis.Exists(key);

    public V Get<V>(string key)
        => Redis.Get<V>(key);

    public IEnumerable<string> GetAllKey<V>()
        // 只查 SqlSugar 用到的 Key（效能注意）
        => Redis.SearchCacheRegex("SqlSugarDataCache.*");

    public V GetOrCreate<V>(string cacheKey, Func<V> create, int cacheDurationInSeconds = int.MaxValue)
    {
        if (ContainsKey<V>(cacheKey))
        {
            var result = Get<V>(cacheKey);
            return result ?? create();
        }
        else
        {
            var result = create();
            Add(cacheKey, result, cacheDurationInSeconds);
            return result;
        }
    }

    public void Remove<V>(string key)
        => Redis.Remove(key);
}
```

### 2.2 MemoryCache 版（無需 Redis，小專案適用）

```csharp
public class SqlSugarMemoryCache : ICacheService
{
    private static readonly MemoryCache _cache = new MemoryCache(new MemoryCacheOptions());
    private static readonly ConcurrentDictionary<string, bool> _keys = new();

    public void Add<V>(string key, V value)
    {
        _cache.Set(key, value);
        _keys[key] = true;
    }

    public void Add<V>(string key, V value, int cacheDurationInSeconds)
    {
        _cache.Set(key, value, TimeSpan.FromSeconds(cacheDurationInSeconds));
        _keys[key] = true;
    }

    public bool ContainsKey<V>(string key) => _cache.TryGetValue(key, out _);

    public V Get<V>(string key) => _cache.TryGetValue(key, out V val) ? val : default;

    public IEnumerable<string> GetAllKey<V>()
        => _keys.Keys.Where(k => k.StartsWith("SqlSugarDataCache"));

    public V GetOrCreate<V>(string key, Func<V> create, int seconds = int.MaxValue)
        => _cache.GetOrCreate(key, entry =>
        {
            entry.AbsoluteExpirationRelativeToNow = TimeSpan.FromSeconds(seconds);
            _keys[key] = true;
            return create();
        });

    public void Remove<V>(string key)
    {
        _cache.Remove(key);
        _keys.TryRemove(key, out _);
    }
}
```

### 2.3 注入設定

```csharp
// Program.cs
ICacheService myCache = new SqlSugarRedisCache();  // 或 SqlSugarMemoryCache

builder.Services.AddSingleton<ISqlSugarClient>(s =>
    new SqlSugarScope(new ConnectionConfig
    {
        DbType                = DbType.SqlServer,
        ConnectionString      = "...",
        IsAutoCloseConnection = true,
        ConfigureExternalServices = new ConfigureExternalServices
        {
            DataInfoCacheService = myCache  // 注入快取服務
        }
    }));
```

## 三、查詢使用快取

### 3.1 自動 Key 模式（以 SQL 語句為 Key）

```csharp
// 預設快取一天
db.Queryable<SystemMenu>()
    .Where(it => it.IsEnabled == true)
    .WithCache()
    .ToList();

// 指定快取秒數（1000 秒）
db.Queryable<SystemDict>()
    .WithCache(1000)
    .ToList();

// 多表聯查也支援
db.Queryable<Order, Customer>((o, c) => new JoinQueryInfos(
        JoinType.Left, o.CustomerId == c.Id))
    .WithCache()
    .Select((o, c) => new { o.OrderNo, c.Name })
    .ToList();

// 條件式快取（語法糖）
db.Queryable<Order>()
    .WithCacheIF(enableCache)  // 等同於 if(enableCache) .WithCache()
    .ToList();
```

### 3.2 追加 Key 模式（自訂快取 Key）

```csharp
// 快取資料
var menus = db.Queryable<SystemMenu>()
    .WithCache("MenuCache")
    .ToList();

// 主動清除指定 Key 的快取
db.DataCache.RemoveDataCache("MenuCache");

// 在刪除操作時順便清除快取
db.Deleteable<SystemMenu>()
    .RemoveDataCache("MenuCache")
    .Where(it => it.Id == 1)
    .ExecuteCommand();
```

## 四、清除快取

### 4.1 半自動清除（推薦，手動加 RemoveDataCache）

```csharp
// 增刪改時加上 RemoveDataCache()，相關查詢快取自動清除
db.Deleteable<SystemDict>()
    .RemoveDataCache()   // 清除 SystemDict 相關所有查詢快取
    .Where(it => it.Id == id)
    .ExecuteCommand();

db.Insertable(newDict)
    .RemoveDataCache()
    .ExecuteCommand();

db.Updateable(dict)
    .RemoveDataCache()
    .ExecuteCommand();
```

### 4.2 全自動清除（所有增刪改自動清快取）

```csharp
// 設定時加上 IsAutoRemoveDataCache = true
new ConnectionConfig
{
    ConnectionString      = "...",
    IsAutoCloseConnection = true,
    MoreSettings          = new ConnMoreSettings
    {
        IsAutoRemoveDataCache = true  // 所有增刪改自動清除相關快取
    }
}
// ⚠️ 缺點：子查詢、MergeTable 等複雜查詢可能無法正確清除，這種情況改用追加 Key 模式
```

### 4.3 根據 Key 關鍵字清除

```csharp
// 清除所有 Key 包含 "MenuCache" 的快取
db.DataCache.RemoveDataCache("MenuCache");
```

## 五、純手動快取（不使用 WithCache）

```csharp
// 自行控制快取邏輯
public List<SystemMenu> GetMenus()
{
    var cached = db.DataCache.Get<List<SystemMenu>>("key_menus");
    if (cached == null)
    {
        var list = db.Queryable<SystemMenu>()
            .Where(it => it.IsEnabled)
            .ToList();
        db.DataCache.Add("key_menus", list);
        return list;
    }
    return cached;
}
```

## 六、效能調優（Key 過多時）

當系統快取 Key 超過 1000 個，代表有大量不適合快取的查詢被快取了，
這時可重寫模糊刪除方法（5.1.4.109+）：

```csharp
// 程式啟動時設定
StaticConfig.CacheRemoveByLikeStringFunc = (service, key) =>
{
    // 根據 key 模糊刪除快取（避免呼叫 GetAllKey() 掃描所有 Key）
    // 例：用 Redis SCAN + DEL 命令模糊刪除
    var pattern = $"*{key}*";
    var redisKeys = RedisHelper.SearchKeys(pattern);
    foreach (var k in redisKeys)
        RedisHelper.Remove(k);
};
```

## 適用場景建議

| 資料類型 | 推薦快取方式 |
|---|---|
| 系統選單、角色、權限 | `WithCache()` 自動 Key + `IsAutoRemoveDataCache = true` |
| 字典表、設定表 | `WithCache("DictCache")` 追加 Key，手動 `RemoveDataCache` |
| 匯率、費率等低頻更新 | `WithCache(3600)` 設定 1 小時快取 |
| 訂單列表、動態查詢 | ❌ 不適合快取（每個請求 SQL 不同）|
| 使用者當前資料（個人化）| ❌ 不適合快取（資料因人而異）|

---

## 注意事項

- 系統同時存在的快取 Key 應控制在 **1000 個以下**，超過代表快取使用不當
- `SugarRedisClient` 必須用**單例模式**，不要在建構函式中 new
- 全自動清除（`IsAutoRemoveDataCache`）不支援子查詢、MergeTable 等複雜查詢
- `WithCache` 的 Key 是 SQL + 參數，SQL 一直在變的查詢不要用
- `RemoveDataCache()` 只支援實體操作，不支援原生 SQL 方式
