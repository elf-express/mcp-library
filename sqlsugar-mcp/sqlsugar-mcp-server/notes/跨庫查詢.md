# SqlSugar — 跨庫查詢筆記

[//]: # (sqlsugar-source)
> 📖 官方文件:[果糖網 SqlSugar 5x 文件](https://www.donet5.com/Home/Doc?typeId=2244)

## 速查表

| 方式 | 適用場景 | 版本需求 |
|---|---|---|
| 跨庫導航（方式一）| 跨服務器、跨資料庫種類，效能最好 | 5.1.3.24+ |
| 手動 AS 跨庫（方式二）| 同伺服器或已設定 dblink，語法彈性最高 | 無限制 |
| 自動同服務器跨庫（方式三）| MySQL / SqlServer 同服務器、PgSQL Schema | 5.1.4.66+ |
| 自動跨服務器（方式四）| 已設定 dblink 的跨服務器 | 5.1.4.72+ |
| 跨庫子查詢 | ThenMapper / Subqueryable / UnionAll | 5.1.4.107+ |

---

## 一、跨庫導航（推薦，5.1.3.24+）

支援跨服務器、跨資料庫種類，效能接近本庫聯表。
缺點：子表無法過濾主表（可 ToList 後在記憶體過濾）。

### 實體設定

```csharp
[Tenant("db1")]   // 標記屬於哪個庫的 ConfigId
public class Order
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }
    public string Name { get; set; }

    [Navigate(NavigateType.OneToMany, nameof(OrderItem.OrderId))]
    public List<OrderItem> Items { get; set; }
}

[Tenant("db2")]
public class OrderItem
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int ItemId { get; set; }
    public int OrderId { get; set; }
    public decimal? Price { get; set; }

    [Navigate(NavigateType.OneToOne, nameof(OrderId))]
    public Order Order { get; set; }
}
```

### 建立多庫 db 實例

```csharp
var db = new SqlSugarClient(new List<ConnectionConfig>
{
    new ConnectionConfig
    {
        ConfigId              = "db1",
        DbType                = DbType.SqlServer,
        ConnectionString      = "連線字串A",
        IsAutoCloseConnection = true
    },
    new ConnectionConfig
    {
        ConfigId              = "db2",
        DbType                = DbType.SqlServer,
        ConnectionString      = "連線字串B",
        IsAutoCloseConnection = true
    }
});
```

### 查詢

```csharp
// 透過 QueryableWithAttr 自動依 [Tenant] 特性路由到對應庫
var list = db.QueryableWithAttr<OrderItem>()
    .Includes(z => z.Order)   // 自動跨庫導航
    .ToList();

// 不透過特性，手動指定跨庫
var list = db.GetConnection("db1").Queryable<OrderItem>()
    .CrossQuery(typeof(Order), "db2")   // Order 在 db2
    .Includes(z => z.Order)
    .ToList();

// 注意：ISqlSugarClient 介面需先轉成 ITenant
var list = db.AsTenant().QueryableWithAttr<OrderItem>()
    .Includes(z => z.Order)
    .ToList();
```

**✅ 實際案例：AIRSET 主庫 + EL 庫跨庫導航查詢**

```csharp
// 主庫（AIRSET）訂單 + EL 庫的電子發票資料
[Tenant("main")]
public class Order { ... }

[Tenant("el")]
public class EInvoice
{
    [Navigate(NavigateType.OneToOne, nameof(OrderNo))]
    public Order Order { get; set; }
}

var list = db.QueryableWithAttr<EInvoice>()
    .Includes(x => x.Order)
    .Where(x => x.InvoiceDate >= startDate)
    .ToList();
```

## 二、手動 AS 跨庫查詢（語法彈性最高）

直接指定完整表名，支援所有資料庫，但跨服務器需先設定 dblink。

### 單表跨庫

```csharp
// SqlServer：庫名.dbo.表名
var list = db.Queryable<Order>().AS("OtherDb.dbo.Order").ToList();
```

### 多表聯查跨庫

```csharp
// SqlServer（需 dblink）
var list = db.Queryable<Order>()
    .LeftJoin<OrderItem>((o, i) => o.Id == i.OrderId, "db2.dbo.OrderItem")
    .LeftJoin<Custom>((o, i, c) => c.Id == o.CustomId, "db3.dbo.Custom")
    .Where((o, i, c) => o.TypeId == 1)
    .Select((o, i, c) => new { o.Id, i.Name })
    .ToList();

// MySQL（只支援同伺服器）
var list = db.Queryable<Order>()
    .LeftJoin<OrderItem>((o, i) => o.Id == i.OrderId, "db2.OrderItem")
    .ToList();

// Oracle（需 dblink）
var list = db.Queryable<Order>()
    .LeftJoin<OrderItem>((o, i) => o.Id == i.OrderId, "\"ORDERITEM\"@dblink_name")
    .ToList();
```

**✅ 實際案例：同 SqlServer 實例，AIRSET 主庫聯 EL 庫**

```csharp
// 同一個 SqlServer 實例下的兩個資料庫（不需要 dblink）
var list = db.Queryable<Shipment>()
    .LeftJoin<EInvoice>((s, e) => s.OrderNo == e.OrderNo,
        "AIRSET_EL.dbo.EInvoice")   // 完整三段式表名
    .Where(s => s.IsDeleted == false)
    .Select((s, e) => new
    {
        s.ShipmentNo,
        e.InvoiceNo,
        e.Amount
    })
    .ToList();
```

## 三、同服務器自動跨庫（5.1.4.66+）

### 3.1 MySQL / SqlServer 同伺服器

```csharp
var db = new SqlSugarClient(new List<ConnectionConfig>
{
    new ConnectionConfig { ConfigId = "A", DbType = DbType.SqlServer, ConnectionString = "...", IsAutoCloseConnection = true },
    new ConnectionConfig { ConfigId = "B", DbType = DbType.SqlServer, ConnectionString = "...", IsAutoCloseConnection = true }
});

// 實體加上 [Tenant] 特性
[Tenant("A")] public class Order { ... }
[Tenant("B")] public class OrderItem { ... }

// QueryableWithAttr 自動依 ConfigId 生成跨庫表名
var list = db.QueryableWithAttr<Order>()
    .LeftJoin<OrderItem>((o, i) => o.Id == i.OrderId)
    .Select((o, i) => new { o.Id, i.Price })
    .ToList();
```

### 3.2 PgSQL 跨 Schema

PgSQL 不支援跨庫語法，改用 searchpath 實現跨 Schema：

```csharp
new ConnectionConfig
{
    ConfigId         = "B",
    DbType           = DbType.PostgreSQL,
    DbLinkName       = "public",                       // Schema 名稱
    ConnectionString = "...;searchpath=public",        // 對應 searchpath
    IsAutoCloseConnection = true
}

// Left Join 時自動加上 public. 前綴
var list = db.QueryableWithAttr<OptRole>()
    .LeftJoin<Role>((x, y) => x.RoleId == y.Id)
    .ToList();
// → Left JOIN public.Role
```

## 四、跨服務器自動查詢（需設定 dblink，5.1.4.72+）

設定好 DbLinkName 後，QueryableWithAttr 自動拼出完整跨服務器表名。

### 4.1 SqlServer dblink

```csharp
new ConnectionConfig
{
    ConfigId              = "B",
    DbType                = DbType.SqlServer,
    DbLinkName            = "db1.ecology2013_SHQC2.dbo",  // Linked Server 名稱.庫.schema
    // DbLinkName         = "[192.168.1.110].targetDb.dbo"  // 也可以用 IP
    ConnectionString      = "...",
    IsAutoCloseConnection = true
}

// 生成 SQL：Left JOIN db1.ecology2013_SHQC2.dbo.OrderItem
```

SqlServer Linked Server 設定（管理者執行一次）：

```sql
-- 新增 Linked Server
EXEC sp_addlinkedserver
    @server     = 'RemoteServer',
    @srvproduct = '',
    @provider   = 'SQLOLEDB',
    @datasrc    = '192.168.1.102'

-- 設定登入
EXEC sp_addlinkedsrvlogin
    'RemoteServer', 'false', NULL, 'sa', '123456'
```

### 4.2 Oracle dblink

```csharp
new ConnectionConfig
{
    ConfigId   = "B",
    DbLinkName = "@dblink_name",   // 必須以 @ 開頭
    ...
}
// 生成 SQL：Left JOIN OrderItem@dblink_name
```

Oracle dblink 建立：

```sql
CREATE DATABASE LINK dblink_name
    CONNECT TO target_user IDENTIFIED BY target_pwd
    USING '(DESCRIPTION=(ADDRESS=(PROTOCOL=TCP)(HOST=目標IP)(PORT=1521))
            (CONNECT_DATA=(SERVICE_NAME=目標服務名)))';
```

### 4.3 PgSQL 外部表（FDW）

```csharp
new ConnectionConfig
{
    ConfigId   = "B",
    DbLinkName = "remote_",   // 必須以 _ 結尾
    ...
}
// 生成 SQL：Left JOIN remote_OrderItem（無轉義符號）
```

PgSQL 外部表設定：

```sql
CREATE EXTENSION postgres_fdw;

CREATE SERVER target_server
    FOREIGN DATA WRAPPER postgres_fdw
    OPTIONS (dbname 'targetdb', host 'targethost', port '5432');

CREATE USER MAPPING FOR current_user
    SERVER target_server
    OPTIONS (user 'targetuser', password 'targetpassword');

-- 外部表欄位結構需與目標表一致
CREATE FOREIGN TABLE IF NOT EXISTS remote_OrderItem (
    ItemId  INT,
    OrderId INT,
    Price   DECIMAL
) SERVER target_server;
```

### DbLinkName 表名生成規則

| DbLinkName 格式 | 生成 SQL 表名 |
|---|---|
| 一般字串（如 db1.dbo）| db1.dbo.[表名] |
| 以 _ 結尾（如 remote_）| remote_表名（無轉義符）|
| 以 @ 開頭（如 @link）| [表名]@link |

## 五、跨庫子查詢

### 5.1 ThenMapper 跨庫

```csharp
var mydb = db.GetConnection("db2");
mydb.ThenMapper(root, item =>
{
    item.Details = mydb.Queryable<OrderItem>()
        .SetContext(d => d.OrderId, () => item.Id, item)
        .ToList();
});
```

### 5.2 Subqueryable 跨庫（5.1.4.107+）

```csharp
db.Queryable<Order>()
    .Select(it => new
    {
        it.Id,
        ItemCount = SqlFunc.Subqueryable<OrderItem>()
                        .AsWithAttr()         // 依 [Tenant] 特性自動找表名
                        .Where(s => s.OrderId == it.Id)
                        .Count()
    })
    .ToList();

// 也可以用 AS 指定表名
SqlFunc.Subqueryable<OrderItem>()
    .AS("db2.dbo.OrderItem")
    .Where(s => s.OrderId == it.Id)
    .Count()
```

### 5.3 UnionAll / 嵌套查詢跨庫

```csharp
// AsWithAttr 強制使用 [Tenant] 特性對應的表名（切換庫時表名不跟著變）
var q1 = db.Queryable<Order>().AsWithAttr().Where(it => it.Id > 10);
var q2 = db.Queryable<Order>().AsWithAttr().Where(it => it.Id < 2);

db.UnionAll(q1, q2).ToList();
```

## 注意事項

- 跨庫導航不支援子表過濾主表（Any/Count 過濾），可 ToList 後在記憶體過濾，或分頁時先查前 1000 筆主表
- ISqlSugarClient 介面操作跨庫需先轉換：db.AsTenant().QueryableWithAttr\<T\>()
- 手動 AS 跨服務器需先在資料庫層設定 Linked Server / dblink / FDW
- UnionAll 跨庫必須用 AsWithAttr，不能用 QueryableWithAttr（切換庫時表名會跟著變）
- PgSQL 不支援跨庫語法，只能透過 searchpath 或 FDW 外部表實現
- DbLinkName 以 _ 結尾生成無轉義符表名（適合 PgSQL FDW），以 @ 開頭生成 @dblink 語法（適合 Oracle）
