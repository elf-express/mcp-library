## 速查表

| 需求 | 語法 |
|---|---|
| 新版打包(推薦) | db.UseTran(()=>{ 多操作 }) |
| 舊版加入佇列 | db.AddQueue(...) |
| 一次送出 | db.SaveQueues() / SaveQueuesAsync() |
| 多查詢返回 | SaveQueues<T>() |
| 失效排查 | Client 非單例 / Scope 非同步上下文不一致 |
| 限制 | 需在同一 db 上下文內 |

---

## 兩種方式比較

| 方式 | 語法 | 版本 | 適用場景 |
|---|---|---|---|
| UseTran（新版推薦）| using var tran = db.UseTran() | 新版 | 一般事務包裝，語法簡潔 |
| AddQueue / SaveQueues（舊版）| .AddQueue() + db.SaveQueues() | 舊版 | 多操作打包 + 多查詢返回 |

---

## 一、新版 UseTran（推薦）

```csharp
// using 不能少！少了意外發生時事務無法釋放
using var tran = db.UseTran();

// 在事務中執行任意操作
var list  = db.Queryable<Order>().Where(it => it.Status == 1).ToList();
var newId = db.Insertable(new Order { Name = "新訂單", Amount = 100m }).ExecuteReturnIdentity();
db.Updateable<Order>()
    .SetColumns(it => new Order { Status = 2 })
    .Where(it => it.Id == newId)
    .ExecuteCommand();

tran.CommitTran();  // 提交；若未呼叫 CommitTran，using 結束時自動 Rollback
```

**✅ 實際案例：電子發票開立（主表 + 明細 + 狀態更新）**

```csharp
public async Task<int> CreateInvoiceAsync(InvoiceDto dto)
{
    using var tran = db.UseTran();

    // 1. 插入發票主表
    var invoiceId = await db.Insertable(dto.Adapt<Invoice>())
        .ExecuteReturnIdentityAsync();

    // 2. 插入發票明細
    var items = dto.Items.Select(i => new InvoiceItem
    {
        InvoiceId   = invoiceId,
        Description = i.Description,
        Amount      = i.Amount
    }).ToList();
    await db.Insertable(items).ExecuteCommandAsync();

    // 3. 更新訂單狀態
    await db.Updateable<Order>()
        .SetColumns(it => new Order { InvoiceStatus = 1, InvoiceId = invoiceId })
        .Where(it => it.Id == dto.OrderId)
        .ExecuteCommandAsync();

    tran.CommitTran();
    return invoiceId;
}
```

## 二、舊版 AddQueue + SaveQueues

### 2.1 基本增刪改打包

```csharp
// 單個操作（帶事務）
db.Insertable(new Order { Name = "A" }).AddQueue();
db.SaveQueues();

// 多個操作（同一事務）
db.Insertable(new Order { Name = "A" }).AddQueue();
db.Updateable(new Order { Id = 1, Name = "B" }).AddQueue();
db.Deleteable<Order>().Where(it => it.Id == 99).AddQueue();
db.SaveQueues();  // 三個操作在同一個事務中提交

// 非同步提交
db.Insertable(new Order { Name = "A" }).AddQueue();
db.Updateable(new Order { Id = 1, Name = "B" }).AddQueue();
await db.SaveQueuesAsync();
```

### 2.2 多查詢返回

```csharp
// 同時查詢多個表並返回結果（類似多結果集）
db.Queryable<Order>().AddQueue();
db.Queryable<Customer>().AddQueue();
db.AddQueue("SELECT * FROM OrderItem WHERE OrderId = @id", new { id = 1 });

var (orders, customers, items) = db.SaveQueues<Order, Customer, OrderItem>();
// 回傳 Tuple，最多支援 N 個型別
```

### 2.3 混合查詢與增改

```csharp
// 查詢 + 插入一起打包
db.Queryable<Order>().Where(it => it.Status == 1).AddQueue();
db.Insertable(new Order { Name = "新訂單" }).AddQueue();
var result = db.SaveQueues<Order>();
// result 包含查詢結果，插入也會執行
```

## 三、AddQueue 失效排查

### 3.1 SqlSugarClient（非單例）

```csharp
// ✅ 正確：同一個 db 實例（ContextId 相同）
var db = GetInstance();
db.Insertable(new Order { Name = "A" }).AddQueue();
db.Updateable(new Order { Id = 1, Name = "B" }).AddQueue();
db.SaveQueues();

// ❌ 錯誤：兩個不同的 db 實例
var db1 = GetInstance();
var db2 = GetInstance();
db1.Insertable(new Order { Name = "A" }).AddQueue();
db2.Updateable(...).AddQueue();  // db2 和 db1 不是同一個 ContextId → 失效
```

### 3.2 SqlSugarScope（單例注入）非同步問題

```csharp
// ❌ 問題：AddQueue 和 SaveQueues 在不同非同步方法中，AsyncLocal 機制導致失效
public async Task MethodA()
{
    db.Insertable(...).AddQueue();  // 這裡
}
public async Task MethodB()
{
    await db.SaveQueuesAsync();  // 和這裡不在同一個非同步呼叫鏈
}

// ✅ 解法1：確保在同一個方法內
public async Task Process()
{
    db.Insertable(...).AddQueue();
    db.Updateable(...).AddQueue();
    await db.SaveQueuesAsync();  // 同一方法
}

// ✅ 解法2：在宣告 SqlSugar 時立即存取 ContextId
var db = scope.GetService<ISqlSugarClient>();
_ = db.ContextId;  // 強制提前建立，確保 AsyncLocal 共享
db.Insertable(...).AddQueue();
```

## 四、使用限制

| 限制 | 說明 |
|---|---|
| 不支援部分自訂 SQL / 預存程序 | AddQueue 對複雜 SQL 相容性較差 |
| 不適合大量操作 | 個別 DB 有參數化上限，大量插入請用 BulkCopy |
| 不要在 AddQueue 中迴圈 | 應直接用 `db.Insertable(list).ExecuteCommand()`，不要 foreach AddQueue |
| AddQueue 必須 SaveQueues | 未呼叫 SaveQueues 則佇列中的操作不執行 |

## 五、UseTran vs AddQueue 選擇

```csharp
// ✅ 推薦：新版 UseTran（語法清晰，無限制）
using var tran = db.UseTran();
// ... 任意操作 ...
tran.CommitTran();

// 舊版 AddQueue（適合需要返回多查詢結果的場景）
db.Queryable<T1>().AddQueue();
db.Queryable<T2>().AddQueue();
var (list1, list2) = db.SaveQueues<T1, T2>();
```

一般情況優先用 **UseTran**；需要同時執行多個查詢並取回結果時，才考慮 AddQueue + SaveQueues。

---

## 注意事項

- `using var tran = db.UseTran()` 的 `using` 不能省略，省略後若發生例外事務無法釋放
- UseTran 未呼叫 CommitTran 則 using 結束時自動 Rollback
- AddQueue 只適合少量操作，不適合大批量（有參數化數量上限）
- SqlSugarScope 單例 + 非同步場景下，AddQueue 和 SaveQueues 必須在同一非同步呼叫鏈中
- 大量插入請用 `db.Insertable(list).ExecuteCommand()` 而非迴圈 AddQueue
