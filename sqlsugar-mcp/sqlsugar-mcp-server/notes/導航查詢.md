## 速查表

| 需求 | 語法 |
|---|---|
| 一對一 | [Navigate(NavigateType.OneToOne, nameof(外鍵欄位))] |
| 多對一 | [Navigate(NavigateType.ManyToOne, nameof(外鍵), nameof(子表.Id))] |
| 一對多 | [Navigate(NavigateType.OneToMany, nameof(子表.外鍵欄位))] |
| 多對多 | [Navigate(typeof(中間表), nameof(中間表.AId), nameof(中間表.BId))] |
| 填充子物件 | .Includes(x => x.導航屬性) |
| 多層級 | .Includes(x => x.A, a => a.B) |
| 子表過濾 | .Includes(x => x.Books.Where(y => y.Id > 0).ToList()) |
| 主表用子表過濾 | .Where(x => x.Books.Any(z => z.Name == "jack")) |
| 自動填充第一層 | .IncludesAllFirstLayer() |
| 同級兩個導航 | .IncludesAllSecondLayer(x => x.Books) |
| 條件性導航 | .IF(條件, it => it.Includes(x => x.導航屬性)) |

---

## 一、導航查詢特點

- 無需外鍵，只需設定特性與主鍵即可使用
- 高效能（5.0.8.1+ 大幅優化，5.1.4.159+ 再次優化）
- 支援大數據分頁導航查詢
- 支援一對一、一對多、多對多、多層級

## 二、導航關係設定

### 2.1 一對一

```csharp
public class StudentA
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int StudentId { get; set; }
    public int SchoolId  { get; set; }

    // 用例1：主鍵模式，SchoolId 對應 SchoolA 的主鍵
    [Navigate(NavigateType.OneToOne, nameof(SchoolId))]
    public SchoolA SchoolA { get; set; }  // 只能是 null，不能手動賦值

    // 用例2：指定兩端欄位（第一個主表，第二個從表）
    [Navigate(NavigateType.OneToOne, nameof(SchoolId), nameof(SchoolA.Id))]
    public SchoolA SchoolItem { get; set; }

    // 用例3：特殊 UI 框架需要預設值時
    [Navigate(NavigateType.OneToOne, nameof(SchoolId))]
    public SchoolA SchoolA { get; set; } = new OneToOneInitializer<SchoolA>();
    // 只能用 OneToOneInitializer，不能 new SchoolA()
}
```

**✅ 實際案例：訂單對應一張客戶資料**

```csharp
public class Order
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id         { get; set; }
    public int CustomerId { get; set; }
    public string OrderNo { get; set; }

    [Navigate(NavigateType.OneToOne, nameof(CustomerId))]
    public Customer Customer { get; set; }
}

var list = db.Queryable<Order>()
    .Includes(x => x.Customer)
    .Where(x => x.Customer.Name == "王小明")
    .ToList();
```

### 2.3 一對多

```csharp
public class StudentA
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }

    // 用例1：正常一對多，BookA.studenId 對應本表主鍵
    [Navigate(NavigateType.OneToMany, nameof(BookA.studenId))]
    public List<BookA> Books { get; set; }  // 禁止手動賦值

    // 用例2：反向導航（第一個從表欄位，第二個主表欄位）
    [Navigate(NavigateType.OneToMany, nameof(BookA.studenId), nameof(Id))]
    public List<BookA> Books { get; set; }
}
```

**一對多查詢範例**

```csharp
// 基本查詢
var list = db.Queryable<StudentA>()
    .Includes(x => x.Books)
    .ToList();

// 子表過濾 + 排序 + 取前幾筆
var list = db.Queryable<StudentA>()
    .Includes(x => x.Books
        .Where(y => y.BookId > 0)
        .OrderBy(y => y.BookId)
        .Take(10)
        .ToList())
    .ToList();

// 主表用子表條件過濾（不需要 Includes）
var list = db.Queryable<StudentA>()
    .Where(x => x.Books.Any(z => z.Name == "jack"))
    .ToList();

// 主表 + 子表都過濾
var list = db.Queryable<StudentA>()
    .Includes(x => x.Books.Where(it => it.Name == "jack").ToList())
    .Where(x => x.Books.Any(z => z.Name == "jack"))
    .ToList();

// Select 指定欄位
var list = db.Queryable<StudentA>()
    .Includes(x => x.Books.Select(z => new BookA { Name = z.Name }).ToList())
    .ToList();
```

**✅ 實際案例：訂單對應多筆明細**

```csharp
public class Order
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }
    public string OrderNo { get; set; }

    [Navigate(NavigateType.OneToMany, nameof(OrderDetail.OrderId))]
    public List<OrderDetail> Details { get; set; }
}

// 查訂單並帶出明細，只取金額大於 0 的項目
var list = db.Queryable<Order>()
    .Includes(x => x.Details
        .Where(d => d.Amount > 0)
        .OrderBy(d => d.LineNo)
        .ToList())
    .Where(x => x.Details.Any())  // 只查有明細的訂單
    .ToList();
```

### 2.4 多對多

```csharp
public class A1
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }

    // 注意順序：中間表的 AId 對應本表，BId 對應目標表
    [Navigate(typeof(ABMapping), nameof(ABMapping.AId), nameof(ABMapping.BId))]
    public List<B1> BList { get; set; }  // 只能是 null
}

public class ABMapping
{
    [SugarColumn(IsPrimaryKey = true)]
    public int AId { get; set; }
    [SugarColumn(IsPrimaryKey = true)]
    public int BId { get; set; }
}
```

**✅ 實際案例：商品對應多個標籤（Tag）**

```csharp
public class Product
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }

    [Navigate(typeof(ProductTag), nameof(ProductTag.ProductId), nameof(ProductTag.TagId))]
    public List<Tag> Tags { get; set; }
}

// 查商品並帶出所有標籤，只取有標籤的商品
var list = db.Queryable<Product>()
    .Includes(x => x.Tags.Where(t => t.IsActive).ToList())
    .Where(x => x.Tags.Any())
    .ToList();
```

### 2.5 多層級導航（省 > 市 > 區）

```csharp
var list = db.Queryable<StudentA>()
    // 兩層：StudentA → SchoolA → RoomList
    .Includes(st => st.SchoolA, sch => sch.RoomList)
    // 一層：StudentA → Books
    .Includes(st => st.Books)
    .ToList();
```

超過 3 層需加 AsNavQueryable()

```csharp
db.Queryable<Order>()
    .Includes(it => it.A)
    .AsNavQueryable()  // 加在這裡
    .Includes(it => it.B, it => it.C, it => it.D, it => it.E)
    .ToList();
// AsNavQueryable 能不用就不用，正常 Includes 重載已夠用
```

**✅ 實際案例：出貨單 → 出貨明細 → 批號**

```csharp
var list = db.Queryable<Shipment>()
    .Includes(s => s.Details, d => d.BatchList)
    .Where(s => s.IsDeleted == false)
    .ToList();
```

## 三、Includes 進階用法

### 同一層有兩個子導航

```csharp
// 手動寫法
.Includes(x => x.Books, x => x.A)
.Includes(x => x.Books, x => x.B)

// 自動寫法（只到第二層）
.IncludesAllSecondLayer(x => x.Books)
```

### 自動填充第一層所有導航

```csharp
.IncludesAllFirstLayer().ToList()

// 排除特定導航屬性
.IncludesAllFirstLayer(nameof(Order.ProjectPhases)).ToList()

// 依名稱指定導航
.IncludesByNameString(nameof(Order.ProjectPhases)).ToList()
```

### 條件性導航（5.1.4.189+）

```csharp
var list = db.Queryable<Order>()
    .IF(needDetails, it =>
    {
        it.Includes(x => x.Details);
    })
    .ToList();
```

## 四、導航方法（不需 Includes）

```csharp
// Any：主表用子表條件過濾
.Where(x => x.Books.Any())
.Where(x => x.Books.Any(z => z.Id == 1))

// Count：根據子表數量過濾
.Where(x => x.Books.Count() > 2)

// 一對一存在判斷（5.1.2.9+）
.Where(x => SqlFunc.Exists(x.SchoolA.Id))
```

## 五、轉 DTO

### 自動轉換（Mapster，推薦）

```csharp
// 安裝 Nuget：Mapster
var list = db.Queryable<StudentA>()
    .Includes(x => x.Books)
    .ToList();

var dtoList = list.Adapt<List<StudentDTO>>();

// 技巧：DTO 欄位命名對應導航屬性
// TreeDTO.ParentName → Tree.Parent.Name（自動映射）
```

### 手動轉 DTO（5.1.4.71+）

```csharp
// 方法一：自動映射其餘屬性
var list = db.Queryable<Student>()
    .Includes(x => x.Books)
    .Select(x => new StudentDTO
    {
        Books = x.Books
    }, true)  // true = 其餘屬性自動映射
    .ToList();

// 方法二：Mapster 轉換子集合
var list = db.Queryable<Student>()
    .Includes(x => x.Books)
    .Select(x => new StudentDTO
    {
        Name  = x.Name,
        Books = x.Books.Adapt<List<BookDTO>>()
    })
    .ToList();

// 方法三：直接用 C# 方法操作子集合
var list = db.Queryable<Student>()
    .Includes(x => x.Books)
    .Select(x => new StudentDTO
    {
        Name     = x.Name,
        BookIds  = x.Books.Select(it => it.Id).ToList(),
        BookDtos = x.Books.Select(it => new BookDTO { Id = it.Id, Name = it.Name }).ToList()
    })
    .ToList();
```

**✅ 實際案例：訂單含明細轉 DTO**

```csharp
var list = db.Queryable<Order>()
    .Includes(x => x.Details)
    .Select(x => new OrderDTO
    {
        OrderNo    = x.OrderNo,
        CreateTime = x.CreateTime,
        // 明細轉 DTO 並只取需要欄位
        Details    = x.Details.Select(d => new OrderDetailDTO
        {
            ProductName = d.ProductName,
            Qty         = d.Qty,
            Amount      = d.Amount
        }).ToList()
    })
    .ToList();
```

## 六、ThenMapper（手動特殊層級）

適合普通導航無法處理的複雜結構，如非標準關聯、跨庫層級

```csharp
// 兩層：Student → SchoolA → RoomList
var list = db.Queryable<StudentA>().ToList();

// 第一層
db.ThenMapper(list, stu =>
{
    stu.SchoolA = db.Queryable<SchoolA>()
        .SetContext(s => s.SchoolId, () => stu.SchoolId, stu)
        .FirstOrDefault();
});

// 第二層
db.ThenMapper(list.Select(x => x.SchoolA), sch =>
{
    sch.RoomList = db.Queryable<RoomA>()
        .SetContext(r => r.SchoolId, () => sch.SchoolId, sch)
        .ToList();
});
```

- 一定要用 SetContext，直接寫 Where 會造成 N+1 迴圈查詢
- db 內外需要是同一個 ConextId

## 七、性能優化建議

- 升級到 5.0.8.1+ 以上（大數據導航優化）
- 主表一次查 1 萬筆以上建議升級到 5.1.4.159+
- 關聯欄位推薦用主鍵，非主鍵欄位務必加索引
- 高並發場景改用聯表查詢手動映射，效能優於導航自動映射
- 大量資料可搭配分批查詢

```csharp
// 老版本大數據分批導航（每次 300 筆）
var list = new List<Tree>();
db.Queryable<Tree>()
    .Includes(it => it.Child)
    .ForEach(it => list.Add(it), 300);
```

## 八、注意事項

- 導航屬性只能是 null，不能手動 new 賦值（一對一例外：可用 OneToOneInitializer）
- 多對多中間表欄位順序不能寫錯
- 一對多第二個參數：[從表欄位, 主表欄位]，和一對一相反
- ThenMapper 必須用 SetContext，否則會產生 N+1 查詢
- IncludesAllFirstLayer 只自動到第一層，第二層需 IncludesAllSecondLayer，第三層以上全手動
- Includes 中的 Select 只能指定欄位，不能放導航物件
