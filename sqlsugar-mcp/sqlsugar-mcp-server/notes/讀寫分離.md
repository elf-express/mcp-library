## 速查表

| 需求 | 語法 |
|---|---|
| 設定從庫 | SlaveConnectionConfigs |
| 自動分流 | 增刪改→主庫,查詢→從庫 |
| 強制走主 | db.MasterQueryable / 事務內自動走主 |
| 反向模式 | 預設關閉,按需 SlaveQueryable |
| 故障轉移 | 從庫掛掉自動降權 |
| 多主多從 | 搭配多租戶 |
| 搭配二級快取 | WithCache |

---

## 核心規則

| 操作類型 | 路由規則 |
|---|---|
| 查詢（Queryable）| 走**從庫**（依 HitRate 機率輪詢）|
| 插入 / 更新 / 刪除 | 走**主庫** |
| 事務中所有操作 | 強制走**主庫** |

## 一、基本設定

```csharp
// 主庫 + 多個從庫（HitRate 越大，被選到的機率越高）
// 範例：從庫A HitRate=5，從庫B HitRate=10
// A 機率 = 5/(5+10) = 33.3%，B 機率 = 10/(5+10) = 66.6%

var db = new SqlSugarClient(new ConnectionConfig
{
    DbType                = DbType.SqlServer,
    ConnectionString      = "Server=主庫IP;Database=MyDb;...",  // 主庫
    IsAutoCloseConnection = true,
    InitKeyType           = InitKeyType.Attribute,

    SlaveConnectionConfigs = new List<SlaveConnectionConfig>
    {
        new SlaveConnectionConfig
        {
            HitRate          = 7,
            ConnectionString = "Server=從庫1IP;Database=MyDb;..."
        },
        new SlaveConnectionConfig
        {
            HitRate          = 3,
            ConnectionString = "Server=從庫2IP;Database=MyDb;..."
        }
    }
});
```

## 二、AlwaysOn AG 場景設定（AIRSET 實際架構）

```csharp
// SQL Server AlwaysOn 可用性群組：Primary + 多個 Secondary
builder.Services.AddSingleton<ISqlSugarClient>(s =>
    new SqlSugarScope(new ConnectionConfig
    {
        DbType                = DbType.SqlServer,
        ConnectionString      = config["DB:Primary"],   // Primary（寫入）
        IsAutoCloseConnection = true,
        InitKeyType           = InitKeyType.Attribute,

        SlaveConnectionConfigs = new List<SlaveConnectionConfig>
        {
            // Secondary 節點（只讀副本）
            new SlaveConnectionConfig
            {
                HitRate          = 10,
                ConnectionString = config["DB:Secondary1"]  // 192.168.30.11
            },
            new SlaveConnectionConfig
            {
                HitRate          = 10,
                ConnectionString = config["DB:Secondary2"]  // 192.168.30.12
            }
        }
    }, db =>
    {
        db.Aop.OnLogExecuting = (sql, pars) =>
            Console.WriteLine(UtilMethods.GetNativeSql(sql, pars));
    }));
```

## 三、強制走主庫的三種方式

```csharp
// 方式 1：MasterQueryable（查詢強制走主庫，5.0.8+）
var list = db.MasterQueryable<Order>()
    .Where(it => it.Id == 1)
    .ToList();
// 適合：剛寫入後馬上查詢（避免主從延遲問題）

// 方式 2：事務（事務中所有操作自動走主庫）
db.BeginTran();
var order = db.Queryable<Order>().First(it => it.Id == id);  // 走主庫
db.Updateable(order).ExecuteCommand();                         // 走主庫
db.CommitTran();

// 方式 3：全局禁用讀寫分離
db.Ado.IsDisableMasterSlaveSeparation = true;
// 適合：維護模式、批次作業等需要全部走主庫的場景
```

## 四、反向模式（預設關閉讀寫分離，按需啟用）

```csharp
// 設定讀寫分離但預設關閉，只有明確呼叫 SlaveQueryable 才走從庫
static SqlSugarScope db = new SqlSugarScope(new ConnectionConfig
{
    ConnectionString      = "主庫連線字串",
    DbType                = DbType.SqlServer,
    IsAutoCloseConnection = true,
    SlaveConnectionConfigs = new List<SlaveConnectionConfig>
    {
        new SlaveConnectionConfig { HitRate = 10, ConnectionString = "從庫連線字串" }
    }
}, db =>
{
    db.Ado.IsDisableMasterSlaveSeparation = true;  // 全局禁用讀寫分離
});

// 一般查詢走主庫（因為已禁用讀寫分離）
db.Queryable<Order>().ToList();

// 明確指定走從庫（注意：事務中仍然走主庫）
db.SlaveQueryable<Order>().ToList();  // 強制從庫
```

## 五、故障轉移（從庫掛掉自動降權）

```csharp
var db = new SqlSugarClient(new ConnectionConfig
{
    ConnectionString      = "主庫連線字串",
    DbType                = DbType.SqlServer,
    IsAutoCloseConnection = true,
    SlaveConnectionConfigs = new List<SlaveConnectionConfig>
    {
        new SlaveConnectionConfig { HitRate = 80, ConnectionString = "從庫1連線字串" },
        new SlaveConnectionConfig { HitRate = 20, ConnectionString = "從庫2連線字串" }
    }
}, cdb =>
{
    // 啟動時檢查從庫1是否可用
    var slave1 = new SqlSugarClient(new ConnectionConfig
    {
        DbType           = DbType.SqlServer,
        ConnectionString = "從庫1連線字串",
        IsAutoCloseConnection = true
    });

    if (!slave1.Ado.IsValidConnection())
    {
        // 從庫1掛掉 → 將 HitRate 設為 0（不再路由到此庫）
        cdb.CurrentConnectionConfig.SlaveConnectionConfigs[0].HitRate = 0;
    }
    // 注意：IsValidConnection() 失敗可能耗時較長，
    // 建議自行實作邏輯：失敗後 5 分鐘內都不讀此庫
});
```

## 六、多主多從（搭配多租戶）

```csharp
var slaveConfigs = new List<SlaveConnectionConfig>
{
    new SlaveConnectionConfig { HitRate = 8, ConnectionString = "從庫1" },
    new SlaveConnectionConfig { HitRate = 2, ConnectionString = "從庫2" }
};

var db = new SqlSugarClient(new List<ConnectionConfig>
{
    new ConnectionConfig
    {
        ConfigId              = "sqlserver",
        DbType                = DbType.SqlServer,
        ConnectionString      = "SqlServer 主庫",
        IsAutoCloseConnection = true,
        SlaveConnectionConfigs = slaveConfigs
    },
    new ConnectionConfig
    {
        ConfigId              = "mysql",
        DbType                = DbType.MySql,
        ConnectionString      = "MySQL 主庫",
        IsAutoCloseConnection = true,
        SlaveConnectionConfigs = slaveConfigs
    }
});

// 取得子 db 操作
var sqlDb   = db.GetConnection("sqlserver");
var mysqlDb = db.GetConnection("mysql");

// 單庫事務
sqlDb.Ado.BeginTran();
sqlDb.Ado.CommitTran();

// 多租戶跨庫事務
db.BeginTran();
sqlDb.Insertable(order).ExecuteCommand();
mysqlDb.Insertable(log).ExecuteCommand();
db.CommitTran();

// AOP 多庫各別設定（不能直接 db.Aop）
db.GetConnection("sqlserver").Aop.OnLogExecuting = (sql, p) => Console.WriteLine("[SQL]" + sql);
db.GetConnection("mysql").Aop.OnLogExecuting     = (sql, p) => Console.WriteLine("[MySQL]" + sql);
```

## 七、AddScoped 模式注意事項

```csharp
// ❌ 錯誤：config 物件單例，多個請求共用同一個 config 可能導致連線字串被改
services.AddScoped<ISqlSugarClient>(s =>
    new SqlSugarClient(sharedConfig));  // sharedConfig 單例 → 危險！

// ✅ 正確：每次 new 時用 CopyConfig 複製一份獨立的設定
services.AddScoped<ISqlSugarClient>(s =>
    new SqlSugarClient(UtilMethods.CopyConfig(config)));

// 多庫版
services.AddScoped<ISqlSugarClient>(s =>
    new SqlSugarClient(configs.Select(c => UtilMethods.CopyConfig(c)).ToList()));
```

## 八、二級緩存搭配讀寫分離

```csharp
// 查詢走從庫 + 快取結合
var list = db.Queryable<SystemMenu>()
    .WithCache(600)  // 快取 600 秒（走從庫取資料後快取）
    .ToList();

// 更新主庫後清除快取
db.Updateable(menu)
    .RemoveDataCache()  // 清除相關查詢快取
    .ExecuteCommand();  // 走主庫
```

## 注意事項

- 資料庫層的主從複製（如 SQL Server AlwaysOn AG 同步延遲）需在資料庫層設定，SqlSugar 只負責路由
- 剛寫入後馬上查詢，需用 `MasterQueryable` 避免主從延遲造成查不到資料
- `IsValidConnection()` 失敗時連線嘗試可能耗時較長，建議自行實作短路邏輯（失敗後 N 分鐘不讀）
- AddScoped 模式需用 `UtilMethods.CopyConfig()` 複製設定，避免 config 物件被共享
- SqlSugarScope 單例模式密碼失效問題：升級到 5.0.5.1+
- 多庫 AOP 設定用 `db.GetConnection("configId").Aop.xxx`，不能直接用 `db.Aop.xxx`
