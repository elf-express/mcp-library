# SqlSugar — 導航更新（級聯保存）筆記

[//]: # (sqlsugar-source)
> 📖 官方文件:[果糖網 SqlSugar 5x 文件](https://www.donet5.com/Home/Doc?typeId=2432)

## 核心邏輯

導航更新 = 更新主表 + 依關聯類型對子表進行不同策略的同步。
不依賴資料庫外鍵，只需設定實體導航特性即可使用。

---

## 速查表

| 需求 | 語法 |
|---|---|
| 多層級導航更新 | db.UpdateNav(list).Include(z => z.子表).ThenInclude(z => z.孫表).ExecuteCommand() |
| 一對一（存在更新，不存在插入）| .Include(z => z.School) |
| 一對多模式一（先刪後加，效能好）| .Include(z => z.Items)（預設）|
| 一對多模式二（插入或更新或刪除）| .Include(z => z.Items, new UpdateNavOptions { OneToManyInsertOrUpdate = true }) |
| 多對多 — 只更新中間表 | .Include(z => z.Roles)（預設）|
| 多對多 — 更新主表 + 中間表 | new UpdateNavOptions { ManyToManyIsUpdateA = true } |
| 多對多 — 更新子表 + 中間表 | new UpdateNavOptions { ManyToManyIsUpdateB = true } |
| 主表插入或更新 | new UpdateNavRootOptions { IsInsertRoot = true } |
| 主表忽略欄位 | new UpdateNavRootOptions { IgnoreColumns = new[] { "CreateTime" } } |
| 主表只更新欄位 | new UpdateNavRootOptions { UpdateColumns = new[] { "Name" } } |
| 禁止更新主表 | new UpdateNavRootOptions { IsDisableUpdateRoot = true } |
| NULL 欄位不更新 | new UpdateNavRootOptions { IsIgnoreAllNullColumns = true } |
| 自動更新第一層所有導航 | .IncludesAllFirstLayer() |

## 一、多層級導航更新語法

一對一、一對多、多對多可以混合使用。

```csharp
var list = new List<Student>
{
    new Student
    {
        Id       = 1,
        Name     = "Jack",
        SchoolId = 2,
        SchoolA = new SchoolA { Id = 1, SchoolName = "北大更新", RoomList = new List<Room>{ ... } },
        Books   = new List<Book> { new Book { BookId = 0, Name = "新書" } }
    }
};

db.UpdateNav(list)
    .Include(z => z.SchoolA).ThenInclude(z => z.RoomList)  // 更新兩層：Root → SchoolA → RoomList
    .Include(z => z.Books)                                   // 更新一層：Root → Books
    .ExecuteCommand();

// 異步
await db.UpdateNav(list).Include(z => z.Books).ExecuteCommandAsync();
```

## 二、一對一導航更新

邏輯：更新主表；子表存在更新，不存在插入。

```csharp
db.UpdateNav(list)
    .Include(z => z.SchoolA)
    .ExecuteCommand();
```

## 三、一對多導航更新

### 3.1 模式一：先刪後加（預設，效能最好）

邏輯：更新主表 → 刪除舊子表 → 插入新子表。
適合子表整批替換的場景。

```csharp
db.UpdateNav(list)
    .Include(z => z.Books)   // 預設模式
    .ExecuteCommand();

// 強制清除所有舊子表（包含孫表的髒資料）
db.UpdateNav(list)
    .Include(z => z.Books, new UpdateNavOptions { OneToManyDeleteAll = true })
    .ThenInclude(z => z.Tags)
    .ExecuteCommand();
```

### 3.2 模式二：插入/更新/刪除（5.1.4.113+）

邏輯：更新主表 → 子表存在更新、不存在插入，同時支援前端刪除的子表。
適合前端傳入差異清單的場景，效能略低於模式一。

```csharp
db.UpdateNav(list)
    .Include(z => z.Books, new UpdateNavOptions { OneToManyInsertOrUpdate = true })
    .ExecuteCommand();

// 注意：子表要走更新邏輯，必須帶有主鍵；無主鍵視為新增
```

### 3.3 其他一對多選項

```csharp
// 禁止更新主表（只更新子表）
db.UpdateNav(list, new UpdateNavRootOptions { IsDisableUpdateRoot = true })
    .Include(z => z.Books)
    .ExecuteCommand();

// NULL 欄位不更新（5.1.4.144+）
db.UpdateNav(list)
    .Include(z => z.Books, new UpdateNavOptions
    {
        OneToManyInsertOrUpdate = true,
        IgnoreNullColumns = true
    })
    .ExecuteCommand();

// 忽略特定欄位
db.UpdateNav(list)
    .Include(z => z.Books, new UpdateNavOptions
    {
        OneToManyInsertOrUpdate = true,
        IgnoreColumns = new[] { "CreateTime" }
    })
    .ExecuteCommand();

// 軟刪除模式（結合過濾器，5.1.4.86+）
db.QueryFilter.AddTableFilter<Book>(it => it.IsDeleted == false);
db.UpdateNav(list)
    .Include(z => z.Books, new UpdateNavOptions { OneToManyEnableLogicDelete = true })
    .ExecuteCommand();
```

**✅ 實際案例：編輯訂單及明細（前端傳入完整明細清單）**

```csharp
[HttpPut("update")]
public async Task<ApiResult> UpdateOrder([FromBody] OrderEditDto dto)
{
    var order = dto.Adapt<Order>();
    order.Items = dto.Items.Adapt<List<OrderItem>>();

    await db.UpdateNav(order)
        .Include(o => o.Items)   // 先刪後加，適合整批替換
        .ExecuteCommandAsync();

    return ApiResult.Success();
}
```

## 四、多對多導航更新

四種模式，用 UpdateNavOptions 控制。

```csharp
// 模式一（預設）：只更新中間表（清空後重新插入）
db.UpdateNav(list)
    .Include(z => z.Roles)
    .ExecuteCommand();
// 技巧：只更新中間表時，A/B 物件只需帶主鍵，其他欄位不需賦值

// 模式二：更新主表（A）+ 更新中間表
db.UpdateNav(list)
    .Include(z => z.Roles, new UpdateNavOptions { ManyToManyIsUpdateA = true })
    .ExecuteCommand();

// 模式三：存在更新子表（B）/不存在插入 + 更新中間表
db.UpdateNav(list)
    .Include(z => z.Roles, new UpdateNavOptions { ManyToManyIsUpdateB = true })
    .ExecuteCommand();

// 模式四：更新主表（A）+ 存在更新/不存在插入子表（B）+ 更新中間表
db.UpdateNav(list)
    .Include(z => z.Roles, new UpdateNavOptions
    {
        ManyToManyIsUpdateA = true,
        ManyToManyIsUpdateB = true
    })
    .ExecuteCommand();

// 中間表額外欄位賦值（5.1.4.86+）
db.UpdateNav(user)
    .Include(z => z.Roles, new UpdateNavOptions
    {
        ManyToManySaveMappingTemplate = new UserRole
        {
            UpdateTime = DateTime.Now,
            OrgId      = currentOrgId
        }
    })
    .ExecuteCommand();

// 多對多軟刪除（5.1.4.89+）
db.UpdateNav(list)
    .Include(z => z.Roles, new UpdateNavOptions { ManyToManyEnableLogicDelete = true })
    .ExecuteCommand();
```

## 五、主表通用選項（UpdateNavRootOptions）

```csharp
db.UpdateNav(list, new UpdateNavRootOptions
{
    IgnoreColumns        = new[] { "CreateTime", "CreateUser" },  // 主表忽略更新欄位
    UpdateColumns        = new[] { "Name", "Status" },             // 主表只更新這些欄位
    IsInsertRoot         = true,   // 主表不存在插入，存在更新（插或更新）
    IgnoreInsertColumns  = new[] { "Id" },  // 主表插入時忽略的欄位
    IsIgnoreAllNullColumns = true, // 主表 NULL 欄位不更新
    IsDisableUpdateRoot  = true,   // 禁止更新主表（只處理子表）
    IsDiffLogEvent       = true,   // 啟用差異日誌
    IsOptLock            = true    // 主表啟用樂觀鎖
})
.Include(z => z.Roles)
.ExecuteCommand();
```

**✅ 實際案例：編輯時不更新建立時間和建立人**

```csharp
await db.UpdateNav(order, new UpdateNavRootOptions
{
    IgnoreColumns = new[] { "CreateTime", "CreateUser" }
})
.Include(o => o.Items)
.ExecuteCommandAsync();
```

## 六、自動更新第一層所有導航（5.1.4.108+）

```csharp
// 自動包含所有第一層導航
db.UpdateNav(list).IncludesAllFirstLayer().ExecuteCommand();

// 排除特定導航
db.UpdateNav(list)
    .IncludesAllFirstLayer(nameof(Student.TempField))
    .ExecuteCommand();

// 自動兩層 + 手動第三層
db.UpdateNav(list)
    .IncludesAllFirstLayer()
    .IncludeByNameString(nameof(Student.SchoolA))
        .ThenIncludeByNameString(nameof(SchoolA.RoomList))
    .ExecuteCommand();
```

## 七、常見問題：多個物件共用同一個子表引用

從資料庫查詢後組裝物件，若多個主表物件的子表指向同一個 List 實例，會造成導航更新異常：

```csharp
// ❌ 問題：tasks 共用同一個 emps 物件
var tasks = db.Queryable<Task>().Where(t => t.Id == 1 || t.Id == 2).ToList();
var emps  = new List<Emp> { new Emp { Id = 101 }, new Emp { Id = 102 } };

foreach (var task in tasks)
    task.EmpList = emps;  // 多個 task 共用同一個 List 實例

// ✅ 解決：用 TranslateCopy 複製成獨立物件
tasks = db.Utilities.TranslateCopy(tasks);
// 對效能要求高可用 AutoMapper 等工具
```

## 注意事項

- 一對多模式一（先刪後加）效能最好，適合整批替換；模式二適合差異更新但效能略低
- 多對多預設只更新中間表，不影響 A/B 兩張實體表
- long 主鍵多服務器部署必須設定不同的 SnowFlakeSingle.WorkId
- UpdateNav 中 IsInsertRoot = true 等於 Storageable 的主表插入或更新
- 子表走更新邏輯需要有主鍵，無主鍵一律視為新增
