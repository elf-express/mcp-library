## 速查表

| 方式 | 語法 | 多庫相容 | AOP |
|---|---|---|---|
| 字典集合刪除 | db.Deleteable\<object\>().AS("表名").WhereColumns(list) | ✅ | ❌ |
| SQL 字串刪除 | db.Deleteable\<object\>().AS("表名").Where("sql") | ⚠️ 較差 | ❌ |
| 動態建類刪除 | db.DeleteableByObject(value).ExecuteCommand() | ✅✅ 最佳 | ✅ |

## 一、字典集合刪除（多庫相容）

```csharp
// 單筆刪除（字典集合方式）
var dc = new Dictionary<string, object> { ["Id"] = 1 };
var list = new List<Dictionary<string, object>> { dc };

db.Deleteable<object>()
    .AS("Order")
    .WhereColumns(list)
    .ExecuteCommand();
// → DELETE FROM Order WHERE Id = @Id

// 批量刪除（多筆）
var deleteList = new List<Dictionary<string, object>>
{
    new Dictionary<string, object> { ["Id"] = 1 },
    new Dictionary<string, object> { ["Id"] = 2 },
    new Dictionary<string, object> { ["Id"] = 3 }
};
db.Deleteable<object>()
    .AS("Order")
    .WhereColumns(deleteList)
    .ExecuteCommand();

// 多欄位條件（複合主鍵）
var dc2 = new Dictionary<string, object> { ["OrderId"] = 1, ["ProductId"] = 101 };
var list2 = new List<Dictionary<string, object>> { dc2 };
db.Deleteable<object>()
    .AS("OrderItem")
    .WhereColumns(list2)
    .ExecuteCommand();
```

**✅ 實際案例：低程式碼平台通用刪除 API**

```csharp
[HttpDelete("{tableName}/{id}")]
public async Task<ApiResult> DeleteRow(string tableName, int id)
{
    if (!_allowedTables.Contains(tableName))
        return ApiResult.Fail("不允許的資料表");

    var list = new List<Dictionary<string, object>>
    {
        new Dictionary<string, object> { ["Id"] = id }
    };

    int rows = await db.Deleteable<object>()
        .AS(tableName)
        .WhereColumns(list)
        .ExecuteCommandAsync();

    return rows > 0 ? ApiResult.Success() : ApiResult.Fail("找不到資料");
}
```

## 二、SQL 字串刪除

多庫相容性較差，適合單一資料庫且有特殊語法需求的場景。

```csharp
// 單筆刪除
db.Deleteable<object>()
    .AS("Order")
    .Where("Id = @id", new { id = 1 })
    .ExecuteCommand();
// → DELETE FROM Order WHERE Id = @id

// IN 批量刪除
db.Deleteable<object>()
    .AS("Order")
    .Where("Id IN (@ids)", new { ids = new int[] { 1, 2, 3 } })
    .ExecuteCommand();
// → DELETE FROM Order WHERE Id IN ('1','2','3')
```

## 三、動態建類刪除（最完整多庫 + AOP）

```csharp
// 建立動態類型（WithCache 快取，啟動時建一次）
var type = db.DynamicBuilder()
    .CreateClass("Order", new SugarTable())
    .CreateProperty("Id",   typeof(int),    new SugarColumn { IsPrimaryKey = true, IsIdentity = true })
    .CreateProperty("Name", typeof(string), new SugarColumn())
    .WithCache()
    .BuilderType();

// 從字典建立物件並刪除
var dic   = new Dictionary<string, object> { ["Id"] = 1, ["Name"] = "Jack" };
var value = db.DynamicBuilder().CreateObjectByType(type, dic);

db.DeleteableByObject(value).ExecuteCommand();

// 同一動態類也支援插入、更新、插入或更新、查詢
db.InsertableByObject(value).ExecuteCommand();
db.UpdateableByObject(value).ExecuteCommand();
db.StorageableByObject(value).ExecuteCommand();  // 插入或更新
db.QueryableByObject(type).ToList();             // 5.1.4.84+
```

更多動態建類用法：https://www.donet5.com/Home/Doc?typeId=2562

## 各方式比較

| 比較項目 | 字典集合 | SQL 字串 | 動態建類 |
|---|---|---|---|
| 批量支援 | ✅ | ✅ | ✅ |
| 多庫相容 | ✅ | ⚠️ 較差 | ✅✅ 最佳 |
| AOP 支援 | ❌ | ❌ | ✅ |
| 程式碼量 | 少 | 最少 | 較多 |
| 適用場景 | 通用低程式碼 | 單一 DB 特殊需求 | 產品級多庫 |

---

## 注意事項

- WhereColumns 接收的是 List\<Dictionary\<string, object\>\>，即使只刪一筆也要包成集合
- SQL 字串的 Where 條件含資料庫特有語法（如 TOP、ROWNUM）時，跨庫會有問題
- 動態建類刪除依賴主鍵，字典中必須包含主鍵欄位
- 無實體刪除沒有軟刪除（IsLogic）選項，軟刪除請改用無實體更新（更新 IsDeleted 欄位）
