# SqlSugar — 導航插入（級聯插入）筆記

[//]: # (sqlsugar-source)
> 📖 官方文件:[果糖網 SqlSugar 5x 文件](https://www.donet5.com/Home/Doc?typeId=2430)

## 與 EF Core 差異

SqlSugar 不依賴資料庫外鍵，只需設定實體導航特性即可使用，不需要在資料庫建立外鍵約束。

---

## 速查表

| 需求 | 語法 |
|---|---|
| 多層級導航插入 | db.InsertNav(list).Include(z => z.子表).ThenInclude(z => z.孫表).ExecuteCommand() |
| 一對一插入 | .Include(z => z.SchoolA) |
| 一對多插入 | .Include(z => z.Books) |
| 一對多存在不插入 | .Include(z => z.Items, new InsertNavOptions { OneToManyIfExistsNoInsert = true }) |
| 多對多插入 | .Include(z => z.Roles) |
| 多對多不清空中間表 | .Include(z => z.Roles, new InsertNavOptions { ManyToManyNoDeleteMap = true }) |
| 中間表額外欄位賦值 | new InsertNavOptions { ManyToManySaveMappingTemplate = new 中間表 { 欄位 = 值 } } |
| 自動插入第一層所有導航 | .IncludesAllFirstLayer() |
| 忽略主表欄位 | db.InsertNav(list, new InsertNavRootOptions { IgnoreColumns = new[] { "FieldName" } }) |
| 自訂雪花 ID | StaticConfig.CustomSnowFlakeFunc = () => 你的方法() |

## 重要前置設定：雪花 ID WorkId

長整數（long）主鍵會自動用雪花 ID 填充，多服務器部署時必須設定不同的 WorkId，避免 ID 衝突：

```csharp
// 程式啟動時設定一次（從設定檔讀取，每個服務器必須不同）
SnowFlakeSingle.WorkId = 唯一數字;  // 例如：1、2、3...
```

## 一、多層級導航插入語法

一對一、一對多、多對多可以混合使用。

```csharp
var list = new List<Student>
{
    new Student
    {
        Id       = 0,
        Name     = "Jack",
        SchoolId = 0,
        // 一對一導航
        SchoolA = new SchoolA
        {
            SchoolName = "北大",
            // 孫層導航
            RoomList = new List<Room>
            {
                new Room { RoomName = "101" }
            }
        },
        // 一對多導航
        Books = new List<Book>
        {
            new Book { Name = "數學" },
            new Book { Name = "英文" }
        }
    }
};

db.InsertNav(list)
    .Include(z => z.SchoolA).ThenInclude(z => z.RoomList)  // Root → SchoolA → RoomList（兩層）
    .Include(z => z.Books)                                   // Root → Books（一層）
    .ExecuteCommand();
```

多層級語法：`.Include(...).ThenInclude(...).ThenInclude(...)`

## 二、一對一導航插入

邏輯：插入主表；子表存在不插入，不存在才插入。

```csharp
// 實體設定
public class Student
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }
    public string Name { get; set; }
    public int SchoolId { get; set; }

    [Navigate(NavigateType.OneToOne, nameof(SchoolId))]
    public School School { get; set; }  // 不能手動賦預設值，只能是 null
}

// 插入
db.InsertNav(list)
    .Include(z => z.School)
    .ExecuteCommand();
```

## 三、一對多導航插入

### 3.1 強制插入子表（預設）

邏輯：主表不存在插入；子表強制插入（不管是否存在）。

```csharp
db.InsertNav(list)
    .Include(z => z.Books)
    .ExecuteCommand();
```

### 3.2 子表存在不插入（5.1.1+）

邏輯：主表不存在插入；子表存在不插入，不存在才插入。

```csharp
db.InsertNav(list)
    .Include(z => z.Books, new InsertNavOptions { OneToManyIfExistsNoInsert = true })
    .ExecuteCommand();
```

**✅ 實際案例：訂單主表 + 明細同時新增**

```csharp
var order = new Order
{
    OrderNo    = GenerateOrderNo(),
    CustomerId = req.CustomerId,
    CreateTime = DateTime.Now,
    Items = req.Items.Select(i => new OrderItem
    {
        ProductId = i.ProductId,
        Qty       = i.Qty,
        Price     = i.Price
    }).ToList()
};

await db.InsertNav(order)
    .Include(o => o.Items)
    .ExecuteCommandAsync();
// 自動：先插入 Order，取得 Id，再設定 Items.OrderId 批量插入
```

### 3.3 多個 Include 共用同一子表時

當多個 Include 指向相同子表（如 Items 下有 Lefts 和 Rights），第二個 Include 需加 `OneToManyIfExistsNoInsert` 避免重複插入：

```csharp
db.InsertNav(main)
    .Include(it => it.Items)
        .ThenInclude(it => it.Lefts)
    .Include(it => it.Items, new InsertNavOptions { OneToManyIfExistsNoInsert = true })
        .ThenInclude(it => it.Rights)
    .ExecuteCommand();
```

## 四、多對多導航插入

邏輯：主表不存在插入；子表不存在插入；插入中間表（預設會先清空再插入中間表）。

### 4.1 實體設定

```csharp
// 中間表
public class UserRole
{
    [SugarColumn(IsPrimaryKey = true)]
    public int UserId { get; set; }
    [SugarColumn(IsPrimaryKey = true)]
    public int RoleId { get; set; }
}

// 主表
public class User
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }
    public string Name { get; set; }
    [Navigate(typeof(UserRole), nameof(UserRole.UserId), nameof(UserRole.RoleId))]
    public List<Role> Roles { get; set; }  // 只能是 null，不能給預設值
}

// 子表
public class Role
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }
    public string Name { get; set; }
}
```

### 4.2 插入

```csharp
// 標準插入（每次清空中間表再重新插入）
db.InsertNav(list)
    .Include(z => z.Roles)
    .ExecuteCommand();

// 不清空中間表（追加模式）（5.1.3.29+）
db.InsertNav(list)
    .Include(z => z.Roles, new InsertNavOptions { ManyToManyNoDeleteMap = true })
    .ExecuteCommand();

// 小技巧：A 和 B 主鍵已存在時，只需傳主鍵值，不會觸發 A/B 的插入
```

### 4.3 中間表額外欄位賦值（5.1.4.86+）

```csharp
db.InsertNav(user)
    .Include(z => z.Roles, new InsertNavOptions
    {
        ManyToManySaveMappingTemplate = new UserRole
        {
            CreateTime = DateTime.Now,  // 中間表額外欄位
            OrgId      = currentOrgId
        }
    })
    .ExecuteCommand();
```

## 五、主表選項設定

```csharp
// 忽略主表指定欄位
db.InsertNav(list, new InsertNavRootOptions
{
    IgnoreColumns  = new[] { "Remark" },   // 忽略這些欄位不插入
    // InsertColumns = new[] { "Name", "Amount" }  // 只插入這些欄位
})
.Include(z => z.Items)
.ExecuteCommand();
```

## 六、自動插入第一層所有導航（5.1.4.108+）

當第一層導航很多時，可以用 IncludesAllFirstLayer 省略逐個 Include：

```csharp
// 自動包含所有第一層導航
db.InsertNav(list).IncludesAllFirstLayer().ExecuteCommand();

// 排除不需要的導航
db.InsertNav(list)
    .IncludesAllFirstLayer(nameof(Student.TempField))  // 排除 TempField
    .ExecuteCommand();

// 自動兩層 + 手動指定第三層
db.InsertNav(list)
    .IncludesAllFirstLayer()
    .IncludeByNameString(nameof(Student.SchoolA))
        .ThenIncludeByNameString(nameof(SchoolA.RoomList))
    .ExecuteCommand();
```

## 七、自訂雪花 ID

若專案已有雪花 ID 實作，可替換 SqlSugar 內建的：

```csharp
// 程式啟動時設定一次
StaticConfig.CustomSnowFlakeFunc = () =>
{
    return YourSnowflakeService.NextId();  // 替換成你自己的實作
};
```

## 效能說明

| 主鍵類型 | 效能 | 說明 |
|---|---|---|
| 自增（Identity）| 最差 | 需逐筆插入後取得 ID，才能設定子表 FK |
| 雪花 ID（long）| 好 | 可預先生成 ID，批量插入 |
| GUID | 好 | 同雪花 ID，可預先生成 |

自增主鍵建議只用於小數據量的導航插入場景。

---

## 注意事項

- 導航屬性（如 Books、Roles）只能初始化為 null，不能給 `new List<>()`，否則空清單也會觸發操作
- 多對多預設每次插入前清空中間表，追加模式需設定 ManyToManyNoDeleteMap = true
- long 主鍵多服務器部署必須設定不同的 SnowFlakeSingle.WorkId
- 多個 Include 指向同一子表時，第二個 Include 需加 OneToManyIfExistsNoInsert 避免重複
