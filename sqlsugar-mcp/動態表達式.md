## 速查表

| 需求 | 語法 |
|---|---|
| 動態 AND 條件 | Expressionable.Create\<T\>().AndIF(條件, it => ...).ToExpression() |
| 動態 OR 條件 | .Or(it => ...) |
| 條件成立才加 | .AndIF(bool, it => ...) / .OrIF(bool, it => ...) |
| 多表動態表達式 | Expressionable.Create\<T1, T2, T3\>().And((x,y,z) => ...) |
| 多值模糊搜尋（IN LIKE）| foreach 迴圈 exp.Or(it => it.Name.Contains(item)) |
| 字串轉表達式 | .Where("it", $"it.Name={"a"}") |
| 表達式轉 SQL | expContext.Resolve(exp, ...) + expContext.Result.GetString() |
| 去掉 1=1 | 檢查 LambdaExpression.Body 是否為 ConstantExpression true |
| 超過 1000 條件 | 用 new Thread(..., int.MaxValue) 設定最大堆疊 |

## 一、動態表達式基本用法

### 1.1 連寫風格（常用）

```csharp
Expression<Func<Order, bool>> exp = Expressionable.Create<Order>()
    .AndIF(p > 0,    it => it.Id == p)
    .AndIF(name != null, it => it.Name == name && it.Sex == 1)
    .ToExpression();  // 不能少這一句

var list = db.Queryable<Order>().Where(exp).ToList();
// 不需要判斷 null，不需要加 1=1，條件不成立時自動略過
```

### 1.2 分段寫法（邏輯複雜時）

```csharp
var expable = Expressionable.Create<Order>();

if (someCondition)
    expable.And(it => it.Status == 1);

if (anotherCondition)
    expable.And(it => it.Amount > 1000);

var exp = expable.ToExpression();  // 用 var 接，不能直接傳
db.Queryable<Order>().Where(exp).ToList();
```

**✅ 實際案例：訂單搜尋介面，多個可選條件**

```csharp
public List<OrderDto> SearchOrders(OrderSearchRequest req)
{
    var exp = Expressionable.Create<Order>()
        .AndIF(!string.IsNullOrEmpty(req.OrderNo),
            it => it.OrderNo.Contains(req.OrderNo))
        .AndIF(req.CustomerId.HasValue,
            it => it.CustomerId == req.CustomerId)
        .AndIF(req.StartDate.HasValue,
            it => it.CreateTime >= req.StartDate)
        .AndIF(req.EndDate.HasValue,
            it => it.CreateTime <= req.EndDate)
        .AndIF(req.Status.HasValue,
            it => it.Status == req.Status)
        .ToExpression();

    return db.Queryable<Order>()
        .Where(exp)
        .OrderBy(it => it.CreateTime, OrderByType.Desc)
        .ToList();
}
```

### 1.3 多表動態表達式

```csharp
var exp = Expressionable.Create<Order, Customer>()
    .And((o, c) => o.CustomerId == c.Id)
    .AndIF(name != null, (o, c) => c.Name.Contains(name))
    .ToExpression();

var list = db.Queryable<Order>()
    .LeftJoin<Customer>((o, c) => o.CustomerId == c.Id)
    .Where(exp)
    .ToList();
```

**✅ 實際案例：出貨單搜尋，可選擇過濾客戶名稱**

```csharp
var exp = Expressionable.Create<Shipment, Customer>()
    .And((s, c) => s.IsDeleted == false)
    .AndIF(!string.IsNullOrEmpty(customerName),
        (s, c) => c.Name.Contains(customerName))
    .AndIF(shipDateStart.HasValue,
        (s, c) => s.ShipDate >= shipDateStart)
    .ToExpression();

var list = db.Queryable<Shipment>()
    .LeftJoin<Customer>((s, c) => s.CustomerId == c.Id)
    .Where(exp)
    .Select((s, c) => new ShipmentDto
    {
        ShipmentNo   = s.ShipmentNo,
        CustomerName = c.Name,
        ShipDate     = s.ShipDate
    })
    .ToList();
```

## 二、OR 條件

### 2.1 基本 OR

```csharp
var exp = Expressionable.Create<Order>()
    .And(it => it.Id == 1)
    .Or(it => it.Id == 100)
    .ToExpression();

var list = db.Queryable<Order>().Where(exp).ToList();
// → WHERE Id = 1 OR Id = 100
```

### 2.2 IN 的模糊搜尋（多值 LIKE OR）

```csharp
// 搜尋 Name LIKE 'a' OR Name LIKE 'b' OR Name LIKE 'c'
var keywords = new string[] { "業務", "採購", "倉管" };
var exp = Expressionable.Create<Employee>();

foreach (var keyword in keywords)
{
    exp.Or(it => it.Name.Contains(keyword));
}

var list = db.Queryable<Employee>()
    .Where(exp.ToExpression())
    .ToList();
// → WHERE Name LIKE '%業務%' OR Name LIKE '%採購%' OR Name LIKE '%倉管%'
```

**✅ 實際案例：搜尋框同時比對多個欄位（任一符合即顯示）**

```csharp
// 搜尋字輸入 "王小明"，同時比對訂單號、客戶名、備註
string keyword = "王小明";
var exp = Expressionable.Create<Order>()
    .Or(it => it.OrderNo.Contains(keyword))
    .Or(it => it.CustomerName.Contains(keyword))
    .Or(it => it.Remark.Contains(keyword));

var list = db.Queryable<Order>()
    .Where(exp.ToExpression())
    .ToList();
```

## 三、WhereIF（更簡單的動態條件）

簡單的動態條件直接用 WhereIF，比 Expressionable 更直覺

```csharp
var list = db.Queryable<Order>()
    .WhereIF(p > 0,       it => it.Id == p)
    .WhereIF(name != null, it => it.Name == name)
    .ToList();
// WhereIF 條件不成立時完全不加到 SQL
```

- 只需要 AND 條件時優先用 WhereIF，需要 OR 時才用 Expressionable

## 四、去掉 1=1

當所有 AndIF 條件都不成立時，Expressionable 會產生 1=1，若不想要可以這樣處理

```csharp
var ex = Expressionable.Create<Order>()
    .AndIF(id > 0, it => it.Id == id)
    .ToExpression();

// 判斷是否為空表達式（1=1）
bool isEmpty = ((ex as LambdaExpression).Body as ConstantExpression)
                    ?.Value?.Equals(true) == true;

var list = db.Queryable<Order>()
    .WhereIF(!isEmpty, ex)
    .ToList();
```

## 五、字串轉表達式（5.1.4.107+）

適合低程式碼、動態欄位、前端傳入條件等場景

### 啟動時設定

```csharp
// Program.cs 或啟動設定，只需執行一次
StaticConfig.DynamicExpressionParserType = typeof(DynamicExpressionParser);
StaticConfig.DynamicExpressionParsingConfig = new ParsingConfig
{
    CustomTypeProvider = new SqlSugarTypeProvider()
};

// 擴充 SqlFunc 識別
public class SqlSugarTypeProvider : DefaultDynamicLinqCustomTypeProvider
{
    public override HashSet<Type> GetCustomTypes()
    {
        var types = base.GetCustomTypes();
        types.Add(typeof(SqlFunc));
        return types;
    }
}
```

### 使用方式

```csharp
// 硬編碼條件
var list = db.Queryable<Order>().Where("it", $"it.Id > 1").ToList();

// 帶參數化變數（防注入）
FormattableString str = FormattableStringFactory.Create("it.Id > {0}", 1);
var list = db.Queryable<Order>().Where("it", str).ToList();

// 動態欄位 + 參數化
string field = "it.Amount";
FormattableString str2 = FormattableStringFactory.Create(field + " > {0}", 1000);

// 導航屬性動態條件
var list = db.Queryable<Order>()
    .Where("it", $"SqlFunc.Exists(it.Customer.Id)")
    .ToList();

// 動態類 + 動態條件（低程式碼場景）
var list = db.QueryableByObject(typeof(Order))
    .Where("it", $"it.Status == {1}")
    .ToList();
```

**✅ 實際案例：低程式碼平台前端傳入動態條件**

```csharp
// 前端傳入：[{ field: "Amount", op: ">", value: 1000 }, { field: "Status", op: "==", value: 1 }]
public List<dynamic> DynamicQuery(string tableName, List<FilterCondition> filters)
{
    var conditions = filters.Select(f =>
        FormattableStringFactory.Create($"it.{f.Field} {f.Op} {{0}}", f.Value)
    ).ToList();

    var query = db.QueryableByObject(Type.GetType(tableName));
    foreach (var cond in conditions)
        query = query.Where("it", cond);

    return query.ToList();
}
```

## 六、表達式轉 SQL（當只能用 ADO 時）

```csharp
// 把 Lambda 表達式解析成 SQL 字串
var expContext = new SqlServerExpressionContext();

Expression<Func<Order, bool>> exp = it => it.Id == 1 && it.Name.Contains("test");
expContext.Resolve(exp, ResolveExpressType.WhereSingle);

string whereSql = expContext.Result.GetString();
var parameters = expContext.Parameters;

// 用原生 ADO 執行
db.Queryable<Order>()
    .Where(whereSql)
    .AddParameters(parameters)
    .ToList();
```

### 各資料庫解析器

```csharp
ExpressionContext expContext = dbType switch
{
    DbType.MySql      => new MySqlExpressionContext(),
    DbType.SqlServer  => new SqlServerExpressionContext(),
    DbType.Sqlite     => new SqliteExpressionContext(),
    DbType.Oracle     => new OracleExpressionContext(),
    DbType.PostgreSQL => new PostgreSQLExpressionContext(),
    DbType.Dm         => new DmExpressionContext(),
    DbType.Kdbndp     => new KdbndpExpressionContext(),
    _                 => throw new Exception("不支援的資料庫類型")
};
```

### 解析類型說明

| ResolveExpressType | 說明 | 範例輸出 |
|---|---|---|
| WhereSingle | 無別名 Where | [Id] = @Id0 |
| WhereMultiple | 有別名 Where（聯表用）| [it].[Id] = @Id0 |
| SelectSingle | Select 解析 | [Id] AS [Id] |
| FieldSingle | 單欄位名稱 | Id |
| FieldMultiple | 聚合函數欄位 | AVG([it].[Id]) |

## 七、常見錯誤與解法

### 條件超過 1000 個報錯

原因：動態表達式底層用遞迴解析，超過 1000 條件遞迴深度超出限制

**方案一：用 List.Any（高版本支援）**

```csharp
// OR 語意
.Where(it => list.Any(s => s.Id == it.Id && s.Name == it.Name2))

// AND 語意
.Where(it => list.All(s => s.Id == it.Id && s.Name == it.Name2))
```

**方案二：設定執行緒最大堆疊**

```csharp
List<Order> result = null;
var thread = new System.Threading.Thread(() =>
{
    var expable = Expressionable.Create<Order>();
    for (int i = 0; i < 2000; i++)
        expable.Or(it => it.Id == i);

    result = db.Queryable<Order>()
        .Where(expable.ToExpression())
        .ToList();
}, int.MaxValue);  // 設定最大堆疊

thread.Start();
Thread.Sleep(2000);  // 等待執行完成
```

## 注意事項

- ToExpression() 這一句不能省略，省略會報錯
- 分段寫法最後要用 var exp = expable.ToExpression() 接，不能直接傳 expable
- WhereIF 只支援 AND，需要 OR 才用 Expressionable
- 字串轉表達式需在啟動時設定 StaticConfig，只設定一次
- 字串條件中欄位用 it.欄位名，值用 {0} 參數化避免 SQL Injection
- 表達式轉 SQL 的別名：WhereSingle 無別名，WhereMultiple 有 [it]. 前綴
