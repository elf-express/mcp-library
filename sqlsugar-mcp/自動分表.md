## 速查表

| 需求 | 語法 |
|---|---|
| 實體標記 | [SplitTable(SplitType.Month)] + [SplitField] |
| 建表同步 | db.CodeFirst.SplitTables().InitTables<T>() |
| 區間查詢 | .SplitTable(時間區間) |
| 最近 N 張 | .SplitTable(tabs=>tabs.Take(N)) |
| 精準單表 | .SplitTable(取特定表) |
| 插入 | db.Insertable(...).SplitTable() |
| 刪除 / 更新 | .SplitTable() |
| 聚合並行 | 各分表並行加總 |
| 索引 | [SugarIndex] |

---

## 適用場景

- ERP 系統用 5 年不卡，10 年後資料暴增變慢
- 每天幾十到上百萬筆資料進庫（如操作日誌、感測器資料）
- 資料庫現有資料超過 1 億筆，索引莫名失效、效能大幅下降

## 一、實體定義

```csharp
[SplitTable(SplitType.Year)]   // 按年分表（支援：年/季/月/週/日）
[SugarTable("OperationLog_{year}{month}{day}")]  // 3 個變數必須都有（為了後期改分表類型相容）
public class OperationLog
{
    [SugarColumn(IsPrimaryKey = true)]
    public long Id { get; set; }          // 主鍵不能用自增，用 long（雪花ID）或 Guid

    public string Content { get; set; }

    [SugarColumn(IsNullable = true)]
    public DateTime UpdateTime { get; set; }

    [SplitField]                          // 分表欄位：插入時依此欄位決定存哪張表
    public DateTime CreateTime { get; set; }
}

// 按年分表格式：OperationLog_20220101
// 按月分表格式：OperationLog_20220501
// 按日分表格式：OperationLog_20220511
```

## 二、建表與同步結構

```csharp
// 程式啟動時執行一次（禁止寫到業務邏輯中多次執行）
// 插入會自動建表，這行主要用於：實體改動後同步多張分表結構
db.CodeFirst
    .SplitTables()           // 標記分表模式
    .InitTables<OperationLog>();  // 若無分表則初始化一張
```

## 三、查詢

### 3.1 時間區間查詢（最常用）

```csharp
// 自動依時間找對應分表，並合併查詢
var beginDate = new DateTime(2024, 1, 1);
var endDate   = new DateTime(2024, 12, 31);

// 簡單查詢
var list = db.Queryable<OperationLog>()
    .SplitTable(beginDate, endDate)
    .ToPageList(1, 20);

// 搭配 Where（條件寫在 SplitTable 前面！）
var list2 = db.Queryable<OperationLog>()
    .Where(it => it.Content.Contains("登入"))  // ✅ Where 寫在前
    .SplitTable(beginDate, endDate)
    .OrderByDescending(it => it.CreateTime)    // ✅ OrderBy 寫在 SplitTable 後
    .ToPageList(1, 20);
```

### 3.2 選取最近 N 張分表

```csharp
// 只查最近 3 年的分表（按年分表時 Take(3) = 最近 3 年）
var list = db.Queryable<OperationLog>()
    .Where(it => it.Id == someId)
    .SplitTable(tabs => tabs.Take(3))
    .ToList();
```

### 3.3 精準定位單張表（效能最高）

```csharp
// 根據分表欄位的值取得精確表名
var tableName = db.SplitHelper<OperationLog>()
    .GetTableName(DateTime.Now);  // 取得今天對應的分表名

// 推薦：表不存在不報錯
var list = db.Queryable<OperationLog>()
    .SplitTable(tabs => tabs.InTableNames(tableName))
    .ToList();
```

### 3.4 表達式過濾表名

```csharp
// 只查表名包含 "2024" 的分表
var list = db.Queryable<OperationLog>()
    .Where(it => it.Content.Contains("error"))
    .SplitTable(tabs => tabs.Where(y => y.TableName.Contains("2024")))
    .ToList();
```

### 3.5 全部分表查詢

```csharp
// 適合用主鍵或有索引欄位查詢（哪怕 100 張分表也很快）
var log = db.Queryable<OperationLog>()
    .Where(it => it.Id == targetId)  // 有索引的條件
    .SplitTable()                    // 無條件 = 全部分表
    .First();
```

### 3.6 分表 JOIN 正常表

```csharp
var list = db.Queryable<OperationLog>()
    .SplitTable(tabs => tabs.Take(3))
    .LeftJoin<SystemUser>((log, u) => log.UserId == u.Id)
    .Select((log, u) => new { log.Content, u.UserName })
    .ToPageList(1, 20);
```

## 四、插入

```csharp
// 單筆插入（雪花ID，表不存在自動建表）
var log = new OperationLog
{
    CreateTime = DateTime.Now,
    Content    = "使用者登入"
};
db.Insertable(log).SplitTable().ExecuteReturnSnowflakeId();

// 批量插入（自動依 CreateTime 分配到對應表）
var logs = new List<OperationLog>
{
    new OperationLog { CreateTime = new DateTime(2022, 1, 1), Content = "A" },
    new OperationLog { CreateTime = new DateTime(2023, 6, 1), Content = "B" },
    new OperationLog { CreateTime = new DateTime(2024, 12, 1), Content = "C" }
};
db.Insertable(logs).SplitTable().ExecuteReturnSnowflakeIdList();
// → 自動識別 3 筆資料，分別插入 3 個不同分表

// 大量資料 BulkCopy（表不存在自動建表）
db.Fastest<OperationLog>().SplitTable().BulkCopy(logs);

// 多 WorkId 設定（多程式/多端口同時插入時避免雪花ID衝突）
// 程式啟動時設定：SnowFlakeSingle.WorkId = int.Parse(config["WorkId"]);
```

## 五、刪除

```csharp
// 推薦：直接根據實體集合刪除（5.0.7.7+）
db.Deleteable(deleteList).SplitTable().ExecuteCommand();

// 精準刪除（效能最高）
var tableName = db.SplitHelper<OperationLog>().GetTableName(targetDate);
db.Deleteable<OperationLog>().AS(tableName).Where(it => it.Id == targetId).ExecuteCommand();

// 範圍刪除（遍歷近 3 張分表）
var tables = db.SplitHelper<OperationLog>().GetTables().Take(3);
foreach (var table in tables)
{
    db.Deleteable<OperationLog>()
        .AS(table.TableName)
        .Where(it => it.CreateTime < DateTime.Now.AddYears(-5))
        .ExecuteCommand();
}
```

## 六、更新

```csharp
// 推薦：直接根據實體集合更新（5.0.7.7+）
db.Updateable(updateList).SplitTable().ExecuteCommand();

// BulkCopy 分表更新
db.Fastest<OperationLog>().SplitTable().BulkUpdate(updateList);

// 精準更新
var tableName = db.SplitHelper<OperationLog>().GetTableName(targetDate);
db.Updateable(entity).AS(tableName).ExecuteCommand();

// 表達式精準更新
db.Updateable<OperationLog>()
    .AS(tableName)
    .SetColumns(it => new OperationLog { Status = 1 })
    .Where(it => it.Id == targetId)
    .ExecuteCommand();
```

## 八、效能優化

### 8.1 基本原則

```csharp
// ✅ Where 寫在 SplitTable 前面（先過濾再合併）
db.Queryable<OperationLog>()
    .Where(it => it.Level == "ERROR")   // 先過濾
    .SplitTable(beginDate, endDate)     // 再分表合併
    .ToList();

// 分表欄位必須建索引
[SugarIndex("index_{split_table}_createtime", nameof(OperationLog.CreateTime), OrderByType.Desc)]
```

### 8.2 聚合統計（並行各分表加總）

```csharp
var tasks  = new List<Task<decimal>>();
var tables = db.SplitHelper<OperationLog>().GetTables();

foreach (var table in tables)
{
    var newDb = db.CopyNew();  // 保證執行緒安全
    tasks.Add(newDb.Queryable<OperationLog>()
        .AS(table.TableName)
        .SumAsync(it => it.Amount));
}
await Task.WhenAll(tasks);

var total = tasks.Sum(t => t.Result);
// Sum  = 各分表 Sum 加總
// Max  = 各分表 Max 中的最大值
// Min  = 各分表 Min 中的最小值
// Count = 各分表 Count 加總
// Avg  = (各分表 Sum 加總) / (各分表 Count 加總)
```

### 8.3 建立索引

```csharp
// 自動在各分表建立索引（分表名佔位符 {split_table}，需 5.1.4.140+）
[SugarIndex("index_{split_table}_createtime", nameof(OperationLog.CreateTime), OrderByType.Desc)]
public class OperationLog { ... }
```

### 8.4 分表名稱快取（5.1.4.170+，大量分表時用）

```csharp
// 程式啟動時設定（避免每次查詢都掃描資料庫表清單）
StaticConfig.SplitTableGetTablesFunc = () =>
{
    if (cache.TryGet("split_tables", out var cached)) return cached;
    var list = db.DbMaintenance.GetTableInfoList(false);
    var result = list.Select(it => new SplitTableInfo { TableName = it.Name }).ToList();
    cache.Set("split_tables", result, TimeSpan.FromMinutes(10));
    return result;
};

// 新分表建立時清除快取
StaticConfig.SplitTableCreateTableFunc = (type, insertObjs) =>
{
    cache.Remove("split_tables");
};
```

## 九、老表資料遷移到分表

```csharp
// 步驟1：將老表改名（避免與分表名衝突）
// 在 SQL Server Management Studio 或 T-SQL 中執行：
// EXEC sp_rename 'OperationLog', 'OperationLog_Old'

// 步驟2：分批讀取老表，寫入新分表
var pageList = new List<OperationLog>();
var pageSize = 200_000;  // 每批 20 萬筆

db.Queryable<object>()
    .AS("OperationLog_Old")
    .Select<OperationLog>()
    .ForEach(it =>
    {
        pageList.Add(it);
        if (pageList.Count == pageSize)
        {
            db.Fastest<OperationLog>().SplitTable().BulkCopy(pageList);
            pageList = new List<OperationLog>();
        }
    }, pageSize);

// 插入最後不足一批的資料
db.Fastest<OperationLog>().SplitTable().BulkCopy(pageList);
```

## 注意事項

- 主鍵**不能用自增**（int Identity），必須用 `long`（雪花ID）或 `Guid`
- `[SugarTable]` 的格式中 `{year}{month}{day}` 三個變數必須都有
- `Where` 條件必須寫在 `SplitTable` 前面，`OrderBy` 寫在後面
- `.SplitTable()` 不能漏掉，否則只操作原始表名（不分表）
- MySql 5.1.3.25+ 新建分表 bool 類型從 bit 改為 tinyint(1)，混用版本需統一
- `InitTables` 禁止寫到業務邏輯中，只在程式啟動時執行一次
- 多程式/多端口同時插入時需設定唯一 `SnowFlakeSingle.WorkId`
