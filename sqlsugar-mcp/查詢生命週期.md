## 速查表

| 需求 | 語法 |
|---|---|
| 何時觸發查詢 | ToList / First / Count 等才執行;Queryable 為延遲 |
| 條件累加 | Queryable 是引用類型,Where 會累加 |
| 多處共用查詢 | .Clone() 避免條件互相污染 |
| 取得 SQL 不執行 | .ToSql() |

---

## 三個核心概念

| 概念 | 說明 |
|---|---|
| 觸發執行 | Queryable 轉成非 Queryable 時才真正執行 SQL |
| 引用類型 | Queryable 是參考型別，Where 條件會累加 |
| Clone 機制 | 同一個 Queryable 要用在多個地方，必須先 Clone |

---

## 一、何時觸發資料庫查詢

Queryable 本身不執行 SQL，只有呼叫「終結方法」時才真正執行。

```csharp
// 還沒執行（只是建立查詢物件）
var query = db.Queryable<Order>().Where(o => o.Id > 0);

// 以下這些呼叫才會觸發執行
query.ToList()         // 執行，回傳 List<T>
query.FirstOrDefault() // 執行，回傳單筆
query.Count()          // 執行，回傳筆數
query.Any()            // 執行，回傳 bool
query.ToPageList(...)  // 執行，回傳分頁
query.ToDataTable()    // 執行，回傳 DataTable
query.ToSql()          // 不執行資料庫，只產生 SQL 字串
```

## 二、Queryable 是引用類型，Where 條件會累加

```csharp
var query = db.Queryable<Order>();

query.Where(o => o.Id > 0);     // 加入第一個條件
query.Where(o => o.Status == 1); // 加入第二個條件

var list = query.ToList();
// → WHERE Id > 0 AND Status = 1
// 兩個 Where 都生效！
```

這和 EF Core 不同，EF 的 Where 不加 var 接住的話不會累加。

**✅ 實際案例：動態組裝查詢條件**

```csharp
var query = db.Queryable<Order>()
    .Where(o => o.IsDeleted == false);

// 根據條件動態加入 Where
if (!string.IsNullOrEmpty(orderNo))
    query.Where(o => o.OrderNo.Contains(orderNo));

if (startDate.HasValue)
    query.Where(o => o.CreateTime >= startDate.Value);

if (status.HasValue)
    query.Where(o => o.Status == status.Value);

// 最終執行時，所有 Where 都會合併
var list = query.OrderBy(o => o.CreateTime, OrderByType.Desc).ToList();
```

## 三、Clone 機制（同一個查詢用在多處）

Queryable 執行一次終結方法後不能再重複使用（ToSql 後就不能 ToList）。
需要多次執行同一個查詢，必須先 Clone。

```csharp
// ❌ 錯誤寫法：同一個 query 直接用兩次
var query = db.Queryable<Order>().Where(o => o.Id > 0);
int count = query.Count();   // 第一次執行後 query 狀態改變
var list  = query.ToList();  // 可能拿到錯誤結果

// ✅ 正確寫法：每次使用前先 Clone
var query = db.Queryable<Order>().Where(o => o.Id > 0);
int count = query.Clone().Count();
var list  = query.Clone().ToList();
```

**✅ 實際案例：同時取分頁資料和總筆數（效能版）**

```csharp
public PageResult<OrderDto> GetOrders(PageRequest req)
{
    // 建立基底查詢（不執行）
    var baseQuery = db.Queryable<Order>()
        .Where(o => o.IsDeleted == false)
        .WhereIF(!string.IsNullOrEmpty(req.OrderNo),
            o => o.OrderNo.Contains(req.OrderNo))
        .WhereIF(req.Status.HasValue,
            o => o.Status == req.Status)
        .Select(o => new OrderDto
        {
            OrderNo    = o.OrderNo,
            Amount     = o.Amount,
            CreateTime = o.CreateTime
        });

    // 兩個地方使用，各自 Clone 獨立執行
    int total = baseQuery.Clone().Count();
    var list  = baseQuery.Clone()
        .OrderBy(o => o.CreateTime, OrderByType.Desc)
        .ToPageList(req.PageIndex, req.PageSize);

    return new PageResult<OrderDto> { Data = list, Total = total };
}
```

## 四、ToSql 取得 SQL 字串（不執行資料庫）

```csharp
var query = db.Queryable<Order>()
    .Where(o => o.Id > 0)
    .OrderBy(o => o.CreateTime, OrderByType.Desc);

// 取得 SQL 字串（不執行）
var sql = query.ToSql();
Console.WriteLine(sql.Key);        // SQL 字串
Console.WriteLine(sql.Value);      // 參數集合

// Clone 後才能繼續 ToList
var list = query.Clone().ToList();
```

---

## 注意事項

- Queryable 是引用類型，直接傳遞變數時要注意 Where 條件累加的副作用
- 同一個 query 要用在兩個地方（如 Count + ToList），一定要 Clone，這是和 EF 的本質差異
- ToSql 只產生 SQL 字串，不執行資料庫，但呼叫後 query 狀態已改變，後續需要 Clone
- 終結方法（ToList / Count / Any 等）呼叫後 query 不可再直接使用
