## 速查表

| 需求 | 語法 |
|---|---|
| 大量插入 | db.Fastest<T>().BulkCopy(list) |
| 大量更新 | db.Fastest<T>().BulkUpdate(list) |
| 插入或更新 | db.Fastest<T>().BulkMerge(list)(5.1.4.109+) |
| 大量查詢 | db.Fastest<T>().BulkQuery |
| 大量刪除 | BulkDelete |
| 分表 | .SplitTable() |
| AOP | 啟用 DataExecuting |
| 分批匯出降記憶體 | ForEach / PageEach |

---

> 需要 SqlSugar 5.0.45+

## 效能比較

| 方案 | 效能排名 | 備註 |
|---|---|---|
| db.Fastest BulkCopy | 🥇 最快 | 比 EFCore.BulkExtension 快 30%，列數越多優勢越大 |
| db.Fastest BulkUpdate | 🥇 最快 | 是 EFCore.BulkExtension 的 2~3 倍 |
| db.Insertable(list) | 🥈 | 一般批量插入，適合萬筆以下 |
| EFCore.BulkExtension | 🥉 | 收費框架效能更高 |
| Dapper | 第四 | |

**原理**：ORM 效能瓶頸在實體轉換，SqlSugar 將轉換效能做到極致，並採用各資料庫最佳 API（SqlServer 用 SqlBulkCopy，MySQL 用 LOAD DATA INFILE 等）。

---

## 一、BulkCopy（大量插入）

```csharp
// 基本插入
db.Fastest<Order>().BulkCopy(list);

// 分頁插入（降低記憶體佔用，適合超大資料集）
db.Fastest<Order>().PageSize(100_000).BulkCopy(list);

// DataTable 插入
db.Fastest<DataTable>().AS("Order").BulkCopy(dataTable);

// 非同步（高並發場景，await 不能少）
await db.Fastest<Order>().BulkCopyAsync(list);

// 搭配分表
db.Fastest<OrderSpliteTest>().SplitTable().BulkCopy(list);

// 雪花ID 手動賦值（BulkCopy 需手動設定 ID）
foreach (var item in list)
    item.Id = SnowFlakeSingle.Instance.NextId();
db.Fastest<Order>().BulkCopy(list);
```

---

## 二、BulkUpdate（大量更新）

```csharp
// 基本更新（依主鍵更新所有欄位）
db.Fastest<Order>().BulkUpdate(list);

// 分頁更新
db.Fastest<Order>().PageSize(100_000).BulkUpdate(list);

// 無主鍵更新（指定更新條件欄位）
db.Fastest<Order>().BulkUpdate(list, new string[] { "OrderNo" });

// 只更新指定欄位（條件欄位, 更新欄位）
db.Fastest<Order>().BulkUpdate(list,
    new string[] { "Id" },           // 更新條件（等同 WHERE Id = @Id）
    new string[] { "Name", "Status", "UpdateTime" });  // 只更新這幾列

// DataTable 更新
db.Fastest<DataTable>().AS("Order")
    .BulkUpdate(dataTable, new string[] { "Id" });  // Id 是主鍵條件
db.Fastest<DataTable>().AS("Order")
    .BulkUpdate(dataTable, new string[] { "Id" }, new string[] { "Name", "Amount" });

// 非同步
await db.Fastest<Order>().BulkUpdateAsync(list);

// 搭配分表
db.Fastest<OrderSpliteTest>().SplitTable().BulkUpdate(list);
```

## 三、BulkMerge（大量插入或更新，5.1.4.109+）

```csharp
// 自動判斷：存在則更新，不存在則插入
// SqlServer / Oracle → 使用 MERGE INTO + BulkCopy
// 其他資料庫 → 使用 Storageable.ExecuteSqlBulkCopy()

db.Fastest<Order>().BulkMerge(list);
db.Fastest<Order>().PageSize(100_000).BulkMerge(list);

// DataTable（5.1.4.147+）
db.Fastest<DataTable>()
    .AS("Order")
    .BulkMerge(dataTable,
        new string[] { "Id" },  // 主鍵欄位
        true);                   // true = 自增主鍵
```

## 四、BulkQuery（大量查詢）

```csharp
// 普通查詢效能已經超過 Dapper，直接用即可
var list = db.Queryable<Order>().ToList();

// 分頁降低記憶體佔用（適合複雜 DTO 轉換和大量匯出）
var orders = new List<Order>();
db.Queryable<Order>()
    .ForEach(it =>
    {
        orders.Add(it);
        // ⚠️ 禁止在這裡操作資料庫（會在迴圈中反覆連線）
    }, 2000);  // 每批 2000 筆
```

---

## 五、BulkDelete（大量刪除）

```csharp
// 分頁刪除（避免一次刪除太多造成鎖定）
db.Deleteable<Order>(list).PageSize(1000).ExecuteCommand();
```

## 六、啟用 DataExecuting AOP

```csharp
// BulkCopy 預設不觸發 DataExecuting（考量效能）
// 需要觸發時加 EnableDataAop()
db.Fastest<Order>().EnableDataAop().BulkCopy(list);
db.Fastest<Order>().EnableDataAop().BulkUpdate(list);
```

## 七、資料庫支援情況

| 資料庫 | BulkCopy | BulkUpdate | BulkMerge | 自增主鍵 | 特殊注意 |
|---|---|---|---|---|---|
| SQL Server | ✅ | ✅ | ✅ | ✅ | 無 |
| MySQL | ✅ | ✅ | ✅ | ✅ | 連線字串加 `AllowLoadLocalInfile=true` |
| PostgreSQL | ✅ | ✅ | ✅ | ❌ | 不支援自增或特殊預設值 |
| Oracle | ✅（.NET Core）| ✅ | ✅ | ❌ | 不支援事務；無視唯一索引可能插入重複。建議用 GUID 或雪花ID |
| SQLite | ✅（5.0.5.3+）| ✅ | ✅ | ✅ | 無 |
| 達夢 / 金倉 | ✅ | ✅ | ✅ | — | 無 |

> ⚠️ 所有資料庫：不支援有**預設值**或**計算列**的欄位

## 八、實際應用場景

### 8.1 Excel 匯入大量資料

```csharp
public async Task ImportExcelAsync(Stream excelStream)
{
    // 讀取 Excel（略）
    var importList = ReadExcel<OrderImportDto>(excelStream);

    // DTO → 實體
    var orders = importList.Adapt<List<Order>>();

    // 手動設定雪花ID
    foreach (var o in orders)
        o.Id = SnowFlakeSingle.Instance.NextId();

    // 分頁大量插入（降低記憶體壓力）
    await db.Fastest<Order>()
        .PageSize(50_000)
        .BulkCopyAsync(orders);
}
```

### 8.2 銀行明細對帳批次更新

```csharp
public async Task UpdateReconcileStatusAsync(List<BankDetail> details)
{
    // 只更新對帳狀態和更新時間
    await db.Fastest<BankDetail>()
        .BulkUpdateAsync(details,
            new string[] { "Id" },                           // WHERE Id = @Id
            new string[] { "ReconcileStatus", "UpdateTime" }); // 只更新這兩列
}
```

### 8.3 資料同步（存在更新，不存在插入）

```csharp
public async Task SyncOrdersAsync(List<Order> orders)
{
    // BulkMerge：有就更新，沒有就插入
    await db.Fastest<Order>()
        .PageSize(100_000)
        .BulkMergeAsync(orders);
}
```

### 8.4 大量資料匯出（ForEach 分批降低記憶體）

```csharp
public async Task ExportToExcelAsync(string filePath)
{
    using var excel = new ExcelPackage();
    var ws = excel.Workbook.Worksheets.Add("訂單");
    int row = 2;

    db.Queryable<Order>()
        .Where(it => it.CreateTime >= DateTime.Today.AddMonths(-1))
        .ForEach(order =>
        {
            ws.Cells[row, 1].Value = order.OrderNo;
            ws.Cells[row, 2].Value = order.Amount;
            row++;
        }, 5000);  // 每批 5000 筆

    await excel.SaveAsAsync(new FileInfo(filePath));
}
```

## 效能選擇指引

| 資料量 | 推薦方式 |
|---|---|
| < 1,000 筆 | `db.Insertable(list).ExecuteCommand()` 普通批量插入 |
| 1,000 ~ 10 萬筆 | `db.Fastest<T>().BulkCopy(list)` |
| > 10 萬筆 | `db.Fastest<T>().PageSize(100000).BulkCopy(list)` 分頁插入 |
| 高並發場景 | `await db.Fastest<T>().BulkCopyAsync(list)` 非同步 |
| 存在更新不存在插入 | `db.Fastest<T>().BulkMerge(list)` |

---

## 注意事項

- 需要 SqlSugar **5.0.45+**
- BulkCopy 不支援**預設值欄位**和**計算列**
- 雪花ID 需手動賦值：`SnowFlakeSingle.Instance.NextId()`
- 多台伺服器同時插入同一個庫需設定唯一 WorkId：`SnowFlakeSingle.WorkId = 機器唯一編號`
- 返回前端的雪花ID 要轉成 **string**，否則前端 JS 精度不足
- MySQL 連線字串需加 `AllowLoadLocalInfile=true`
- 高並發時用非同步（`BulkCopyAsync`），**await 不能少**
- 資料量小於 1000 筆時 BulkCopy 效能反而不如普通插入
