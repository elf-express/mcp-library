# 修改選擇項配置方式 ​

在設計器中，`select`、`radio`、`checkbox`、`cascader`、`treeSelect`、`transfer` 等“依賴 options 的元件”，右側屬性面板通常會提供一組“選項配置方式”（例如：靜態結構、遠端請求、全域資料來源）。這些配置方式並不是散落在每個 DragRule 裡，而是由通用工具方法統一生成：

- `makeOptionsRule(...)`：普通列表 options（對應 `TableOptions`）
- `makeTreeOptionsRule(...)`：樹形 options（對應 `TreeOptions`）

本文講清楚如何二開上述方法，達到兩類目標：

- **新增一種選項配置方式**（示例：增加“指令碼/結構體編輯 Struct”）
- **刪除一種預設內建方式**（示例：移除“全域資料來源”）

## 你要改哪些地方？ ​

影響“選項配置方式”的核心檔案是：

- `/src/utils/index.js`

- `makeOptionsRule(t, to, label, value)`
- `makeTreeOptionsRule(t, to, label, value)`

這些方法內部引用的關鍵配置元件（便於理解與排查）：

- **遠端請求配置**：`/src/designer/fetch/FetchConfig.vue`（元件名：`FetchConfig`）
- **樹形靜態配置**：`/src/designer/TreeOptions.vue`（元件名：`TreeOptions`）
- **全域資料來源選擇**：`/src/designer/GlobalFetchSelect.vue`（元件名：`GlobalFetchSelect`）
- **全域資料來源管理彈窗**：`/src/designer/fetch/GlobalFetchConfig.vue`

## 機制說明：`_optionType` + `control` ​

`makeOptionsRule` / `makeTreeOptionsRule` 的核心是一個 `radio` 規則：

- **欄位名固定為**：`field: '_optionType'`
- **預設選中值**：`value: 2`（預設“靜態結構”）
- **選項列表**：`options: [...]`
- **聯動渲染**：`control: [...]`

其中 `control` 的含義是：

- 當 `_optionType === 某个 value` 時，把對應的 `rule: [...]` 動態插入到面板中（也就是切換不同的配置方式 UI）。

預設內建的 3 種方式（當前實現）：

- **1：全域資料來源（global）**

- UI：`GlobalFetchSelect`
- 儲存：`field: 'formCreateEffect>fetch'`（物件，內部帶 `key/to/_uni` 等欄位）
- **3：遠端請求（fetch）**

- UI：`FetchConfig`
- 儲存：`field: 'formCreateEffect>fetch'`（物件，內部會寫入 `to`）
- **2：靜態結構（struct）**

- 普通 options：`TableOptions`（`makeOptionsRule`）
- 樹形 options：`TreeOptions`（`makeTreeOptionsRule`）
- 儲存：`field: 'formCreate' + upper(to).replace('.', '>')`（寫入對應的 `rule.props.xxx`）

### `to` 引數是什麼意思？ ​

`to` 用於描述“選項資料應該寫回到哪一個欄位”，例如：

- 元件透過 `rule.props.data` 接收 options，則 `to` 應傳：`'props.data'`
- 元件透過 `rule.options` 接收 options（少見），則 `to` 可傳：`'options'`

對於靜態方式（TableOptions/TreeOptions），`to` 會被轉換成面板內部的 field（`formCreateProps>data` 這類路徑），最終落到對應的 rule 上。

對於遠端/全域方式（FetchConfig/GlobalFetchSelect），`to` 會寫入 `formCreateEffect>fetch.to`，用於告知執行時“請求結果要寫到哪個目標欄位”。

## 示例一：新增一種配置方式 ​

目標：在“靜態結構（TableOptions/TreeOptions）”之外，再提供一個更靈活的方式，讓高階使用者直接用指令碼/結構體編輯 options（支援複製貼上大段資料、支援更復雜的值型別）。

這裡複用內建 `Struct` 元件：

- 元件檔案：`/src/designer/Struct.vue`
- 元件名：`Struct`

### 1. 修改 `makeOptionsRule`（普通 options） ​

在 `/src/utils/index.js` 的 `makeOptionsRule` 中：

- 在 `options` 陣列裡新增一項（value 例：`4`）
- 在 `control` 陣列裡新增對應的 `rule`

示例改法（僅展示關鍵結構）：

js

```
// 新增一种选项配置方式：Struct（脚本/结构体编辑）
options.push({ label: t('fetch.optionsType.structEditor'), value: 4 });

control.push({
  value: 4,
  rule: [
    {
      type: 'Struct',
      field: 'formCreate' + upper(to).replace('.', '>'),
      title: t('props.options'),
      _fc_important_prop: true,
      wrap: { show: false },
    },
  ],
});
```

### 2. 修改 `makeTreeOptionsRule`（樹形 options） ​

同理在 `makeTreeOptionsRule` 中追加 value=4 的分支（樹形 options 的目標欄位一致）：

js

```
options.push({ label: t('fetch.optionsType.structEditor'), value: 4 });

control.push({
  value: 4,
  rule: [
    {
      type: 'Struct',
      field: 'formCreate' + upper(to).replace('.', '>'),
      title: t('props.options'),
      _fc_important_prop: true,
      wrap: { show: false },
    },
  ],
});
```

### 3. 補齊多語言（必須） ​

由於新增了 `fetch.optionsType.structEditor`，需要在語言包中補齊該 key（至少 `zh-cn`，並建議同步 `en` 等其他語言）。

示例（`/src/locale/zh-cn.js`）：

js

```
fetch: {
  // ...
  optionsType: {
    fetch: '远程数据',
    global: '全局数据源',
    struct: '静态数据',
    structEditor: '脚本编辑',
  },
},
```

### 4. （可選）把新增方式設為預設 ​

如果你希望新方式成為預設選中的配置方式，把 `value: 2` 改成 `value: 4` 即可：

js

```
return {
  type: 'radio',
  field: '_optionType',
  value: 4, // 默认选中“脚本编辑”
  // ...
};
```

## 示例二：刪除一種預設內建方式 ​

目標：某些專案不允許引用全域資料來源（避免跨表單複用導致治理困難），希望右側面板只保留“靜態結構”和“遠端請求”兩類方式。

要點：**必須同時刪除**兩處，否則會出現“能顯示選項，但切換後沒有對應配置 UI”的問題：

- `options`：刪掉 `value: 1` 的那一項
- `control`：刪掉 `value: 1` 的那一段

### 1. 在 `makeOptionsRule` 中移除 global 分支 ​

從 `options` 中移除：

js

```
{ label: t('fetch.optionsType.global'), value: 1 }
```

並從 `control` 中移除：

js

```
{
  value: 1,
  rule: [
    {
      type: 'GlobalFetchSelect',
      field: 'formCreateEffect>fetch',
      props: { to },
    },
  ],
}
```

### 2. 在 `makeTreeOptionsRule` 中移除 global 分支 ​

與上面完全一致：同步移除 `options` 與 `control` 中的 `value: 1` 分支。

## 常見問題排查 ​

### 1. 選項切換後沒有任何配置 UI ​

- **最可能原因**：只改了 `options` 沒改 `control`（或 value 對不上）
- **定位路徑**：`/src/utils/index.js` 的 `makeOptionsRule` / `makeTreeOptionsRule`

### 2. 靜態 options 改了，但執行態元件拿不到資料 ​

- **最可能原因**：`to` 傳錯（例如應該傳 `'props.data'`，但傳了 `'data'` 或 `'options'`）
- **定位路徑**：元件 DragRule 的 `props()` 裡呼叫 `makeOptionsRule(t, to, ...)` 的那一行

### 3. 新增方式文案顯示為空 ​

- **最可能原因**：多語言 key 缺失（`fetch.optionsType.xxx` 未補齊）
- **定位路徑**：`/src/locale/zh-cn.js`（以及其他語言包）
