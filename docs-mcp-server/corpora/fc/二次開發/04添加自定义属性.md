# 新增自定義屬性 ​

渲染器屬性允許您為表單渲染器新增自定義的屬性處理器，透過屬性擴展，您可以處理元件配置、注入自定義邏輯、管理動態樣式等。本文將以 **級聯樣式（CSS）** 功能作為參考樣例，詳細說明如何擴展一個新的渲染器屬性功能。

在[渲染器文件](https://form-create.com/v3/element-ui/)中可以檢視內建表單元件及其所有可配置引數。

在原始碼中可以參考現有的屬性擴展實現（如 `src/utils/css.js`、`src/utils/behavior.js`、`src/utils/easySlots.js`）。

注意

渲染器屬性擴展需要在**執行態渲染器**和**設計態渲染器**中同時註冊，確保設計器和執行環境都能正常工作。

## 擴展渲染器屬性要改哪些地方？ ​

在設計器中擴展一個新的渲染器屬性功能，通常涉及 3 層：

- **屬性處理器實現（Attribute Handler）**：定義屬性的載入、監聽、清理邏輯（如 `src/utils/css.js`）
- **渲染器註冊（formCreate.register）**：在 PC/移動端渲染器中註冊屬性處理器（如 `src/form/elm.js`、`src/form/mobile.js`）
- **設計器配置元件（可選）**：如果需要視覺化配置介面，需要建立配置元件（如 `src/designer/style/CssInput.vue`）

## 參考樣例：級聯樣式（CSS）的接入鏈路 ​

級聯樣式功能是一個典型的"屬性擴展 + 視覺化配置"的例子。相關檔案一般可以在以下位置找到：

- **屬性處理器**：`src/utils/css.js` - 處理 CSS 樣式注入和清理
- **設計器配置元件**：`src/designer/style/CssInput.vue` - 提供視覺化 CSS 編輯介面
- **PC 渲染器註冊**：`src/form/elm.js` - 註冊 `css` 屬性處理器
- **移動端渲染器註冊**：`src/form/mobile.js` - 註冊 `css` 屬性處理器
- **表單工具註冊**：`src/utils/form.js` - 同時註冊到執行態和設計態渲染器

### 1. 屬性處理器結構 ​

一個標準的屬性處理器需要實現以下介面：

js

```
export default function (fc) {
    const attr = {
        name: 'yourAttrName',  // 属性名称，对应 rule.$yourAttrName
        load(attr, rule, api) {
            // 属性加载时的处理逻辑
            // attr: 属性对象，可通过 attr.getValue() 获取值
            // rule: 当前规则对象
            // api: 表单 API 对象
        },
        watch(attr, rule, api) {
            // 属性值变化时的处理逻辑（可选，默认使用 load）
        },
        deleted(attr) {
            // 属性被删除时的清理逻辑（可选）
        },
    };
    return attr;
}
```

### 2. CSS 屬性處理器要點 ​

以 CSS 樣式功能為參照，推薦你的屬性處理器至少具備：

- **動態樣式注入**：透過建立 `<style>` 標籤動態注入樣式
- **作用域隔離**：為每個元件生成唯一的類名，避免樣式衝突
- **生命週期管理**：在元件刪除時自動清理相關樣式
- **值變化監聽**：支援樣式值變化時實時更新

### 3. 設計器配置元件 ​

如果需要在設計器右側面板提供視覺化配置介面，可以建立配置元件：

- 元件位置：`src/designer/style/YourAttrInput.vue`
- 透過 `inject: ['designer']` 獲取設計器上下文
- 透過 `designer.setupState.activeRule.$yourAttr` 訪問和修改屬性值

## 擴展示例：新增一個 `dataAttributes` 資料屬性擴展功能 ​

下面給出一個可直接照抄的示例：新增一個 `dataAttributes` 屬性擴展，允許為元件動態新增 HTML data 屬性，用於整合第三方庫或實現自定義互動。

### 1. 建立屬性處理器 ​

新建 `src/utils/dataAttributes.js`：

js

```
/**
 * 数据属性扩展处理器
 * 允许为组件动态添加 HTML data-* 属性
 */
export default function (fc) {
    const dataAttributes = {
        name: 'dataAttributes',

        /**
         * 属性加载时的处理逻辑
         * @param {Object} attr - 属性对象
         * @param {Object} rule - 当前规则对象
         * @param {Object} api - 表单 API 对象
         */
        load(attr, rule, api) {
            const value = attr.getValue();
            const prop = attr.getProp();

            // 如果之前有设置过 data 属性，先清理
            if (prop._dataAttributes) {
                dataAttributes.deleted(attr);
            }

            // 如果有值，将 data 属性合并到组件的 props 中
            if (value && typeof value === 'object') {
                const dataProps = {};
                Object.keys(value).forEach(key => {
                    // 确保 key 符合 data-* 属性命名规范
                    const dataKey = key.startsWith('data-') ? key : `data-${key}`;
                    dataProps[dataKey] = value[key];
                });

                // 将 data 属性合并到组件的 props 中
                if (!prop.props) {
                    prop.props = {};
                }
                Object.assign(prop.props, dataProps);

                // 保存原始值，用于后续清理
                prop._dataAttributes = value;

                // 同步规则变化
                api.sync(rule);
            }
        },

        /**
         * 属性值变化时的处理逻辑
         */
        watch(attr, rule, api) {
            // 重新加载以应用新值
            dataAttributes.load(attr, rule, api);
        },

        /**
         * 属性被删除时的清理逻辑
         */
        deleted(attr) {
            const prop = attr.getProp();
            if (prop._dataAttributes && prop.props) {
                // 移除之前添加的 data 属性
                Object.keys(prop._dataAttributes).forEach(key => {
                    const dataKey = key.startsWith('data-') ? key : `data-${key}`;
                    delete prop.props[dataKey];
                });
                delete prop._dataAttributes;
            }
        },
    };

    return dataAttributes;
}
```

### 5. 建立設計器配置元件（可選） ​

如果需要視覺化配置介面，新建 `src/designer/DataAttributesInput.vue`：

vue

```
<template>
    <div class="_fd-data-attributes-input">
        <el-badge type="info" is-dot :hidden="!configured">
            <div class="_fd-data-attributes-btn" @click="visible = true">
                <i class="fc-icon icon-setting"></i>
                {{ t('class.dataAttributes') }}
            </div>
        </el-badge>
        <el-dialog
            class="_fd-data-attributes-input-dialog _fd-config-dialog"
            v-model="visible"
            destroy-on-close
            :close-on-click-modal="false"
            append-to-body
            width="600px"
        >
            <template #header>
                {{ t('class.dataAttributes') }}
                <Warning :tooltip="t('warning.dataAttributes')"></Warning>
            </template>
            <el-form :model="formData" label-width="100px">
                <el-form-item
                    v-for="(item, index) in formData.items"
                    :key="index"
                    :label="`属性 ${index + 1}`"
                >
                    <div style="display: flex; gap: 8px; width: 100%;">
                        <el-input
                            v-model="item.key"
                            placeholder="属性名（如：track-id）"
                            style="flex: 1;"
                        />
                        <el-input
                            v-model="item.value"
                            placeholder="属性值"
                            style="flex: 1;"
                        />
                        <el-button
                            type="danger"
                            :icon="Delete"
                            @click="removeItem(index)"
                            circle
                        />
                    </div>
                </el-form-item>
                <el-form-item>
                    <el-button type="primary" :icon="Plus" @click="addItem">
                        添加属性
                    </el-button>
                </el-form-item>
            </el-form>
            <template #footer>
                <div>
                    <el-button @click="visible = false" size="default">
                        {{ t('props.cancel') }}
                    </el-button>
                    <el-button type="primary" @click="onOk" size="default">
                        {{ t('props.ok') }}
                    </el-button>
                </div>
            </template>
        </el-dialog>
    </div>
</template>

<script>
import { defineComponent } from 'vue';
import { Plus, Delete } from '@element-plus/icons-vue';
import Warning from '../Warning.vue';

export default defineComponent({
    name: 'DataAttributesInput',
    components: {
        Warning,
    },
    inject: ['designer'],
    computed: {
        t() {
            return this.designer.setupState.t;
        },
        activeRule() {
            return this.designer.setupState.activeRule;
        },
        configured() {
            const attrs = this.activeRule?.$dataAttributes;
            return attrs && Object.keys(attrs).length > 0;
        },
    },
    data() {
        return {
            visible: false,
            formData: {
                items: [],
            },
        };
    },
    watch: {
        visible(n) {
            if (n) {
                // 打开对话框时，将当前值转换为表单数据
                const attrs = this.activeRule?.$dataAttributes || {};
                this.formData.items = Object.keys(attrs).map(key => ({
                    key: key.replace(/^data-/, ''),  // 移除 data- 前缀用于编辑
                    value: attrs[key],
                }));
                // 如果没有数据，至少显示一个空行
                if (this.formData.items.length === 0) {
                    this.formData.items.push({ key: '', value: '' });
                }
            }
        },
    },
    methods: {
        addItem() {
            this.formData.items.push({ key: '', value: '' });
        },
        removeItem(index) {
            this.formData.items.splice(index, 1);
        },
        onOk() {
            // 将表单数据转换为对象格式
            const attrs = {};
            this.formData.items.forEach(item => {
                if (item.key && item.value !== undefined) {
                    // 确保 key 符合 data-* 命名规范
                    const key = item.key.startsWith('data-') 
                        ? item.key 
                        : `data-${item.key}`;
                    attrs[key] = item.value;
                }
            });

            // 更新规则属性
            this.activeRule.$dataAttributes = Object.keys(attrs).length > 0 ? attrs : undefined;
            this.visible = false;
        },
    },
});
</script>
```

### 6. 整合到設計器配置面板（可選） ​

如果建立了配置元件，需要將其新增到設計器的基礎配置中。**該專案的配置面板項需要加在 `/src/config/base/field.js`** 裡。

js

```
import { localeOptions } from '../../utils';

export default function field({ t }) {
    return [
        // ... 其他配置项 ...
        {
            type: 'FieldInput',
            field: 'field',
            value: '',
            title: t('form.field'),
            warning: t('warning.field'),
        },
        // ... 其他配置项 ...

        // 添加数据属性配置组件
        {
            type: 'DataAttributesInput',
            field: '$dataAttributes',
            title: '',
            wrap: { show: false },
            col: { show: false },
        },
    ];
}
```

**說明：**

- `type: 'DataAttributesInput'`：使用你建立的配置元件型別
- `field: '$dataAttributes'`：對應規則中的屬性欄位（以 `$` 開頭）
- `wrap: { show: false }` 和 `col: { show: false }`：隱藏表單項的包裝，讓元件獨立顯示
- `title: ''`：不顯示標題，因為元件內部已經包含了標題和按鈕

**注意：** 確保 `DataAttributesInput` 元件已經在設計器表單中完成元件註冊：

js

```
// 在 designer3.0 的入口文件或组件注册文件中
import DataAttributesInput from './designer/DataAttributesInput.vue';

// 注册到设计器表单
designerForm.component('DataAttributesInput', DataAttributesInput);
```

## 使用示例 ​

### 在規則中直接配置 ​

js

```
const rules = [
    {
        type: 'input',
        field: 'username',
        title: '用户名',
        // 配置数据属性
        $dataAttributes: {
            'data-track-id': 'user-input',
            'data-analytics': 'true',
            'data-custom': 'my-value',
        },
    },
];
```

### 在設計器中配置 ​

1. 在設計器中選中元件
2. 在右側配置面板的樣式區域找到"資料屬性"按鈕
3. 點選開啟配置對話方塊
4. 新增需要的 data 屬性鍵值對
5. 儲存後，屬性會自動應用到元件上

### 渲染結果 ​

配置後，元件會渲染為：

html

```
<input 
    data-track-id="user-input"
    data-analytics="true"
    data-custom="my-value"
    class="el-input__inner"
    ...
/>
```

## 常見問題排查 ​

### 1. 屬性配置了但沒有生效 ​

- **檢查註冊位置**：確認在 `src/form/elm.js`、`src/form/mobile.js` 和 `src/utils/form.js` 中都註冊了屬性處理器
- **檢查屬性名稱**：確認規則中使用的是 `$dataAttributes`（以 `$` 開頭）
- **檢查屬性值格式**：確認屬性值是物件格式，且鍵值對正確

### 2. 移動端和 PC 端表現不一致 ​

- **檢查雙端註冊**：確認在 PC 端和移動端渲染器中都註冊了相同的屬性處理器
- **檢查平臺差異**：某些屬性在不同平臺可能有不同的處理方式，需要做平臺判斷

### 3. 對已有規則的影響 ​

- 新增的屬性擴展功能只對**新配置的規則**生效
- 已儲存的規則不會自動應用新屬性，需要手動編輯規則或重新配置

### 版本相容 ​

- **Vue2 版本**：屬性處理器的實現方式相同，但需要注意 Vue2 和 Vue3 的 API 差異
- **Vue3 版本**：使用 Composition API 時，需要確保屬性處理器相容新的響應式系統
