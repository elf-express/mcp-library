# 全域事件 ​

全域事件用於定義元件事件，供元件進行配置和選擇，方便在不同元件中統一使用預設事件，簡化事件的管理和分發；如果需要從程式碼側主動觸發這些事件，也可以繼續檢視[操作全域資料](https://pro.form-create.com/doc/global-data-api)中的全域事件呼叫方式。

## 建立事件 ​

點選建立事件，可選擇“自定義”或者“事件流”。

- 自定義事件是為滿足個性化業務需求，脫離內建事件之外建立的專屬事件，支援定義、觸發、解綁，核心解決 “內建事件不夠用” 的問題；
- 事件流是事件觸發後的執行規則，管控回撥的執行順序、傳播範圍，支援透過 `return false` 中斷執行，核心解決 “事件執行混亂” 的問題；

實際接入時，公共確認彈窗、統一取數、批次回填、提交流程攔截等邏輯都很適合沉澱到全域事件裡，後續再由具體元件按需引用。

### 選擇條件 ​

選擇新增條件，即可設定條件判斷

![image](https://pro.form-create.com/doc/img/help/event-05.png)

條件判斷中分為邏輯條件和計算公式；當判斷規則只是做真假分支時，通常直接使用邏輯條件即可；若需要寫更長的表示式、複用函式或處理計算過程，可以繼續參考[擴展計算函式](https://pro.form-create.com/doc/formula)與[內建公式函式](https://pro.form-create.com/doc/formula_function)。

邏輯條件透過定義條件，元件的狀態和資料值會根據條件是否成立自動調整。

![image](https://pro.form-create.com/doc/img/help/event-06.png)

計算公式透過公式和表單資料，動態計算元件的狀態和資料值。

![image](https://pro.form-create.com/doc/img/help/event-07.png)

## 元件中如何引用全域事件 ​

全域事件建立完成後，可在元件事件配置中直接選擇引用。適合把“開啟模組”“統一校驗”“公共回寫邏輯”等能力沉澱為可複用資源；若事件內部還需要讀寫表單值、校驗欄位或發起請求，通常會繼續配合[表單 API](https://pro.form-create.com/doc/api)使用。

![image](https://pro.form-create.com/doc/img/help/event-19.png)

## inject 引數說明 ​

全域事件編輯區中的 `inject` 可用於獲取表單 API、當前引數和執行時上下文，適合在統一事件邏輯裡讀取表單資料、操作欄位狀態或呼叫外部能力；如果你想進一步確認 `api`、`rule`、`self`、`args` 等欄位的結構和含義，可以直接檢視[$inject](https://pro.form-create.com/doc/inject)。
