# 屬性配置 ​

元件屬性配置功能提供了對元件屬性管理，使用者可以設定元件的各種配置項，以控制元件的行為和外觀；當配置項需要從執行時資料動態驅動時，通常還會與[變數](https://pro.form-create.com/doc/variable)、[匯入外部資料](https://pro.form-create.com/doc/get-data)或[預定義全域資料](https://pro.form-create.com/doc/global-data)配合使用。

![image](https://pro.form-create.com/doc/img/help/overview-05.png)

## 繫結變數 ​

從變數讀取配置項資料，支援透過從變數（例如 Cookie、localStorage、外部資料來源或自定義變數）中讀取資料來設定配置項，實現資料的動態繫結和靈活管理；如果你想先了解變數來源、模板語法與執行時更新方式，可以繼續檢視[變數](https://pro.form-create.com/doc/variable)。

![image](https://pro.form-create.com/doc/img/help/overview-06.png)

## 屬效能力總覽 ​

屬性配置是元件最常用的配置區域，通常會覆蓋以下能力：

- 文字與提示：佔位文字、提示語、標籤說明。
- 輸入限制：最大長度、輸入型別、禁用與只讀。
- 資料來源：靜態資料、遠端資料、全域資料來源、變數繫結。
- 互動細節：清除按鈕、預設值、展示方式等。

其中“資料來源”這一類配置最常見的組合方式，是先透過[匯入外部資料](https://pro.form-create.com/doc/get-data)或[預定義全域資料](https://pro.form-create.com/doc/global-data)準備資料，再在具體屬性項裡選擇繫結來源。

## 繫結變數支援來源 ​

屬性配置中的變數讀取通常支援 Cookie、`localStorage`、外部資料來源、自定義變數以及部分表單上下文資料。對於需要跨元件複用或動態回填的配置項，優先建議使用變數繫結；如果後續還需要在事件裡繼續讀寫這些資料，也可以配合[表單 API](https://pro.form-create.com/doc/api)中的 `getData`、`setData` 等方法處理。
