# 元件列表 ​

元件列表是視覺化表單構建的核心，所有可用於搭建表單的基礎元件、業務元件均集中在此展示，是從零開始搭建新表單的起點，主要分為元件、欄位、大綱三塊操作區域；如果你後續準備擴展自己的元件分組、拖拽元件或模板，這裡也是最直接的落點。

- 元件區域提供了豐富的元件和模板管理功能，主要包含模板、基礎元件、子表單元件、容器元件、圖表元件、輔助元件、佈局元件等七個類別。選擇合適的元件型別非常重要，關係到系統業務的使用者體驗，對使用者操作、資料收集、資料分析都有很大的影響；如果你還想進一步檢視每個內建分組下具體有哪些預設元件、它們對應的元件標識和分組 ID，可以繼續閱讀[內建分組和元件](https://pro.form-create.com/doc/components)。
- 欄位列表區域提供了管理和操作欄位的功能；
- 表單大綱區域提供了一目瞭然的表單結構檢視。

### 選型建議 ​

- 需要錄入欄位值時，優先從基礎元件裡選擇。
- 需要做頁面分組、分頁籤、彈窗或佈局排版時，優先看佈局元件和容器類能力。
- 需要做說明、按鈕、媒體展示、統計資訊時，優先看輔助元件。
- 需要複用固定結構時，優先考慮模板，而不是每次手工拖拽多個元件。

## 拖拽建立表單 ​

元件列表中的元件支援直接拖入工作區，完成欄位建立和佈局搭建。對於大多數表單設計任務，常見的操作鏈路就是“選擇元件 -> 拖入畫布 -> 配置屬性 -> 預覽結果”。

## 模板與高階元件入口 ​

元件列表不僅是基礎欄位入口，也是模板、高階元件和佈局能力的統一入口。例如表格表單、分步表單、計算公式、資料表格等能力，都適合從元件列表中快速定位並插入。開發側如果要理解這些分類是如何組織的，可以參考[擴展元件分組](https://pro.form-create.com/doc/menu)中的分組機制；當前預設分組也和原始碼裡的選單定義保持一致。

## 自定義模板 / 元件 ​

當內建模板或元件無法完全滿足業務需求時，可以透過擴展方式沉澱自定義模板和自定義元件。這樣可以把常用業務結構複用到多個表單中，減少重複搭建成本。具體做法可以繼續檢視[擴展元件](https://pro.form-create.com/doc/component)與[擴展模板](https://pro.form-create.com/doc/template)；如果你還需要從程式碼側理解最終生成的規則資料結構，可以再結合[TS型別定義](https://pro.form-create.com/doc/ts)閱讀。
