# 高階配置 ​

高階配置是對錶單基礎配置的補充與擴展，核心為表單控制元件賦予動態行為能力，支援根據業務規則、使用者操作、表單資料狀態等動態調整控制元件的顯示、編輯、校驗、資料關聯邏輯，使表單從 “靜態填寫容器” 升級為適配複雜業務場景的 “智慧互動表單”，既滿足個性化業務需求，又提升表單填寫效率與資料錄入規範性。通常來說，只有當基礎屬性、事件和驗證已經不夠表達業務邏輯時，才會進入這一層配置。

![image](https://pro.form-create.com/doc/img/help/overview-07.png)

## 高階配置能力 ​

- 邏輯條件：負責判斷條件是否成立，是隱藏、必填、禁用、資料聯動和元件聯動的基礎，可繼續檢視[使用邏輯條件](https://pro.form-create.com/doc/help/component-advanced/condition)。
- 計算公式：負責根據欄位和函式動態計算結果，可用於算值，也可參與條件控制，可繼續檢視[使用計算公式](https://pro.form-create.com/doc/help/component-advanced/formula)。
- 資料聯動：負責把來源欄位或外部資料回填到目標欄位，可繼續檢視[資料聯動](https://pro.form-create.com/doc/help/component-advanced/data-linkage)。
- 元件聯動：負責控制目標欄位的顯示、載入、禁用或必填狀態。

## 配置建議 ​

當需求是“根據條件控制元件狀態”時，優先考慮邏輯條件或元件聯動；當需求是“根據條件自動賦值”時，優先使用資料聯動；當需求需要複雜運算或複用函式時，優先使用計算公式。若某些邏輯已經超出視覺化配置的表達範圍，也可以回到[事件配置](https://pro.form-create.com/doc/help/component/event)、[$inject](https://pro.form-create.com/doc/inject)和[表單 API](https://pro.form-create.com/doc/api)中用程式碼補足。
