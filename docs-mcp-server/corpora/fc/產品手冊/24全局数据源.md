# 全域資料來源 ​

全域資料來源用於新增一些常用的資料來源，支援靜態資料和遠端資料。可以在`checkbox`，`radio`，`select`等元件配置中使用，支援擴展。

## 建立資料來源 ​

1. 輸入自定義資料來源名稱
2. 操作按鈕，支援新增和刪除

![image](https://pro.form-create.com/doc/img/help/data-source-01.png)

## 設定遠端資料 ​

### 請求配置 ​

用於配置遠端資料的請求連結、請求方式等資訊；如果你想進一步瞭解請求結構、請求頭、變數替換或 `beforeFetch` 的處理方式，可以繼續檢視[遠端請求](https://pro.form-create.com/doc/fetch)。

![image](https://pro.form-create.com/doc/img/help/data-source-02.png)

### 前置處理 ​

在請求傳送前，可透過處理函式修改請求引數；若需終止請求，直接返回 false 或一個拒絕的 Promise。這類前置邏輯通常會和 Token、使用者資訊或業務上下文一起使用，相關的資料匯入方式可參考[匯入外部資料](https://pro.form-create.com/doc/get-data)。

![image](https://pro.form-create.com/doc/img/help/data-source-03.png)

### 資料處理 ​

自定義針對遠端介面資料的處理邏輯，返回處理後的資料。預設讀取`res.data`；當介面返回結構和選項元件要求不一致時，建議在這裡統一做轉換，避免在多個元件裡重複處理。

![image](https://pro.form-create.com/doc/img/help/data-source-04.png)

### 錯誤處理 ​

自定義介面請求失敗後的處理邏輯

![image](https://pro.form-create.com/doc/img/help/data-source-05.png)

## 設定靜態資料 ​

1. 資料內容區，複雜的資料結構時可以透過JSON的方式快速設定資料
2. 儲存按鈕，修改配置後需要點選按鈕儲存修改後的內容

![image](https://pro.form-create.com/doc/img/help/data-source-06.png)

## 使用資料來源 ​

1. 從全域資料來源中選擇資料
2. 重新整理按鈕，點選按鈕可以重新載入全域資料
3. 管理按鈕，快捷開啟全域資料來源管理介面

![image](https://pro.form-create.com/doc/img/help/data-source-07.png)

在元件側實際使用時，通常會與[選項資料](https://pro.form-create.com/doc/help/component-props/options)一起出現：先在這裡統一維護資料，再回到元件屬性中選擇具體的資料來源。

## 適用元件與使用入口 ​

全域資料來源通常用於 `checkbox`、`radio`、`select`、級聯選擇器等選項類元件。配置完成後，可在對應元件的選項資料配置區域直接選擇全域資料來源；如果你希望從開發側批次匯入或預置全域資料，也可以繼續檢視[預定義全域資料](https://pro.form-create.com/doc/global-data)。

## 資料集管理說明 ​

除了新增資料來源外，日常維護中還會涉及資料集的刪除、重新命名、擴展欄位和重新整理載入。對於多個表單共用的選項配置，建議優先使用全域資料來源統一維護，這樣既能減少重複配置，也更方便開發者統一調整介面和資料格式。
