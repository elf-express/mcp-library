# 屬性欄 ​

右側屬性欄是表單設計的核心配置區，分為元件配置和表單配置。元件配置是對元件的專屬設定，會根據選中元件動態切換內容；表單配置是對全域表單的整體批次配置。這裡的多數視覺化項最終都會落到規則物件上，因此它既是產品側最常用的配置入口，也是和開發側規則結構銜接最緊密的區域。

![属性栏](https://pro.form-create.com/doc/img/help/attributes-01.png)

## 元件配置項概覽 ​

選中元件後，屬性欄通常可配置以下內容：

- 基礎資訊：標題、欄位標識、提示文案、預設值、寬度等。
- 屬性配置：佔位文字、最大輸入長度、清除按鈕、輸入型別、禁用與只讀等。
- 事件與動作：元件事件、全域事件引用、動作配置。
- 驗證能力：必填、長度、正則、自定義驗證和觸發方式。
- 高階配置：邏輯條件、計算公式、資料聯動、隱藏條件、必填條件、禁用條件等。
- 樣式配置：全域 class、自定義樣式、佈局與間距、尺寸和視覺效果。

如果你想逐項深入檢視這些配置的具體用法，可以繼續閱讀[基礎配置](https://pro.form-create.com/doc/help/component/base)、[屬性配置](https://pro.form-create.com/doc/help/component-props/overview)、[事件配置](https://pro.form-create.com/doc/help/component/event)、[驗證配置](https://pro.form-create.com/doc/help/component/validate)、[高階配置](https://pro.form-create.com/doc/help/component-advanced/overview)和[樣式配置](https://pro.form-create.com/doc/help/component/style)。從開發角度看，這些內容也都可以在[TS型別定義](https://pro.form-create.com/doc/ts)裡的 `Rule`、`props`、`validate`、`computed` 等欄位中找到對應位置。

## 表單配置項概覽 ​

切換到表單配置後，屬性欄主要處理整個表單的全域設定：

- 表單整體配置：尺寸、標籤位置、標籤寬度、按鈕顯示與佈局方式。
- 表單樣式：標籤預設樣式、全域展示風格和表單外觀。
- 表單事件：提交、重置、建立、載入、修改、請求前置等生命週期事件。
- 錄入資料與預設值：配置初始資料、重置與清空策略，便於後續測試和複用。

其中表單事件和全域設定通常會繼續與[表單事件配置](https://pro.form-create.com/doc/help/form/event)、[全域概覽](https://pro.form-create.com/doc/help/global/overview)、[全域資料來源](https://pro.form-create.com/doc/help/global/data-source)、[全域變數](https://pro.form-create.com/doc/help/global/variable)等能力配合使用；若要在事件中繼續讀寫這些配置，可以再結合[表單 API](https://pro.form-create.com/doc/api)理解執行時行為。
