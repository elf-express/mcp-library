# 覆蓋內建元件 ​

FormCreate 允許您透過 `formCreate.component()` 方法掛載同名元件來覆蓋內建元件。這種方式可以讓您完全自定義內建元件的行為和外觀，滿足特定的業務需求。

獲取 formCreate 物件

在使用 `formCreate.component()` 之前，您需要先獲取 formCreate 物件。可以透過以下方式獲取：

- 從設計器包中匯入：`import { formCreate } from 'path/to/fcDesignerPro'`
- 從設計器例項獲取：`FcDesigner.formCreate`

## 基本用法 ​

### 覆蓋單個元件 ​

js

```
import formCreate from '@form-create/element-ui';
import CustomInput from './CustomInput.vue';

// 覆盖内置的 input 组件
formCreate.component('input', CustomInput);
```

### 覆蓋多個元件 ​

js

```
import formCreate from '@form-create/element-ui';
import CustomInput from './CustomInput.vue';
import CustomSelect from './CustomSelect.vue';

// 覆盖多个内置组件
formCreate.component('input', CustomInput);
formCreate.component('select', CustomSelect);
```

## 使用示例 ​

### 示例 1：覆蓋輸入框元件 ​

建立一個自定義的輸入框元件，新增自動格式化功能：

vue

```
<!-- CustomInput.vue -->
<template>
  <el-input
    :modelValue="formattedValue"
    @update:modelValue="handleChange"
    v-bind="$attrs"
  />
</template>

<script>
import { defineComponent, computed } from 'vue';

export default defineComponent({
  name: 'CustomInput',
  props: {
    modelValue: [String, Number],
    disabled: Boolean,
    // 添加自定义属性：自动格式化
    autoFormat: {
      type: Boolean,
      default: false
    }
  },
  emits: ['update:modelValue', 'change'],
  setup(props, { emit }) {
    // 格式化值（例如：自动添加千分位）
    const formattedValue = computed(() => {
      if (!props.autoFormat || !props.modelValue) {
        return props.modelValue;
      }
      // 格式化逻辑
      return String(props.modelValue).replace(/\B(?=(\d{3})+(?!\d))/g, ',');
    });

    const handleChange = (val) => {
      // 移除格式化字符
      const rawValue = props.autoFormat 
        ? val.replace(/,/g, '') 
        : val;

        emit('update:modelValue', rawValue);
      emit('change', rawValue);
    };

    return {
      formattedValue,
      handleChange
    };
  }
});
</script>
```

在應用啟動時覆蓋內建元件：

js

```
// main.js
import formCreate from '@form-create/element-ui';
import CustomInput from './components/CustomInput.vue';

// 覆盖内置的 input 组件
formCreate.component('input', CustomInput);
```

### 示例 2：覆蓋選擇器元件 ​

建立一個自定義的選擇器元件，新增搜尋和遠端載入功能：

vue

```
<!-- CustomSelect.vue -->
<template>
  <el-select
    :modelValue="modelValue"
    @update:modelValue="handleChange"
    filterable
    remote
    :remote-method="remoteMethod"
    :loading="loading"
    v-bind="$attrs"
  >
    <el-option
      v-for="item in options"
      :key="item.value"
      :label="item.label"
      :value="item.value"
    />
  </el-select>
</template>

<script>
import { defineComponent, ref } from 'vue';
import axios from 'axios';

export default defineComponent({
  name: 'CustomSelect',
  props: {
    modelValue: [String, Number, Array],
    disabled: Boolean,
    // 自定义属性：远程搜索接口
    remoteUrl: String
  },
  emits: ['update:modelValue', 'change'],
  setup(props, { emit }) {
    const options = ref([]);
    const loading = ref(false);

    const remoteMethod = async (query) => {
      if (!props.remoteUrl) {
        return;
      }

      loading.value = true;
      try {
        const response = await axios.get(props.remoteUrl, {
          params: { keyword: query }
        });
        options.value = response.data;
      } catch (error) {
        console.error('加载选项失败:', error);
      } finally {
        loading.value = false;
      }
    };

    const handleChange = (val) => {
      emit('update:modelValue', val);
      emit('change', val);
    };

    return {
      options,
      loading,
      remoteMethod,
      handleChange
    };
  }
});
</script>
```

覆蓋內建元件：

js

```
// main.js
import formCreate from '@form-create/element-ui';
import CustomSelect from './components/CustomSelect.vue';

// 覆盖内置的 select 组件
formCreate.component('select', CustomSelect);
```

## 在設計器中使用 ​

覆蓋內建元件後，設計器中所有使用該元件的地方都會自動使用您的自定義元件：

js

```
// 表单规则中使用覆盖后的组件
const rule = [
  {
    type: 'input',  // 使用覆盖后的自定义 input 组件
    field: 'username',
    title: '用户名',
    props: {
      autoFormat: true  // 使用自定义属性
    }
  },
  {
    type: 'select',  // 使用覆盖后的自定义 select 组件
    field: 'category',
    title: '分类',
    props: {
      remoteUrl: '/api/categories'  // 使用自定义属性
    }
  }
];
```

## 注意事項 ​

重要提示

1. **元件名稱必須完全匹配**：覆蓋內建元件時，元件名稱必須與內建元件的識別符號完全一致。常見的內建元件識別符號包括：`input`、`select`、`elButton`、`textarea`、`radio`、`checkbox` 等。完整列表請檢視[內建元件列表](https://pro.form-create.com/doc/components)
2. **覆蓋時機**：`formCreate.component()` 應該在應用啟動時呼叫，確保在所有表單例項建立之前完成覆蓋
3. **元件規範**：自定義元件必須遵循 FormCreate 的元件規範，包括：

- 必須支援 `modelValue` 和 `update:modelValue`（v-model 雙向繫結）
- 必須支援 `disabled` 屬性
- 必須觸發 `change` 事件
- 使用 `v-bind="$attrs"` 透傳其他屬性
4. **向後相容**：覆蓋元件時，建議保持與內建元件相同的 API 介面，確保現有表單規則可以正常工作
5. **全域影響**：覆蓋內建元件會影響整個應用中所有使用該元件的地方，請謹慎使用

透過這種方式，您可以完全自定義內建元件的行為，同時保持與 FormCreate 的相容性。
