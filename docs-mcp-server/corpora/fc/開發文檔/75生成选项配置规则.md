# 生成選項配置規則 ​

本文詳細介紹瞭如何為元件（如穿梭框、樹形選擇器等）配置選項資料，包括普通列表和樹形結構兩種常見形式。透過內建的 `makeOptionsRule` 和 `makeTreeOptionsRule` 方法，開發者可以快速生成標準化的選項配置規則。

## 普通選項配置 ​

![make-rule1.png](https://pro.form-create.com/doc/img/make-rule1.png)

### 配置步驟 ​

1. **確定接收選項的屬性**  
 查閱元件文件確認接收選項的屬性名，如 `elTransfer` 元件透過 `props.data` 接收選項。
2. **使用 `makeOptionsRule` 方法**  
 該方法用於生成普通選項的配置規則。

注意

**to 引數**需與元件接收選項的 props 欄位對應，例如元件中透過 props.data 接收時，需傳 `"props.data"`。

### 程式碼示例 ​

js

```
import uniqueId from '@form-create/utils/lib/unique';
import {makeOptionsRule} from '/path/to/fcDesignerPro/dist/index.es.js';

const label = '穿梭框';
const name = 'elTransfer';

export default {
    // 组件基础配置
    menu: 'main',
    icon: 'icon-transfer',
    label,
    name,
    input: true,
    event: ['change', 'leftCheckChange', 'rightCheckChange'],
    validate: ['array'],
    // 默认规则配置
    rule({t}) {
        return {
            type: name,
            field: uniqueId(),
            title: '穿梭框',
            info: '',
            $required: false,
            props: {
                //初始数据
                data: [
                    {label: "选项01", value: "1"},
                    {label: "选项02", value: "2"}, 
                    {label: "选项03", value: "3"}
                ]
            }
        };
    },
    // 属性配置
    props(_, {t}) {
        return [
            // 生成选项配置规则
            makeOptionsRule(t, 'props.data', 'label', 'value'),
            {
                type: 'switch',
                title: '是否可搜索',
                field: 'filterable'
            }
        ];
    }
};
```

### 資料結構 ​

typescript

```
/**
 * 生成普通选项配置规则
 * @param t 国际化函数
 * @param to 选项数据存放位置（如 'props.data'）
 * @param label 标签字段名（默认 'label'）
 * @param value 值字段名（默认 'value'）
 * @returns 生成的配置规则数组
 */
makeOptionsRule(t: T, to: string, label?: string, value?: string): Rule[];
```

## 樹形選項配置 ​

![make-rule2.png](https://pro.form-create.com/doc/img/make-rule2.png)

### 配置步驟 ​

1. **確定接收選項的屬性**  
 如 `elTreeSelect` 元件透過 `props.data` 接收樹形選項。
2. **使用 `makeTreeOptionsRule` 方法**  
 專為樹形結構選項設計的配置方法。

注意

**to 引數**需與元件接收選項的 props 欄位對應，例如元件中透過 props.data 接收時，需傳 `"props.data"`。

### 程式碼示例 ​

js

```
import uniqueId from '@form-create/utils/lib/unique';
import {makeTreeOptionsRule} from '/path/to/fcDesignerPro/dist/index.es.js';

const label = '树形选择';
const name = 'elTreeSelect';

export default {
    // 组件基础配置
    menu: 'main',
    icon: 'icon-tree-select',
    label,
    name,
    input: true,
    event: ['change', 'visibleChange', 'removeTag', 'clear', 'blur', 'focus'],
    validate: ['string', 'number', 'array'],
    // 默认规则配置
    rule({t}) {
        return {
            type: name,
            field: uniqueId(),
            title: '树形选择',
            info: '',
            $required: false,
            props: {
                nodeKey: 'value',
                showCheckbox: true,
                //初始数据
                data: [
                    {
                        label: "选项201",
                        value: "1",
                        children: [
                            {label: "选项1-101", value: "2"}
                        ]
                    }
                ]
            }
        };
    },
    // 属性配置
    props(_, {t}) {
        return [
            // 生成树形选项配置规则
            makeTreeOptionsRule(t, 'props.data', 'label', 'value'),
            {type: 'switch', title: '多选', field: 'multiple'}
        ];
    }
};
```

### 資料結構 ​

typescript

```
/**
 * 生成树形选项配置规则
 * @param t 国际化函数
 * @param to 选项数据存放位置
 * @param label 标签字段名（默认 'label'）
 * @param value 值字段名（默认 'value'）
 * @returns 生成的配置规则数组
 */
makeTreeOptionsRule(t: T, to: string, label?: string, value?: string): Rule[];
```

## 注意事項 ​

1. 確保路徑 `/path/to/fcDesignerPro/dist/index.es.js` 已替換為實際路徑
2. 欄位名引數（label/value）需與實際資料結構保持一致
