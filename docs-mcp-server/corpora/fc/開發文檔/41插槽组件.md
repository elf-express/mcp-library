# 插槽元件 ​

插槽元件允許您透過 Vue 的插槽功能擴展表單內顯示區域，提供了更大的靈活性和自定義能力。透過插槽元件，您可以在設計器和表單執行時動態注入自定義內容，實現更靈活的表單佈局和互動功能。

![slot-component.png](https://pro.form-create.com/doc/img/slot-component.png)

## 基礎使用 ​

### 在設計器中配置插槽元件 ​

在表單設計器中，您可以透過左側元件面板拖拽"插槽區域"元件到表單中。插槽元件預設會自動生成一個唯一的插槽名稱（格式為 `block_` + 唯一ID），您也可以手動修改這個名稱。

插槽元件的主要配置項：

- **name**：插槽名稱，用於匹配對應的插槽內容。預設格式為 `block_` + 唯一ID

### 在設計器中使用插槽 ​

在 `fc-designer` 設計器中，您可以透過插槽自定義設計器的顯示內容。插槽的名稱為 `#block_[name]` 格式，其中 `[name]` 為插槽元件配置的 name 值。

**基礎用法**：

vue

```
<template>
  <fc-designer ref="designer">
    <template #block_Fmrym0sz8orudpc="scope">
      这里是自定义内容
    </template>
  </fc-designer>
</template>
```

**新增自定義操作按鈕**：

vue

```
<template>
  <fc-designer ref="designer">
    <template #block_Fmrym0sz8orudpc="scope">
      <el-button @click="handleCustomAction(scope)">自定义操作</el-button>
    </template>
  </fc-designer>
</template>
<script setup>
import { ElMessage } from 'element-plus';

const handleCustomAction = (scope) => {
  ElMessage.info('自定义操作触发');
  console.log('当前表单数据:', scope.api.formData());
  console.log('当前组件规则:', scope.rule);
};
</script>
```

**根據預覽模式顯示不同內容**：

vue

```
<template>
  <fc-designer ref="designer">
    <template #block_Fmrym0sz8orudpc="scope">
      <div v-if="scope.preview" class="preview-mode">
        <p>预览模式下显示：{{ scope.rule.title }}</p>
      </div>
      <div v-else class="edit-mode">
        <el-input v-model="inputValue" placeholder="编辑模式下可输入" />
      </div>
    </template>
  </fc-designer>
</template>
<script setup>
import { ref } from 'vue';
const inputValue = ref('');
</script>
```

### 在表單中使用插槽 ​

在 `form-create` 渲染的表單中，您同樣可以透過插槽自定義表單顯示內容。使用方式與設計器中完全相同。

vue

```
<template>
  <form-create :rule="rule" :option="option">
    <template #block_Fmrym0sz8orudpc="scope">
      <div class="custom-slot-content">
        <h3>自定义内容区域</h3>
        <p>您可以在这里放置任何自定义内容</p>
        <el-button @click="handleSubmit(scope)">提交表单</el-button>
      </div>
    </template>
  </form-create>
</template>
<script setup>
import { ref } from 'vue';
import formCreate from '@form-create/element-ui';

const rule = ref([
  // 表单规则配置
]);
const option = ref({});

const handleSubmit = (scope) => {
  scope.api.submit();
};
</script>
```

### 資料繫結和更新 ​

當插槽元件配置了 `field` 屬性後，您可以透過 `scope.model` 進行資料的讀取和更新：

vue

```
<template>
  <fc-designer ref="designer">
    <template #block_Fmrym0sz8orudpc="scope">
      <div>
        <p>当前值：{{ scope.model.value }}</p>
        <el-button @click="updateValue(scope)">更新值</el-button>
      </div>
    </template>
  </fc-designer>
</template>
<script setup>
const updateValue = (scope) => {
  const newValue = Date.now();
  scope.model.callback(newValue);
};
</script>
```

## Scope 物件 ​

`Scope` 物件提供了插槽元件的上下文資料，包括元件的生成規則、渲染規則、表單 API 等。以下是 Scope 的完整資料結構：

ts

```
type Scope = {
    //插槽组件的生成规则
    rule: Rule,
    //插槽组件的最终渲染规则
    prop: Rule,
    //是否为阅读模式
    preview: Boolean,
    //表单的API
    api: Api,
    //获取组件的值和同步值，仅在规则中定义 field 时可用
    model: {
        //组件的值
        value: any;
        //更新值
        callback: (value: any) => void;
    },
}
```

**欄位說明：**

- `rule`：元件的原始生成規則，包含元件的完整配置資訊
- `prop`：元件的最終渲染規則，經過所有轉換和處理後的實際渲染配置
- `preview`：布林值，標識當前是否為預覽模式，您可以根據此值顯示不同的內容
- `api`：表單 API 物件，提供表單操作的各種方法，如獲取表單資料、設定表單值、驗證表單等
- `model`：當插槽元件定義了 `field` 屬性時，此物件可用，提供對元件值的讀取和更新能力

## 配置項 ​

插槽元件提供了以下配置選項，您可以透過在設計器中配置屬性來自定義插槽元件的行為。

| 屬性名 | 型別 | 預設值 | 必需 | 說明 |
| --- | --- | --- | --- | --- |
| name | String | block_default | 否 | 插槽名稱，用於匹配對應的插槽內容 |
| field | String | - | 否 | 欄位名，定義後可使用model進行資料繫結 |

## 方法 ​

插槽元件內部提供了以下方法供內部使用。

| 方法名 | 引數 | 說明 | 返回值 |
| --- | --- | --- | --- |
| getSlot | - | 獲取對應的插槽函式（內部使用） | Function |

## 使用示例 ​

### 示例 1：顯示動態內容 ​

透過 `scope.api` 可以獲取表單資料並展示動態內容：

vue

```
<template>
  <fc-designer ref="designer">
    <template #block_userInfo="scope">
      <div class="user-info-card">
        <h4>用户信息</h4>
        <p>表单数据：{{ JSON.stringify(scope.api.formData()) }}</p>
      </div>
    </template>
  </fc-designer>
</template>
```

### 示例 2：條件顯示內容 ​

根據表單狀態或其他條件顯示不同的內容：

vue

```
<template>
  <fc-designer ref="designer">
    <template #block_conditional="scope">
      <div v-if="scope.preview">
        <el-tag type="info">预览模式</el-tag>
      </div>
      <div v-else>
        <el-tag type="success">编辑模式</el-tag>
      </div>
    </template>
  </fc-designer>
</template>
```

### 示例 3：表單資料操作 ​

透過 `scope.api` 執行表單操作：

vue

```
<template>
  <fc-designer ref="designer">
    <template #block_actions="scope">
      <el-button-group>
        <el-button @click="scope.api.submit()">提交</el-button>
        <el-button @click="scope.api.resetFields()">重置</el-button>
        <el-button @click="handleClear(scope)">清空</el-button>
      </el-button-group>
    </template>
  </fc-designer>
</template>
<script setup>
const handleClear = (scope) => {
  const formData = scope.api.formData();
  Object.keys(formData).forEach(key => {
    scope.api.clearValidateState(key);
  });
};
</script>
```

### 示例 4：響應式資料更新 ​

使用 `scope.model` 實現雙向資料繫結：

vue

```
<template>
  <fc-designer ref="designer">
    <template #block_counter="scope">
      <div class="counter-widget">
        <el-button @click="decrease(scope)">-</el-button>
        <span style="padding: 0 10px">{{ scope.model.value || 0 }}</span>
        <el-button @click="increase(scope)">+</el-button>
      </div>
    </template>
  </fc-designer>
</template>
<script setup>
const increase = (scope) => {
  const value = scope.model.value || 0;
  scope.model.callback(value + 1);
};

const decrease = (scope) => {
  const value = scope.model.value || 0;
  if (value > 0) {
    scope.model.callback(value - 1);
  }
};
</script>
```

## 注意事項 ​

1. **插槽名稱必須唯一**：在設計器和表單中，插槽名稱必須與插槽元件的 name 屬性完全匹配才能正確顯示內容。
2. **Scope 物件只讀**：雖然 scope 物件提供了豐富的資料和方法，但建議不要直接修改 scope 物件的屬性，而是透過提供的方法進行操作。
3. **資料繫結使用場景**：只有在插槽元件配置了 `field` 屬性後，才可以使用 `scope.model` 進行資料繫結，否則該物件為空物件。
4. **插槽查詢機制**：插槽元件會向上查詢父元件的插槽，支援多層巢狀的表單結構，確保在複雜場景下也能正確找到對應的插槽內容。

透過使用插槽元件，您可以輕鬆地擴展和定製表單和設計器中的顯示內容，以滿足您的具體需求。
