# 獲取表單結構樹 ​

在設計器中，您可以使用 `getDescription` 和 `getFormDescription` 方法獲取當前拖拽表單的層級結構。這些方法幫助您檢視和操作表單元件的組織結構。

注意

層級結構與表單最終的渲染規則可能存在不一致的情況。例如，某些巢狀元件層級結構和最終規則不同。

## 示例 ​

![description.png](https://pro.form-create.com/doc/img/description.png)

vue

```
<fc-designer ref="designer" />
```

## 所有元件的層級結構 ​

透過`getDescription`方法獲取當前設計器中所有元件的層級結構，包括設計器內的所有元件，無論是表單元件還是非表單元件。

vue

```
<template>
    <fc-designer ref="designer" />
</template>

<script>
    export default {
        methods: {
            getDescription() {
                // 调用 getDescription 方法获取所有组件的层级结构
                const tree = this.$refs.designer.getDescription();
                console.log(tree);
            }
        }
    }
</script>
```

## 資料結構 ​

ts

```
export type DescriptionData = Array<{
    //唯一值
    _fc_id: string;
    //组件类型
    type: string;
    //字段id
    field?: string;
    //组件名称
    title?: string;
    //插槽名称
    slot?: string;
    //组件属性配置
    props: Object;
    //子级
    children?: DescriptionData;
}>
```

- **_fc_id：** 唯一識別符號，用於唯一識別每個元件。
- **type：** 元件型別，如 input、slider 等。
- **field：** 欄位 ID，表單中每個元件的欄位標識。
- **title：** 元件名稱，用於描述元件功能。
- **slot：** 插槽名稱，用於指定元件插槽。
- **props：** 元件的屬性配置，包含元件的所有配置資訊。
- **children：** 子級元件的層級結構，表示當前元件下的所有子元件。
