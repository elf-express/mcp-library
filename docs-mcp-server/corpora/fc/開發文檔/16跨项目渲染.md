# 跨專案渲染 ​

跨專案渲染時可能出現元件無法渲染問題，本文提供完整解決方案。

注意

高階版設計器基於開源渲染器擴展了功能並新增元件，使用時需從設計器包中匯入對應的專用渲染器。

## 跨專案使用 ​

當您需要在其他專案中使用設計器生成的表單規則時，只需要匯入對應的渲染器即可，無需完整匯入設計器。高階版設計器基於開源渲染器擴展了功能並新增元件，因此需要使用設計器包中的專用渲染器。

### 渲染器檔案說明 ​

設計器包中提供了不同框架的渲染器檔案：

**PC 端渲染器**（位於 `dist/render` 目錄）：

- Element Plus/Element UI: `dist/render/element-plus/form-create.[es|umd].js`
- Ant Design Vue: `dist/render/ant-design-vue/form-create.[es|umd].js`

**移動端渲染器**（位於 `dist/mobile` 目錄）：

- Vant: `dist/mobile/form-create.vant.[es|umd].js`

注意

跨專案使用時只需要匯入渲染器，不需要匯入完整的設計器包。渲染器檔案體積更小，僅包含表單渲染所需的功能。

### 使用步驟 ​

**1. 複製渲染器檔案**

將設計器包中對應的渲染器檔案複製到目標專案中，例如複製到 `src/libs` 目錄：

**Element Plus/Element UI 版本：**

```
fcDesignerPro/dist/render/element-plus/form-create.es.js
→ 目标项目/src/libs/form-create.es.js
```

**Ant Design Vue 版本：**

```
fcDesignerPro/dist/render/ant-design-vue/form-create.es.js
→ 目标项目/src/libs/form-create.es.js
```

**移動端版本：**

```
fcDesignerMobile/dist/mobile/form-create.vant.es.js
→ 目标项目/src/libs/form-create.vant.es.js
```

**2. 替換渲染器匯入**

在目標專案中，將開源版渲染器的匯入替換為高階版渲染器。

**Element Plus/Element UI 版本：**

替換前：

js

```
import formCreate from "@form-create/element-ui"
```

替換後：

js

```
import formCreate from "@/libs/form-create.es.js"
// 或使用相对路径
// import formCreate from "src/libs/form-create.es.js"
```

**Ant Design Vue 版本：**

替換前：

js

```
import formCreate from "@form-create/ant-design-vue"
```

替換後：

js

```
import formCreate from "@/libs/form-create.es.js"
// 或使用相对路径
// import formCreate from "src/libs/form-create.es.js"
```

**移動端版本：**

替換前：

js

```
import formCreate from "@form-create/vant"
```

替換後：

js

```
import formCreate from "@/libs/form-create.vant.es.js"
// 或使用相对路径
// import formCreate from "src/libs/form-create.vant.es.js"
```

### 完整使用示例 ​

**Element Plus/Element UI 版本：**

在 `main.js` 中掛載：

js

```
import { createApp } from 'vue'
import App from './App.vue'
import formCreate from '@/libs/form-create.es.js'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'

const app = createApp(App)

// 挂载 ElementPlus
app.use(ElementPlus)
// 挂载 formCreate
app.use(formCreate)

app.mount('#app')
```

在元件中使用：

vue

```
<template>
    <form-create v-model="formData" :rule="rule" v-model:api="fApi" @submit="handleSubmit"></form-create>
</template>

<script setup>
import { ref } from 'vue'

const formData = ref({})
const fApi = ref(null)
const rule = ref([
    // 从设计器导出的表单规则
])

function handleSubmit(data) {
    console.log('表单数据:', data)
}
</script>
```

## 擴展元件 ​

### 入口檔案 ​

element-plus/element-ui 版本對應:

```
/src/form/elm.js
```

ant-design-vue 版本對應:

```
/src/form/antdv.js
```

移動端版本對應:

```
/src/form/mobile.js
```

![build2](https://pro.form-create.com/doc/img/build2.png)

### 掛載元件 ​

將自定義擴展元件匯入上述檔案並完成掛載，具體操作可參考示例元件。例如匯入[擴展元件](https://pro.form-create.com/doc/component)中的示例元件

注意

**1.若表單依賴全域方法或變數，需在目標專案中同步定義對應依賴。**

**2.自定義元件和打包檔案中禁止使用框架全域方法，避免跨專案遷移時功能失效。**

匯入自定義Vue元件（Title和Input），並使用 formCreate.component 方法將它們掛載到表單中

js

```
//导入组件
import Title from 'Title.vue';
import Input from 'Input.vue';

//挂载组件
formCreate.component('Title', Title);
formCreate.component('FcInput', Input);
```

### 更新渲染器版本 ​

透過 npm 命令更新渲染器至最新版本，可提升穩定性和修復已知問題。

element-plus/element-ui

sh

```
npm update @form-create/element-ui
```

ant-design-vue

sh

```
npm update @form-create/ant-design-vue
```

移動端

sh

```
npm update @form-create/vant
```

### 重新打包渲染器 ​

在設計器專案根目錄執行打包命令

element-plus/element-ui

sh

```
npm run build:elm
```

ant-design-vue

sh

```
npm run build:antdv
```

移動端

sh

```
npm run build:mobile
```

![build3](https://pro.form-create.com/doc/img/build3.png)
