# 分步表單 ​

分步表單元件透過將複雜表單拆解為多個步驟，引導使用者逐步完成填寫。每個步驟都可以包含獨立的表單內容，支援資料驗證和步驟切換，有助於提高使用者填寫體驗和資料準確性。

![step-form.png](https://pro.form-create.com/doc/img/step-form.png)

## 基礎使用 ​

### 切換步驟 ​

透過注入的 `api.el` 方法可以操作分步表單，動態設定其啟用狀態。例如，切換到步驟2：

js

```
function onChange($inject) {
    $inject.api.el('ref_F2vulxvqc841dac').active = 2;
}
```

將 `ref_F2vulxvqc841dac` 替換為您的分步表單元件的ID，將 `2` 替換為目標步驟索引（從0開始）。透過設定 `active` 屬性，您可以實現步驟的精確跳轉。

### 獲取子表單API ​

分步表單內部的表單例項可以透過 `subApi` 屬性獲取，方便您進行資料操作或驗證：

js

```
function getSubApi($inject) {
    // 获取分步表单实例
    const stepFormInstance = $inject.api.el('ref_F2vulxvqc841dac');
    // 获取子表单API
    const subApi = stepFormInstance.subApi;

    // 获取当前表单数据
    const formData = subApi.formData();
    console.log('当前步骤表单数据:', formData);

    // 验证当前步骤
    subApi.validate().then(() => {
        console.log('验证通过');
    }).catch(() => {
        console.log('验证失败');
    });
}
```

透過 `subApi`，您可以訪問當前活躍步驟的表單API，進行資料讀取、欄位驗證、值設定等操作。

## 配置項 ​

分步表單提供了豐富的配置選項，您可以透過在設計器中配置屬性來自定義分步表單的行為和外觀。

| 屬性名 | 型別 | 預設值 | 必需 | 說明 |
| --- | --- | --- | --- | --- |
| stepsProps | Object | - | 否 | 步驟條屬性配置 |
| modelValue | Object | - | 否 | 雙向繫結的表單資料 |
| autoValidate | Boolean | - | 否 | 是否在切換步驟時自動驗證 |
| submitBtn | Boolean | true | 否 | 是否顯示提交按鈕 |
| steps | Array | [] | 是 | 步驟配置陣列 |
| options | Object | { submitBtn: false, resetBtn: false } | 否 | 表單配置選項 |

## 事件 ​

分步表單元件提供了豐富的事件，方便您監聽表單狀態變化並執行相應的處理。

| 事件名 | 引數 | 說明 |
| --- | --- | --- |
| update:modelValue | value | 表單資料更新事件，當任意步驟的表單資料變化時觸發 |
| change | value | 表單資料變化事件，與update:modelValue類似 |
| itemMounted | api | 子表單掛載完成事件，當子表單初始化完成時觸發 |
| submit | formData, api | 表單提交事件，當使用者點選提交按鈕時觸發 |
| next | {active, api} | 下一步切換事件，當切換到下一步時觸發 |
| validateFail | api | 表單驗證失敗事件，當自動驗證失敗時觸發 |

### 事件使用示例 ​

**監聽步驟切換**：

js

```
function handleNext({ active, api }) {
    console.log('当前步骤:', active);
    console.log('表单数据:', api.formData());
    // 在特定步骤执行特定操作
    if (active === 1) {
        // 第二步时的特殊处理
        console.log('进入第二步');
    }
}
```

**監聽表單提交**：

js

```
function handleSubmit(formData, api) {
    console.log('最终提交的数据:', formData);
    // 提交数据到后台
    // submitToBackend(formData);
}
```

**監聽表單資料變化**：

js

```
function handleChange(value) {
    console.log('当前所有步骤的数据:', value);
    // 可以实时保存草稿
    // saveDraft(value);
}
```

## 方法 ​

分步表單提供了多種方法，方便您進行程式化控制。

| 方法名 | 引數 | 說明 | 返回值 |
| --- | --- | --- | --- |
| init | - | 初始化步驟表單 | - |
| onPrev | - | 執行上一步操作 | - |
| validate | - | 驗證當前步驟的表單資料 | Promise |
| onNext | - | 執行下一步操作 | - |
| submit | - | 提交整個表單 | - |
| addSubApi | api | 新增子表單API例項（內部使用） | - |
| formData | value | 處理表單資料變化（內部使用） | - |
| setValue | value | 設定表單值 | - |

### 方法使用示例 ​

**程式化驗證當前步驟**：

js

```
function validateCurrentStep($inject) {
    const stepForm = $inject.api.el('ref_F2vulxvqc841dac');
    const subApi = stepForm.subApi;
    subApi.validate().then(() => {
        console.log('当前步骤验证通过');
        // 继续下一步或其他操作
    }).catch(errors => {
        console.log('当前步骤验证失败:', errors);
        // 提示用户修复错误
    });
}
```

**程式化切換步驟**：

js

```
function goToStep($inject, stepIndex) {
    const stepForm = $inject.api.el('ref_F2vulxvqc841dac');
    stepForm.active = stepIndex;
}
```

**程式化提交表單**：

js

```
function submitForm($inject) {
    const stepForm = $inject.api.el('ref_F2vulxvqc841dac');
    const subApi = stepForm.subApi;
    // 验证所有步骤
    subApi.validate().then(() => {
        const formData = subApi.formData();
        console.log('提交数据:', formData);
        // 提交到后台
    }).catch(() => {
        console.log('验证失败，无法提交');
    });
}
```

## Data ​

分步表單元件的內部資料屬性如下：

| 資料名 | 初始值 | 說明 |
| --- | --- | --- |
| active | 0 | 當前啟用的步驟索引 |
| cacheRule | [] | 快取的分步規則 |
| cacheValue | {} | 快取的表單資料 |
| subApi | {} | 子表單API例項 |
| Form | markRaw(formCreateInject.form.$form()) | Form元件例項 |

**資料說明**：

- **active**：當前啟用的步驟索引，從0開始。可以透過修改此值來實現步驟跳轉
- **cacheRule**：快取的分步規則陣列，包含了所有步驟的表單規則配置
- **cacheValue**：快取的表單資料物件，儲存了所有步驟的資料，確保步驟切換時資料不丟失
- **subApi**：子表單API例項，提供了訪問和操作當前步驟表單的方法
- **Form**：Form元件例項，用於渲染表單內容
