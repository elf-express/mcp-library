# 儲存表單設計 ​

提供了儲存表單設計的功能，使開發者能夠持久化儲存表單設計，以便使用者在頁面重新整理或過載後能夠恢復先前的狀態。這對於複雜表單的管理和持久化至關重要。

注意!!!!!

**必須使用 `formCreate.parseJson` 代替 `JSON.parse` 方法，使用 `formCreate.toJson` 代替 `JSON.stringify` 方法來轉換 JSON 資料，以確保資料格式正確。**

## 顯示儲存按鈕 ​

設計器中內建了一個儲存按鈕，透過 `config.showSaveBtn` 配置項來控制其顯示。此功能允許使用者在表單設計完成後方便地觸發儲存操作。

![save.png](https://pro.form-create.com/doc/img/save.png)

以下程式碼展示瞭如何在配置中啟用儲存按鈕：

js

```
{
    config: {
        showSaveBtn: true
    }
}
```

## 儲存資料 ​

當使用者點選儲存按鈕時，會觸發 `save` 事件。開發者可以自定義處理該事件以實現儲存功能，將設計好的表單規則和配置傳送到伺服器或本地儲存。

### 儲存資料示例 ​

以下是儲存資料的一個簡單實現示例：

vue

```
<template>
    <fc-designer ref="designer" @save="handleSave" :config="config"/>
</template>
<script setup>
    const config = {
        showSaveBtn: true
    }
    function handleSave ({rule, options}) {
        // 示例：调用后端接口保存表单规则和配置
        axios.post('/api/saveForm', {
            rules: rule, //JSON字符串
            options: options //JSON字符串
        }).then(response => {
            // 处理保存成功的逻辑
            console.log('表单保存成功', response.data);
        }).catch(error => {
            // 错误处理
            console.error('表单保存失败', error);
        });
    }
</script>
```

透過上述程式碼，當使用者點選儲存按鈕時，表單的規則和配置以 JSON 格式傳送到指定的 API 介面。

## 回顯資料 ​

為了在載入頁面時回顯之前設計的表單，需要從儲存中載入之前儲存的 JSON 規則和配置，將其應用到設計器中。

### 回顯資料示例 ​

以下是回顯資料的實現示例：

vue

```
<template>
    <fc-designer ref="designer" @save="handleSave" :config="config"/>
</template>
<script setup>
    const designer = ref(null)
    onMounted(() => {
        try {
            // 示例：从服务器端获取保存的JSON规则
            const { data } = await axios.get('/api/getForm');
            const { ruleJson, optionsJson } = data;
            // 回显设计的表单
            designer.value.setOptions(optionsJson);
            designer.value.setRule(ruleJson);
        } catch (error) {
            console.error('加载表单数据失败', error);
        }
    });
</script>
```

在以上程式碼中，應用程式會在元件掛載後立即從伺服器載入之前儲存的資料，並將其應用到當前的設計器例項以恢復表單狀態。

透過這些方法和實踐，開發者可以輕鬆構建一個功能完善的表單管理系統，支援表單的建立、儲存、載入及更新，使使用者能夠更高效地使用設計工具。
