# 移動端自適應 ​

`FcDesigner` 支援將 PC 端設計的 **ElementPlus / AntDesignVue** 表單規則在移動端渲染為**VantUI**風格的表單，提供了一致的使用者體驗，並確保在不同裝置上都能良好展示。

[功能演示](https://pro.form-create.com/view)

![mobile](https://pro.form-create.com/doc/img/example2.gif)

## 功能概述 ​

透過移動端自適應功能，您可以確保在不同裝置上的表單顯示效果一致。無論是在 PC 端還是移動端，表單都能根據裝置型別自動調整樣式和佈局。具體而言，移動端的表單將使用 Vant 風格元件，以最佳化在小螢幕上的展示效果和互動體驗。

注意

當前移動端自適應功能僅支援部分基礎表單元件和佈局元件，尚未覆蓋所有元件型別。

### 圖表與模板 ​

圖表元件（折線圖、餅圖、柱狀圖等）及 Mermaid 圖表、模板（三列柵格、金額等）在移動端同樣支援渲染。

UI 庫差異說明

PC 端使用 **Element Plus / Ant Design Vue**，移動端使用 **Vant**。由於不同 UI 庫的元件能力和 API 存在差異，**部分配置項在移動端可能存在不相容或表現不一致的情況**。不支援自適應的元件在移動端會原樣顯示（無法正確渲染為 Vant 風格）。

## 使用方式 ​

**1. 安裝和載入元件**

先在專案中引入移動端專用渲染器 `dist/render/vant/form-create.[es|umd].js`，如果你把高階版放在其他位置，請同步調整。

diff

```
- import formCreateMobile from "@form-create/vant";
+ import formCreateMobile from "./fcDesignerPro/dist/render/vant/form-create.es";
```

**2. 渲染移動端表單**

在 Vue 元件中，使用 form-create-mobile 元件來渲染表單。該元件會自動根據裝置型別渲染為適合的樣式。

vue

```
<template>
    <!-- Element Plus版本使用driver="elm"，Ant Design Vue版本使用driver="antd" -->
    <form-create-mobile
    driver="elm"
    v-model="formData"
    v-model:api="fapi"
    :rule="rule"
    :option="option"
    @submit="onSubmit"
    ></form-create-mobile>
</template>
```

重要提醒

**必須正確配置 driver 屬性**：在移動端渲染時，必須根據您使用的 UI 框架正確設定 driver 屬性，這是確保正確渲染為 Vant UI 風格的關鍵配置。

- ✅ Element Plus 版本：`driver="elm"` - 渲染為 Vant UI 風格
- ✅ Ant Design Vue 版本：`driver="antd"` - 渲染為 Vant UI 風格
- ❌ 錯誤：不設定 driver - 可能導致渲染失敗
- ❌ 錯誤：driver 配置與使用的 UI 框架不匹配 - 可能導致樣式異常

vue

```
<script setup>
    import { ref } from 'vue';
    import formCreateMobile from "./fcDesignerPro/dist/render/vant/form-create.es";
    const formData = ref({});
    const fapi = ref({});
    //加载json数据
    const rule = ref(formCreateMobile.parseJson('/*json*/'));
    const option = ref(formCreateMobile.parseJson('/*json*/'));
    function onSubmit(data) {
        console.log('Form submitted:', data);
    }
</script>
```

## 元件適配 ​

透過在不同端掛載功能相同但 UI 不同的元件。業務層使用統一的元件介面，底層根據執行環境自動選擇對應的 UI 實現。

假設當前有一個基於 ElementUI 的自定義 `UnifiedSelect` 業務元件

vue

```
<template>
  <!-- Element UI 版本 -->
  <el-select
    v-model="innerValue"
    :placeholder="placeholder"
    :disabled="disabled"
    :clearable="clearable"
    :multiple="multiple"
    @change="handleChange"
  >
    <el-option
      v-for="item in options"
      :key="item.value"
      :label="item.label"
      :value="item.value"
    />
  </el-select>
</template>

<script>
export default {
  name: 'UnifiedSelect',
  props: {
    value: [String, Number, Array],
    options: {
      type: Array,
      default: () => []
    },
    placeholder: String,
    disabled: Boolean,
    clearable: Boolean,
    multiple: Boolean,
  },
  data() {
    return {
      innerValue: this.value,
      showPicker: false,
      selectedLabel: ''
    }
  },
  watch: {
    value(newVal) {
      this.innerValue = newVal
    }
  },
  methods: {
    handleChange(value) {
      this.$emit('input', value)
      this.$emit('change', value)
    },
  }
}
</script>
```

**（Vant UI 實現）**

實現移動端適配需建立功能相同的同名元件，並在移動端渲染時掛載該元件。

ts

```
<template>
  <!-- Vant UI 版本 -->
  <van-field
    v-model="selectedLabel"
    readonly
    :placeholder="placeholder"
    :disabled="disabled"
    @click="showPicker = true"
  />
  <van-popup v-model="showPicker" round position="bottom">
    <van-picker
      :columns="pickerOptions"
      @confirm="handleVantConfirm"
      @cancel="showPicker = false"
    />
  </van-popup>
</template>

<script>
export default {
  name: 'UnifiedSelect',
  props: {
    value: [String, Number, Array],
    options: {
      type: Array,
      default: () => []
    },
    placeholder: String,
    disabled: Boolean,
    clearable: Boolean,
    multiple: Boolean,
  },
  data() {
    return {
      innerValue: this.value,
      showPicker: false,
      selectedLabel: ''
    }
  },
  computed: {
    pickerOptions() {
      return this.options.map(item => ({
        text: item.label,
        value: item.value
      }))
    }
  },
  watch: {
    value(newVal) {
      this.innerValue = newVal
      this.updateSelectedLabel()
    },
    options() {
      this.updateSelectedLabel()
    }
  },
  methods: {
    handleChange(value) {
      this.$emit('input', value)
      this.$emit('change', value)
    },
    handleVantConfirm(value) {
      this.innerValue = value.value
      this.handleChange(value.value)
      this.showPicker = false
    },
    updateSelectedLabel() {
      const selected = this.options.find(item => item.value === this.innerValue)
      this.selectedLabel = selected ? selected.label : ''
    }
  }
}
</script>
```

**在移動端掛載該元件**

在移動端使用 formCreateMobile.component 方法掛載自定義元件

ts

```
formCreateMobile.component('UnifiedSelect', UnifiedSelect);
```

## Driver 配置說明 ​

根據您專案使用的 UI 框架，需要正確配置 driver 屬性：

### Element Plus 專案 ​

vue

```
<form-create-mobile driver="elm" />
```

### Ant Design Vue 專案 ​

vue

```
<form-create-mobile driver="antd" />
```

配置要點

- **Element Plus 專案**：使用 `driver="elm"`
- **Ant Design Vue 專案**：使用 `driver="antd"`
- 兩種配置都會在移動端渲染為 Vant UI 風格
- 配置錯誤可能導致元件渲染異常

透過 FcDesigner 的移動端自適應功能，您可以輕鬆實現跨平臺的表單展示。移動端使用 Vant 元件最佳化展示效果，同時確保 PC 端和移動端都能獲得一致且良好的使用者體驗。
