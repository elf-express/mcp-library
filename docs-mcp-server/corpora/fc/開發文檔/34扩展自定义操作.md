# 擴展自定義操作 ​

在 `FcDesigner` 設計器中，您可以透過多種方式擴展自定義操作按鈕，以增強業務邏輯和使用者體驗。以下是如何實現這些功能的詳細示例。

![扩展自定义操作](https://pro.form-create.com/doc/img/handle.png)

## 儲存表單 ​

您可以透過在設計器中新增一個儲存按鈕來實現儲存功能。點選儲存按鈕後會觸發 `save` 事件，您可以在事件處理函式中編寫儲存邏輯。

vue

```
<template>
    <fc-designer ref="designer" @save="handleSave" :config="config"></fc-designer>
</template>
<script>
    export default {
        name: 'app',
        data() {
            return {
                config: {
                    // 显示保存按钮
                    showSaveBtn: true
                },
            };
        },
        methods: {
            // 保存事件处理函数
            handleSave() {
                // 处理表单保存逻辑
                console.log('保存按钮被点击');
                // 例如，您可以将表单数据发送到服务器
                // axios.post('/save', this.$refs.designer.getFormData());
            }
        }
    };
</script>
```

## 配置擴展 ​

您可以透過 `props.handle` 配置擴展操作按鈕。每個按鈕都有一個 `label` 和一個 `handle` 函式，用於定義按鈕的行為。

vue

```
<template>
    <fc-designer ref="designer" :handle="handle"></fc-designer>
</template>
<script>
    export default {
        name: 'app',
        data() {
            return {
                handle: [
                    {
                        // 自定义操作按钮：中英切换
                        label: '中英切换',
                        handle: () => {
                            // 处理中英文切换逻辑
                            console.log('中英切换按钮被点击');
                            // 例如，切换界面语言
                        },
                    },
                    {
                        // 自定义操作按钮：选择表单
                        label: '选择表单',
                        handle: () => {
                            // 处理选择表单逻辑
                            console.log('选择表单按钮被点击');
                            // 例如，打开表单选择对话框
                        },
                    },
                ],
            };
        }
    };
</script>
```

## 插槽擴展 ​

設計器提供了多個插槽，允許您在不同位置插入自定義內容，提供更高的靈活性。

### handle 插槽 ​

您可以透過 `handle` 插槽擴展自定義按鈕。使用插槽可以插入任意自定義元件或按鈕。

vue

```
<template>
    <fc-designer ref="designer">
        <template #handle>
            <el-button @click="customAction">自定义按钮</el-button>
        </template>
    </fc-designer>
</template>
<script>
    export default {
        name: 'app',
        methods: {
            // 自定义按钮点击事件处理函数
            customAction() {
                console.log('自定义按钮被点击');
                // 执行自定义操作
            }
        }
    };
</script>
```

### header 插槽 ​

設計器頂部區域現已支援自定義插槽，可用於擴展品牌資訊、快捷操作入口或業務相關的自定義內容。該插槽位於設計器主內容區域的頂部，在表單渲染之前顯示。


**使用場景：**

- **業務資訊**：顯示當前表單狀態、版本資訊、操作提示等
- **導航入口**：新增返回、幫助、設定等導航按鈕

## 使用場景示例 ​

### 表單資料儲存 ​

在企業應用中，通常需要在表單設計器中實現動態資料的儲存。例如，在一個OA系統中，使用者可以在設計器中編輯表單並儲存其設定。使用 `showSaveBtn` 配置，您可以在設計器中新增一個儲存按鈕，使用者點選後自動將表單資料提交到伺服器。

vue

```
<template>
    <fc-designer ref="designer" @save="handleSave" :config="config"></fc-designer>
</template>
<script>
    export default {
        name: 'app',
        data() {
            return {
                config: {
                    showSaveBtn: true
                },
            };
        },
        methods: {
            handleSave() {
                const formData = this.$refs.designer.getFormData();
                console.log('表单数据:', formData);
                // 将数据提交到服务器
                axios.post('/save', formData).then(response => {
                    console.log('保存成功:', response.data);
                }).catch(error => {
                    console.error('保存失败:', error);
                });
            }
        }
    };
</script>
```

### 表單選擇對話方塊 ​

在某些應用中，使用者可能需要在設計器中選擇預定義的表單模板。您可以透過插槽擴展自定義按鈕來實現表單選擇對話方塊

vue

```
<template>
    <fc-designer ref="designer">
        <template #handle>
            <el-button @click="openFormSelectDialog">选择表单</el-button>
        </template>
    </fc-designer>

    <!-- 表单选择对话框 -->
    <el-dialog title="选择表单" :visible.sync="dialogVisible">
        <el-select v-model="selectedForm" placeholder="请选择表单">
            <el-option
                v-for="form in formOptions"
                :key="form.value"
                :label="form.label"
                :value="form.value">
            </el-option>
        </el-select>
        <span slot="footer" class="dialog-footer">
            <el-button @click="dialogVisible = false">取消</el-button>
            <el-button type="primary" @click="confirmSelection">确定</el-button>
        </span>
    </el-dialog>
</template>

<script>
    export default {
        name: 'app',
        data() {
            return {
                dialogVisible: false,
                selectedForm: '',
                formOptions: [
                    { label: '表单模板 A', value: 'formA' },
                    { label: '表单模板 B', value: 'formB' }
                ]
            };
        },
        methods: {
            openFormSelectDialog() {
                this.dialogVisible = true;
            },
            confirmSelection() {
                console.log('选中的表单:', this.selectedForm);
                this.dialogVisible = false;
            }
        }
    };
</script>
```

透過這些方法，您可以根據業務需求自定義和擴展 `FcDesigner` 設計器的操作按鈕，使其更符合實際使用場景。
