# 常見問題 ​

本文件彙集了一些使用者在使用表單設計器時常遇到的問題，為您提供快速解決方案和使用建議。

## 頁面樣式錯亂/功能不正常 ​

**問題分析**：如果頁面樣式出現錯亂或按鈕點選沒有任何響應，通常是因為某些 UI 元件沒有正確掛載。

### 解決方法： ​

1. **檢視控制檯警告**：開啟瀏覽器的開發者工具，檢視控制檯輸出。如果出現黃色警告，如 `"[Vue warn]: Failed to resolve component el-button"`，則說明有元件未掛載。

![image](https://pro.form-create.com/doc/img/question1.png)

1. **掛載缺失元件**：在 `main.js` 中匯入並掛載缺失的元件。例如：

javascript

```
// 导入未挂载的组件
import { ElBadge } from 'element-plus';

const app = createApp(App);
// 挂载组件，有两种方式
app.use(ElBadge);
// 或者
app.component(ElBadge.name, ElBadge);
```

透過上述步驟，您可以有效解決 UI 元件未掛載導致的問題，從而恢復頁面的正常功能和樣式。

## 手動觸發表單提交 ​

有時候可能需要在事件中手動觸發表單提交，可按以下步驟操作：

### 解決方法： ​

透過呼叫 api.submit 方法可觸發表單提交（需透過表單驗證後才會執行）。

js

```
api.submit()
```

## 判斷元件當前是在設計器環境還是渲染器環境中被渲染 ​

### 解決方法： ​

透過在元件中定義`designer`判斷環境。

vue

```
<script>
    import {defineComponent} from 'vue';
    export default defineComponent({
        name: 'FieldList',
        inject: ['designer'],
        methods: {
            nodeClick(node) {
                 if(this.designer) {
                     //设计器环境
                 } else {
                     //渲染器环境
                 }
            }
        }
    });
</script>
```

## 在元件中獲取設計器當前選中的規則 ​

### 解決方法： ​

透過在元件中定義`designer`來獲取設計器的例項, 透過例項可以獲取到當前選中的規則和其他資訊。

vue

```
<template>
    <input type="text" v-model="title" @change="onChange">
</template>
<script>
    import {defineComponent} from 'vue';
    export default defineComponent({
        name: 'FieldList',
        inject: ['designer'],
        data() {
            return {
                title: ''
            }  
        },
        computed: {
            activeRule() {
                return this.designer.setupState.activeRule;
                //return this.designer.activeRule; (vue2)
            }  
        },
        methods: {
            onChange() {
                //直接修改当前选中的规则
                 this.activeRule.title = this.title;
            }
        }
    });
</script>
```

## 規則中的函式被編譯的問題 ​

**問題描述**：在表單規則中使用普通函式定義（如 `function (get, api) { ... }`）時，在專案打包時這些函式可能會被編譯或最佳化，導致儲存後的規則無法正常執行。

### 解決方法： ​

使用 `new Function()` 建構函式代替普通函式定義，這樣可以避免在專案打包時被編譯，確保函式能夠正確序列化和執行。

**不推薦的方式**（可能被編譯）：

js

```
function (get, api) {
    return get('$cookie.token') || 'default Token';
}
```

**推薦的方式**（使用 new Function）：

js

```
new Function('get', 'api', "return get('$cookie.token') || 'default Token'")
```

### 使用示例 ​

**1. 在全域變數中使用**：

js

```
designer.value.setGlobalVariable({
  "var_Fppdlz6gytmzb1c": {
    label: "token",
    deletable: false,
    handle: new Function('get', 'api', "return get('$cookie.token') || 'default Token'")
  }
});
```

**2. 在事件處理函式中使用**：

js

```
{
  type: 'button',
  field: 'submitBtn',
  title: '提交',
  on: {
    click: new Function('$inject', `
      const api = $inject.api;
      const formData = api.formData();
      console.log('表单数据:', formData);
    `)
  }
}
```

**3. 在計算屬性中使用**：

js

```
{
  type: 'input',
  field: 'total',
  title: '总价',
  value: new Function('get', 'api', `
    const quantity = get('$form.quantity') || 0;
    const price = get('$form.price') || 0;
    return quantity * price;
  `)
}
```

**4. 在資料解析函式中使用**：

js

```
designer.value.setGlobalData({
  "data_Fs1elui4kttlacc": {
    action: "http://192.168.1.4:8081/",
    method: "GET",
    label: "自定义接口数据",
    type: "fetch",
    parse: new Function('res', 'return res.data;')
  }
});
```

注意事項

1. **字串格式**：`new Function()` 的最後一個引數是函式體字串，注意字串的引號轉義
2. **引數順序**：`new Function()` 的引數順序是：引數1, 引數2, ..., 函式體
3. **程式碼可讀性**：雖然 `new Function()` 可以避免編譯問題，但程式碼可讀性較差，建議在註釋中說明函式功能
4. **錯誤處理**：在函式體字串中新增適當的錯誤處理邏輯

## 表單響應式失效/賦值失敗 ​

### 解決方法： ​

vue

```
<template>
    <form-create v-if="rule.length" v-model="formData" v-model:api="fApi" @submit="handleSubmit" :rule="rule"
                 :option="option"></form-create>
</template>
```

1. `rule`, `formData`, `options` 必須被 `ref` 包裹或定義在 `data` 物件中
2. 給 `<form-create/>` 標籤增加 `v-if="rule.length"` 確保透過最新的規則渲染表單
3. `rule`規則不能複用, 如果在彈窗中使用, 彈窗開啟時深複製一份原始規則出來使用
4. 透過 `api` 修改表單資料時, 將 `api.setValue` 方法包裹在 `nextTick` 方法中

## 表單/元件的事件沒有執行或者報錯 ​

### 解決方法： ​

必須使用 `formCreate.parseJson` 代替 `JSON.parse` 方法，使用 `formCreate.toJson` 代替 `JSON.stringify` 方法來轉換 JSON 資料，以確保資料格式正確。

## 表單的生命週期事件沒有觸發 ​

**問題描述**：在例項化表單時，您可能發現生命週期事件（如 `onMounted`、`onCreated`）沒有按預期觸發。

### 解決方法： ​

這個問題通常發生在 `options` 賦值時，`FormCreate` 元件已完成初始化。為了解決這一問題，可以採用以下策略：

**使用 `v-if` 控制渲染**：給 `form-create` 標籤增加 `v-if` 條件，確保在所有資料準備好之後再渲染表單。

例如：

html

```
<form-create v-if="isReady" :rule="rule" :option="option"></form-create>
```

設定 `isReady` 為 `false`，確保在配置完成並準備好後，將其設定為 `true`。這樣可以確保表單元件在適當的時機再次進行初始化，並正確觸發生命週期事件。

## 表格表單中操作同一行的資料 ​

**問題背景**：在複雜表單中，可能需要操作同一行的表單資料，比如更新或獲取單行資料。此時，我們可以使用 FormCreate 提供的方法來實現。

### 解決方法： ​

FormCreate 提供了一系列方法來方便地獲取和修改同一行的元件規則和資料：

- **`getParentSubRule`**：獲取父規則的子元件規則。
- **`getChildrenRuleList`**：獲取子元件規則列表。
- **`getChildrenFormData`**：獲取子元件的表單資料。
- **`setChildrenFormData`**：設定子元件的表單資料。
