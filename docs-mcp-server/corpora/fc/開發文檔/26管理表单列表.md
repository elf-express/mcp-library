# 管理表單列表 ​

在表單設計器中，您可以透過 `props.list` 配置來管理左側的表單列表。表單列表功能使使用者能夠高效地載入和切換不同的表單模板或業務表單。

![管理表单列表](https://pro.form-create.com/doc/img/book/image-20241016214139856.png)

## 資料結構 ​

`FormList` 定義了欄位列表的結構：

ts

```
type FormItem = {
    //表单名称
    label: string;
    //是否禁用
    disabled?: boolean;
    //表单生成规则
    rule?: string | Rule[];
    //表单配置
    options?: string | Options;
    //加载表单规则
    load?: (item: FormItem) => ({
        rule?: string | Rule[];
        options?: string | Options;
    } | Promise<{
        rule?: string | Rule[];
        options?: string | Options;
    }>);
    //子表单
    children?: FormItem[];
}

type FormList = FormItem[];
```

**注意: rule 的優先順序高於 load，如果同時定義，使用 rule 生成表單。**

## 切換表單事件 ​

切換表單時自動觸發 switchForm 事件，可監聽到當前切換的表單變化。

html

```
<template>
    <fc-designer ref="designer" @switchForm="handleSwitch"></fc-designer>
</template>
<script setup>
    function handleSwitch(formItem) {
        //todo
    }
</script>
```

## 自定義表單列表區域 ​

透過`form-list`插槽可以實現對錶單列表區域的自定義

html

```
<template>
    <fc-designer ref="designer" :list="list">
        <template #form-list>
            自定义表单列表区域内容
        </template>
    </fc-designer>
</template>
```

## 使用場景 ​

- **載入表單:**

方便地載入預設的表單模板或業務表單，為不同應用場景快速提供表單基礎，減少設計時間。

- **切換表單:**

支援在多個表單模板或業務表單之間快速切換，提升設計效率和靈活性，使用者可以根據具體需求迅速調整表單結構。

- **動態配置:**

利用 load 方法，可以動態載入表單規則和配置，適應不同的業務場景和使用者需求。

- **模組化設計:**

將表單分為多個子表單，使得設計更加清晰、結構化，便於後續的維護和擴展。
