## 事件 ​

元件事件列表及詳細說明：

| 事件名稱 | 描述 | 引數說明 |
| --- | --- | --- |
| active | 元件被選中時觸發。 | rule: Rule - 選中元件的規則物件 |
| create | 建立新元件時觸發。 | rule: Rule - 新建立的元件規則物件 |
| copy | 元件被複制時觸發。 | rule: Rule - 被複制的元件規則物件 |
| delete | 元件被刪除時觸發。 | rule: Rule - 被刪除的元件規則物件 |
| drag | 拖拽新元件到設計器時觸發。 | e: {item: Object, dragRule: Object, rule: Rule} - 包含拖拽規則和元件資訊 |
| inputData | 錄入資料模式下儲存資料時觸發。檢視預設值。 | formData: Object - 當前錄入的表單資料 |
| inputPageData | 彈窗中儲存錄入資料時觸發。 | formData: Object - 彈窗中錄入的表單資料 |
| save | 點選儲存按鈕時觸發。檢視儲存JSON規則。 | data: {rule: string, options: string} - 包含當前表單規則和配置 |
| clear | 設計表單被清空時觸發。 | - |
| changeDevice | 修改區域顯示方式時觸發。檢視移動端自適應。 | - |
| switchForm | 切換表單時觸發。檢視管理表單列表。 | - |
| copyRule | 複製規則時觸發。 | e: {event: Event, rule: Rule} - 包含事件物件和被複制的規則物件 |
| pasteRule | 貼上規則時觸發。 | e: {event: Event, copyRule: Rule} - 包含事件物件和被貼上的規則物件 |
| sortUp | 元件上移時觸發。 | e: {event: Event, rule: Rule} - 包含事件物件和移動的規則物件 |
| sortDown | 元件下移時觸發。 | e: {event: Event, rule: Rule} - 包含事件物件和移動的規則物件 |
| previewSubmit | 預覽彈窗中提交表單時觸發。 | formData: Object - 提交的表單資料api: Object - 表單API物件 |
| previewReset | 預覽彈窗中重置表單時觸發。 | api: Object - 表單API物件 |
| changeField | 修改元件的欄位 ID 時觸發。檢視管理欄位ID。 | e: {field: string, oldField: string, rule: Rule} - 包含新欄位ID、舊欄位ID和元件規則 |
| saveField | 儲存欄位時觸發。檢視儲存欄位。 | field: string - 儲存的欄位ID |

## 示例 ​

### 監聽元件 ID 變化 ​

透過監聽 changeField 事件，實時獲取表單元件ID（field）的變更資訊，包括新ID、舊ID和元件規則，以便開發者進行相應的邏輯處理。

vue

```
<template>
    <fc-designer @changeField="handle"></fc-designer>
</template>
<script>
    export default {
        name: 'Component',
        methods: {
            handle({field, oldField, rule}) {
                // TODO
            }
        }
    };
</script>
```

### 監聽預覽彈窗中表單提交 ​

透過監聽 previewSubmit 事件，可以捕獲在表單設計器的預覽彈窗中提交表單時所產生的資料，從而實現對錶單資料的儲存或其他自定義處理邏輯。

vue

```
<template>
    <fc-designer @previewSubmit="handle"></fc-designer>
</template>
<script>
    export default {
        name: 'Component',
        methods: {
            handle(formData, api) {
                console.log('预览提交数据:', formData);
                console.log('表单API:', api);
                // 保存数据
            }
        }
    };
</script>
```

### 監聽元件建立事件 ​

透過監聽 create 事件，可以在使用者建立新元件時執行自定義邏輯。

vue

```
<template>
    <fc-designer @create="handleCreate"></fc-designer>
</template>
<script>
    export default {
        name: 'Component',
        methods: {
            handleCreate(rule) {
                console.log('创建新组件:', rule);
                // 可以在这里添加自定义逻辑，比如记录日志、发送统计等
            }
        }
    };
</script>
```

### 監聽元件拖拽事件 ​

透過監聽 drag 事件，可以在使用者拖拽元件到設計器時執行自定義邏輯。

vue

```
<template>
    <fc-designer @drag="handleDrag"></fc-designer>
</template>
<script>
    export default {
        name: 'Component',
        methods: {
            handleDrag(e) {
                console.log('拖拽的组件:', e.item);
                console.log('拖拽规则:', e.dragRule);
                console.log('规则对象:', e.rule);
                // 可以在这里执行自定义逻辑
            }
        }
    };
</script>
```

### 監聽元件排序事件 ​

透過監聽 sortUp 和 sortDown 事件，可以在使用者調整元件順序時執行自定義邏輯。

vue

```
<template>
    <fc-designer @sortUp="handleSortUp" @sortDown="handleSortDown"></fc-designer>
</template>
<script>
    export default {
        name: 'Component',
        methods: {
            handleSortUp(e) {
                console.log('组件上移:', e.rule);
                // 处理上移逻辑
            },
            handleSortDown(e) {
                console.log('组件下移:', e.rule);
                // 处理下移逻辑
            }
        }
    };
</script>
```

### 監聽複製和貼上事件 ​

透過監聽 copyRule 和 pasteRule 事件，可以在使用者複製和貼上元件時執行自定義邏輯。

vue

```
<template>
    <fc-designer @copyRule="handleCopy" @pasteRule="handlePaste"></fc-designer>
</template>
<script>
    export default {
        name: 'Component',
        methods: {
            handleCopy(e) {
                console.log('复制规则:', e.rule);
                // 处理复制逻辑
            },
            handlePaste(e) {
                console.log('粘贴规则:', e.copyRule);
                // 处理粘贴逻辑
            }
        }
    };
</script>
```

## 型別定義 ​

**展開**typescript

```
// 组件操作相关事件
type Active = (rule: Rule) => void;
type Create = (rule: Rule) => void;
type Copy = (rule: Rule) => void;
type Delete = (rule: Rule) => void;
type Drag = (e: { item: Object, dragRule: Object, rule: Rule }) => void;

// 数据操作相关事件
type InputData = (formData: Record<string, Object>) => void;
type InputPageData = (formData: Record<string, Object>) => void;
type Save = (data: { rule: string, options: string }) => void;
type Clear = () => void;

// 界面操作相关事件
type ChangeDevice = () => void;
type SwitchForm = () => void;

// 规则操作相关事件
type CopyRule = (e: { event: Event, rule: Rule }) => void;
type PasteRule = (e: { event: Event, copyRule: Rule }) => void;
type SortUp = (e: { event: Event, rule: Rule }) => void;
type SortDown = (e: { event: Event, rule: Rule }) => void;

// 字段操作相关事件
type ChangeField = (e: { field: string, oldField: string, rule: Rule }) => void;
type SaveField = (field: string) => void;

// 预览相关事件
type PreviewSubmit = (formData: Record<string, Object>, api: Object) => void;
type PreviewReset = (api: Object) => void;
```
