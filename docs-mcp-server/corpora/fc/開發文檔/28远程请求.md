# 遠端請求 ​

在表單設計器中，有時我們需要在發起遠端請求之前動態修改請求的引數，例如新增認證 Token 或使用者 ID。透過 `beforeFetch` 事件和變數，我們可以實現這一需求。

![request.png](https://pro.form-create.com/doc/img/request.png)

## 資料結構 ​

ts

```
type FetchConfig = {
    //请求地址
    action: String;
    //请求方式
    method?: 'GET' | 'POST';
    //GET 参数
    query?: Object;
    //发送请求时携带的数据
    data?: Object;
    //携带数据的发送方式
    dataType?: 'json' | 'formData';
    //发送请求的请求头
    headers?: Object;
    //后置数据数据回调
    parse?: Function;
    //请求成功后的回调
    onSuccess: (body: any) => void
    //请求失败后的回调
    onError?: (e: Error | ProgressEvent) => void;
}
```

注意

- **資料格式**：dataType 為 'json' 時，data 會被 JSON.stringify() 序列化
- **請求頭**：headers 中的 Content-Type 會根據 dataType 自動設定
- **非同步處理**：所有 fetch 操作都是非同步的，返回 Promise 物件

## 傳送GET請求 ​

傳送 GET 請求示例，透過 name 引數獲取對應 Select 元件的配置資料。

js

```
const formData = api.formData();
api.fetch({
    action: '/api/getdata',
    query: {
        name: formData.name
    }
}).then(res=>{
    api.getRule('select').props.options = res.data; 
});
```

GET 請求注意事項

- **引數編碼**：query 引數會自動進行 URL 編碼

### GET 請求 ​

js

```
api.fetch({
    action: '/api/getdata',
    query: {
        name: formData.name,
    }
}).then(res => {
    api.getRule('select').props.options = res.data;
}).catch(error => {
    console.error('数据获取失败:', error);
});
```

## 傳送POST請求 ​

傳送 POST 請求示例，提交表單資料。

js

```
const formData = api.formData();
//外部变量
const token = api.getData('globalToken');
api.fetch({
    action: '/api/submit',
    method: 'POST',
    dataType: 'json',
    data: formData,
    headers: {
        token: token
    }
}).then(res=>{
    // 表单提交成功
});
```

POST 請求注意事項

- **資料格式**：dataType 為 'json' 時，data 會被 JSON.stringify() 序列化
- **Content-Type**：dataType 為 'json' 時，會自動設定 Content-Type 為 'application/json'

### 表單資料提交 ​

js

```
api.fetch({
    action: '/api/submit',
    method: 'POST',
    dataType: 'json',
    data: formData,
    headers: {
        'Content-Type': 'application/json',
        'X-CSRF-Token': getCsrfToken()
    },
}).then(res => {
    console.log('提交成功:', res);
}).catch(error => {
    console.error('提交失败:', error);
});
```

## 在請求中使用變數 ​

在配置請求時，可以使用雙大括號 {{ }} 來引用變數。支援在連結、請求頭和資料中使用變數。例如，{{token}} 會在請求時被替換為變數 token 的值。如果變數的值是物件，可以透過 {{variableName.attributeName}} 訪問物件中的屬性值。

![fetch.png](https://pro.form-create.com/doc/img/fetch.png)

變數使用注意事項

- **變數存在性**：確保引用的變數存在，否則會被替換為空字串
- **巢狀訪問**：使用點號訪問物件屬性時，確保物件結構正確

### 變數作用域 ​

js

```
// 支持以下变量作用域：
// $form - 表单数据
// $cookie - Cookie 数据  
// $localStorage - 本地存储数据
// $sessionStorage - 会话存储数据
// $global - 全局变量
// $data - 外部传入数据
```

**示例**

假設我們有一個遠端 API 請求需要動態插入使用者的 Token 和 ID，我們可以在配置中使用雙大括號 {{ }} 語法來實現這一點。

**請求 URL 示例(GET 方式)**

如果您的請求 URL 中需要插入動態變數，例如使用者 ID，可以使用雙大括號語法來指定變數：

js

```
api.fetch({
    action: 'https://api.example.com/user/{{$form.userId}}/details',
    method: 'GET',
    headers: {
        'Authorization': 'Bearer {{$cookie.token}}'
    },
    data: {},
}).then(res=>{
    //todo
});
```

在實際請求執行時， {{userId}} 和 {{token}} 會被替換為實際的變數值。

**請求資料示例(POST 方式)**

請求的資料部分同樣可以使用變數進行動態設定：

js

```
api.fetch({
    action: 'https://api.example.com/update',
    method: 'POST',
    dataType: 'json',
    headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer {{$cookie.token}}'
    },
    data: {
        name: '{{$form.userName}}',
        age: '{{$form.userAge}}'
    },
}).then(res=>{
    //todo
});
```

**變數在請求中的替換**

在實際請求過程中，設計器會自動替換雙大括號中的變數值。例如：

- {{$cookie.token}} 替換為cookie中的token欄位
- {{$form.userId}} 替換為表單中的userId欄位
- {{$form.userName}} 替換為表單中的userName欄位
- {{$form.userAge}} 替換為表單中的userAge欄位
- **支援使用 {{變數名}} 的方式直接讀取內建變數、全域變數和外部傳入的資料，具體定義方法請查閱[變數使用文件](https://pro.form-create.com/doc/variable)**

## 透過前置事件修改請求 ​

`beforeFetch` 事件是在傳送請求之前觸發的回撥函式。您可以在這個回撥中對請求引數進行修改，比如增加請求頭、修改請求 URL、新增查詢引數等。

重要說明

`beforeFetch` 事件中獲取的是最終請求資料，不再支援模板變數解析。此時變數已經被替換為實際值。

beforeFetch 事件注意事項

- **執行時機**：在請求傳送前最後執行，變數已被替換
- **引數修改**：可以直接修改 config 物件的屬性
- **返回值**：函式不需要返回值，直接修改 config 即可
- **錯誤處理**：在 beforeFetch 中丟擲錯誤會阻止請求傳送

**1. 開啟表單事件配置彈窗**

進入表單設計器，點選配置面板中的“事件”選項，開啟事件配置彈窗。

![beforeFetch1.png](https://pro.form-create.com/doc/img/beforeFetch1.png)

**2. 配置beforeFetch事件**

在事件配置彈窗中，選擇 beforeFetch 事件，並輸入：

![beforeFetch2.png](https://pro.form-create.com/doc/img/beforeFetch2.png)

**3. 修改引數**

![beforeFetch3.png](https://pro.form-create.com/doc/img/beforeFetch3.png)

例如，增加 token 到請求頭：

js

```
function getCookie(name) {
    name = name + '=';
    const decodedCookie = decodeURIComponent(document.cookie);
    const cookieArray = decodedCookie.split(';');
    for (let i = 0; i < cookieArray.length; i++) {
        let cookie = cookieArray[i];
        while (cookie.charAt(0) === ' ') {
            cookie = cookie.substring(1);
        }
        if (cookie.indexOf(name) === 0) {
            cookie = cookie.substring(name.length, cookie.length);
            try {
                return JSON.parse(cookie);
            } catch (e) {
                return cookie;
            }
        }
    }
    return null;
}
config.headers.token = getCookie('user-token');
```

例如，新增使用者 ID 到請求 URL：

js

```
function getLocalStorage(name) {
    const value = localStorage.getItem(name);
    if (value) {
        try {
            return JSON.parse(value);
        } catch (e) {
            return value;
        }
    }
    return null;
}
config.action += ('?uid=' + getLocalStorage('user_id'));
```

例如，新增自定義 token 變數到 headers 中：

js

```
config.headers.token += data.api.getData('token');
```

## 請求後置回撥 ​

`parse` 是一個後置請求回撥函式，用於在請求完成後處理返回的資料。您可以在 `parse` 函式中對資料進行格式化、轉換或其他處理操作，以便在元件中使用。

![fetch2.png](https://pro.form-create.com/doc/img/fetch2.png)

**示例：解析和處理請求結果**

假設您有一個 API 介面返回的資料格式如下：

json

```
{
    "status": "success",
    "data": {
        "items": [
            { "id": 1, "name": "Item 1" },
            { "id": 2, "name": "Item 2" }
        ],
        "total": 2
    }
}
```

您可以使用 `parse` 函式來處理這些資料。例如，您可能想要提取 `data.items` 並格式化成另一種結構。

**示例程式碼**

以下是如何使用 `parse` 函式來處理 API 請求返回的資料，並將其格式化為一個包含專案名稱的陣列：

js

```
function parse (data) {
    if (data.status === 'success') {
        // 提取并格式化数据
        return data.data.items.map(item => ({
            id: item.id,
            name: item.name
        }));
    } else {
        throw new Error('请求失败');
    }
}
</script>
```

## 重寫內建的請求方法 ​

可以透過重寫 formCreate 的 fetch 方法來修改內建傳送請求的方法，從而動態修改請求引數。

注意

檔案上傳操作同樣會呼叫 fetch 方法，需注意其特殊適配要求，具體實現可參考[內建請求模組](https://github.com/xaboy/form-create/blob/next/packages/core/src/frame/fetch.js#L28)。

js

```
import FcDesigner from '/path/to/fcDesignerPro/dist/index.es.js'
const globalFetch = FcDesigner.designerForm.fetch;

FcDesigner.designerForm.fetch = FcDesigner.formCreate.fetch = (config) => {
    config.headers.token = getCookie('user-token');
    return globalFetch(config);
}
```
