## 速查表

| 需求 | 語法 |
|---|---|
| 根因 | 99% 為執行緒不安全(await 漏寫 / Task.WhenAll / Job) |
| Client 模式 | 每次 new,IsAutoCloseConnection=true |
| 取執行緒安全副本 | db.CopyNew() |
| Scope 模式 | 同上下文自動安全,適合 Web API |
| 排查1 | 每個 async 呼叫都要 await |

---

## 核心原則

偶發性錯誤（偶發報錯、偶發回傳錯誤資料）**99% 都是執行緒不安全造成的**，
主要發生在：非同步 await 漏寫、`Task.WhenAll`、定時任務 Job、`List.ForEach` 中使用非同步。

---

## 一、兩種模式比較

| 模式 | 執行緒安全 | 效能 | 適合場景 |
|---|---|---|---|
| SqlSugarClient | ❌ 本身不安全，需每次 new | 🚀 超高（等同 Dapper/EF Core）| 有經驗、每個請求手動 new |
| SqlSugarScope | ✅ 同一上下文自動安全 | ⚡ 中上 | 一般 Web API，IOC 單例注入 |

## 二、SqlSugarClient 模式注意事項

```csharp
// ✅ 正確：每次請求都 new 一個新實例
public class OrderService
{
    public List<Order> GetList()
    {
        var db = new SqlSugarClient(new ConnectionConfig
        {
            DbType                = DbType.SqlServer,
            ConnectionString      = "...",
            IsAutoCloseConnection = true  // ⚠️ 必須設定 true
        });
        return db.Queryable<Order>().ToList();
    }
}

// ✅ 或用 db.CopyNew() 取得執行緒安全的副本
var newDb = db.CopyNew();
await newDb.Queryable<Order>().ToListAsync();

// ❌ 錯誤：SqlSugarClient 存放在靜態變數（執行緒不安全）
public static SqlSugarClient Db = new SqlSugarClient(...);  // 危險！
```

## 三、SqlSugarScope 排查三步驟（100% 覆蓋所有場景）

### 排查 1：await 漏寫

```csharp
// ❌ 問題：await 漏寫，導致上下文混亂
public async Task Process()
{
    var list = _db.Queryable<Order>().ToListAsync();  // 漏了 await！
    _db.Insertable(new Order()).ExecuteCommandAsync();  // 漏了 await！
}

// ✅ 修正：每個非同步呼叫都要加 await
public async Task Process()
{
    var list = await _db.Queryable<Order>().ToListAsync();
    await _db.Insertable(new Order()).ExecuteCommandAsync();
}
```

### 排查 2：async void / Action 不支援 await

```csharp
// ❌ 問題：List.ForEach 的 Action 不受 await 等待
var orders = new List<Order> { ... };
orders.ForEach(async o =>
{
    await _db.Insertable(o).ExecuteCommandAsync();  // 不受外層 await 控制！
});

// ✅ 修正：改用普通 foreach
foreach (var o in orders)
{
    await _db.Insertable(o).ExecuteCommandAsync();
}

// ❌ 問題：async void 不受 await 等待
public async void DoWork() { ... }          // 危險！

// ✅ 修正：返回 Task
public async Task DoWork() { ... }
```

### 排查 3：Task.WhenAll / 第三方 Job / BackgroundService → 用 CopyNew()

```csharp
// ❌ 問題：Task.WhenAll 並行執行，共用 db 不安全
await Task.WhenAll(
    _db.Queryable<Order>().ToListAsync(),
    _db.Queryable<Customer>().ToListAsync()  // 並發使用同一個 db！
);

// ✅ 修正：各自用 CopyNew()
await Task.WhenAll(
    _db.CopyNew().Queryable<Order>().ToListAsync(),
    _db.CopyNew().Queryable<Customer>().ToListAsync()
);

// ✅ BackgroundService / Hangfire / Quartz 定時任務
public class MyJob : BackgroundService
{
    private readonly ISqlSugarClient _db;

    public MyJob(ISqlSugarClient db) { _db = db; }

    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        while (!ct.IsCancellationRequested)
        {
            // Job 中一定要用 CopyNew()
            var newDb = _db.CopyNew();
            var list  = await newDb.Queryable<Order>()
                .Where(it => it.Status == 0)
                .ToListAsync();

            foreach (var order in list)
            {
                await newDb.Updateable(order)
                    .SetColumns(it => it.Status == 1)
                    .ExecuteCommandAsync();
            }

            await Task.Delay(5000, ct);
        }
    }
}
```

## 四、CopyNew() 詳細用法

```csharp
// 基本用法（在當前業務方法中使用，不要賦值給全域變數）
await db.CopyNew().Queryable<Order>().ToListAsync();

// ❌ 錯誤：賦值給全域或公用方法（破壞執行緒安全）
base.Context = db.CopyNew();  // 錯誤！

// ✅ 事務中的正確用法（宣告局部變數）
var newDb = db.CopyNew();
newDb.BeginTran();
try
{
    await newDb.Insertable(order).ExecuteCommandAsync();
    await newDb.InsertableRange(items).ExecuteCommandAsync();
    newDb.CommitTran();
}
catch
{
    newDb.RollbackTran();
    throw;
}

// ✅ 倉儲中使用（5.1.4.78+）
rep.CopyNew().Insert(entity);                    // ORM 內建方法
rep.CopyNew<MyRepository<Order>>().MyMethod(p);  // 自訂倉儲方法

// ✅ 多租戶 + CopyNew（5.1.4.106+）
var newDb = db.CopyNew();
newDb.GetConnection("log").Insertable(log).ExecuteCommand();  // ✅
// newDb.GetConnectionScope("log")  ← ❌ 不要用 Scope 版本
```

## 五、AsyncLock 非同步鎖（特殊場景）

適合 CopyNew() 不方便修改時，點對點保證執行緒安全。

```csharp
// 同一請求內確保不並發（建議使用場景 ≤ 5 個地方）
using (await db.AsyncLock())  // 預設 30 秒逾時
{
    // 區塊內的 await 不能少
    await _rep.InsertAsync(order);
    await _rep.InsertAsync(orderItem);
}
```

⚠️ 注意：AsyncLock 是同一個實例，使用地方不能太多（建議 5 個以下），只用於特殊無法改動的情況。

## 六、決策流程

```
遇到偶發性錯誤？
    │
    ▼
是否在 Task.WhenAll / ParallelAsync / 定時 Job 中？
    → 是 → 用 db.CopyNew()
    │
    ▼
是否有 async void 或 List.ForEach(async ...) ?
    → 是 → 改為 async Task + 普通 foreach
    │
    ▼
是否有任何 await 漏寫？
    → 是 → 補上所有 await
    │
    ▼
以上都沒有，但還是報錯？
    → 用 db.CopyNew() 暫時解決，再慢慢排查根本原因
```

## 七、CopyNew 使用限制

| 情況 | 說明 |
|---|---|
| ✅ 適合使用 | Task.WhenAll / 定時 Job / BackgroundService / 個別方法偶發報錯 |
| ❌ 不適合使用 | 賦值給全域變數或公用方法（會破壞事務） |
| ⚠️ 多租戶 | CopyNew 後用 GetConnection()，不要用 GetConnectionScope() |
| ⚠️ 事務 | CopyNew 後的 newDb 要在當前方法內管理事務，不要跨方法 |

---

## 注意事項

- `async void` 和 `Action<T>` 中的非同步不受外層 `await` 控制，一律改為 `async Task`
- `List.ForEach(async ...)` 是 `Action`，不受 await 控制，改用 `foreach`
- `CopyNew()` 要用在當前業務方法裡，禁止寫到公用方法或賦值給靜態/全域變數
- `SqlSugarClient` 模式必須設定 `IsAutoCloseConnection = true`
- 偶發性錯誤 99% 都是上述問題，按三個排查步驟依序確認即可
