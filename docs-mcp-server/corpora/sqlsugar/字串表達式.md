## 速查表

| 需求 | 語法 |
|---|---|
| 啟動設定 | StaticConfig.DynamicExpressionParserType=...(啟動時設一次) |
| 核心 | 用 FormattableString 構造條件 |
| Where | Where("it", $"it.Name==\"a\"") |
| Select | Select("it", "...") |
| 聯表(動態建類) | QueryableByObject + 字串表達式 |
| 函數 | SqlFunc.Exists 等需註冊型別 Provider |

---

## 功能說明

字串表達式讓你在 Where / Select 中使用**字串形式的 Lambda**，最大優點是可以存取導航屬性，
彌補無實體多庫查詢 API（ObjectFuncModel）無法直接用導航屬性的缺陷。
兩者互補，1 + 1 > 2。

| 情境 | 推薦方式 |
|---|---|
| 需要導航屬性過濾 / 選取 | 字串表達式（本篇）|
| 不需要導航、多庫通用條件構建 | ObjectFuncModel（無實體多庫查詢）|
| 動態建類 + 導航屬性過濾 | 兩者合用 |

## 啟動設定（程式啟動時執行一次）

```csharp
// Program.cs / Startup.cs（5.1.4.106+）
StaticConfig.DynamicExpressionParserType = typeof(DynamicExpressionParser);

// 若需要在字串表達式中使用 SqlFunc，需擴展 TypeProvider
StaticConfig.DynamicExpressionParsingConfig = new ParsingConfig
{
    CustomTypeProvider = new SqlSugarTypeProvider()
};

// SqlSugarTypeProvider 實作
public class SqlSugarTypeProvider : DefaultDynamicLinqCustomTypeProvider
{
    public override HashSet<Type> GetCustomTypes()
    {
        var types = base.GetCustomTypes();
        types.Add(typeof(SqlFunc));  // 加入 SqlFunc 支援
        return types;
    }
}
```

注意：SqlFunc 擴展支援一般查詢，但**不支援子查詢（Subqueryable）**。

## 一、FormattableString 構造（核心概念）

所有字串表達式都是 `FormattableString` 型別，有三種寫法：

```csharp
// 硬編碼（靜態條件）
FormattableString expr1 = $"it.Id > 1";

// 固定欄位 + 參數化變數（防 SQL Injection）
int threshold = 100;
FormattableString expr2 = FormattableStringFactory.Create("it.Amount > {0}", threshold);
// → WHERE Amount > @p0  （參數化，安全）

// 動態欄位 + 參數化變數
string fieldName = "it.Amount";
FormattableString expr3 = FormattableStringFactory.Create(fieldName + " > {0}", threshold);
// → WHERE Amount > @p0

// 多個變數
string name = "Jack";
int status = 1;
FormattableString expr4 = FormattableStringFactory.Create(
    "it.Name == {0} && it.Status == {1}", name, status);
```

## 二、Where 條件

```csharp
// 單表必須加 "it" 前綴（避免快取錯誤）
var list = db.Queryable<Order>()
    .Where("it", $"it.Status == 1")
    .ToList();

// 導航屬性過濾（這是字串表達式最核心的優勢）
var list2 = db.Queryable<Student>()
    .Where("it", $"it.School.Name == {"北大"}")
    .ToList();
// → WHERE School.Name = @p0  （自動 JOIN）

// 使用 SqlFunc
var list3 = db.Queryable<Student>()
    .Where("it", $"SqlFunc.Exists(it.Address.Id)")
    .ToList();

// 導航屬性集合過濾
var list4 = db.Queryable<Student>()
    .Where("it", $"it.Books.Any()")
    .ToList();
// 有任何書籍的學生

// 動態建類 + 導航過濾（QueryableByObject）
var list5 = db.QueryableByObject(typeof(Student))
    .Where("it", $"it.Address.Id == {1}")
    .ToList();

// 動態欄位名 + 參數化
string field = "it.Amount";
decimal min = 100m;
FormattableString cond = FormattableStringFactory.Create(field + " > {0}", min);
var list6 = db.Queryable<Order>().Where("it", cond).ToList();
```

## 三、Select 查詢

```csharp
// 單一欄位查詢（返回基本型別）
List<int> ids = db.Queryable<Order>()
    .Select<int>("it", $"it.Id", typeof(Order), typeof(int))
    .ToList();

// 多欄位查詢（返回匿名 / 動態物件）
var list = db.Queryable<Order>()
    .Select("it", $"new(it.Id as Id, it.Name as Name, it.Amount as Amount)", typeof(Order))
    .ToList();

// 含導航屬性的 Select
var list2 = db.Queryable<Student>()
    .Select("it", $"new(it.Name as Name, it.School.Name as SchoolName)", typeof(Student))
    .ToList();
```

## 四、聯表查詢（結合動態建類）

聯表時 Where / Select 中**不要加 `=>` 箭頭語法**。

```csharp
// 建立多表短名稱對應
var shortNames = DynamicParameters.Create(
    "x",  typeof(Order),
    "u",  typeof(OrderItem),
    "u2", typeof(OrderItem));

var list = db.QueryableByObject(typeof(Order), "x")
    .AddJoinInfo(
        typeof(OrderItem),
        DynamicParameters.Create("x", typeof(Order), "u", typeof(OrderItem)),
        $"x.Id == u.OrderId",
        JoinType.Left)
    .AddJoinInfo(
        typeof(OrderItem),
        DynamicParameters.Create("x", typeof(Order), "u", typeof(OrderItem), "u2", typeof(OrderItem)),
        $"x.Id == u2.OrderId",
        JoinType.Left)
    .Where(shortNames, $"x.Id == u.OrderId")
    .Select(shortNames,
        $"new(x.Name as Name, u.OrderId as OrderId)",
        typeof(ViewOrder))
    .ToList();

// 注意：聯表不要使用 it=> 箭頭語法
```

## 五、與無實體多庫查詢的組合

```csharp
// Where：字串表達式（用於導航屬性）
// Select：ObjectFuncModel（用於函數計算）
var selector = new List<SelectModel>
{
    new SelectModel { FiledName = "Name", AsName = "Name" },
    new SelectModel
    {
        FiledName = ObjectFuncModel.Create("AggregateCount", "Id"),
        AsName    = "OrderCount"
    }
};

var list = db.QueryableByObject(typeof(Order), "it")
    .Where("it", $"it.Customer.Status == {1}")  // 字串表達式：導航過濾
    .Select<object>(selector)                    // ObjectFuncModel：聚合計算
    .ToList();
```

## 注意事項

- 單表查詢必須指定別名 `"it"`，否則快取可能出錯：`.Where("it", $"it.xxx")`
- 聯表查詢中的條件和選取**不要加 `=>` 箭頭**語法
- 字串插值 `{變數}` 會自動參數化，防 SQL Injection
- SqlFunc 擴展需要設定 SqlSugarTypeProvider，且**不支援 Subqueryable 子查詢**
- `FormattableStringFactory.Create` 用於動態組合欄位名稱 + 參數值
- 此功能需要安裝 `System.Linq.Dynamic.Core` NuGet 套件
