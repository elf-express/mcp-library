## 速查表

| 方式 | 語法 | 多庫相容 | AOP |
|---|---|---|---|
| 字典更新 | db.Updateable(dc).AS("表名").WhereColumns("id") | ✅ | ❌ |
| 批量字典更新 | db.Updateable(dcList).AS("表名").WhereColumns("id") | ✅ | ❌ |
| 匿名物件更新 | db.UpdateableByDynamic(new {...}).AS("表名").WhereColumns("id") | ✅ | ❌ |
| ExpandoObject 更新 | db.Updateable(new Dictionary\<string,object\>(ex)).AS("表名") | ✅ | ❌ |
| SQL 字串更新 | db.Updateable\<object\>().AS("表名").SetColumns(...).Where("sql") | ⚠️ 較差 | ❌ |
| DataTable BulkUpdate | db.Fastest\<DataTable\>().AS("表名").BulkUpdate(dt, new[]{"id"}) | ✅ | ❌ |
| 動態建類更新 | db.UpdateableByObject(value).ExecuteCommand() | ✅✅ 最佳 | ✅ |
| 表達式字串更新 | db.UpdateableByObject(typeof(T)).SetColumns(...).Where(...) | ✅ | ❌ |

## 一、字典更新（最常用）

```csharp
// 單筆字典更新（WhereColumns 指定條件欄位，條件欄位不會被更新）
var dc = new Dictionary<string, object>
{
    ["Id"]         = 1,
    ["Name"]       = "新名稱",
    ["Amount"]     = 888m,
    ["UpdateTime"] = DateTime.Now
};
db.Updateable(dc).AS("Order").WhereColumns("Id").ExecuteCommand();
// → UPDATE Order SET Name=@Name, Amount=@Amount, UpdateTime=@UpdateTime WHERE Id=@Id

// 批量字典更新
var dcList = new List<Dictionary<string, object>> { dc1, dc2 };
db.Updateable(dcList).AS("Order").WhereColumns("Id").ExecuteCommand();

// 多個條件欄位
db.Updateable(dc).AS("Order").WhereColumns("Id", "OrgId").ExecuteCommand();
```

**✅ 實際案例：低程式碼平台通用編輯 API**

```csharp
[HttpPut("{tableName}/update")]
public async Task<ApiResult> UpdateRow(string tableName, [FromBody] Dictionary<string, object> row)
{
    if (!_allowedTables.Contains(tableName))
        return ApiResult.Fail("不允許的資料表");

    row["UpdateTime"] = DateTime.Now;
    row["UpdateUser"] = CurrentUserId;

    int rows = await db.Updateable(row).AS(tableName)
        .WhereColumns("Id")
        .ExecuteCommandAsync();

    return rows > 0 ? ApiResult.Success() : ApiResult.Fail("找不到資料");
}
```

## 二、匿名物件更新

```csharp
// 單筆（不支援批量）
db.UpdateableByDynamic(new { Id = 1, Name = "新名稱", Amount = 100m })
    .AS("Order")
    .WhereColumns("Id")
    .ExecuteCommand();
// → UPDATE Order SET Name=@Name, Amount=@Amount WHERE Id=@Id

// 注意：不支援批量，批量請改用字典集合
```

## 三、ExpandoObject（動態物件）更新

```csharp
ExpandoObject ex  = new ExpandoObject();
var           dic = (IDictionary<string, object>)ex;
dic["Id"]     = 1;
dic["Name"]   = "更新名稱";
dic["Amount"] = 300m;

db.Updateable(new Dictionary<string, object>(ex))
    .AS("Order")
    .WhereColumns("Id")
    .ExecuteCommand();
```

## 四、SQL 字串方式更新

多庫相容性較差，只適合單一資料庫場景。

```csharp
// SetColumns + Where 字串
db.Updateable<object>()
    .AS("Order")
    .SetColumns("Name", "新名稱")
    .SetColumns("Status", 2)
    .Where("Id = 1")
    .ExecuteCommand();
// → UPDATE Order SET Name=@Const0, Status=@Const1 WHERE Id=1
```

## 五、DataTable BulkUpdate（大量資料）

```csharp
// 根據指定欄位作為條件批量更新（需要較新版本）
db.Fastest<DataTable>()
    .AS("Order")
    .BulkUpdate(dataTable, new string[] { "Id" });
// 第二個參數為條件欄位陣列
```

## 六、動態建類更新（最完整多庫 + AOP）

```csharp
// 建立動態類型（WithCache 快取，啟動時建一次）
var type = db.DynamicBuilder()
    .CreateClass("Order", new SugarTable())
    .CreateProperty("Id",         typeof(int),      new SugarColumn { IsPrimaryKey = true, IsIdentity = true })
    .CreateProperty("Name",       typeof(string),   new SugarColumn())
    .CreateProperty("Amount",     typeof(decimal),  new SugarColumn())
    .CreateProperty("UpdateTime", typeof(DateTime), new SugarColumn())
    .WithCache()
    .BuilderType();

// 從字典建立物件並更新
var dic   = new Dictionary<string, object>
{
    ["Id"]         = 1,
    ["Name"]       = "Jack 更新",
    ["Amount"]     = 200m,
    ["UpdateTime"] = DateTime.Now
};
var value = db.DynamicBuilder().CreateObjectByType(type, dic);

db.UpdateableByObject(value).ExecuteCommand();

// 同一動態類可插入、刪除、插入或更新、查詢
db.InsertableByObject(value).ExecuteCommand();
db.DeleteableByObject(value).ExecuteCommand();
db.StorageableByObject(value).ExecuteCommand();  // 插入或更新
db.QueryableByObject(type).ToList();             // 5.1.4.84+
```

更多動態建類用法：https://www.donet5.com/Home/Doc?typeId=2562

## 七、表達式字串更新（5.4.130-preview10+）

動態字串表達式，適合需要在執行時期動態組合更新邏輯的場景。

```csharp
// 程式啟動時執行一次
StaticConfig.DynamicExpressionParserType = typeof(DynamicExpressionParser);

// 無實體動態表達式更新
var oldVal = "a";
var newVal = "c";

db.UpdateableByObject(typeof(Student))
    .SetColumns("it", $"it.Name == it.Name.Replace({oldVal},{newVal})")
    .Where("it",      $"it.Name.Contains({oldVal})")
    .ExecuteCommand();

// 硬編碼表達式
FormattableString expr = $"it => it.Id > 1";

// 動態產生表達式（帶參數）
FormattableString dynExpr = FormattableStringFactory.Create("it => it.Id > {0}", 0);
```

## 各方式比較

| 比較項目 | 字典 | 匿名物件 | SQL 字串 | 動態建類 |
|---|---|---|---|---|
| 批量支援 | ✅ | ❌ | ❌ | ✅ |
| 多庫相容 | ✅ | ✅ | ⚠️ | ✅✅ 最佳 |
| AOP 支援 | ❌ | ❌ | ❌ | ✅ |
| 程式碼量 | 少 | 最少 | 少 | 較多 |
| 適用場景 | 通用低程式碼 | 單筆快速更新 | 單一 DB 特殊需求 | 產品級多庫 |

---

## 注意事項

- WhereColumns 指定的欄位只作為 WHERE 條件，不會被更新
- UpdateableByDynamic（匿名物件）不支援批量，批量請改用字典集合
- SQL 字串方式（SetColumns + Where 字串）不支援多庫，Where 中有特殊語法時各資料庫可能有差異
- 動態建類 WithCache 根據「表名 + 欄位名」快取，相同結構只建一次
