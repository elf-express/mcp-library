# SqlSugar — 更新數據筆記

[//]: # (sqlsugar-source)
> 📖 官方文件:[果糖網 SqlSugar 5x 文件](https://www.donet5.com/Home/Doc?typeId=1191)

## 兩種更新模式

| 模式 | 語法 | 特色 |
|---|---|---|
| 實體物件更新 | db.Updateable(entity) | 像 ORM，需要主鍵，適合全欄位更新 |
| 表達式更新 | db.Updateable\<T\>().SetColumns(...).Where(...) | 像 SQL，適合指定欄位更新 |

---

## 速查表

| 需求 | 語法 |
|---|---|
| 根據主鍵更新全部欄位 | db.Updateable(entity).ExecuteCommand() |
| 批量更新 | db.Updateable(list).ExecuteCommand() |
| 大數據批量更新 | db.Fastest\<T\>().BulkUpdate(list) |
| 忽略指定欄位 | .IgnoreColumns(it => new { it.CreateTime }) |
| 只更新指定欄位 | .UpdateColumns(it => new { it.Name, it.Status }) |
| NULL 欄位不更新 | .IgnoreColumns(ignoreAllNullColumns: true) |
| 無主鍵自訂條件列 | .WhereColumns(it => new { it.OrderNo }) |
| 表達式指定欄位更新 | .SetColumns(it => it.Name == "A").Where(...) |
| 表達式多欄位更新 | .SetColumns(it => new Order { Name = "A", Status = 1 }) |
| 欄位 +1 | .SetColumns(it => it.Num == it.Num + 1) |
| 條件成立才更新 | .SetColumnsIF(條件, it => it.Name == "A") |
| 批量 IN 條件更新 | .Where(it => ids.Contains(it.Id)) |
| 聯表更新 | .InnerJoin\<T2\>(...).SetColumns((x,y) => ...) |
| 字典更新 | db.Updateable(dc).AS("表名").WhereColumns("id") |
| 取得資料庫時間 | SqlFunc.GetDate() 或 UpdateServerTime = true |

## 一、實體物件更新

### 1.1 單筆 / 批量

```csharp
// 單筆（根據主鍵，實體有多少欄位更新多少欄位）
int rows = db.Updateable(order).ExecuteCommand();
bool changed = db.Updateable(order).ExecuteCommandHasChange(); // 回傳 bool

// 批量（非參數化，效能好）
db.Updateable(list).ExecuteCommand();

// 批量（參數化，相容性好）
db.Updateable(list).UseParameter().ExecuteCommand();

// 分頁批量更新（5.1.4.129+）
db.Updateable(list).PageSize(1000).ExecuteCommand();

// 異步
await db.Updateable(order).ExecuteCommandAsync();
```

**✅ 實際案例：審核訂單，更新狀態 + 審核時間**

```csharp
public async Task<bool> ApproveOrderAsync(int orderId, string approver)
{
    var order = await db.Queryable<Order>()
        .Where(o => o.Id == orderId)
        .FirstOrDefaultAsync();

    if (order == null) return false;

    order.Status     = 2;           // 已審核
    order.Approver   = approver;
    order.ApproveTime = DateTime.Now;

    return await db.Updateable(order)
        .UpdateColumns(it => new { it.Status, it.Approver, it.ApproveTime })
        .ExecuteCommandHasChangeAsync();
}
```

### 1.2 忽略指定欄位

```csharp
// Lambda 方式
db.Updateable(order)
    .IgnoreColumns(it => new { it.CreateTime, it.CreateUser })
    .ExecuteCommand();

// 字串方式
db.Updateable(order)
    .IgnoreColumns("CreateTime", "CreateUser")
    .ExecuteCommand();

// 特性方式（實體設定，永遠不更新此欄位）
[SugarColumn(IsOnlyIgnoreUpdate = true)]
public DateTime CreateTime { get; set; }
```

---

### 1.3 只更新指定欄位

```csharp
// Lambda 方式（最常用）
db.Updateable(order)
    .UpdateColumns(it => new { it.Status, it.Remark, it.UpdateTime })
    .ExecuteCommand();

// 字串方式
db.Updateable(order)
    .UpdateColumns("Status", "Remark", "UpdateTime")
    .ExecuteCommand();
```

---

### 1.4 NULL 欄位不更新

```csharp
// 忽略 null 欄位
db.Updateable(order)
    .IgnoreColumns(ignoreAllNullColumns: true)
    .ExecuteCommand();

// 忽略 null 且忽略預設值（int=0、bool=false 也不更新）
db.Updateable(order)
    .IgnoreColumns(ignoreAllNullColumns: true, ignoreAllDefaultValue: true)
    .ExecuteCommand();
```

---

### 1.5 無主鍵 / 自訂條件列

```csharp
// 用 WhereColumns 指定作為條件的欄位（條件欄位不會被更新）
db.Updateable(order)
    .WhereColumns(it => new { it.OrderNo })
    .ExecuteCommand();
// → UPDATE Order SET ... WHERE OrderNo = @OrderNo

// 多欄位條件
db.Updateable(list)
    .WhereColumns(it => new { it.OrderNo, it.CustomerId })
    .ExecuteCommand();
```

---

### 1.6 差異追蹤更新（只更新改過的欄位）

```csharp
var order = db.Queryable<Order>().Where(o => o.Id == 1).First();

db.Tracking(order);       // 建立追蹤快照

order.Name   = "新名稱";  // 只改 Name
order.Status = 2;         // 改 Status

// 只有 Name 和 Status 被更新，其他欄位不動
db.Updateable(order).ExecuteCommand();

// 清空追蹤
db.ClearTracking(); // 5.1.4.108+
```

---

### 1.7 欄位累加（PublicSetColumns）

```csharp
// 固定值累加：批量更新時，Num 欄位 SET num = num + 1
db.Updateable(list)
    .PublicSetColumns(it => it.Num, it => it.Num + 1)
    .ExecuteCommand();

// 集合變量累加：price = price + 每筆集合中的 price
db.Updateable(list)
    .PublicSetColumns(it => it.Price, "+")
    .ExecuteCommand();
// MySQL 若用到 float 欄位需設定精度
```

---

### 1.8 ReSetValue（在已有值基礎上修改）

```csharp
db.Updateable(order)
    .ReSetValue(it =>
    {
        it.Name       = it.Name + "_備份";
        it.UpdateTime = DateTime.Now;
    })
    .ExecuteCommand();
// 注意：這是在 C# 物件值基礎上修改，不是資料庫欄位 +1
```

## 二、表達式更新（像 SQL）

### 2.1 指定欄位更新

```csharp
// 更新多欄位
db.Updateable<Order>()
    .SetColumns(it => new Order { Name = "A", Status = 1, UpdateTime = DateTime.Now })
    .Where(it => it.Id == 1)
    .ExecuteCommand();
// → UPDATE Order SET Name='A', Status=1, UpdateTime=... WHERE Id=1

// 更新單欄位（SetColumns 可叠加）
db.Updateable<Order>()
    .SetColumns(it => it.Name == "jack")
    .SetColumns(it => it.Status == 2)
    .Where(it => it.Id == 1)
    .ExecuteCommand();
// → UPDATE Order SET Name='jack', Status=2 WHERE Id=1
```

**✅ 實際案例：軟刪除**

```csharp
await db.Updateable<Order>()
    .SetColumns(it => new Order
    {
        IsDeleted  = true,
        DeleteTime = DateTime.Now,
        DeleteUser = currentUser
    })
    .Where(it => it.Id == orderId)
    .ExecuteCommandAsync();
```

---

### 2.2 欄位 +1（資料庫層面自增）

```csharp
// 資料庫欄位自增（不依賴 C# 物件的值）
db.Updateable<Order>()
    .SetColumns(it => it.ViewCount == it.ViewCount + 1)
    .Where(it => it.Id == 1)
    .ExecuteCommand();
// → UPDATE Order SET ViewCount = ViewCount + 1 WHERE Id = 1
```

---

### 2.3 條件成立才更新（SetColumnsIF）

```csharp
db.Updateable<Order>()
    .SetColumns(it => it.Name == it.Name)          // 必要的保底欄位
    .SetColumnsIF(remark != null, it => it.Remark == remark)
    .SetColumnsIF(status.HasValue, it => it.Status == status.Value)
    .Where(it => it.Id == orderId)
    .ExecuteCommand();
```

---

### 2.4 批量 IN 條件更新

```csharp
var ids = new int[] { 1, 2, 3 };

db.Updateable<Order>()
    .SetColumns(it => it.Status == 2)
    .Where(it => ids.Contains(it.Id))
    .ExecuteCommand();
// → UPDATE Order SET Status=2 WHERE Id IN (1,2,3)
```

---

### 2.5 無實體表達式更新

```csharp
// 字串欄位更新
db.Updateable<object>()
    .AS("Order")
    .SetColumns("Name", "新名稱")
    .Where("Id = 1")
    .ExecuteCommand();

// MappingColumn 嵌入 SQL（5.1.4.143+）
db.Updateable<object>()
    .AS("Order")
    .SetColumns(
        it => SqlFunc.MappingColumn<string>("Amount"),
        it => SqlFunc.MappingColumn<string>("Amount + 100"))
    .Where("Status = 1")
    .ExecuteCommand();
// → UPDATE Order SET Amount = Amount + 100 WHERE Status = 1
```

---

### 2.6 分頁查詢後更新（In + Subquery）

```csharp
var subQuery = db.Queryable<Order>()
    .OrderByDescending(it => it.CreateTime)
    .Take(10).Skip(10)
    .Select(it => it.Id);  // 只能 Select 單欄位

db.Updateable<Order>()
    .SetColumns(it => new Order { Status = 2, UpdateTime = DateTime.Now })
    .In(it => it.Id, subQuery)
    .ExecuteCommand();
```

## 三、聯表更新

```csharp
// 多庫相容（子查詢方式）
db.Updateable<Order>()
    .SetColumns(it => new Order
    {
        CustomerId = SqlFunc.Subqueryable<Customer>()
            .Where(c => c.Id == it.CustomerId)
            .Select(c => c.Id),
        Name = "新名稱"
    })
    .ExecuteCommand();

// 優雅聯表寫法（SqlServer / MySQL / PgSQL / Oracle 支援）
db.Updateable<Order>()
    .InnerJoin<Customer>((o, c) => o.CustomerId == c.Id)
    .SetColumns((o, c) => new Order { Name = c.Name, Amount = c.CreditLimit })
    .Where((o, c) => o.Status == 1)
    .ExecuteCommand();
```

## 五、預設值（系統時間）

### 特性設定（5.1.3.42+）

```csharp
public class Order
{
    // 更新時自動取資料庫當前時間（支援多庫）
    [SugarColumn(UpdateServerTime = true, IsOnlyIgnoreInsert = true)]
    public DateTime UpdateTime { get; set; }

    // 更新時執行指定 SQL
    [SugarColumn(UpdateSql = "getdate()")]
    public DateTime UpdateTime2 { get; set; }

    [SugarColumn(UpdateSql = "num+1")]
    public int Num { get; set; }
}
```

表達式更新若要觸發 UpdateSql，SetColumns 需加 true：

```csharp
db.Updateable<Order>()
    .SetColumns(it => new Order { Name = "A" }, true)  // true = 觸發 UpdateSql 設定
    .Where(it => it.Id == 1)
    .ExecuteCommand();
```

## 七、更新主鍵

ORM 預設不支援更新主鍵，兩種方案：

- 刪除舊記錄 + 新增新記錄（推薦，主鍵無外部引用時）
- 建立無主鍵實體，用 WhereColumns 指定條件欄位更新

---

## 注意事項

- 實體更新必須設定主鍵，否則需用 WhereColumns 指定條件欄位
- WhereColumns 指定的欄位只作為 WHERE 條件，不會被更新
- NULL 欄位不更新（ignoreAllNullColumns: true）因每筆欄位數不同，批量效能差
- 差異追蹤（Tracking）批量筆數多時效能差，適合單筆場景
- SetColumnsIF 第一個條件不成立時，整個 SetColumns 不加入 SQL
- UpdateServerTime 和 AOP 方式不要同時用，會衝突
- 大數據更新用 db.Fastest\<T\>().BulkUpdate()，效能最強
