## 功能介紹

Storageable 高級保存，適用於：
- 大量 Excel 資料導入（含驗證）
- 大量通用資料保存
- 可編輯表格的批量保存（增刪改混合）
- 需要分組統計（成功、錯誤、忽略各幾筆）的場景

---

## 分組優先級（由高到低）

```
Ignore > Other（預留）> Error > Delete > Update > Insert
```

一筆資料同時符合多個條件，依優先級決定歸入哪一組。
TotalList = InsertList + UpdateList + ErrorList + IgnoreList + OtherList + DeleteList

---

## 速查表

| 分組方法 | 說明 |
|---|---|
| .SplitError(條件, "錯誤訊息") | 符合條件的歸入錯誤清單 |
| .SplitInsert(條件) | 符合條件的歸入插入清單（優先級最低）|
| .SplitUpdate(條件) | 符合條件的歸入更新清單（it.Any() = 主鍵存在資料庫）|
| .SplitDelete(條件) | 符合條件的歸入刪除清單 |
| .SplitIgnore(條件) | 符合條件的歸入忽略清單（不處理）|
| .WhereColumns(it => it.欄位) | 指定非主鍵欄位作為 Any() 的查找條件 |
| x.AsInsertable.ExecuteCommand() | 執行插入 |
| x.AsUpdateable.ExecuteCommand() | 執行更新 |
| x.AsDeleteable.ExecuteCommand() | 執行刪除 |
| x.InsertList / UpdateList / ErrorList | 取得各分組的資料清單 |
| x.BulkCopy() | 大數據插入（搭配分頁使用）|
| x.BulkUpdate() | 大數據更新（搭配分頁使用）|

## 一、完整用例（含驗證、插入、更新、刪除）

```csharp
// 一次性查出驗證用的字典資料（不要在 SplitError 中逐筆查庫）
var sexList = db.Queryable<DictItem>().Where(d => d.TypeCode == "SEX").ToList();

var x = db.Storageable(importList)  // 建議不超過 2 萬筆，超過用分頁（標題五）
    // 【錯誤驗證】不查庫的記憶體驗證（效能好）
    .SplitError(it => string.IsNullOrEmpty(it.Item.Name),   "名稱不能為空")
    .SplitError(it => string.IsNullOrEmpty(it.Item.Phone),  "電話不能為空")
    .SplitError(it => !sexList.Any(z => z.Code == it.Item.Sex), "性別不存在字典中")
    .SplitError(it => it.Item.CreateTime < DateTime.Now.AddYears(-1), "非今年的資料")

    // 【刪除】符合條件的刪除（優先級高於更新和插入）
    .SplitDelete(it => it.Item.CreateTime < DateTime.Now.AddYears(-10))

    // 【更新】主鍵存在資料庫則更新
    .SplitUpdate(it => it.Any())   // it.Any() = 根據主鍵確認資料庫是否已存在

    // 【插入】其餘全部插入（優先級最低，不符合以上條件就是插入）
    .SplitInsert(it => true)

    .ToStorage();  // 執行分組

// 統計結果
Console.WriteLine($"插入:{x.InsertList.Count} 更新:{x.UpdateList.Count} " +
                  $"錯誤:{x.ErrorList.Count} 刪除:{x.DeleteList.Count} 總計:{x.TotalList.Count}");

// 輸出錯誤明細
foreach (var item in x.ErrorList)
    Console.WriteLine($"第 {item.Item.Id} 筆：{item.StorageMessage}");

// 執行資料庫操作
x.AsInsertable.ExecuteCommand();
x.AsUpdateable.ExecuteCommand();
x.AsDeleteable.ExecuteCommand();
```

**✅ 實際案例：Excel 發票資料導入**

```csharp
[HttpPost("import")]
public ApiResult ImportInvoices([FromBody] List<InvoiceImportDto> dtoList)
{
    // 驗證用字典（一次查完）
    var customerList = db.Queryable<Customer>().Select(c => new { c.Code, c.Name }).ToList();
    var taxRateList  = db.Queryable<TaxRate>().ToList();

    var importList = dtoList.Adapt<List<Invoice>>();

    var x = db.Storageable(importList)
        .SplitError(it => string.IsNullOrEmpty(it.Item.InvoiceNo),   "發票號碼不能為空")
        .SplitError(it => string.IsNullOrEmpty(it.Item.CustomerCode), "客戶代碼不能為空")
        .SplitError(it => !customerList.Any(c => c.Code == it.Item.CustomerCode), "客戶代碼不存在")
        .SplitError(it => it.Item.Amount <= 0,  "金額必須大於 0")
        .SplitError(it => it.Item.TaxRate < 0 || it.Item.TaxRate > 1, "稅率範圍錯誤")
        .SplitUpdate(it => it.Any())   // 發票號碼已存在則更新
        .SplitInsert(it => true)
        .WhereColumns(it => it.InvoiceNo)  // 以發票號碼（非主鍵）查找是否存在
        .ToStorage();

    if (x.ErrorList.Any())
    {
        var errors = x.ErrorList
            .Select(e => $"第 {dtoList.IndexOf(dtoList.First(d => d.InvoiceNo == e.Item.InvoiceNo)) + 1} 行：{e.StorageMessage}")
            .ToList();
        return ApiResult.Fail(string.Join("\n", errors));
    }

    x.AsInsertable.ExecuteCommand();
    x.AsUpdateable.ExecuteCommand();

    return ApiResult.Success($"導入成功：新增 {x.InsertList.Count} 筆，更新 {x.UpdateList.Count} 筆");
}
```

## 二、it.Any() 原理說明

```csharp
// it.Any() 等於：
it.Any(y => y.Id == it.Item.Id)
// 等於（底層只查一次庫）：
db.Queryable<T>().Where(t => ids.Contains(t.Id)).ToList().Any(y => y.Id == it.Item.Id)

// Any 只在主鍵已存在的資料中過濾，不查全表
// 多次 Any 也只讀一次資料庫（SqlSugar 內部優化）
```

## 三、驗證名稱是否重複（WhereColumns 用法）

```csharp
// ❌ 錯誤寫法：it.Any() 只根據主鍵，無法用名稱做重複驗證
var x = db.Storageable(list)
    .SplitError(it => it.Any(y => y.Name == it.Item.Name), "名稱已存在")  // 錯！
    .SplitInsert(it => true)
    .ToStorage();

// ✅ 正確寫法：用 WhereColumns 改變 Any() 的查找條件
var x = db.Storageable(list)
    .SplitError(it => it.Any(), "名稱已存在")  // 此時 Any() 根據 WhereColumns 指定的欄位判斷
    .SplitInsert(it => true)
    .WhereColumns(it => it.Name)  // 指定以 Name 欄位在資料庫查找是否存在
    .ToStorage();

x.AsInsertable.ExecuteCommand();

if (x.ErrorList.Any())
    foreach (var item in x.ErrorList)
        Console.WriteLine(item.StorageMessage);
```

## 四、As 功能（分組後套用增刪改方法）

AsInsertable / AsUpdateable / AsDeleteable 轉換後支援 100% 增刪改的所有方法：

```csharp
// 插入時忽略特定欄位
x.AsInsertable.IgnoreColumns(it => new { it.CreateTime }).ExecuteCommand();

// 更新時只更新特定欄位
x.AsUpdateable.UpdateColumns(it => new { it.Name, it.Status }).ExecuteCommand();

// 返回自增 ID
int newId = x.AsInsertable.ExecuteReturnIdentity();

// 取出分組資料另行處理
var insertItems = x.InsertList.Select(z => z.Item).ToList();
var updateItems = x.UpdateList.Select(z => z.Item).ToList();
```

## 五、大數據分頁處理（超過 2 萬筆）

```csharp
db.Utilities.PageEach(list, 2000, pageList =>
{
    var x = db.Storageable(pageList)
        .SplitUpdate(it => it.Any())
        .SplitInsert(it => true)
        .ToStorage();

    x.BulkCopy();    // 大數據插入
    x.BulkUpdate();  // 大數據更新（5.0.4.6+）
});

// 效能注意事項：
// 條件欄位 varchar 長度不超過 50（大字段嚴重影響效能）
// 資料有索引且唯一性高時效能最佳
// varchar vs nvarchar 效能差異大，選最優類型
```

## 六、DataTable 高級保存

```csharp
var x = db.Storageable(dt)
    .SplitUpdate(it => it.Any())
    .SplitInsert(it => true)
    .SplitDelete(it => Convert.ToInt32(it["Id"]) == 100)
    .WhereColumns("Id")
    .ToStorage();

// 注意：DataTable 刪除優先級高於更新高於插入
x.AsDeleteable.ExecuteCommand();
x.AsInsertable.IgnoreColumns("Id").ExecuteCommand();  // 自增欄位需忽略
x.AsUpdateable.ExecuteCommand();
```

var x = db.Storageable(list)

## 注意事項

- SplitError 中需要查庫的驗證（如查重），效能差，建議改用「一次查出清單 → 在記憶體 Any() 過濾」
- it.Any() 只查主鍵是否存在，要用其他欄位查重需搭配 WhereColumns 改變查找依據
- 超過 2 萬筆資料建議用 PageEach 分頁 + BulkCopy / BulkUpdate 處理
- SplitInsert(it => true) 可以不寫 NotAny 條件，因為 Insert 優先級最低，不符合其他條件就一定是插入
- TotalList = 所有分組的總合，用來驗算分組是否完整
