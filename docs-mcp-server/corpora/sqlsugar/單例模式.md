## 速查表

| 需求 | 語法 |
|---|---|
| IOC 單例(推薦) | AddSingleton<ISqlSugarClient>(new SqlSugarScope(...)) |
| 靜態類單例 | static SqlSugarScope Db = new SqlSugarScope(...) |
| 原理 | Scope 用 AsyncLocal 隔離每個請求/執行緒的 db |
| 驗證單例 | 比對 db.ContextId / hashcode |
| Client vs Scope | Client 非安全需每次 new;Scope 單例安全 |

---

## SqlSugarScope 原理

`SqlSugarScope` 內部機制：

- **同一上下文**（同一個請求/執行緒）中，真實的 SqlSugar 物件是**同一個**（Scope 模式）
- **不同上下文**會自動 new 出不同的物件，無需人工處理
- 解決了單例死鎖問題
- 解決了非單例的執行緒安全問題
- 支援跨方法事務

> ⚠️ 非同步第三方組件和 `Task.WhenAll` 場景需特別注意，見官方偶發性錯誤文件（typeId=1224）

## 一、IOC 單例（推薦，Web API 標準做法）

```csharp
// Program.cs
services.AddHttpContextAccessor();  // Furion 框架可省略此行

services.AddSingleton<ISqlSugarClient>(s =>
{
    var sqlSugar = new SqlSugarScope(new ConnectionConfig
    {
        DbType                = DbType.SqlServer,
        ConnectionString      = "Server=.;Database=MyDb;Trusted_Connection=True",
        IsAutoCloseConnection = true,
        InitKeyType           = InitKeyType.Attribute
    }, db =>
    {
        // 單例參數設定，所有上下文（所有請求）共用
        db.Aop.OnLogExecuting = (sql, pars) =>
        {
            // 取得不要求同上下文的 IOC 物件（直接從 s 取）
            // var logger = s.GetService<ILogger<Program>>();

            // 取得要求同上下文的 IOC 物件（透過 HttpContext）
            var accessor = s.GetService<IHttpContextAccessor>();
            var logger   = accessor?.HttpContext?.RequestServices
                .GetService<ILogger<Program>>();
            logger?.LogDebug(UtilMethods.GetNativeSql(sql, pars));
        };
    });
    return sqlSugar;
});
```

## 二、靜態類單例（非 Web 場景，如 WinForm / Console）

### ✅ 正確寫法

```csharp
// 公用靜態輔助類（不能是泛型類）
public class SqlSugarHelper
{
    // ✅ 正確：public static 欄位 = new ...()
    public static SqlSugarScope Db = new SqlSugarScope(new ConnectionConfig
    {
        DbType                = DbType.SqlServer,
        ConnectionString      = "Server=.;Database=MyDb;Trusted_Connection=True",
        IsAutoCloseConnection = true,
        InitKeyType           = InitKeyType.Attribute
    }, db =>
    {
        db.Aop.OnLogExecuting = (sql, pars) =>
            Console.WriteLine(UtilMethods.GetNativeSql(sql, pars));
    });
}

// 使用
var list = SqlSugarHelper.Db.Queryable<Order>().ToList();
```

### ❌ 錯誤寫法（會導致多次建立實例）

```csharp
// ❌ 錯誤：用 => 屬性，每次存取都 new 一個
public static SqlSugarScope Db => new SqlSugarScope(...);

// ❌ 錯誤：get 屬性，每次都 new
public static SqlSugarScope Db { get { return new SqlSugarScope(...); } }

// ❌ 錯誤：方法形式，每次呼叫都 new
public static SqlSugarScope GetDb() => new SqlSugarScope(...);

// ❌ 錯誤：SqlSugarClient 不能單例，只有 SqlSugarScope 才能單例
public static SqlSugarClient Db = new SqlSugarClient(...);

// ❌ 錯誤：在泛型類中 new，DbContext<T>.Db 隨 T 不同會建多個實例
public class DbContext<T>
{
    public static SqlSugarScope Db = new SqlSugarScope(...);  // 錯誤！
}

// ❌ 錯誤：在建構函式中 new，每次 new DbContext 都會建一個
public class DbContext
{
    public DbContext()
    {
        Db = new SqlSugarScope(...);  // 錯誤！
    }
    public static SqlSugarScope Db;
}
```

## 三、使用範例

### 3.1 繼承方式（DbContext 基底類）

```csharp
// 基底類（非泛型）
public class DbContext
{
    protected static SqlSugarScope Db = new SqlSugarScope(new ConnectionConfig
    {
        DbType                = DbType.SqlServer,
        ConnectionString      = "Server=.;Database=MyDb;Trusted_Connection=True",
        IsAutoCloseConnection = true
    }, db =>
    {
        db.Aop.OnLogExecuting = (sql, p) => Console.WriteLine(sql);
    });
}

// 業務類繼承 DbContext，直接用 Db
public class OrderManager : DbContext
{
    public List<Order> GetList()
    {
        return Db.Queryable<Order>().ToList();
    }

    public int Save(Order order)
    {
        return Db.Insertable(order).ExecuteReturnIdentity();
    }
}

// 注意：若 DbContext 是泛型類，需把 Db 提取到非泛型類
public class DbContext<T>
{
    // ✅ 從非泛型類取，不自己 new
    protected static SqlSugarScope Db = SqlSugarHelper.Db;
}
```

### 3.2 靜態類呼叫方式

```csharp
public class OrderManager
{
    public List<Order> GetList()
    {
        return DbContext.Db.Queryable<Order>().ToList();
    }
}

public class DbContext
{
    public static SqlSugarScope Db = new SqlSugarScope(new ConnectionConfig
    {
        DbType                = DbType.SqlServer,
        ConnectionString      = "Server=.;Database=MyDb;Trusted_Connection=True",
        IsAutoCloseConnection = true
    }, db =>
    {
        db.Aop.OnLogExecuting = (s, p) => Console.WriteLine(s);
    });
}
```

## 四、驗證單例是否成功

```csharp
// 服務啟動後，多次呼叫 GetHashCode() 結果相同 → 單例成功
Console.WriteLine(SqlSugarHelper.Db.GetHashCode());
Console.WriteLine(SqlSugarHelper.Db.GetHashCode());
// 兩次輸出相同值 → ✅ 單例正確
// 兩次輸出不同值 → ❌ 每次都在 new，寫法有問題
```

## 五、SqlSugarClient vs SqlSugarScope

| 項目 | SqlSugarClient | SqlSugarScope |
|---|---|---|
| 可單例 | ❌ 不能 | ✅ 可以 |
| 執行緒安全 | ❌ 不安全 | ✅ 安全 |
| 跨方法事務 | ❌ 不支援 | ✅ 支援 |
| 適合場景 | 每個請求 new 一個（AddScoped）| 全應用程式單例（AddSingleton）|
| IOC 生命週期 | AddScoped | AddSingleton |

---

## 注意事項

- 只有 `SqlSugarScope` 能單例，`SqlSugarClient` **不能**單例
- 靜態欄位必須是 `= new ...()` 的形式，不能是屬性（`=>`）或方法
- 泛型類中的靜態欄位不能 new，要從非泛型類引用
- 非同步場景（Task.WhenAll / 第三方非同步組件）需特別注意上下文問題，見偶發性錯誤文件
