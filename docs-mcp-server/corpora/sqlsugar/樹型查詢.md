## 速查表

| 需求 | 語法 |
|---|---|
| 建構樹（主鍵關聯）| .ToTree(it => it.Child, it => it.ParentId, 根節點值) |
| 建構樹（非主鍵）| .ToTree(it => it.Child, it => it.ParentCode, 0, it => it.Code) |
| 非主鍵用特性標記 | [SugarColumn(IsTreeKey = true)] |
| 樹型權限過濾 | .ToTree(it => it.Child, it => it.ParentId, 0, inIds) |
| 查所有上級 | .ToParentList(it => it.ParentId, 節點Id) |
| 查所有下級（扁平）| .ToChildList(it => it.ParentId, 節點Id) |
| 異步建構樹 | .ToTreeAsync(...) |
| List 直接建構樹 | UtilMethods.BuildTree(db, list, pk, parentIdName, childName, rootValue) |
| 樹 + 聯表 | .LeftJoin\<T\>(...).Select(...).ToTree(..., it => it.Id) 需 5.1.4.110+ |
| 樹 + 導航（查上級）| .Includes(x => x.Parent).ToTree(...) |

## 一、實體設計

樹型結構的實體需具備三個元素：主鍵、父級欄位、子節點集合

```csharp
public class Tree
{
    [SugarColumn(IsPrimaryKey = true)]
    public int Id { get; set; }

    public string Name { get; set; }

    public int ParentId { get; set; }  // 父節點 Id，根節點設為 0 或 null

    [SugarColumn(IsIgnore = true)]     // 不對應資料庫欄位
    public List<Tree> Child { get; set; }
}
```

- 欄位名稱可以自定義，不一定要叫 Child / ParentId
- 根節點的 ParentId 通常設為 0 或 null

## 二、基本樹型查詢

```csharp
// 主鍵關聯，根節點 ParentId = 0
var tree = db.Queryable<Tree>()
    .ToTree(it => it.Child, it => it.ParentId, 0);

// 異步
var tree = await db.Queryable<Tree>()
    .ToTreeAsync(it => it.Child, it => it.ParentId, 0);
```

**✅ 實際案例：選單權限樹**

```csharp
// 系統選單：根節點 ParentId = 0
public class Menu
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id       { get; set; }
    public string Name  { get; set; }
    public string Path  { get; set; }
    public int ParentId { get; set; }
    public int Sort     { get; set; }

    [SugarColumn(IsIgnore = true)]
    public List<Menu> Children { get; set; }
}

var menuTree = db.Queryable<Menu>()
    .Where(m => m.IsDeleted == false)
    .OrderBy(m => m.Sort)
    .ToTree(it => it.Children, it => it.ParentId, 0);
```

## 三、非主鍵關聯的樹

當樹的關聯欄位不是主鍵（如用 Code 關聯）時有兩種做法

### 方式一：第四個參數直接指定

```csharp
db.Queryable<Tree>()
    .ToTree(it => it.Child, it => it.ParentCode, 0, it => it.Code)
```

### 方式二：用特性標記（支援 ToChildList 等所有方法）

```csharp
public class Tree
{
    [SugarColumn(IsPrimaryKey = true)]
    public int Id { get; set; }

    public string Name { get; set; }

    [SugarColumn(IsTreeKey = true)]   // 標記此欄位為樹關聯鍵
    public string Code { get; set; }

    public string ParentCode { get; set; }

    [SugarColumn(IsIgnore = true)]
    public List<Tree> Child { get; set; }
}

db.Queryable<Tree>()
    .ToTree(it => it.Child, it => it.ParentCode, 0);
```

**✅ 實際案例：科目代碼樹（用科目代碼關聯）**

```csharp
public class Account
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int Id { get; set; }

    [SugarColumn(IsTreeKey = true)]
    public string AccCode { get; set; }   // 如 1001、1001-01

    public string ParentAccCode { get; set; }
    public string AccName       { get; set; }

    [SugarColumn(IsIgnore = true)]
    public List<Account> Children { get; set; }
}

var accTree = db.Queryable<Account>()
    .Where(a => a.IsActive == true)
    .ToTree(it => it.Children, it => it.ParentAccCode, null);
// 根節點 ParentAccCode 為 null
```

## 四、樹型權限過濾（5.1.3.28+）

根據 Id 過濾時 SqlSugar 會自動補全父節點，確保樹結構完整

```csharp
// 指定要顯示的節點 Id 陣列
object[] inIds = new object[] { 11, 12 };
var tree = db.Queryable<Tree>()
    .ToTree(it => it.Child, it => it.ParentId, 0, inIds);

// 先用 Where 條件篩選出節點 Id，再傳入建構樹
object[] inIds = db.Queryable<Tree>()
    .Where(it => it.Name.Contains("業務"))
    .Select(it => it.Id)
    .ToList()
    .Cast<object>()
    .ToArray();

var tree = db.Queryable<Tree>()
    .ToTree(it => it.Child, it => it.ParentId, 0, inIds);

// 如果 Where 條件不影響樹結構（如按 Type 過濾），可以直接用 Where
var tree = db.Queryable<Tree>()
    .Where(it => it.Type == 1)
    .ToTree(it => it.Child, it => it.ParentId, 0);
```

**✅ 實際案例：根據使用者權限顯示部分組織架構**

```csharp
// 使用者只能看到特定部門及其子部門
var allowedIds = userPermissions.DeptIds.Cast<object>().ToArray();

var deptTree = db.Queryable<Department>()
    .ToTree(it => it.Children, it => it.ParentId, 0, allowedIds);
```

## 五、查所有上級（ToParentList）

從指定節點往上找，回傳扁平清單

```csharp
// 查 Id = 11 的節點的所有上級
var parents = db.Queryable<Tree>()
    .ToParentList(it => it.ParentId, 11);
```

**✅ 實際案例：顯示當前分類的麵包屑路徑**

```csharp
// 查當前分類 Id = 15 的所有祖先，組成麵包屑
var breadcrumb = db.Queryable<Category>()
    .ToParentList(it => it.ParentId, 15);

// breadcrumb 由上到下：[一級分類, 二級分類, 三級分類]
var breadcrumbText = string.Join(" > ", breadcrumb.Select(c => c.Name));
```

## 六、查所有下級（ToChildList，扁平無層級）

ToTree 是有層級巢狀的，ToChildList 是把所有子孫攤平成一個 List

```csharp
// 查 Id = 2 節點下的所有子孫（扁平）
var allChildren = db.Queryable<Tree>()
    .ToChildList(it => it.ParentId, 2);
```

**✅ 實際案例：刪除部門時先找出所有子部門 Id**

```csharp
// 刪除部門前，找出所有子孫部門一併刪除
var allDeptIds = db.Queryable<Department>()
    .ToChildList(it => it.ParentId, deptId)
    .Select(d => d.Id)
    .ToList();

allDeptIds.Add(deptId);  // 加入自身

db.Deleteable<Department>()
    .In(allDeptIds)
    .ExecuteCommand();
```

## 七、樹 + 聯表（5.1.4.110+）

```csharp
var tree = db.Queryable<Tree>()
    .LeftJoin<ExtraInfo>((t, e) => t.Id == e.TreeId)
    .Select((t, e) => new TreeDto
    {
        Id       = t.Id,
        ParentId = t.ParentId,
        Name     = t.Name,
        Remark   = e.Remark   // 從聯表取額外欄位
    })
    .ToTree(it => it.Children, it => it.ParentId, 0, it => it.Id);
// 第四個參數指定 Id 欄位（DTO 沒有主鍵特性時必填）
```

**✅ 實際案例：部門樹帶出負責人姓名**

```csharp
var tree = db.Queryable<Department>()
    .LeftJoin<Employee>((d, e) => d.ManagerId == e.Id)
    .Select((d, e) => new DeptDto
    {
        Id          = d.Id,
        ParentId    = d.ParentId,
        DeptName    = d.Name,
        ManagerName = e.Name   // 負責人姓名
    })
    .ToTree(it => it.Children, it => it.ParentId, 0, it => it.Id);
```

## 八、樹 + 導航（查上級）

```csharp
public class Tree1
{
    [SugarColumn(IsPrimaryKey = true)]
    public int Id { get; set; }
    public string Name   { get; set; }
    public int ParentId  { get; set; }

    [Navigate(NavigateType.OneToOne, nameof(ParentId))]
    public Tree1 Parent { get; set; }   // 上級節點

    [SugarColumn(IsIgnore = true)]
    public List<Tree1> Child { get; set; }
}

var tree = db.Queryable<Tree1>()
    .Includes(x => x.Parent)           // 同時填充上級
    .ToTree(x => x.Child, x => x.ParentId, 0);
```

## 九、樹 + 導航 + 聯表（最完整）

```csharp
var tree = await db.Queryable<MaterialTree>()
    .Includes(p => p.Parent)                               // 導航：查上級
    .LeftJoin<MaterialInfo>((t1, t2) => t1.Code == t2.Code)  // 聯表：取額外欄位
    .Select((t1, t2) => new MaterialDto
    {
        Code         = t1.Code,
        ParentCode   = t1.ParentCode,
        Spec         = t2.Spec,         // 聯表欄位
        Unit         = t2.Unit,
        Parent       = t1.Parent        // 導航上級
    })
    .ToTreeAsync(p => p.Children, p => p.ParentCode, null, p => p.Code);
```

## 注意事項

- ToTree 的根節點值（第三個參數）通常是 0 或 null，依實際資料決定
- ToChildList 回傳扁平清單，ToTree 回傳巢狀結構
- 樹 + 聯表或 DTO 無主鍵時，第四個參數必須指定 Id 欄位（5.1.4.110+）
- 權限過濾的 inIds 陣列需轉成 object[]，用 Cast\<object\>().ToArray()
- Where 條件若只過濾節點屬性（不影響樹結構）可直接用，若按 Id 過濾則用 inIds 方式確保樹完整
- IsTreeKey 特性標記後，ToChildList、ToParentList 都可以正確使用非主鍵關聯
