## 速查表

| 需求 | 語法 |
|---|---|
| 實體類 UNION ALL | db.UnionAll(q1, q2).ToList() |
| 匿名物件 UNION ALL（5.1.4.147+）| db.UnionAll(q1, q2).ToList() |
| 匿名物件 UNION ALL（舊版）| Select(it => (object) new { ... }) 後 db.UnionAll(q1, q2).Select\<T\>().ToList() |
| 單一欄位 UNION ALL | Select(it => (object) it.Name) 後 .Select\<string\>().ToList() |
| 跨庫 UNION ALL | .AsWithAttr() 後 db.UnionAll(q1, q2).ToList() |

---

## 核心觀念

UNION ALL 把多個查詢結果**垂直堆疊**，保留重複資料。每個子查詢的 Select 欄位**數量與名稱必須一致**，否則會報錯。

---

## 一、實體類方式（最常用）

```csharp
var q1 = db.Queryable<Student>().Select(it => new Model { Name = it.Name });
var q2 = db.Queryable<School>().Select(it => new Model { Name = it.Name });

var list = db.UnionAll(q1, q2).ToList();
```

生成 SQL：

```sql
SELECT * FROM (
    SELECT [Name] AS [name] FROM [Student]
    UNION ALL
    SELECT [Name] AS [name] FROM [School]
) unionTable
```

- Select 必須用 new Class 或 (object)new {} 確保回傳型別是參考型別
- 兩個子查詢 Select 出來的欄位結構要完全一致

**✅ 實際案例：應收帳款與應付帳款合併成一張流水帳**

```csharp
// 應收帳款（收入）
var receivable = db.Queryable<AccountReceivable>()
    .Where(r => r.CompanyId == companyId)
    .Select(r => new LedgerDto
    {
        TxDate    = r.InvoiceDate,
        TxType    = "應收",
        Amount    =  r.Amount,   // 正數
        Reference = r.InvoiceNo
    });

// 應付帳款（支出）
var payable = db.Queryable<AccountPayable>()
    .Where(p => p.CompanyId == companyId)
    .Select(p => new LedgerDto
    {
        TxDate    = p.DueDate,
        TxType    = "應付",
        Amount    = -p.Amount,   // 負數
        Reference = p.VoucherNo
    });

// 合併後依日期排序
var list = db.UnionAll(receivable, payable)
    .OrderBy(it => it.TxDate)
    .ToList();
```

## 二、匿名物件方式

### 5.1.4.147+ 版本（直接使用）

```csharp
var q1 = db.Queryable<Order>().Select(it => new { id = it.Id, name = it.Name });
var q2 = db.Queryable<Order>().Select(it => new { id = it.Id, name = it.Name });

var list = db.UnionAll(q1, q2).ToList();
```

### 舊版本（需加 object 轉換）

```csharp
var q1 = db.Queryable<Order>().Select(it => (object)new { id = it.Id });
var q2 = db.Queryable<Order>().Select(it => (object)new { id = it.Id });

var list = db.UnionAll(q1, q2).Select<Order>().ToList();
```

**✅ 實際案例：合併本月與上月訂單做比對清單**

```csharp
var thisMonth = db.Queryable<Order>()
    .Where(o => o.CreateTime >= thisStart && o.CreateTime < thisEnd)
    .Select(o => new { o.OrderNo, o.Amount, Period = "本月" });

var lastMonth = db.Queryable<Order>()
    .Where(o => o.CreateTime >= lastStart && o.CreateTime < lastEnd)
    .Select(o => new { o.OrderNo, o.Amount, Period = "上月" });

var list = db.UnionAll(thisMonth, lastMonth)
    .OrderBy(it => it.Period)
    .ToList();
```

## 三、單一欄位

```csharp
var q1 = db.Queryable<Order>().Where(it => it.Id > 10).Select(it => (object)it.Name);
var q2 = db.Queryable<Order>().Where(it => it.Id == 2).Select(it => (object)it.Name);

var list = db.UnionAll(q1, q2).Select<string>().ToList();
```

**✅ 實際案例：取得所有客戶名稱與廠商名稱合併清單（去重用 Distinct）**

```csharp
var customerNames = db.Queryable<Customer>()
    .Select(c => (object)c.Name);

var supplierNames = db.Queryable<Supplier>()
    .Select(s => (object)s.Name);

// UNION ALL 保留重複，若要去重可再加 .Distinct()
var allNames = db.UnionAll(customerNames, supplierNames)
    .Select<string>()
    .ToList();
```

## 四、UNION ALL 後接 Where / OrderBy / 分頁

UnionAll 結果可以繼續串接過濾與排序

```csharp
var q1 = db.Queryable<Order>().Select(it => new OrderDto { Id = it.Id, Name = it.Name });
var q2 = db.Queryable<Order>().Select(it => new OrderDto { Id = it.Id, Name = it.Name });

var list = db.UnionAll(q1, q2)
    .Where(it => it.Id > 5)
    .OrderBy(it => it.Name)
    .ToList();

// 分頁
var total = 0;
var pageList = db.UnionAll(q1, q2)
    .ToPageList(1, 20, ref total);
```

**✅ 實際案例：多倉庫庫存合併後分頁查詢**

```csharp
var warehouseA = db.Queryable<InventoryA>()
    .Select(i => new InventoryDto
    {
        ProductCode  = i.ProductCode,
        ProductName  = i.ProductName,
        Qty          = i.Qty,
        WarehouseName = "倉庫A"
    });

var warehouseB = db.Queryable<InventoryB>()
    .Select(i => new InventoryDto
    {
        ProductCode  = i.ProductCode,
        ProductName  = i.ProductName,
        Qty          = i.Qty,
        WarehouseName = "倉庫B"
    });

var total = 0;
var list = db.UnionAll(warehouseA, warehouseB)
    .Where(it => it.Qty > 0)
    .OrderBy(it => it.ProductCode)
    .ToPageList(pageIndex, pageSize, ref total);
```

## 五、跨庫 UNION ALL

跨庫時需使用 AsWithAttr 強制轉換表名，不能用 QueryableWithAttr

```csharp
var q1 = db.Queryable<Order>().AsWithAttr();
var q2 = db.Queryable<Order>().AsWithAttr();

db.UnionAll(q1, q2).ToList();
```

---

## 注意事項

- 每個子查詢的 Select 欄位數量、名稱、順序必須完全一致，否則 SQL 報錯
- Select 必須用 new Class 或 new {} 確保型別是參考型別，不能直接 Select 實體
- UNION ALL 保留重複資料，若要去除重複改用 UNION（SqlSugar 目前主要支援 UNION ALL）
- UnionAll 後可以繼續串接 Where、OrderBy、ToPageList
- 跨庫時必須用 AsWithAttr，不能用 QueryableWithAttr
- MySQL 使用 Skip / Take 搭配 UNION ALL 需升級到最新版本
