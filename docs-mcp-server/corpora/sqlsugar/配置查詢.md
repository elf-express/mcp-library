# SqlSugar — 配置查詢筆記

[//]: # (sqlsugar-source)
> 📖 官方文件:[果糖網 SqlSugar 5x 文件](https://www.donet5.com/Home/Doc?typeId=2309)

## 解決的問題

- 多個地方都要聯同一張字典表，只為了取一個 Name，每次都要寫 Join 很浪費時間
- 字典表存性別、學歷、縣市等分類資料，透過 Type 區分，傳統做法要聯表或載入記憶體，各有缺點

## 速查表

| 需求 | 語法 |
|---|---|
| 字典導航（5.1.2+）| [Navigate(NavigateType.OneToOne, nameof(欄位), nameof(字典.Code), "type='xxx'")] |
| 設定字典表配置 | db.ConfigQuery.SetTable\<T\>(keyField, nameField, uniqueTag, condition) |
| 設定一般聯表配置 | db.ConfigQuery.SetTable\<T\>(keyField, nameField) |
| 查詢時取配置值 | .GetConfigValue\<T\>("uniqueTag") |
| 查詢時取配置值（無 Tag）| .GetConfigValue\<T\>() |
| 取得所有欄位 | .SelectAll() 或 Id.SelectAll() |

---

## 一、字典導航（推薦，5.1.2+）

透過導航屬性直接配置字典表關聯，支援多層級一對一，語法最直覺

### 實體設定

```csharp
public class DataMain
{
    public int    Id        { get; set; }
    public string Name      { get; set; }
    public string SexCode   { get; set; }
    public string Province  { get; set; }

    // 字典導航：SexCode 對應 DataDictionary.Code，且 type='sex'
    [Navigate(NavigateType.OneToOne,
        nameof(SexCode), nameof(DataDictionary.Code), "type='sex'")]
    public DataDictionary SexInfo { get; set; }

    // 字典導航：Province 對應 DataDictionary.Code，且 type='province'
    [Navigate(NavigateType.OneToOne,
        nameof(Province), nameof(DataDictionary.Code), "type='province'")]
    public DataDictionary ProvinceInfo { get; set; }
}
```

### 查詢

```csharp
// Includes 填充導航屬性
var list = db.Queryable<DataMain>()
    .Includes(x => x.SexInfo)
    .Includes(x => x.ProvinceInfo)
    .ToList();

// 用導航屬性過濾（子查詢實現，不需要 Includes）
var list = db.Queryable<DataMain>()
    .Where(x => x.SexInfo.Name == "男")
    .ToList();
```

缺點：字典表 Code 非全域唯一時（如性別和縣市都用 01、02），無法動態配置，需依賴主鍵和導航設定

## 二、ConfigQuery 配置查詢（字典表 Code 不唯一時使用）

### 2.1 字典表結構範例

```csharp
public class DataDictionary
{
    public string Code { get; set; }  // 代碼（同類型下唯一）
    public string Name { get; set; }  // 名稱
    public string Type { get; set; }  // 分類（sex / city / province）
}
```

### 2.2 啟動時初始化配置（只執行一次）

```csharp
// 保證整個程式生命週期只設定一次（ConfigQuery 是全域靜態儲存）
if (!db.ConfigQuery.Any())
{
    lock (lockObj)  // 高並發時加鎖避免重複設定
    {
        // 取出所有 Type 種類
        var types = db.Queryable<DataDictionary>()
            .Select(it => it.Type)
            .Distinct()
            .ToList();

        foreach (var type in types)
        {
            // 參數說明：
            // 1. 關聯欄位（Code）
            // 2. 顯示文字（Name）
            // 3. 唯一標識（type 值，例如 "sex"）
            // 4. 查詢條件（限定這個 type）
            db.ConfigQuery.SetTable<DataDictionary>(
                it => it.Code,
                it => it.Name,
                type,
                it => it.Type == type);
        }
    }
}
```

- 建議放在 程式啟動（Program.cs / Startup.cs）或第一次請求時初始化
- 不能重複更新，只能設一次

### 2.3 查詢時使用

```csharp
var list = db.Queryable<Person>()
    .Select(it => new Person
    {
        Id           = it.Id.SelectAll(),   // Oracle 單表需指定別名 SelectAll("別名")
        SexName      = it.SexId.GetConfigValue<DataDictionary>("sex"),
        ProvinceName = it.ProvinceId.GetConfigValue<DataDictionary>("province"),
        CityName     = it.CityId.GetConfigValue<DataDictionary>("city")
    })
    .ToList();
```

生成 SQL：

```sql
SELECT *,
    (SELECT Name FROM DataDictionary WHERE type='sex'      AND Code = t.SexId)      AS SexName,
    (SELECT Name FROM DataDictionary WHERE type='province' AND Code = t.ProvinceId) AS ProvinceName,
    (SELECT Name FROM DataDictionary WHERE type='city'     AND Code = t.CityId)     AS CityName
FROM Person t
```

- 支援寫在 Where 或 OrderBy 中

**✅ 實際案例：人員清單顯示性別與縣市文字**

```csharp
// Person 表存 SexId、CityId（都是 01、02 這種代碼）
// 用 ConfigQuery 不需要每次 Join DataDictionary

var list = db.Queryable<Person>()
    .Where(it => it.IsDeleted == false)
    .OrderBy(it => it.Name)
    .Select(it => new PersonDto
    {
        Id           = it.Id.SelectAll(),
        SexName      = it.SexId.GetConfigValue<DataDictionary>("sex"),
        CityName     = it.CityId.GetConfigValue<DataDictionary>("city"),
        ProvinceName = it.ProvinceId.GetConfigValue<DataDictionary>("province")
    })
    .ToList();
```

## 三、ConfigQuery 配置一般聯表（Code 全域唯一時）

當關聯欄位在整張表中是唯一的（不需要 Type 區分），可以一行搞定

```csharp
// 啟動時設定
if (!db.ConfigQuery.Any())
{
    // 參數1：關聯欄位（Id）
    // 參數2：顯示文字（Name）
    // 不需要第3、4參數
    db.ConfigQuery.SetTable<Order>(it => it.Id, it => it.Name);
}

// 查詢時使用
var list = db.Queryable<OrderItem>()
    .Select(it => new OrderItem
    {
        ItemId    = it.ItemId.SelectAll(),
        OrderName = it.OrderId.GetConfigValue<Order>()  // 不需要傳 Tag
    })
    .ToList();
```

生成 SQL：

```sql
SELECT *,
    (SELECT Name FROM [Order] WHERE Id = t.OrderId) AS OrderName
FROM OrderItem t
```

**✅ 實際案例：明細表帶出訂單名稱**

```csharp
// 訂單 Id 是全域唯一，不需要 Type 區分
if (!db.ConfigQuery.Any())
{
    db.ConfigQuery.SetTable<Order>(it => it.Id, it => it.Name);
    db.ConfigQuery.SetTable<Customer>(it => it.Id, it => it.Name);
    db.ConfigQuery.SetTable<Warehouse>(it => it.Id, it => it.Name);
}

var list = db.Queryable<ShipmentDetail>()
    .Select(it => new ShipmentDetailDto
    {
        DetailId      = it.Id.SelectAll(),
        OrderName     = it.OrderId.GetConfigValue<Order>(),
        CustomerName  = it.CustomerId.GetConfigValue<Customer>(),
        WarehouseName = it.WarehouseId.GetConfigValue<Warehouse>()
    })
    .ToList();
// 每個 GetConfigValue 都生成一個子查詢，不需要寫任何 Join
```

## 四、SetTable 參數說明

| 參數 | 說明 | 必填 |
|---|---|---|
| 參數一 | 關聯欄位（子表的 Key，如 Code 或 Id） | 是 |
| 參數二 | 要顯示的文字欄位（如 Name） | 是 |
| 參數三 | 唯一標識（同一張表有多種查詢時用來區分） | 否 |
| 參數四 | 額外查詢條件（如 it.Type == "sex"） | 否 |

## 五、兩種方式比較

| 比較項目 | 字典導航（Navigate）| ConfigQuery |
|---|---|---|
| 最低版本 | 5.1.2+ | 無限制 |
| Code 不唯一（需 Type）| 支援（特性寫死條件）| 支援（動態迴圈設定）|
| 支援多層級 | 支援 | 不支援 |
| 動態條件 | 不支援 | 支援 |
| Where / OrderBy 用導航 | 支援 | 支援 |
| 初始化複雜度 | 低（特性直接標） | 中（需啟動時初始化）|

---

## 注意事項

- ConfigQuery 是全域靜態儲存，設定後整個應用程式共用，不能重複更新
- 高並發初始化時需加 lock 防止多次設定
- Oracle 單表查詢時 SelectAll 需傳入別名，例如 Id.SelectAll("t")
- GetConfigValue 底層生成子查詢，多個欄位就會有多個子查詢，大量資料時注意效能
- 字典表資料量建議不超過 10000 筆，超過建議改用 Join 或 ThenMapper
