## 速查表

| 需求 | 語法 |
| --- | --- |
| Left Join | .LeftJoin\<T>((a, b) => a.Id == b.AId) |
| Inner Join | .InnerJoin\<T>((a, b) => a.Id == b.AId) |
| 語法糖二多表 | db.Queryable\<T1, T2>((a, b) => new JoinQueryInfos(...)) |
| 全 Inner Join | db.Queryable\<T1, T2>((a, b) => a.Id == b.AId) |
| 動態 Join 條件 | Expressionable.Create\<T1, T2>() |
| 條件成立才 Join | .LeftJoinIF\<T>(條件, ...) |
| 導航屬性 Join | .IncludeLeftJoin(x => x.導航屬性) |
| 設定資料表別名 | .AS("表別名") 或 .LeftJoin\<T>(..., "別名") |
| 超過 12 張表 | 先 MergeTable() 再繼續 Join |
| Cross Join | JoinType.Cross（僅支援語法糖二） |

## 一、三種 Join 語法糖

### 語法糖一（推薦，5 表以內最清晰）

優點：直觀好懂，支援全部功能  
缺點：超過 5 張表時 (a,b,c,d,e...) 參數列會很長

```
var list = db.Queryable<Order>()
    .LeftJoin<Custom>((o, cus) => o.CustomId == cus.Id)
    .LeftJoin<OrderDetail>((o, cus, od) => o.Id == od.OrderId)
    .Where(o => o.Id == 1)
    .Select((o, cus, od) => new ViewOrder
    {
        Id         = o.Id,
        CustomName = cus.Name
    })
    .ToList();

// InnerJoin 改成 .InnerJoin<T>() 即可
// FullJoin 需要較新版本，用法相同
```

*   別名只能按順序：(a,b)、(a,b,c)，不能跳著用 (a,c)

**✅ 實際案例：出貨單聯客戶、聯倉庫**

```
var list = db.Queryable<Shipment>()
    .LeftJoin<Customer>((s, c) => s.CustomerId == c.Id)
    .LeftJoin<Warehouse>((s, c, w) => s.WarehouseId == w.Id)
    .Where(s => s.IsDeleted == false)
    .Select((s, c, w) => new ShipmentDto
    {
        ShipmentNo    = s.ShipmentNo,
        CustomerName  = c.Name,
        WarehouseCode = w.Code,
        ShipDate      = s.ShipDate
    })
    .ToList();
```

---

### 語法糖二（適合 5 張表以上）

優點：多表時參數整潔，容易封裝成通用方法，支援 Cross Join  
缺點：不支援 LeftJoin(queryable) 嵌套寫法

```
var list = db.Queryable<Order, OrderItem, Custom>((o, i, c) => new JoinQueryInfos(
    JoinType.Left, o.Id == i.OrderId,
    JoinType.Left, o.CustomId == c.Id
))
.Select((o, i, c) => new ViewModel { Name = o.Name })
.ToList();

// T 的數量 = JOIN 數 + 1
// 3 個 T = 2 個 JOIN，8 個 T = 7 個 JOIN，不能多也不能少
```

**✅ 實際案例：訂單聯六張表**

```
var list = db.Queryable<Order, Customer, Warehouse, Product, Employee, Department>(
    (o, c, w, p, e, d) => new JoinQueryInfos(
        JoinType.Left, o.CustomerId  == c.Id,
        JoinType.Left, o.WarehouseId == w.Id,
        JoinType.Left, o.ProductId   == p.Id,
        JoinType.Left, o.EmployeeId  == e.Id,
        JoinType.Left, e.DeptId      == d.Id
    ))
    .Select((o, c, w, p, e, d) => new OrderFullDto
    {
        OrderNo      = o.OrderNo,
        CustomerName = c.Name,
        WarehouseName= w.Name,
        ProductName  = p.Name,
        EmployeeName = e.Name,
        DeptName     = d.Name
    })
    .ToList();
```

### 語法糖三（全部 Inner Join 時的簡潔寫法）

```
var list = db.Queryable<Order, OrderItem, Custom>(
    (o, i, c) => o.Id == i.OrderId && c.Id == o.CustomId)
    .Select((o, i, c) => new Class1
    {
        Id         = o.Id,
        Name       = o.Name,
        CustomName = c.Name
    })
    .ToList();
// → FROM [Order] o, [OrderItem] i, [Custom] c WHERE ...
```

## 二、Where / OrderBy / GroupBy 聯表寫法

*   用到哪幾張表，Lambda 就寫幾個參數
*   一定要寫在 Select 前面

```
.Where(o => o.Id == 1)          // 只用主表
.Where((o, i) => i.Price > 100) // 用到第二張表

.OrderBy(o => o.Id)             // 只用主表
.OrderBy((o, i) => i.Price, OrderByType.Desc)

.GroupBy(o => o.Id)
.GroupBy((o, i) => i.CategoryId)
```

## 三、Select 用法

Select 正常順序：.Where(...).OrderBy(...).Select(...).ToList()  
若要在 Select 後接 Where，需加 .MergeTable() 再接

### 手動映射（推薦，效能最好）

```
.Select((o, i) => new OrderDto
{
    Id    = o.Id,
    Name  = o.Name,
    Price = i.Price
}).ToList();
```

**✅ 實際案例：訂單聯明細，只取需要的欄位**

```
var list = db.Queryable<Order>()
    .LeftJoin<OrderDetail>((o, d) => o.Id == d.OrderId)
    .LeftJoin<Customer>((o, d, c) => o.CustomerId == c.Id)
    .Select((o, d, c) => new OrderListDto
    {
        OrderNo      = o.OrderNo,
        CustomerName = c.Name,
        Amount       = d.Amount,
        CreateTime   = o.CreateTime
    })
    .ToList();
```

---

### 實體自動映射一（5.1.3.35+，語法最簡潔）

```
var list = db.Queryable<SchoolA>()
    .LeftJoin<StudentA>((x, y) => x.SchoolId == y.SchoolId)
    .Select((x, y) => new UnitView01()
    {
        Name  = x.SchoolName,  // 手動指定
        Count = 100             // 手動指定
    }, true)  // true = 其餘欄位依名稱自動映射
    .ToList();
```

---

### 實體自動映射二（SelectAll / o.\*）

```
// 主表全取 + 額外欄位
var list = db.Queryable<Order>()
    .LeftJoin<OrderItem>((o, i) => o.Id == i.OrderId)
    .LeftJoin<Custom>((o, i, c) => o.CustomId == c.Id)
    .Select((o, i, c) => new ViewOrder
    {
        Id         = o.Id.SelectAll(), // 等於 o.*
        CustomName = c.Name
    })
    .ToList();
// → SELECT o.*, [c].[Name] AS [CustomName] FROM ...
```

*   SelectAll 建議只用在一張表，多表容易欄位重名

---

### 實體自動映射三（依命名規則自動對應）

```
// 命名規則：
// 主表欄位直接寫欄位名：Name → Order.Name
// 從表欄位寫 Class+欄位名：CustomName → Custom.Name
//                          OrderItemPrice → OrderItem.Price

public class ViewOrder
{
    public string Name           { get; set; } // Order.Name
    public string CustomName     { get; set; } // Custom.Name
    public string OrderItemPrice { get; set; } // OrderItem.Price
}

var list = db.Queryable<Order>()
    .LeftJoin<OrderItem>((o, i) => o.Id == i.OrderId)
    .LeftJoin<Custom>((o, i, c) => o.CustomId == c.Id)
    .Select<ViewOrder>()
    .ToList();
```

*   每個欄位都要能匹配到，否則無法按規則對應
*   高並發場景建議改用手動映射，效能較高

## 四、導航屬性聯表

設定好導航屬性後，不需要手寫 Join

```
// 實體設定
public class StudentA
{
    [SugarColumn(IsPrimaryKey = true, IsIdentity = true)]
    public int StudentId { get; set; }
    public int SchoolId  { get; set; }

    [Navigate(NavigateType.OneToOne, nameof(SchoolId))]
    public SchoolA SchoolA { get; set; }  // 不能預設賦值，只能是 null
}

// IncludeLeftJoin：生成 Left Join SQL
var list = db.Queryable<StudentA>()
    .IncludeLeftJoin(x => x.SchoolA)
    .Where(x => x.SchoolA.SchoolName == "北大")
    .ToList();

// 不加 IncludeLeftJoin，Where 中直接用（子查詢過濾）
var list = db.Queryable<StudentA>()
    .Where(x => x.SchoolA.SchoolName == "北大")
    .ToList();

// 不加 IncludeLeftJoin，Select 中直接用（子查詢實現）
var list = db.Queryable<StudentA>()
    .Select(x => new
    {
        name       = x.Name,
        SchoolName = x.SchoolA.SchoolName
    })
    .ToList();
```

## 五、設定資料表別名

```
var list = db.Queryable<Order>().AS("Order001")
    .LeftJoin<OrderItem>((o, i) => o.Id == i.OrderId, "OrderItem001")
    .LeftJoin<Custom>((o, i, c) => c.Id == o.CustomId, "Custom001")
    .Select((o, i, c) => new { oid = o.Id, iname = i.Name })
    .ToList();
```

**✅ 實際案例：同一張實體對應多張實體資料表**

```
// 例如歷史資料表和當前資料表同一個實體，用別名區分
var list = db.Queryable<Order>().AS("Order_2024")
    .LeftJoin<Order>((a, b) => a.Id == b.ParentId, "Order_2023")
    .Select((a, b) => new { Current = a.OrderNo, History = b.OrderNo })
    .ToList();
```

## 六、Queryable 和表互相 Join

### 表 JOIN Queryable

```
var rightQuery = db.Queryable<Custom>()
    .LeftJoin<OrderItem>((c, i) => c.Id == i.ItemId)
    .Select(c => c);

var list = db.Queryable<Order>()
    .LeftJoin(rightQuery, (o, j) => o.CustomId == j.Id)
    .Select(o => o)
    .ToList();
```

### Queryable JOIN 表

```
var query = db.Queryable<Order>();
var list  = db.Queryable(query)
    .LeftJoin<OrderDetail>((o, d) => o.Id == d.OrderId)
    .Select(o => o)
    .ToList();
```

## 七、超過 12 張表的聯表

先 MergeTable 合併結果，再繼續 Join

```
var list = db.Queryable<Order>()
    .LeftJoin<OrderItem>((x, y) => x.Id == y.ItemId)
    // ... 省略多個 Join
    .Select((x, y, ...) => new { xid = x.Id, yid = y.ItemId })
    .MergeTable()                                          // 合併成單表
    .LeftJoin<OrderItem>((x, y) => x.yid == y.ItemId)     // 繼續 Join
    .ToList();
```

## 八、動態 Join 條件

### 條件動態（動態決定 ON 的條件）

```
var exp = Expressionable.Create<Order, Custom>();
exp.And((o, c) => o.CustomId == c.Id);
exp.AndIF(name != null, (o, c) => o.Name == name);  // 條件成立才加

var list = db.Queryable<Order>()
    .LeftJoin<Custom>(exp.ToExpression())
    .LeftJoin<OrderDetail>((o, c, od) => o.Id == od.OrderId)
    .ToList();
```

**✅ 實際案例：查詢時依參數決定是否加入日期條件**

```
var exp = Expressionable.Create<Order, Customer>();
exp.And((o, c) => o.CustomerId == c.Id);
exp.AndIF(startDate.HasValue, (o, c) => o.CreateTime >= startDate);
exp.AndIF(endDate.HasValue,   (o, c) => o.CreateTime <= endDate);

var list = db.Queryable<Order>()
    .LeftJoin<Customer>(exp.ToExpression())
    .ToList();
```

---

### 表動態（條件成立才加入這張表）

```
var list = db.Queryable<Order>()
    .LeftJoinIF<Custom>(needJoinCustom, (o, c) => o.CustomId == c.Id)
    .LeftJoin<OrderDetail>((o, c, od) => o.Id == od.OrderId)
    .ToList();
// needJoinCustom = false 時，不生成 Left JOIN [Custom] 這段 SQL
```

## 九、Cross Join

只支援語法糖二，條件固定為 true

```
var list = db.Queryable<Product, Category>((p, c) => new JoinQueryInfos(
    JoinType.Cross, true
))
.Select((p, c) => new { p.Name, CategoryName = c.Name })
.ToList();
// → FROM [Product] p Cross JOIN [Category] c
```

## 注意事項

*   Where、OrderBy、GroupBy 都要寫在 Select 前面
*   若要在 Select 後面接 Where，需加 MergeTable() 轉成單表
*   別名參數只能按順序 (a,b)、(a,b,c)，不能跳著寫
*   語法糖二的 T 數量必須等於 JOIN 數 + 1，多一個或少一個都會報錯
*   SelectAll 建議只用在一張表，多表容易欄位重名
*   高並發場景建議手動映射，效能優於自動映射
