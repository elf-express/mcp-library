# SqlSugar — 查詢過濾器筆記

[//]: # (sqlsugar-source)
> 📖 官方文件:[果糖網 SqlSugar 5x 文件](https://www.donet5.com/Home/Doc?typeId=1205)

## 用途

設定全局查詢條件，所有匹配的 Queryable 操作都會自動附加此條件。

最常用場景：軟刪除過濾（IsDeleted == false）、多租戶隔離（OrgId == 當前使用者 OrgId）

---

## 速查表

| 需求 | 語法 |
|---|---|
| 新增表過濾器 | db.QueryFilter.AddTableFilter\<T\>(it => ...) |
| 條件成立才加過濾器 | db.QueryFilter.AddTableFilterIF\<T\>(bool, it => ...) |
| 介面過濾器（繼承介面的類都生效）| db.QueryFilter.AddTableFilter\<IDeleted\>(it => ...) |
| 動態字串過濾器 | db.QueryFilter.AddTableFilter(typeof(T), "it", $"it=>it.Id=={1}") |
| 單次查詢清除所有過濾器 | .ClearFilter() |
| 單次查詢清除指定過濾器 | .ClearFilter\<IDeleted\>() |
| 當前請求清除過濾器 | db.QueryFilter.Clear() |
| 當前請求清除並還原 | db.QueryFilter.ClearAndBackup() / db.QueryFilter.Restore() |
| 刪除啟用過濾器 | .EnableQueryFilter() |
| 更新啟用過濾器 | .EnableQueryFilter() |
| 子查詢啟用過濾器 | SqlFunc.Subqueryable\<T\>().EnableTableFilter() |
| 設定過濾條件加在 ON 後面 | AddTableFilter\<T\>(..., FilterJoinPosition.On) |
| 設定過濾條件加在 WHERE 後面 | AddTableFilter\<T\>(..., FilterJoinPosition.Where) |

## 一、設定表過濾器（推薦）

### 1.1 在初始化時設定（最佳實踐）

```csharp
var db = new SqlSugarClient(new ConnectionConfig
{
    ConnectionString  = "連線字串",
    DbType            = DbType.SqlServer,
    IsAutoCloseConnection = true
},
db =>
{
    // 介面過濾器：繼承 IDeleted 介面的實體都自動過濾
    db.QueryFilter.AddTableFilter<IDeleted>(it => it.IsDeleted == false);

    // 多租戶過濾
    // db.QueryFilter.AddTableFilter<ITenant>(it => it.OrgId == currentOrgId);
});
```

### 1.2 手動新增過濾器

```csharp
// 特定實體類型過濾
db.QueryFilter.AddTableFilter<Order>(it => it.Name.Contains("A"));

// 條件成立才加過濾器（如非管理員才過濾）
db.QueryFilter.AddTableFilterIF<Order>(isAdmin == false, it => it.OrgId == currentOrgId);

// 介面過濾器（5.1.3.47+，繼承介面的類都生效）
db.QueryFilter.AddTableFilter<IDeleted>(it => it.IsDeleted == false);
```

**✅ 實際案例：軟刪除 + 多租戶雙過濾**

```csharp
// 定義兩個介面
public interface IDeleted   { bool IsDeleted { get; set; } }
public interface ITenant    { int  OrgId     { get; set; } }

// 實體繼承介面
public class Order : IDeleted, ITenant
{
    public int  Id        { get; set; }
    public bool IsDeleted { get; set; }
    public int  OrgId     { get; set; }
    // ...
}

// 啟動時設定
db.QueryFilter
    .AddTableFilter<IDeleted>(it => it.IsDeleted == false)
    .AddTableFilterIF<ITenant>(isAdmin == false, it => it.OrgId == currentOrgId);

// 之後所有查詢自動附加條件，不需要每次手寫 Where
var list = db.Queryable<Order>().ToList();
// → WHERE IsDeleted = 0 AND OrgId = @OrgId
```

## 二、清除過濾器

### 2.1 單次查詢清除（只影響這一行）

```csharp
// 清除所有過濾器
db.Queryable<Order>().ClearFilter().ToList();

// 只清除指定介面過濾器
db.Queryable<Order>().ClearFilter<IDeleted>().ToList();

// 清除多個
db.Queryable<Order>().ClearFilter<IDeleted, ITenant>().ToList();
```

### 2.2 當前請求清除（不影響其他請求）

```csharp
// 清除全部
db.QueryFilter.Clear();

// 清除指定介面
db.QueryFilter.Clear<IDeleted>();
```

### 2.3 清除後還原（適合中途需要特殊查詢再恢復）

```csharp
db.QueryFilter.ClearAndBackup();        // 備份並清除

var allData = db.Queryable<Order>().ToList();  // 不帶過濾器

db.QueryFilter.Restore();              // 還原過濾器

var filteredData = db.Queryable<Order>().ToList();  // 帶過濾器
```

**✅ 實際案例：管理員查詢所有資料（包含已刪除）**

```csharp
public List<Order> GetAllIncludeDeleted()
{
    db.QueryFilter.ClearAndBackup();    // 暫時清除過濾器
    var list = db.Queryable<Order>().ToList();
    db.QueryFilter.Restore();           // 還原
    return list;
}
```

## 三、刪除 / 更新啟用過濾器

過濾器預設只對查詢生效，刪除和更新需要手動啟用或全域設定。

### 3.1 局部啟用

```csharp
// 刪除時啟用過濾器
db.Deleteable<Order>()
    .EnableQueryFilter()
    .Where(it => it.Id == 0)
    .ExecuteCommand();

// 表達式更新時啟用
db.Updateable<Order>()
    .EnableQueryFilter()
    .SetColumns(it => it.Name == "A")
    .Where(it => it.Id == 1)
    .ExecuteCommand();

// 物件更新（需要逐筆 PageSize(1)）
db.Updateable(updateObj)
    .PageSize(1)
    .EnableQueryFilter()
    .ExecuteCommand();
```

### 3.2 全域啟用（5.1.4.62+）

```csharp
new ConnectionConfig
{
    DbType            = DbType.SqlServer,
    ConnectionString  = Config.ConnectionString,
    IsAutoCloseConnection = true,
    MoreSettings = new ConnMoreSettings
    {
        IsAutoDeleteQueryFilter = true,  // 刪除自動套用過濾器
        IsAutoUpdateQueryFilter = true   // 更新自動套用過濾器
    }
}

// 全域設定下，若某次不想套用過濾器
db.QueryFilter.Clear();
db.Deleteable<Order>().Where(it => it.Id == 0).ExecuteCommand();
```

## 四、子查詢啟用過濾器

```csharp
// 子查詢中預設不套用過濾器，需手動啟用
SqlFunc.Subqueryable<Order>().EnableTableFilter().Where(s => s.Id == it.Id).Any()
```

## 五、聯表過濾器位置設定

```csharp
// 預設：加在 ON 後面（適合大多數情況）
db.QueryFilter.AddTableFilter<Order>(it => it.OrgId == 1, QueryFilterProvider.FilterJoinPosition.On);

// 加在 WHERE 後面（LeftJoin 從表需要強制過濾主表時使用）
db.QueryFilter.AddTableFilter<Order>(it => it.OrgId == 1, QueryFilterProvider.FilterJoinPosition.Where);
```

## 六、動態字串過濾器（5.1.4.106+）

適合動態租戶、動態權限等場景，需安裝 Dynamic.Core 組件。

```csharp
// 啟動時設定（只需一次）
StaticConfig.DynamicExpressionParserType = typeof(DynamicExpressionParser);

// 使用（Type 可以是介面或類）
// 注意：第二個參數必須有 $，值用 ${} 傳入
db.QueryFilter
    .AddTableFilter(typeof(IUser), "it", $"it=>it.UserName={"jack"}")
    .AddTableFilter(typeof(IOrg),  "it", $"it=>it.OrgId=={1} || it.OrgId=={2}");

// 支援常用 C# 方法
// $"it=>it.Name.Contains({"A"})"
// $"it=>it.CreateTime.Date=={DateTime.Today}"
```

## 七、不依賴實體的過濾器（最靈活）

適合所有 Queryable 都要加條件，但不針對特定實體時使用。

```csharp
// 單表查詢過濾
db.QueryFilter.Add(new SqlFilterItem
{
    FilterValue = it => new SqlFilterResult { Sql = "IsDeleted = 0" },
    IsJoinQuery = false  // 只對單表 Queryable 生效
});

// 多表查詢過濾
db.QueryFilter.Add(new SqlFilterItem
{
    FilterValue = it => new SqlFilterResult { Sql = "o.OrgId = 1" },
    IsJoinQuery = true   // 對聯表 Queryable 生效
});
```

## 注意事項

- 過濾器設定在 db 實例上，Furion/DI 注入時確保同一個 db 實例才能共享過濾器
- 多租戶場景建議用 GetConnection(tenantId) 再加過濾器，避免串台
- ClearAndBackup / Restore 只影響當前請求，不影響其他並發請求
- 全域更新過濾器（IsAutoUpdateQueryFilter）對物件方式更新建議先查詢再更新，表達式更新才直接套用
- 子查詢不自動套用過濾器，需手動 .EnableTableFilter()
- 動態字串過濾器值必須用 $"{}" 格式傳參數，直接硬編碼字串會有快取問題
