## 非同步的作用

釋放主執行緒的佔用、提升吞吐量（高並發、資料庫請求時間過長等場景）。
並非所有地方都需要非同步，全部寫非同步也不會有大問題，但程式碼難度會提升，建議合適場景使用。

## 一、正確與錯誤寫法速查

```csharp
// ❌ 錯誤：方法返回值沒有 Task
public void GetAll() { await 非同步方法; }

// ❌ 錯誤：沒有 await
public async Task GetAll() { 非同步方法; }

// ❌ 錯誤：async void（沒有等待效果，需考慮執行緒安全）
public async void GetAll() { await 非同步方法; }

// ✅ 正確：async Task
public async Task GetAll() { await 非同步方法; }

// ✅ 正確：async Task<T>
public async Task<List<Order>> GetAll() { return await 非同步方法; }

// ❌ 錯誤：Action 等同於 void，不支援非同步
Action a = async () => await 非同步方法;

// ✅ 正確：Func<Task> 才支援非同步
Func<Task> a = async () => await 非同步方法;

// ❌ 錯誤：Parallel.ForEach 是 Action，不支援非同步
Parallel.ForEach(list, async item => await 非同步方法);

// ❌ 錯誤：List.ForEach 是 Action，不支援非同步
list.ForEach(async item => await 非同步方法);

// ✅ 正確：改用普通 foreach
foreach (var item in list)
{
    await 非同步方法;
}

// ❌ 錯誤：Task.Run 中未加 await
public Task TestAsync()
{
    Task.Run(() => { 非同步方法; });  // 未 await Task.Run，且內部也未 await
}

// ✅ 正確：唯一一種返回值非 Task 的非同步用法（fire-and-forget）
public void FireAndForget()
{
    Task.Run(async () =>
    {
        // 如果是 SqlSugarClient 需要 new 或 CopyNew
        await 非同步方法;
    });
}

// ✅ 正確：await Task.Run
public async Task TestAsync()
{
    await Task.Run(async () =>
    {
        await 非同步方法;
    });
}
```

## 二、基本非同步用法

```csharp
// 查詢
var first = await db.Queryable<Order>().FirstAsync();
var list  = await db.Queryable<Order>().Where(it => it.Status == 1).ToListAsync();
var count = await db.Queryable<Order>().CountAsync();
var any   = await db.Queryable<Order>().AnyAsync(it => it.Status == 0);

// 分頁（注意 total 的特殊寫法）
RefAsync<int> total = 0;
var pageList = await db.Queryable<Order>()
    .Where(it => it.Status == 1)
    .OrderByDescending(it => it.CreateTime)
    .ToPageListAsync(1, 20, total);
int totalCount = total;  // 取出總筆數

// 插入
await db.Insertable(order).ExecuteCommandAsync();
var newId = await db.Insertable(order).ExecuteReturnIdentityAsync();

// 更新
await db.Updateable(order).ExecuteCommandAsync();

// 刪除
await db.Deleteable<Order>().Where(it => it.Id == id).ExecuteCommandAsync();
```

## 三、完整正確的非同步方法範例

```csharp
// ✅ 完整正確寫法（每個非同步方法都有 await）
public async Task<List<Order>> GetOrdersAsync(int customerId)
{
    var db = GetInstance();

    // 每個非同步呼叫都要 await
    await ValidateAsync();

    var items = await db.Queryable<OrderItem>()
        .Where(it => it.CustomerId == customerId)
        .ToListAsync();

    return await db.Queryable<Order>()
        .Where(it => it.CustomerId == customerId)
        .ToListAsync();
}

// ❌ 錯誤：漏掉 await 會造成偶發性錯誤
public async Task<List<Order>> GetOrdersWrong(int customerId)
{
    var db = GetInstance();

    ValidateAsync();  // 漏了 await！

    db.Queryable<OrderItem>().ToListAsync();  // 漏了 await！

    return await db.Queryable<Order>().ToListAsync();
}

// ✅ 非同步中等待用 Task.Delay，不能用 Thread.Sleep
public async Task ProcessAsync()
{
    // ❌ 錯誤
    // Thread.Sleep(1000);

    // ✅ 正確
    await Task.Delay(1000);
}
```

## 四、CancellationToken 取消令牌

```csharp
// 設定取消令牌（支援請求取消）
public async Task<List<Order>> GetWithCancelAsync(CancellationToken token)
{
    db.Ado.CancellationToken = token;

    var list1 = await db.Queryable<Order>().ToListAsync();
    var list2 = await db.Queryable<OrderItem>().ToListAsync();

    db.Ado.RemoveCancellationToken();  // 用完記得移除
    return list1;
}

// Controller 中使用
[HttpGet]
public async Task<IActionResult> Get(CancellationToken cancellationToken)
{
    var list = await _orderService.GetWithCancelAsync(cancellationToken);
    return Ok(list);
}
```

## 六、特殊場景強制執行緒安全

### 6.1 不加 await 的非同步（fire-and-forget）

```csharp
// 沒有 await 一定要加 CopyNew()
db.CopyNew().Queryable<Order>().ToListAsync();  // 不等待，用 CopyNew 保安全

// 有事務的寫法（CopyNew 必須在當前方法內使用）
var newDb = db.CopyNew();
newDb.BeginTran();
await newDb.Insertable(order).ExecuteCommandAsync();
newDb.CommitTran();
```

### 6.2 Task.WhenAll 並行執行

```csharp
// ✅ 每個 Task 用 CopyNew() 保證執行緒安全
var task1 = db.CopyNew().Queryable<Order>().ToListAsync();
var task2 = db.CopyNew().Queryable<Customer>().ToListAsync();
var task3 = db.CopyNew().Queryable<OrderItem>().ToListAsync();

await Task.WhenAll(task1, task2, task3);

var orders    = task1.Result;
var customers = task2.Result;
var items     = task3.Result;
```

### 6.3 定時 Job / BackgroundService

```csharp
protected override async Task ExecuteAsync(CancellationToken ct)
{
    while (!ct.IsCancellationRequested)
    {
        // Job 中一定要 CopyNew()
        var list = await db.CopyNew().Queryable<Order>()
            .Where(it => it.Status == 0)
            .ToListAsync();

        await Task.Delay(5000, ct);
    }
}
```

## 七、Parallel.ForEach → Task.WhenAll

```csharp
// ❌ Parallel.ForEach 不支援非同步（Action 不是 Func<Task>）
Parallel.ForEach(list, async item =>
{
    await db.Insertable(item).ExecuteCommandAsync();  // 不受等待！
});

// ✅ 改用 Task.WhenAll（每個用 CopyNew 保安全）
var tasks = list.Select(item =>
    db.CopyNew().Insertable(item).ExecuteCommandAsync()
);
await Task.WhenAll(tasks);
```

## 八、兩種強制執行緒安全方式

| 方式 | 適用場景 | 寫法 |
|---|---|---|
| `db.CopyNew()` | SqlSugarClient 和 SqlSugarScope 都可用 | `db.CopyNew().Queryable<T>().ToListAsync()` |
| `ServiceProvider.CreateScope()` | 只支援 SqlSugarClient，推薦用於 IOC 場景 | 詳見偶發性錯誤文件 |

## 注意事項

- 非同步方法的返回值只能是 `Task` 或 `Task<T>`，不能是 `void`
- 每個非同步方法呼叫前都必須加 `await`，漏掉就會出現偶發性錯誤
- `List.ForEach` / `Parallel.ForEach` 是 `Action`，不支援 async，改用普通 `foreach`
- 非同步中等待用 `await Task.Delay()`，禁止用 `Thread.Sleep()`
- 不加 await（fire-and-forget）、Task.WhenAll、定時 Job 這三種場景必須加 `db.CopyNew()`
- 分頁的 total 用 `RefAsync<int> total = 0`，不是普通 `int`
