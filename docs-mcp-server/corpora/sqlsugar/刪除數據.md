## 速查表

| 需求 | 語法 |
|---|---|
| 根據主鍵刪除 | .In(1).ExecuteCommand() |
| 根據主鍵陣列刪除 | .In(new int[]{ 1, 2 }).ExecuteCommand() |
| 根據實體刪除 | db.Deleteable\<T\>(entity).ExecuteCommand() |
| 批量實體刪除 | db.Deleteable\<T\>(list).ExecuteCommand() |
| 表達式條件刪除 | .Where(it => it.Id == 1).ExecuteCommand() |
| 無主鍵自訂條件欄位 | .In(it => it.OrderNo, "A001").ExecuteCommand() |
| 聯表刪除（子查詢）| .Where(it => SqlFunc.Subqueryable\<T2\>()...Any()) |
| 無實體刪除 | db.Deleteable\<object\>().AS("表名").Where("sql") |
| 軟刪除（內建）| .In(id).IsLogic().ExecuteCommand() |
| 軟刪除（過濾器相容）| db.Updateable\<T\>().SetColumns(IsDeleted=true).Where(...) |
| 全域過濾器刪除 | .EnableQueryFilter().Where(...).ExecuteCommand() |
| 截斷表（清空+重置自增）| db.DbMaintenance.TruncateTable\<T\>() |
| 導航刪除（級聯）| db.DeleteNav\<T\>(exp).Include(...).ExecuteCommand() |
| 分頁刪除（大量資料）| db.Deleteable\<T\>(list).PageSize(500).ExecuteCommand() |

## 一、根據實體刪除

```csharp
// 單筆（需配置主鍵）
db.Deleteable<Order>(new Order { Id = 1 }).ExecuteCommand();

// 批量實體刪除
var list = new List<Order>
{
    new Order { Id = 1 },
    new Order { Id = 2 }
};
db.Deleteable<Order>(list).ExecuteCommand();

// 分頁批量刪除（大量資料）
db.Deleteable<Order>(list).PageSize(500).ExecuteCommand();

// 異步
await db.Deleteable<Order>(entity).ExecuteCommandAsync();

// 返回 bool
bool changed = db.Deleteable<Order>(entity).ExecuteCommandHasChange();
```

## 二、根據主鍵刪除

```csharp
// 單一主鍵
db.Deleteable<Order>().In(1).ExecuteCommand();

// 主鍵陣列
db.Deleteable<Order>().In(new int[] { 1, 2, 3 }).ExecuteCommand();

// 無主鍵實體，指定條件欄位
db.Deleteable<Order>().In(it => it.OrderNo, "A001").ExecuteCommand();
db.Deleteable<Order>().In(it => it.OrderNo, new[] { "A001", "A002" }).ExecuteCommand();

// 注意：Oracle 用 .In() 只支援 1000 筆以內
// Oracle 超過 1000 筆改用：.Where(it => ids.Contains(it.Id))
```

## 三、根據表達式刪除

```csharp
// 單條件
db.Deleteable<Order>()
    .Where(it => it.Id == 1)
    .ExecuteCommand();

// 多條件
db.Deleteable<Order>()
    .Where(it => it.Status == 0 && it.CreateTime < DateTime.Now.AddYears(-1))
    .ExecuteCommand();
// → DELETE FROM Order WHERE Status = 0 AND CreateTime < @p
```

**✅ 實際案例：刪除指定用戶的過期草稿**

```csharp
await db.Deleteable<Order>()
    .Where(it =>
        it.UserId == currentUserId &&
        it.Status == 0 &&
        it.CreateTime < DateTime.Now.AddDays(-7))
    .ExecuteCommandAsync();
```

// 用 WhereColumns 指定條件欄位（5.1.4.60+）
db.Deleteable<Order>()
    .WhereColumns(list, it => new { it.OrderNo })

## 五、聯表刪除（子查詢）

```csharp
// 刪除有對應學校的學生（子查詢 EXISTS）
db.Deleteable<Student>()
    .Where(p => SqlFunc.Subqueryable<School>()
        .Where(s => s.Id == p.SchoolId)  // 關聯條件不能少！
        .Any())
    .ExecuteCommand();
// 注意：關聯條件（s.Id == p.SchoolId）不能省略，否則會全刪
```

## 六、無實體刪除

```csharp
// 直接 SQL 條件
db.Deleteable<object>()
    .AS("Order")
    .Where("id = @id", new { id = 1 })
    .ExecuteCommand();

// IN 批量刪除
db.Deleteable<object>()
    .AS("Order")
    .Where("id IN (@id)", new { id = new int[] { 1, 2, 3 } })
    .ExecuteCommand();

// 字典集合刪除
var dcList = new List<Dictionary<string, object>>
{
    new Dictionary<string, object> { ["Id"] = 1 },
    new Dictionary<string, object> { ["Id"] = 2 }
};
db.Deleteable<object>().AS("Order").WhereColumns(dcList).ExecuteCommand();

// DataTable 轉字典刪除
var dcList = db.Utilities.DataTableToDictionaryList(dataTable);
db.Deleteable<object>().AS("Order").WhereColumns(dcList).ExecuteCommand();
```

## 七、軟刪除（邏輯刪除）

### 方式一：內建 IsLogic()（簡單直接）

```csharp
// 自動尋找 IsDelete 或 IsDeleted 欄位設為 true
db.Deleteable<Order>().In(1).IsLogic().ExecuteCommand();

// 指定欄位名稱
db.Deleteable<Order>().In(1).IsLogic().ExecuteCommand("IsRemoved");

// 指定欄位 + 更新時間
db.Deleteable<Order>().In(1).IsLogic()
    .ExecuteCommand("IsRemoved", true, DateTime.Now, "DeleteTime");

// 指定欄位 + 更新時間 + 操作人
db.Deleteable<Order>().In(1).IsLogic()
    .ExecuteCommand("IsRemoved", true, DateTime.Now, "DeleteTime", "DeleteUser", userId);

// 說明：用法和普通刪除一樣，只是多加 .IsLogic()
```

### 方式二：Updateable 軟刪除（可結合全域過濾器）

```csharp
// 定義介面
public interface IDeleted
{
    bool IsDeleted { get; set; }
}

// 實體繼承介面
public class Order : IDeleted
{
    public int Id { get; set; }
    public bool IsDeleted { get; set; }
    // ...
}

// 泛型軟刪除方法（可與更新過濾器同時生效）
public void SoftDelete<T>(Expression<Func<T, bool>> exp)
    where T : class, IDeleted, new()
{
    db.Updateable<T>()
        .SetColumns(it => new T { IsDeleted = true }, true)  // true = 觸發 AOP 過濾器
        .Where(exp)
        .ExecuteCommand();
}

// 使用
SoftDelete<Order>(o => o.Id == 1);
```

**✅ 實際案例：Service 層軟刪除（含操作人與時間）**

```csharp
public async Task<bool> DeleteOrderAsync(int orderId, string operatorId)
{
    return await db.Updateable<Order>()
        .SetColumns(it => new Order
        {
            IsDeleted   = true,
            DeleteTime  = DateTime.Now,
            DeleteUser  = operatorId
        })
        .Where(it => it.Id == orderId && it.IsDeleted == false)
        .ExecuteCommandHasChangeAsync();
}
```

## 八、全域過濾器刪除

全域過濾器預設只對查詢生效，刪除需手動 .EnableQueryFilter()：

```csharp
// 設定過濾器
db.QueryFilter.AddTableFilter<Order>(it => it.OrgId == currentOrgId);

// 刪除時啟用過濾器（只會刪除當前租戶的資料）
db.Deleteable<Order>()
    .EnableQueryFilter()
    .Where(it => it.Id == 1)
    .ExecuteCommand();
// → DELETE FROM Order WHERE OrgId = @OrgId AND Id = 1
```

// 刪除主表時同步刪除關聯子表
db.DeleteNav<Order>(o => o.Id == 1)

## 十一、常用案例

### 11.1 保留最新 N 筆，刪除其餘

```csharp
// 保留最新 10 筆，刪除其他
db.Deleteable<Order>()
    .In(it => it.Id,
        db.Queryable<Order>()
            .OrderByDescending(it => it.CreateTime)
            .Skip(10)                  // 跳過最新 10 筆
            .Select(it => it.Id))      // 注意：不要 ToList()！
    .ExecuteCommand();
```

### 11.2 刪除第 N 到第 M 筆

```csharp
// 刪除按時間倒序排列的第 11~20 筆
db.Deleteable<Order>()
    .In(it => it.Id,
        db.Queryable<Order>()
            .OrderByDescending(it => it.CreateTime)
            .Skip(10).Take(10)
            .Select(it => it.Id))
    .ExecuteCommand();
```

### 11.3 大量資料分頁刪除

```csharp
// 分批刪除避免鎖表
db.Deleteable<Order>(largeList)
    .PageSize(500)
    .ExecuteCommand();
```

## 注意事項

- 聯表刪除的關聯條件不能省略，否則會刪除全表
- Oracle 用 .In() 最多 1000 筆，超過改用 .Where(it => ids.Contains(it.Id))
- 案例 11.1/11.2 的 Select 不要接 .ToList()，否則變成兩次查詢，效能變差
- IsLogic 軟刪除和方式二不建議混用，選一種統一
- TruncateTable 是 DDL 操作，不在事務保護範圍內，無法回滾
- 全域過濾器對刪除預設不生效，需手動加 .EnableQueryFilter()
